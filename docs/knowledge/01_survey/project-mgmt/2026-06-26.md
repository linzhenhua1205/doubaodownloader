## 1️⃣ Linear: 「Issue tracking is dead」— 迈向 Context + Agency 新时代

**来源**: [Issue tracking is dead — Linear Blog](https://linear.app/blog/issue-tracking-is-dead) (2026-03-24)
**作者**: Karri Saarinen, Linear CEO

### 1.1 核心论点

> **「The next era of product development is built on context and agency. 下一时代的产品开发建立在上下文和自主行动能力之上。」**

Linear 认为传统 Issue tracking 的范式（创建 → 分配 → 更新状态 → 关闭）已不足以支撑 AI 时代的开发流程。核心变化：

| 旧范式 | 新范式 |
|:-------|:-------|
| Issue 作为「待办事项」 | Issue 作为「上下文容器」— Agent 可直接理解并执行 |
| 人类是唯一执行者 | Agent 是一等公民，可 Assign、可 @Mention、可自动执行 |
| 流程靠人驱动（更新状态、流转） | 流程靠 Agent 编排（自动 triage、auto-fix、自动更新） |
| 工具收敛于「跟踪工作」 | 工具发散到「理解上下文 + 自主行动」 |

### 1.2 产品落地：Linear Agent → Coding Sessions

**来源**: [Coding sessions in Linear — Linear Changelog](https://linear.app/changelog) (2026-06-10)
**作者**: Karri Saarinen

Linear Agent 从 2026 年初发布后，经历了三次关键迭代：

**① Code Intelligence（2026-05-14）** — Agent 能读取整个代码库并回答问题。从「只能看 Issue 上下文」扩展到「理解产品 + 代码的全貌」。

**② Linear Diffs（2026-05-27）** — 代码审查直接集成到 Linear 内部。背景：Agent 产生大量代码 → 人类的审查负担激增 → 需要将审查嵌入开发工作流。

**③ Coding Sessions（2026-06-10）** — Agent 从 triage 到修复、审查、合并的全流程闭环。Karri Saarinen: 「Linear Agent can now write code. With Coding Sessions, it can take a bug from triage all the way to a reviewed fix, without the work ever leaving Linear.」

**④ Auto-fix bugs in triage（2026-06-12）** — Igor Sechyn 展示：部分 bug 在被工程师捡起前，Agent 已在 triage 阶段自动捕获并修复。**「Some bugs at Linear now get fixed before a single engineer picks them up.」**

**⑤ Agent assisted project updates（2026-06-17）** — Agent 自动从 Issue、文档、讨论中提取最新变更，为项目和倡议生成状态更新。

### 1.3 关键客户数据

| 客户 | 迁移/采用 | 关键数字 |
|:-----|:---------|:---------|
| **Ramp** | Linear 为底层产品上下文层 | coding agent 产出 **60%+ merged PRs** |

## 2️⃣ Atlassian: Jira → Agent 编排平台

**来源**: [Claude Agent for Jira — Atlassian Blog](https://www.atlassian.com/blog/announcements/claude-agent-for-jira) (2026-06-18)
**作者**: Himanshu Singh, Geoff Clitheroe

### 2.1 Claude Agent for Jira 架构

基于 Anthropic Claude Managed Agents 基础设施：

```
Jira Issue (验收标准/目标仓库/设计标准)
  → 分配 Claude（Assignee/@mention/自动化规则）
  → Anthropic 沙箱（clone → 分析 → implement → push）
  → GitHub Draft PR
  → 实时状态流回 Jira Issue
  → 人类 Code Review → 合并
```

**设计要点**:
1. **Agent 是 Jira 一等公民** — 可 Assign、@Mention、自动化规则触发
2. **隔离沙箱执行** — 不在本地环境运行，避免安全/权限问题
3. **Human-in-the-loop** — Agent 只到 Draft PR，最终合并由人控制
4. **双向集成** — 既支持 Jira Cloud 托管，也支持本地 CLI/IDE

### 2.2 关键数据：10-15% 天花板 vs 60%+ PRs

**来源**: Atlassian 内部 longitudinal study — 400 engineering orgs

| 指标 | 数值 |
|:-----|:----:|
| AI coding 采用率 | ≤ 90% |
| 整体生产力提升 | 10-15%（**平稳**） |
| 对比 Ramp Agent PRs | 60%+ |
| 差异根因 | 上下文结构化程度 + 任务范围窄度 |

> **推论**: Agent 生产力跃升需要「结构化上下文 + 窄任务范围 + 高效审查流」三者到位。仅拼接入率是不够的。

### 2.3 Rovo Dev in Jira: 两周 120 PRs

一名工程师用 Rovo Dev，两周交付 120 个 PR。Agent 承担低中复杂度任务，工程师聚焦高阶设计。

### 2.4 Cursor in Jira — Agent 编排平台定位

Atlassian 战略：「Jira, your agent orchestration platform」。从 Jira 直接分配工作项给 Cursor，与 Claude Agent 策略一致。

### 2.5 AI Alert Grouping: 839 小时/28 天

**来源**: [Jira Service Management Blog](https://www.atlassian.com/blog/jira-service-management)

JSM AI Alert Grouping 将告警自动分组，Atlassian 内部节省 **839 小时/28 天**，即约 30 小时/天。

### 2.6 Forrester Wave SPM Q2 2026

Atlassian 被评为 Strategic Portfolio Management Tools Strong Performer，Vision & Innovation 维度得分提升最大。

### 2.7 Flex 商业模型

推出 Flex 价值导向商业模型，适应 AI 时代的弹性和企业采购需求。

### 2.8 Agentic Pipelines: 多 Agent 支持

Bitbucket Pipelines 已支持 Claude Code + OpenAI Codex，CI/CD 中 Agent 编排从「单 Agent」走向「多 Agent 协作」。

## 3️⃣ 交叉分析

### 📊 本日五大信号

| # | 信号 | 来源 | 数据支撑 | 强度 |
|:-:|:-----|:-----|:---------|:----:|
| 1 | **Issue tracking 范式终结** — 从「跟踪工作」转向「上下文 + 自主行动」 | Linear CEO | 产品全线转型 + Coding Sessions | 🔴 |
| 2 | **Agent 全流程闭环已工程化** — triage → coding → review → merge | Linear | Auto-fix bug、Coding Sessions、Diffs | 🔴 |
| 3 | **Jira → Agent 编排平台** — 可分配、可触发、可追踪 | Atlassian | Claude Agent/Cursor/Rovo Dev 三重集成 | 🔴 |
| 4 | **Agent 生产力天花板 10-15%（平均）** — 拼接入率不够 | Atlassian | 400 org 纵向研究 | 🔴 |
| 5 | **Agent 是三分技术、七分组织** — 上下文结构化是关键 | 综合 | Ramp 60% vs 均值 10-15% | 🔴 |

### 📈 三大趋势判断

1. **项目管理工具的竞争焦点从「工作跟踪」转向「Agent 编排能力」**。Linear 以 Coding Sessions 实现端到端，Atlassian 以 Claude/Cursor/Rovo Dev 三管齐下作为开放编排平台。Agent 是 Jira 和 Linear 中的一等公民这一决策，标志行业进入「Human-Agent 共同工作」的新阶段。

2. **Agent 生产力取决于上下文质量，而不是采用率**。400 org 研究显示，90% 采用率仅带来 10-15% 提升，而 Ramp 实现了 60%+ Agent PRs。差距根源在于 Product Context 的结构化程度——这正是 Linear 的「Issue as context container」和 Atlassian 的 Teamwork Graph / MCP 集成所解决的问题。

3. **Code Review 成为新的稀缺资源**。Agent 批量生成代码加速了「写」但让「审」成了瓶颈。Linear Diffs、Draft PR 模式、Atlassian 的 Agentic Pipelines 都在解决同一问题：**如何在不牺牲质量的前提下,规模化地审查 Agent 产出**。
