## 1️⃣ 🔥 Linear Agent Coding Sessions: 闭环从 Triage 到修复 — 内部 30% Bug 首通率

**来源**：Linear Now + Changelog，2026-06-11  
**技术详情**：Linear Agent 现在可以用 **Claude Code 和 Codex 直接编写代码**，从 triage → 规划 → 审查 → 发布，全过程不离开 Linear。核心能力包括：

- **云端运行**：无需本地开发环境，原型和修改直接在云中执行
- **Diff 返回**：变更就绪后，Linear 直接返回新 diff 供审查；可分享预览链接、邀请团队成员讨论、多轮迭代
- **自动第一道防线**：新 Bug 上报时，Agent 可自动调查并尝试修复，在到达工程师之前完成首轮处理
- **MCP 集成**：通过 MCP 获取 Sentry、Datadog 等工具的上下文证据，结合代码库追踪根因

**数据**：Linear 内部用此工作流解决约 **30% 的传入 Bug 报告**，且大部分在首次尝试即完成修复。

**PM 意义**：项目管理系统（Linear）不再仅仅是 "Issue 跟踪"，而是变成了 **Agent 驱动的端到端工程闭环**。产品经理在 Linear 中起草 Spec → Agent 自动拆解为 Issue → 自动编码 → 自动提交 Diff 审查。Issue tracking is dead 宣言的落地兑现。

**定价**：Basic / Business / Enterprise 计划可用，消耗 AI Credits，workspace admin 可在设置中管理访问和使用量。

## 2️⃣ 🔥 Linear Team Documents + Shared Skills: 团队级知识结构化

**来源**：Linear Now + Changelog，2026-06-03  
**技术详情**：两个重要产品能力同时推出：

**Team Documents**：每支团队获得一个专属文档页面，用于保存"不属于特定 Issue 或 Project" 但团队需要的上下文——笔记、文档、共享参考资料。支持：
- 从团队主页直接创建文档
- 将已有项目/issue 文档添加到团队空间
- 按章节分组组织资源
- 支持 Cycle 文档分组显示

**Shared Skills for Linear Agent**：给 Agent 的指导指令（Skills）现在可在团队内共享。之前 Skills 是个人化的，现在团队可以将优秀工作流标准化：
- 写 Spec 的规范流程
- 每周规划的固定格式
- 项目更新草案的模板
- 可用 slash command 或在设置中分享

**PM 意义**：Agent 工作流的可重复性和可治理性关键提升。个人的 Agent 使用经验（Skills）可以"复制给所有人"，组织级 AI 最佳实践不再是口口相传。

## 3️⃣ Atlassian Teamwork Collection 完整功能矩阵（Team '26）

**来源**：Atlassian Blog，Built for the Next Era of Teamwork，2026-05-06  
**作者**：Sanchan S Saxena（Head of Product, Teamwork Collection）

Atlassian Team '26 发布的全产品线 AI 更新完整功能矩阵：

| 功能 | 范围 | 状态 |
|:-----|:-----|:-----|
| **Agents in Jira** | Agent 可在 Jira 中认领任务、分配 Issue、更新代码（Studio + 第三方：Amplitude/Canva/Cursor/Figma/Gamma/GitHub Copilot） | **GA** ✅ |
| **Third-Party Agents in Confluence** | @mention 第三方 Agent 在 Confluence 页面（Lovable/Replit/Databricks/Gamma），读上下文后跨工具行动 | 公开 Beta |
| **Remix with Rovo** | 选中 Confluence 内容 → 一键转为 Chart/Timeline/Infographic/Geo Map/Org Chart/Quadrant/Flip Card **不改变原文**。含视觉元素的页面阅读率约 **2×** | Beta |
| **Confluence Slides** | Rovo 从 Teamwork Graph 拉取上下文，AI 生成演示幻灯片，可直接在 Confluence 中呈现 | Beta（5月） |
| **Create with Rovo in Jira** | Confluence 文档/会议摘要/邮件 → 结构化 Jira 工作项，团队启动速度提升 **～30%** | Beta |
| **Loom Agent Briefings** | 录制 walkthrough，语音/画面/点击操作 → 多模态输入 → 结构化 Agent Prompt → Jira 工作项一键生成 | Coming Soon |
| **Bug Reporting Loom + Jira** | 录制 Bug 时自动捕获设备信息/Console 日志/网络数据 → 自动填入 Jira Ticket → 分配给 Rovo Dev 自动修复 | **GA** ✅ |

**PM 意义**：Teamwork Collection 是 Atlassian 最完整的 AI 产品矩阵发布。"Loom → Jira → Confluence → Rovo" 的 Agent 工作流闭环打通，数据来源于 Teamwork Graph（团队知识图谱）。**"Create with Rovo" +30% 启动速度** 是可直接量化评估的效率数据。

## 5️⃣ Atlassian Flex: AI 时代的价值定价模型

**来源**：Atlassian Blog，Announcing Flex: A Flexible Commercial Model for the AI Era
**技术详情**：Flex 是 Atlassian 新推出的企业商业模式：
- **价值导向**：不再仅按 seat 收费，而是与客户实际从 AI 中获取的价值挂钩
- **企业敏捷性**：帮助客户在 AI 变革中快速调整规模的灵活合同模式
- 背景：AI Agent 自动完成工作 → 传统按人（seat）计费模式被颠覆 → 需要新的商业逻辑

**PM 意义**：这是项目管理 SaaS 行业首次从 **"按 seat 收人头税"向 "按价值产出收费"** 的重大商业模式变迁。如果成功，可能被其它 PM 工具效仿。
