## 🔥 TOP2: SpaceX Colossus 1 遭遇**互联延迟瓶颈**——被迫转租给 Anthropic & Google

> **来源**: Bloomberg（经 The Verge 引用, 2026-06-12）
> **链接**: https://www.theverge.com/ai-artificial-intelligence

**背景**：Elon Musk 原计划利用分布在孟菲斯地区的三个数据中心园区，以集群方式训练 Grok AI 模型

**核心问题**：
- 三个园区之间**距离 >10 英里**（约 16 公里）
- 连接时遭遇 **严重延迟问题**（latency issues）
- 叠加**老化的网络基础设施**（aging network infrastructure）
- **多站点互联 → 延迟墙**：物理距离 × 光速 → 训练集群性能无法达标

**被迫转租**：
| 客户 | 租金规模 |
|:-----|:---------|
| **Anthropic** | **~$150 亿/年**（据 Bloomberg） |
| **Google** | **$9.2 亿/月**（$110 亿/年） |

> **关键洞察**：即使是 Musk 级别、以 Colossus 1（全球最大单体 GPU 集群）为后盾，**多站互联训练的延迟墙仍然是硬约束**。SpaceX 最终转而开发**卫星 AI 服务器方案**

**行业影响**：这解释了为什么 Anthropic 和 Google 都在抢租算力——OpenAI 签 10GW 俄亥俄、Anthropic 租 Colossus 1、Google 签 $303 亿 SpaceX 大单，三家巨头在同一周内全部转向**租算力模式**

## 💧 TOP4: Amazon 首次披露数据中心用水——全球占比极小, 局部影响极大

> **来源**: Ars Technica — Kyle Orland (2026-06-12) | 原始来源: Amazon 博客
> **链接**: https://arstechnica.com/tech-policy/2026/06/when-it-comes-to-total-water-use-ai-data-centers-are-a-drop-in-the-bucket/

### 全球数字对比

| 来源 | 年用水量 | 备注 |
|:-----|:---------|:------|
| **Amazon DC** (2025) | **25 亿加仑** | 首次披露 |
| **Google DC** (2024) | **61 亿加仑** | |
| **Microsoft DC** (2024) | **27.5 亿加仑** | |
| **Meta DC** (2024) | **14 亿加仑** | |
| 全美 DC（2021 Nature 研究） | **~1,630 亿加仑** | 含间接水耗 |
| Texas DC（2024 估算） | **250-490 亿加仑** | 2030 年预计达 3,990 亿 |
| ⚡ vs 美国草坪浇灌 | **3.3 万亿加仑/年** | 量级差异 ~20× |
| ⚡ vs 加州杏仁种植 | **1.3 万亿加仑/年** | |
| ⚡ vs 美国高尔夫球场 | **5,310 亿加仑/年** | |

### 局部影响——真正严峻的问题

| 案例 | 数据 |
|:-----|:------|
| **Meta Newton County, Georgia** | 单数据中心占整个县 **10% 供水量** |
| **北弗吉尼亚数据中心集群** | 占区域总耗水 **8%**，2050 年预计升至 **29%** |
| **全美普适** | **40% 计划中/已建 DC 位于"高"或"极高"水稀缺区域**（World Resources Institute） |

**行业应对**：Amazon 通过升高机柜温度减少水冷需求，承诺投入 50 个水回馈项目（每年回补 58 亿加仑）；Google 计划 2030 年前回补 190 亿加仑
