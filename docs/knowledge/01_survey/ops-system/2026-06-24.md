## 1️⃣ 🔴 K8s v1.36: PSI (Pressure Stall Information) 正式 GA

**来源**: Kubernetes Blog (May 12, 2026)
**链接**: https://kubernetes.io/blog/2026/05/12/psi-ga/

- **PSI (Pressure Stall Information)** 源自 Linux 内核 2018 年引入，在 Kubernetes v1.34 进入 Beta，v1.36 正式 GA
- **核心价值**: 在资源耗尽导致故障 **之前**，通过高保真信号提前感知饱和
- 覆盖三种资源压力: CPU、Memory、IO
- 与传统的 utilization 指标互补：utilization 告诉「用了多少」，PSI 告诉「等了多少」
- **运维实操价值**: 提前发现节点资源瓶颈，做预防性 Scale-up 或 Pod 驱逐，减少实际 outage
- 与 Prometheus `node_pressure_*` 指标体系对齐


## 2️⃣ 🔴 Linux 7.2 合并窗口进展

**来源**: LWN.net (Jun 18, 2026)
**链接**: https://lwn.net/Articles/987300/

- 6 月 14 日发布 7.1 内核后，7.2 合并窗口已拉入 **~7,000 changesets**
- **关键特性（运维相关）**:
  - **mTHP 自动创建** — 7.1 已合入 mTHP 支持，7.2 进一步实现自动创建机制，减少容器场景的内存调优开销
  - **BPF 协程化** (Suspend/Resume BPF Programs) — Kumar Kartikeya Dwivedi 工作，允许 BPF 程序以协程方式表达，简化长时间运行 BPF 任务的编写
  - **KASAN for JIT-compiled BPF** — 为 BPF JIT 编译器添加内核内存访问检查
  - **RMR/BRMR 块复制协议** — 基于 RDMA 的单跳块复制，面向云厂商虚拟块设备
  - **OSPM 2026 报告已发布** — 调度器/电源管理峰会内容


## 3️⃣ 🔴 AURpocalypse 深度复盘: 开源包管理信任模型冲击

**来源**: LWN.net (Jun 19, 2026)
**链接**: https://lwn.net/Articles/987913/

- **攻击手法**: 攻击者创建多个新账户 → 认领无人维护的包 → 推送恶意更新（安装恶意软件）
- **持续时间**: 数天，维护者陷入 Whac-A-Mole 式对抗
- **应对措施**: AUR 已暂停新用户注册
- **深层影响**: 社区开始质疑 AUR 的合作模型是否需要结构性重构
- **与 Cilium CI/CD 安全三部曲交叉参考**: CNCF Blog 同期发布了 Cilium CI/CD 供应链安全系列（Part 1 访问控制, Part 2 依赖锁定），形成开源供应链安全的完整图景


## 4️⃣ Systemd v261: 云 IMDS 子系统

**来源**: LWN.net (Jun 19, 2026)
**链接**: https://lwn.net/Articles/987820/

- **新功能**: 新增 Cloud Instance Metadata Service (IMDS) 子系统
- **意义**: 减少对云厂商 agent 的依赖，在多云/混合云 K8s 节点标准化场景有直接价值
- 其他 v261 特性: Live Update Orchestration (LUO) / Kexec Handover (KHO) 支持、Boot Secret（无物理 TPM 场景）


## 5️⃣ Jaeger v2.18 ClickHouse Backend（运维视角）

**来源**: CNCF Blog (Jun 23, 2026)

- **8.6× 压缩率** → 同样存储成本下 retention 延长近一个数量级
- >50k spans/sec → 中大规模集群全量追踪成为可能
- **运维价值**: 降低全量追踪采样成本，延长历史数据留存周期，改善 MTTR 的根因回溯能力
- Materialized Views 预聚合 → 查询延迟从秒级降至毫秒级
- 目前 Alpha 阶段，**不推荐生产使用**
