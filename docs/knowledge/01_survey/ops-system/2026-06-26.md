## 1️⃣ Linux 7.2: Cache Aware Scheduling 合并 — MySQL 最高 360% 提升 🔴

**来源**: [Phoronix](https://www.phoronix.com/news/Extended-Cache-Aware-Sched) — 2026-06-25

**Cache Aware Scheduling（CAS）** 是 Linux 2026 年最令人兴奋的调度创新之一，已合并至 Linux 7.2：

**性能数据**:
- **MySQL**: 扩展版 CAS 达到最高 **360% 吞吐提升**
- 通过在调度层面感知 CPU cache 拓扑，将相关任务调度到共享 cache 的核上以减少 cache miss
- 初始版本已合并至 7.2，新 patch 系列正在进行扩展优化

**相关**: Linux 7.2 同时也带来 MGLRU 改进，**MongoDB 吞吐提升 30~100%**（[Phoronix](https://www.phoronix.com/news/Linux-7.2-MM)）。

**运维影响**: 对数据库密集型和延迟敏感的容器化工作负载可能产生显著影响，K8s 节点级别可受益于调度优化。

## 2️⃣ Linux 7.2: EROFS 增强 — 高效处理大型稀疏 AI 数据集 🟡

**来源**: [Phoronix](https://www.phoronix.com/news/EROFS-Sparse-AI-Datasets) — 2026-06-23

EROFS（Enhanced Read-Only File System）在 Linux 7.2 中进行了 AI 负载专项优化：

**关键改进**:
- **大型稀疏 AI 数据集处理**: 更好地处理 AI 训练数据中的稀疏文件模式
- **更高效的 I/O**: 针对 AI 数据集的访问模式优化读取性能
- 适用于容器镜像分发和 AI 数据集部署场景

**运维关联**: 与容器运行时（containerd/runc）结合，EROFS 可作为只读层的高效文件系统选择，特别适合 AI 推理镜像的快速部署场景。

## 3️⃣ Akrites 项目: 保护开源软件免受 AI 驱动的漏洞利用 🟡

**来源**: [Phoronix](https://www.phoronix.com/news/Akrites) — 2026-06-25

**Linux Foundation** 联合 **Amazon、Anthropic、OpenAI、NVIDIA、Microsoft、Red Hat** 等推出 **Akrites** 项目：

**目标**: 防御 AI/LLM 驱动的安全漏洞发现对关键开源软件的威胁
- 随着 LLM 发现漏洞的速度空前提升，开源项目的应急响应能力需要同步升级
- 确保 AI 暴露的漏洞在被恶意利用前得到有效修复
- 成立**开源安全漏洞修复协作网络**，快速响应 AI 发现的 0day

**运维影响**: 对容器镜像扫描和安全运维流程提出新要求——AI 驱动的漏洞发现速度将大幅快于传统模式。

## 4️⃣ Linux 7.2: IMA/EVM 后量子 ML-DSA 签名支持 🟢

**来源**: [Phoronix](https://www.phoronix.com/news/Linux-7.2-Integrity) — 2026-06-24

Linux 7.2 在完整性子系统中增加 **后量子 ML-DSA（Module-Lattice-Based Digital Signature Algorithm）签名支持**：

**技术要点**:
- IMA（Integrity Measurement Architecture）和 EVM（Extended Verification Module）支持后量子签名
- 为未来后量子密码学迁移做准备
- 对使用 IMA/EVM 进行运行时完整性度量的安全敏感环境有重要意义

## 5️⃣ LWN 本周: Linux 7.2 合入窗口与 OSPM 2026 报告 🟢

**来源**: [LWN.net Weekly Edition](https://lwn.net/) — 2026-06-25

**OSPM 2026 核心议题**:
- **Device Frequency Scaling**: 设备频率缩放策略
- **sched_ext**: 用户态调度器持续升温，但 Linus 不满代码组织方式（"disgusting thing"），已重构
- **多集群 Arm 调度域**: 复杂 Arm SoC 的调度域管理
- **LAVD 调度器**: 基于时间片时长做 CPU 选择
- **BPF coroutines**: 允许 BPF 程序表达为协程（来自 LSM-BFSummit）
- **Fedora 2FA**: 因 AI agent 账户入侵事件推动 provenpackager 强制 2FA

**其他运维相关**:
- **systemd v261**: 新增 Cloud IMDS 子系统（Instance Metadata Service）、boot secret 功能（无需 TPM 的加密密钥）、支持 Kernel LUO/KHO
- **Podman 6.0**: 支持容器多静态 IP、网络隔离提升（Docker 兼容性改进）、配置文件重写（大量 breaking changes）
- **MinIO 归档**: 2025年12月宣布维护模式，2026年2月归档，用户转向 Ceph 和 Garage
- **GIMP 0.54.1 Flatpak**（有意思但无关紧要）
