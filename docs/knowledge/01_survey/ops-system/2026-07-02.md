## 1️⃣ 🔴 Kubernetes AI 政策正式发布

**来源**: LWN.net / Kubernetes Blog (2026-06-29)
**链接**: https://lwn.net/Articles/1032317/

**关键信息**:

- K8s 项目官方发布 AI 政策，核心三条：
  1. **必须披露 AI 辅助**: 使用 AI 工具生成代码的贡献必须明确披露
  2. **禁止 AI 列为合著者**: 不能将 AI 作为 PR 的作者/合著者
  3. **禁止 "assisted-by" / "co-developed" 署名**: 不允许在 commit trailer 中引用 LLM 工具作为贡献者
- **动机**: AI 生成的代码量快速增长但代码维护能力没有同步提升；大量 PR 因 AI 使用问题偏离技术讨论

**影响判断**: K8s 作为 CNCF 旗舰项目，其 AI 政策将成为云原生社区的参考标准。对于企业内部分支/定制 K8s，这类政策也影响 CI/CD 流程中的 AI 辅助代码审查和 PR 管理。

## 3️⃣ 🟡 Secure Boot 证书到期——运维警报

**来源**: LWN.net (2026-07-01)
**链接**: https://lwn.net/Articles/1032457/

**关键信息**:

- Microsoft 的 Secure Boot 证书（Microsoft Windows Production PCA 2011）已到期
- **不会导致已启动的系统崩溃**，但以下场景会受影响：
  - 新安装系统时 Secure Boot 验证失败
  - 更新 bootloader/shim 时需要新证书签名
  - 2027 年后的旧二进制签名将完全失效
- **运维建议**: 系统管理员需要确保 bootloader、shim 和内核的签名已更新到新证书链

**影响判断**: 这是**每 ~5 年一次的证书轮换**。对服务器运维团队意味着需要检查 BMC/BIOS 的 Secure Boot 证书库更新，以及 PXE 部署镜像的签名状态。

## 5️⃣ 🟡 OSPM 2026 Day 3 报告：GPU 亲和性调度 + Profile-Guided Scheduling

**来源**: LWN.net (2026-06-26)
**链接**: https://lwn.net/Articles/1031847/

**关键信息**:

- **GPU Affinity Scheduling（GPU 亲和性调度）**: 在 CPU 调度器中显式考虑 GPU 的 NUMA 拓扑和缓存层次，减少 GPU 驱动/中断处理延迟
- **Profile-Guided Scheduling**: 系统级 profiling → 调度策略自动调优
- **QoS 多租户**: 在共享集群中保证关键 workload 的调度延迟

**影响判断**: 这些特性直接服务于 AI 训练/推理场景的调度优化——GPU 亲和性调度对 NCCL 集合通信延迟有直接影响。
