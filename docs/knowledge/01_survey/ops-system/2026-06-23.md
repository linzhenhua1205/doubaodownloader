## 🔴 今日亮点

### 1. K8s v1.36: PSI 压力失血指标正式 GA

**来源**: [Kubernetes Blog](https://kubernetes.io/blog/2026/05/12/kubernetes-v1-36-psi-metrics-for-kubernetes-graduates-to-ga/) · 2026-05-12

Pressure Stall Information (PSI) 自 Linux 4.20（2018）引入内核，在 K8s 中经过 Beta 阶段（v1.34）后正式 GA：

- **检测资源饱和前兆**：CPU/memory/IO 三类压力信号，比 OOM/CPU throttling 早数秒至数分钟触发告警
- **Kubelet 原生暴露**：通过 `/metrics/resource` endpoint 提供 `node_pressure_*` 和 `pod_pressure_*` 指标
- **运维意义**：配合 Prometheus 告警规则可以实现"资源饱和前感知"，而非传统的"故障后响应"
- **与 Memory QoS 协同**：v1.36 同时引入 Tiered Memory Protection，PSI 信号可作为 Memory QoS 策略的反馈输入

**关联知识库**：PSI 指标与 [Prometheus 告警规则库](../../02_rd/05_software/prometheus-alert-rules.md) 中资源饱和类规则互补

### 2. K8s v1.36: Pod-Level Resource Managers（Alpha）+ Tiered Memory Protection

- **Pod-Level Resource Managers**（Alpha）：将资源管理粒度从 Node 级细化为 Pod 级，允许各 Pod 独立拥有资源管理器实例（CPU Manager / Memory Manager）
- **Tiered Memory Protection**：为多层级内存（本地DRAM + CXL内存 + PMem）提供 QoS 保证，适用于 AI 推理场景中的 KV Cache 分层存储
- **In-Place Vertical Scaling for Pod-Level Resources**（Beta）：在原 Pod 内动态调整 CPU/memory 分配，无需重启

这些特性组合为 **AI 推理服务的精细化资源运营** 提供了基础设施：精细隔离 → 动态调整 → 压力感知

### 3. Jaeger ClickHouse 后端：可观测性管道的新范式

**来源**: CNCF Blog 2026-06-23（详见[云原生跟踪](../bmc-system/2026-06-23.md)）

从运维视角提取的关键数字：
- **8.6× 压缩比** → 存储成本显著降低，更长 retention 成为可能
- **50k+ spans/sec** 单节点写入 → 生产级吞吐
- **5级属性索引**（resource/scope/span/event/link）→ 查询复杂度可控

**对 SRE 团队的价值**：解决追踪系统中长期存在的"存不起、查不快"困境，降低 Tracing 全量采样的运营成本门槛

### 4. Linux 7.2 合并窗口：~7,000 changesets

**来源**: [LWN.net](https://lwn.net/) · 2026-06-18

关键进展：
- **单跳块复制（RMR/BRMR）**：基于 RDMA 的可靠多播 + 块设备层，为云厂商提供低开销持久化虚拟块设备
- **BPF 协程化**：Kumar Kartikeya Dwivedi 的工作允许 BPF 程序以协程方式执行，使长运行 BPF 任务更易编写
- **OSPM 2026 报告**：idle-state selection、sched_ext 用户态调度器、锁持有者抢占等议题
- **Buffer-heads 清理**：Jan Kara 的元数据 buffer head 追踪基础设施改进

### 5. AURpocalypse: AUR 供应链攻击深度复盘

**来源**: [LWN.net](https://lwn.net/) · 2026-06-19

- 攻击者批量注册新账号 → 认领孤包（orphaned packages）→ 推送含恶意软件的更新
- 维护团队多日打地鼠式响应（Whac-A-Mole）
- 当前措施：**暂停 AUR 新用户注册**
- 长期担忧：AUR 现有的协作模式（信任第一贡献者/包的自动提升）在规模化攻击下难以维系

**运维警示**：与 [Cilium CI/CD 安全三部曲](../cloud-native/2026-06-12.md) 中的 supply chain 防护形成呼应——开源包管理的信任模型正面临结构性质疑

### 6. Systemd v261: 云 IMDS 子系统

**来源**: [LWN.net](https://lwn.net/) · 2026-06-19

- 新增 **Cloud Instance Metadata Service (IMDS) 子系统**：减少对云厂商 agent 的依赖
- 利好多云/混合云场景下的 K8s 节点标准化
- 其他新特性：`boot secret`（无 TPM 系统的密钥引导）、KHO/Kexec Handover 支持
