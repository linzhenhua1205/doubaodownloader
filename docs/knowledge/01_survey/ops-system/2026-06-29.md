## 1️⃣ Kernel 7.2-rc1 合并窗口关闭 🔴

**来源**: [LWN.net](https://lwn.net/) — 2026-06-28, corbet

Linus Torvalds 发布 **Linux 7.2-rc1**，标志着 7.2 合并窗口关闭。

### 关键信息

| 指标 | 说明 |
|:-----|:------|
| **状态** | rc1 已发布，开始测试阶段 |
| **Linus 评价** | "Things look reasonably normal for this release" |
| **预计稳定版** | ~7-8 周后（8 月中旬） |

### 已合入的关键特性（来自之前报道）
- **分配令牌（Allocation Tokens）**: 动态分配的结构体被随机放置在内存中，增加漏洞利用难度（bootpatch-SLR 项目仍在推进中）
- **BPF arenas 辅助库 libarena**: 通用 BPF arena 数据结构库，已合入内核但早期阶段
- **BPF JIT KASAN 支持**: 对 JIT 编译的 BPF 代码启用内存访问检查

### 同步的稳定更新

| 版本 | 说明 |
|:-----|:------|
| **7.0.14** | **7.0.x 系列的最终版本**，建议用户升级到 7.1.x 或 7.2-rc |
| **7.1.2** | 最新的 7.1 稳定更新 |
| **6.18.37** | LTS 分支安全更新 |

## 2️⃣ Ceph 和 Garage 作为 MinIO 替代方案 🟡

**来源**: [LWN.net — A look at MinIO alternatives: Ceph and Garage](https://lwn.net/) — 2026-06-25, tjl

**背景**: MinIO 于 2025 年 12 月宣布进入维护模式，2026 年 2 月完全归档。用户正在寻找替代方案。

### 替代方案对比

| 方案 | 优势 | 劣势 |
|:-----|:------|:------|
| **Ceph (RADOS Gateway)** | 成熟稳定，大规模生产验证，完整 S3 兼容，自带纠删码 | 部署复杂，资源消耗大，小规模场景过重 |
| **Garage** | 轻量级（Rust 实现），设计简洁，Geo-distributed 原生支持，低资源占用 | 较新，生态不如 Ceph 丰富，功能集较小 |

### 运维意义
- ✅ MinIO 归档后，**S3 兼容对象存储的选择格局已变** — Ceph 和 Garage 成为主要竞争者
- ✅ 对 AI 训练的数据存储层影响：很多 MLflow/Kubeflow 工作流依赖 MinIO 做 artifact 存储，需要迁移规划
- ✅ 对中小规模集群，Garage 的 Rust 实现和低资源占用值得关注

## 3️⃣ Podman 6.0 发布：多项 Breaking Changes 🟡

**来源**: [LWN.net](https://lwn.net/) — 2026-06-25, jzb

Podman 6.0.0 发布，包含多项重大变更。

### 关键新特性

| 特性 | 说明 |
|:-----|:------|
| **多静态 IP** | 容器可设置多个静态 IP 地址 |
| **网络隔离增强** | 与 Docker 的兼容性大幅改善 |

- ✅ Quadlet（systemd 集成）的变更影响使用 Podman 管理系统的运维方式
- ⚠️ Breaking changes 较多，建议在非生产环境充分测试后再升级

## 4️⃣ systemd v261：Cloud IMDS + Boot Secret 🟡

**来源**: [LWN.net](https://lwn.net/) — 2026-06-26, corbet / Lennart Poettering (Mastodon)

systemd v261 包含多项重要功能更新。

### 关键新特性

| 特性 | 说明 |
|:-----|:------|
| **Cloud IMDS 支持** | systemd 现在可以原生查询云平台的 Instance Metadata Service（IMDS），无需额外工具 |
| **Boot Secret** | 启动时安全注入密钥/凭证到 systemd 服务 |
| **Kernel LUO/KHO** | 配合内核的 Kexec Handover 机制，实现更干净的启动切换 |

### 运维意义
- ✅ Cloud IMDS 原生支持 → 减少云环境初始化对 cloud-init 的依赖
- ✅ Boot Secret → 安全的启动时密钥注入，适合加密存储/TPM 集成场景
- ✅ 对服务器 BMC 刷新、固件升级后的重启流程有潜在优化空间

## 6️⃣ OSPM 2026 Day 3 报告：GPU 调度 + Profile-Guided Scheduling 🟡

**来源**: [LWN.net](https://lwn.net/) — 2026-06-26, corbet

OSPM (Power Management and Scheduling in the Linux Kernel) 峰会的第三天报告发布。

### 关键议题

| 议题 | 内容 |
|:-----|:------|
| **GPU Affinity Scheduling** | 调度器感知 GPU 亲和性，将进程调度到靠近其 GPU 的 CPU 核心，减少 PCIe 延迟 |
| **Profile-Guided Scheduling** | 通过运行时 profile 数据指导调度决策，类似 PGO (Profile-Guided Optimization) 的思路 |
| **QoS 多租户** | 更细粒度的服务质量保证机制在调度层的实现 |
| **Paravirtualization Scheduling** | 虚拟化环境中的调度优化 |

### 运维意义
- ✅ **GPU 亲和性调度**直接对标 K8s 的 NUMA-aware scheduling + device plugin 的协同优化
- ✅ Profile-Guided Scheduling 的思路可以类比到 MLOps 场景：基于历史训练任务的 profiling 数据优化调度
- ✅ 多租户 QoS 对 AI 推理服务的 SLO 保障有重要参考价值
