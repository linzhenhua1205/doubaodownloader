### 1️⃣ 🔧 CNCF 项目 Arm64 自托管 CI Runner 基础设施

**来源**: [CNCF Blog · 2026-06-15](https://www.cncf.io/blog/2026/06/15/improving-arm64-support-in-cncf-projects-with-oci-credits/)
**相关项目**: CNCF Infra, OCI, GitHub Actions

> 核心内容已在 cloud-native/2026-06-15.md 中完整覆盖。此处仅提取运维运营视角。

#### 运维视角

CNCF Infra 团队将 **OCI Arm64 实例** 配置为 **自托管 GitHub Actions Runner**，供 CNCF 项目使用的这套机制，为运维团队提供了 **多架构 CI 基础设施的可复用参考模式**：

| 运维要点 | 说明 |
|:---------|:------|
| **Runner 规模灵活** | 从轻量到任意大小的 Arm64 实例均可，按需申请 |
| **额度管理** | 默认 **$5,000/月** 的 OCI 信用额度上限，超出可申请例外 |
| **接入流程** | CNCF Service Desk 工单提交 → Infra 团队配置 → 在工作流中引用 Runner 名 |
| **运维支持** | CNCF Slack 上的 Infra 团队提供全程支持 |

#### 对运维团队的意义

| 场景 | 可复用的模式 |
|:-----|:-------------|
| 自托管 Runner 管理 | CNCF 的 OCI Runner 池模式（多项目共享、自助申请） |
| 多架构 CI 成本控制 | 信用额度 + 合理上限的治理模式 |
| 开源项目的 CI 基础设施 | 使用 OCI / 其他云提供商的信用额度计划 |

#### ⭐ 信号强度

⭐⭐⭐ — 对运行多架构 CI 管线的运维团队有实践参考价值。核心启示：Arm64 CI 已从"额外负担"变为"基础设施标配"。

### 2️⃣ 📊 K8s SIG Storage：Volume Health 可观测性 + 存储运维挑战

**来源**: [K8s Blog · 2026-06-15](https://kubernetes.io/blog/2026/06/15/sig-storage-spotlight-2026/)
**相关功能**: VolumeGroupSnapshot (GA), CSI Changed Block Tracking (Beta), Volume Health (规划中)

> 核心内容已在 cloud-native/2026-06-15.md 中完整覆盖。此处仅提取运维运营视角。

#### 运维洞察：三大存储运维挑战

SIG Storage 联合主席 Xing Yang 在接受采访时明确了 **K8s 有状态工作负载的三大运维挑战**：

| 挑战 | 运维含义 | 当前可用的运维手段 |
|:-----|:---------|:------------------|
| **数据引力** | 节点故障时本地存储 Pod 被卡住 | Volume Health 监控 → 自动化恢复决策（规划中） |
| **Day 2 复杂度** | 数据库 Schema 升级、引擎补丁、集群升级 | CSI 快照 + VolumeGroupSnapshot 备份 + CBT 增量备份 |
| **数据移动性** | 跨存储层/可用区/集群迁移 | VolumeAttributesClass（动态 IOPS 调优）+ Mutable PV Affinity（Alpha） |

#### Volume Health 特性预览

这是 SIG Storage 正在开发的存储可观测性特性：

| 能力 | 状态 |
|:-----|:------|
| 存储驱动和 K8s 控制面报告 PV 健康状态 | 当前：非持久化 Events |
| 自动化修复决策 | 未来目标（正在调研增强方案） |

**运维启示**：目前 PV 健康信息仅通过 K8s Events 报告（非持久化）。当 Volume Health 达到可观测性级别时，将为 **Prometheus Exporter + Persistent Volume 告警规则**提供新的数据源。

#### 存储运维最佳实践提醒

以下是从 SIG Storage 访谈中提取的可操作建议：

| 场景 | 推荐实践 |
|:-----|:---------|
| 数据库备份 | 使用 **VolumeGroupSnapshot**（v1.36 GA）实现多卷原子级 crash-consistent 快照 |
| 增量备份 | 使用 **CSI Changed Block Tracking**（v1.36 Beta）减少数据传输量 |
| IOPS/吞吐量调优 | 使用 **VolumeAttributesClass**（v1.34 GA）动态调优，无需重建卷 |
| 存储健康监控 | 监控 K8s Events 中 Volume Health 相关信息（暂无 Prometheus 指标） |

#### ⭐ 信号强度

⭐⭐⭐ — 对运行有状态工作负载的平台工程/SRE 团队有价值。Volume Health 特性是 K8s 存储可观测性的重要演进方向。
