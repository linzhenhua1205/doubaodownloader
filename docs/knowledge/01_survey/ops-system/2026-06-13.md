## 🔥 重点信号

### 1️⃣ 🔍 OTel GenAI 可观测性：从 LLM 调用到生产可观测（May 14）

**来源**: [OTel Blog · 2026-05-14](https://opentelemetry.io/blog/2026/genai-observability/)
**作者**: James Newton-King（Microsoft）
**项目**: OpenTelemetry GenAI Semantic Conventions

#### 为什么 GenAI 可观测性不同

传统可观测性追踪的是"请求→服务→响应"的确定性路径。AI Agent 的执行路径却是：

```
用户问了一个简单的提问 → 45秒后回答
到底是模型慢？工具调用超时？重试循环？
```

OTel GenAI 语义约定标准化了 GenAI 操作的记录方式——模型名称、输入/输出 token 数、以及在启用时的 Prompt 和 Tool 调用完整内容。

#### 三款 AI 编程工具的可观测性支持

| 工具 | OTel 支持 |
|:-----|:----------|
| **VS Code Copilot** | 支持 traces, metrics, events，每步 Agent 交互均可观测 |
| **OpenAI Codex** | 导出自定义结构化日志 + OTel 指标（API 请求、工具调用、会话） |
| **Claude Code** | 指标和日志事件通过 OTel 导出，trace 支持 beta |

> ⚠️ **对于运维团队**：这意味着 AI 编程工具不再是黑盒——你可以追踪 Copilot 的每次 LLM 调用、工具调用链、token 消耗。当团队 AI 编程工具支出增长时，这些数据对于**成本归因和性能调优**至关重要。

#### 配置方法（Copilot 示例）

```json
// settings.json
"github.copilot.chat.otel.enabled": true,
"github.copilot.chat.otel.captureContent": true, // 默认不捕获内容（隐私安全）
"github.copilot.chat.otel.otlpEndpoint": "http://localhost:4318"
```

**安全设计**：默认**不捕获 Prompt 内容**（仅 metadata：模型名、token 数、时长）。内容捕获需显式启用。

#### 生产级预览展示

文章使用 **Aspire Dashboard**（开源 Docker 容器）作为后端接收 OTLP 数据：

```bash
docker run --rm -p 18888:18888 -p 4317:18889 -p 4318:18890 -d \
  --name aspire-dashboard \
  -e ASPIRE_DASHBOARD_UNSECURED_ALLOW_ANONYMOUS=true \
  mcr.microsoft.com/dotnet/aspire-dashboard:latest
```

**追踪视图**展示：
- `invoke_agent` span → 子 span `chat`（每次 LLM 调用）+ `execute_tool`（每次工具调用）
- 属性：`gen_ai.request.model`（如 gpt-4o）、`gen_ai.usage.input_tokens`、`gen_ai.usage.output_tokens`、`gen_ai.response.finish_reasons`
- **GenAI 可视化器**渲染 Chat 视图：系统 Prompt、用户消息、助手响应、工具调用参数和结果

**指标视图**展示：
- `gen_ai.client.operation.duration` — LLM 调用延迟直方图，可按 model 过滤
- `gen_ai.client.token.usage` — token 消耗直方图，可按方向（input/output）过滤

#### 运维意义

| 场景 | OTel GenAI 提供 |
|:-----|:----------------|
| Cost Attribution | 每个 LLM 调用的 token 消耗 → 估算每次 Agent 会话成本 |
| 延迟诊断 | 辨别是模型慢还是工具调用慢 |
| Token Hungry 检测 | 发现那些消耗异常多 token 的 Prompt 模式 |
| 模型对比 | 按 model 过滤延迟和 token 消耗 |
| 合规审计 | 完整的事件链（需启用内容捕获） |

#### ⭐ 信号强度

⭐⭐⭐⭐ — 这不是概念文章，而是**可直接复现的操作指南**。对正在引入 AI 编程工具（Copilot/Codex/Claude Code）的研发团队，建立 AI 可观测性应该成为基础设施的一部分。特别是成本归因——顶级 AI 企业每月每员工 AI 支出 $7,500，**没有可观测性就无法管理这个支出**。

### 🛡️ 维度一：供应链安全（CI/CD Pipeline Security）

| 日期 | 项目 | 内容 | 关键措施 |
|:-----|:-----|:-----|:---------|
| Jun 12 | Cilium CI/CD Part 2 | **依赖锁定** | SHA Pin + Vendoring + 5天 Cooldown + 静态分析 |
| Jun 4 | Cilium CI/CD Part 1 | **访问控制** | Ariane 机器人 + 双阶段 Checkout + CODEOWNERS |

**关键运维模式**：供应链安全的四层防线：
1. **谁触发** → Ariane + org-member 白名单

3. **依赖拉取什么** → SHA-pin + Vendoring
4. **CI 凭据保护** → 最小权限 + OIDC
