> **什么是 DATER**: 首次系统化定义数据架构的技术需求维度，统一评估 6 种现代架构

- **6 种架构 + 评估维度**:
  | 架构 | 核心特征 | DATER 适用场景 |
  |:-----|:---------|:-------------|
  | Data Warehouse | 结构化, schema-on-write | 企业 BI 报表 |
  | (Semantic) Data Lake | schema-on-read, 原始存储 | 数据探索/科学 |
  | Data Lakehouse | ACID + 开放格式 + BI | **当前主流，覆盖最广** |

- **分析意义**: 为数据架构选型提供可对比的技术维度清单，支持研究人员和实践者在架构范式间导航

## 🔬 arXiv 论文精选

### ④ SQLConductor: 搜索到策略学习（2606.23537, Jun 22）

> **核心创新**: 将 Text-to-SQL 从"固定多阶段管线"升级为"搜索→策略蒸馏"范式，BIRD 73.2%

- **问题**: 现有固定管线（schema linking → decomposition → SQL gen）无法适应不同查询的复杂度差异
- **方案: SQLConductor**:
  1. **动作空间定义**: Text-to-SQL 子任务编码为专门动作（schema linking, decomposition, SQL gen, verification…）
  2. **MCTS 搜索**: 探索候选工作流的执行效果，通过稳定性估计识别健壮的监督信号
  3. **策略蒸馏**: 稳定性加权 SFT + 课程 RL → 可部署的 step-wise 编排策略
- **结果**: BIRD-Dev **73.2%**（冻结的大动作模型协调），超越直接训练同等或更大模型的方法
- **意义**: 开启 Text-to-SQL 的"Search-to-Policy"新范式——离线搜索最优工作流，在线轻量执行，用更少计算资源达成更强效果

### ⑤ CAPER: Clause-Aligned Process Reward Model（2606.03327, Jun 2）

> **核心创新**: 自动对 SQL AST 做反事实干预推导 clause 级错误根因，训练轻量 Clause-PRM

- **问题**: query-level 执行正确率无法定位错误 clause；token-level 监督不适合 SQL（等价的 SQL 可能形不同）
- **方案 — CAPER**:
  1. **自动推导 clause 级监督**: 对 SQL AST 做 counterfactual intervention（替换/移除 clause），观察执行结果变化
  2. **轻量 Clause-PRM**: CAPER-9B 在 clause 边界给出细粒度反馈
  3. **策略优化**: clause-level 奖励用于 policy optimization + candidate verification
- **结果**:
  - GPT-5.4 上提升 **15.3%** 相对 EX
  - 错误定位能力：84.53% 准确率，90.60% MRR
- **意义**: 与 EXPO-SQL（同日发布昨天已跟踪）形成完美互补——EXPO-SQL 用执行结果定位错误 clause，CAPER 用反事实干预自动标注，两条路线共同推动 Text-to-SQL 进入 clause 级时代

### ⑥ UniQL: 跨 16 种 SQL 方言的统一基准（2606.08018, Jun 6）

> **核心创新**: 首次构建跨 SQL 方言的 Text-to-SQL 基准，覆盖 16 种方言 24,544 条查询

- **问题**: 现有基准（Spider, BIRD）全部基于 SQLite，无法评估模型对异构 SQL 方言的泛化能力
- **方案 — UniQL**:
  - 1,534 条自然语言问题 × 16 种 SQL 方言 = **24,544 条方言特定查询**
  - 所有方言共享相同意图、对齐的 schema 和数据库内容
  - 构建管线：数据库迁移 → SQL 翻译 → 执行验证 → 规则汇总 → 人工校验
- **关键发现**: 当前模型远非"方言通用"，SQLite 上的成功向其他方言迁移有限
- **意义**: 填补了 Text-to-SQL 领域缺失的关键基准，为未来"跨数据库系统"的通用 SQL 生成提供评测基础

| **Text-to-SQL 精细化** | Clause 级信号成为新主线 | CAPER (clause PRM, +15.3%) + EXPO-SQL (clause RL) + Progress-SQL (ODT 结构 profile) 三线并进 |
