## 🏢 Databricks — Data + AI Summit 2026 Day 2 (6/16)

### ⭐ [1] Lakehouse//RT: 湖仓原生实时数仓（最大发布）

> **来源**: [Databricks Blog](https://www.databricks.com/blog/introducing-lakehousert-real-time-performance-unified-lakehouse) | 6/16/2026 | Nong Li, Shoumik Palkar, Shant Hovsepian, Mostafa Mokhtar, Reynold Xin

**核心**: 基于全新实时引擎 **Reyden**，在湖仓上直接实现毫秒级查询响应，**无需数据搬迁**、无需独立 serving 层。

**性能数据**:
- 对比独立实时 serving 层：**最高 16× 性能提升**
- 小数据集响应：**~10ms**；大数据集：**sub-100ms**
- 标准分析基准测试：**12,000 QPS** 下保持 **sub-100ms 延迟**
- TPCDS 复杂查询：一个竞品 **25× 更慢**，大规模下完全失败
- TPCH 多表 JOIN：随数据量增长延迟保持低平，竞品在大数据规模下 2/3 无法运行

**客户实测**:
| 客户 | 场景 | 加速效果 |
|:-----|:-----|:---------|
| **SES**（太空通信） | 数十亿行遥测数据 | **20×** 更快，无需独立 serving 层 |
| **Bally's**（游戏/博彩） | 60TB Delta 表 | **7× faster**, sub-second |
| **Halcyon**（安全） | 百万级端点 | **4×** 更快 |
| **Enverus**（能源数据） | 2.7PB, 3.5 亿+ 记录 | **最高 100×** 快于专用实时引擎 |
| **Equilibrium Energy** | AI Agent 交易 | **3.6×** 中位延迟 vs SQL Serverless |
| **PointClickCare**（医疗） | 平均 **> 1/3 更快**, 部分 **10×** |
| **Cisco**（威胁查询） | **5×** 响应时间提升 |

**架构创新**: AUTO sizing + 增量自动扩展（按节点增减而非整机副本），Unity Catalog 统一治理。

**定价**: Beta 阶段，至 2027 年 1 月 **30% 折扣**。

### ⭐ [2] Lakeflow: Agentic Data Engineering 统一平台

> **来源**: [Databricks Blog](https://www.databricks.com/blog/lakeflow-new-era-agentic-data-engineering) | 6/16/2026 | Bilal Aslam, Ray Zhu 等

**六大新品**:

1. **Genie Code + Lakeflow Designer (GA)** — 可视化/自然语言驱动的 Agentic 流水线开发，拖拽式画布+自然语言提示，可直接生成生产级 Spark Declarative Pipeline
2. **Genie ZeroOps** — 内置 AI Agent 自动监控数据/AI 资产，自动根因分析(RCA)，在 Unity Catalog 沙箱中验证修复方案，人工确认后执行
3. **Lakeflow Connect — 100+ 原生连接器**:
   - 知识管理: Jira(Beta)·GitHub(Beta)·Confluence(GA)·SharePoint(GA)·Google Drive(Beta)·Outlook(Beta)
   - 营销技术: Meta Ads(Beta)·TikTok Ads(Beta)·Google Ads(Beta)·HubSpot(GA)
   - IT/安全: 数据库查询捕获+Lakehouse Federation 源(GA)
   - **Community Connectors (Beta)**: 开源社区连接器框架
4. **Zerobus Ingest** — Kafka-free 高吞吐流式写入:
   - 近实时 **< 5s 写入**，**> 10GB/s 单表**吞吐
   - **Kafka-Compatible API (Beta)**: 无需改代码，Kafka 生产者直推 Databricks
   - gRPC/REST API (GA), OpenTelemetry (Public Preview)
   - **Meta** 实测: "将端到端流水线延迟压缩到 1 分钟以内"
5. **Spark Declarative Pipelines Real-Time Mode (Public Preview)** — **5ms** 端到端延迟，无需 Flink 等独立引擎
   - Coinbase, DraftKings, MakeMyTrip 已在使用
6. **Lakeflow Jobs (50+ 集成)** — 数据就绪驱动编排，而非 Cron 推测
   - 外部编排: 可触发 Snowflake 任务/自定义 REST API/Slack/PagerDuty
   - **40+ 操作示例** GitHub 开源

### ⭐ [3] Full-Text Search Indexes on Databricks (Beta)

> **来源**: [Databricks Blog](https://www.databricks.com/blog/accelerate-search-queries-full-text-search-indexes-databricks) | 6/16/2026 | Yu Xu, Yingyi Bu, Ivan Vezilić

**核心**: 一行 SQL 创建全文搜索索引，自动加速 substring/keyword 查询，无需 query hint。

**性能**: 安全/合规团队在 **PB 级表**上实测：**100×+ 加速** substring 搜索

**架构关键点**:
- 索引存储在基表外，异步维护，**零写入性能影响**
- 被动索引（落后基表）时仍保证查询正确性
- 支持 **Delta 和 Iceberg** 两种格式
- **Liquid Clustering + 全文搜索索引互补**: 前者优化列值过滤，后者优化列内文本搜索
- DBR 18.2 Beta → 18.3 计划: 自动索引维护(Predictive Optimization)

### ⭐ [4] Lakebase Search: Agent-Native 检索（Postgres）

> **来源**: [Databricks Blog](https://www.databricks.com/blog/announcing-lakebase-search-agent-native-retrieval-built-lakebase-postgres) | 6/16/2026 | Pranav Aurora, Zhou Sun, Jinjing Zhou

**核心**: 混合向量+全文检索，天然嵌入 Lakebase Postgres。Agent 操作 Lakebase 数据库的数量是 **人类的 4 倍**。

**两个新 Postgres 扩展**:

| 扩展 | 技术 | 优势 |
|:-----|:-----|:-----|
| **lakebase_vector** | RaBitQ 量化 + 分层存储 | **32× 压缩**, 10 亿向量规模, 1 亿向量索引从 300GB→**< 10GB** RAM |
| **lakebase_text** | 原生 BM25 排名 | 无需 GIN 索引内存膨胀，与向量搜索同 SQL 混合执行 |

**经济性对比 (Per TB/month, cloud list price)**:

| 存储层级 | 成本 |
|:---------|:-----|
| RAM | ~$3,000 / TB / month |
| Local NVMe (cache) | ~$100 / TB / month |
| Object storage (cold tail) | **~$20 / TB / month** |

**LAION-100M (1 亿条 768 维向量) 基准测试**:
- pgvector HNSW: 需要 512GB RAM (64 CPU) 实例, **~40h** 构建索引, 冷启动数分钟
- lakebase_vector: 192GB (24 CPU) 实例, **1.5h** 构建, 冷缓存首次查询仅 **1.13s**
- Recall@10: 0.95 时 P99 53ms, 0.92 时 P99 14ms, 最高 142 QPS

> **来源**: arXiv:2606.14201 | 6/12/2026 | Chao Deng, Ju Fan 等

## 📋 趋势判断

1. **Lakehouse//RT 终结"湖+专用实时层"的双架构时代** — Reyden 引擎在 12,000 QPS + TPCDS 复杂查询下保持 sub-100ms，7 个客户实测验证。这是自 Delta Lake 发布以来 Databricks 在湖仓架构上最重要的产品发布。

2. **Agentic Data Engineering 进入生产阶段** — Lakeflow 的 5 大组件（Genie Code/Designer/ZeroOps/Connect/Jobs）标志着数据工程从"人写 Pipeline"进入"Agent 编排管线"时代，Genie ZeroOps 的自动 RCA + 沙箱修复是大规模数据运维的关键突破。

3. **Postgres 不可逆地进入 Lakehouse 生态** — Lakebase Search 的 32× 向量压缩、BM25 内置、冷尾 20 美元/TB 的成本模型，正在改变"传统搜索需要内存绑定"的基本经济学。

4. **Text-to-SQL 进入工程化博弈阶段** — TAHOE(提示工程化)/TACO(开放域)/Robust Active Learning(少样本精选) 三篇论文代表从"模型能力"转向"数据/提示工程"的迁移趋势。

5. **安全成为 Data Agent 瓶颈** — Data Agents Under Attack 系统研究了 8 个特有风险，Sophrosyne 提出了"探索需要监管"的论点—Agent 过度探索导致 4.6× 不必要的查询。
