## 三、Apache Spark 4.0.0 正式发布 🚀

### 发布概览
- **Spark 4.0.0**：4.x 系列的首次发布，**5,100+ tickets**，**390+ 贡献者**
- 同时已有 **4.1.2** (May 21, 2026) 和 **4.2.0 预览版**

### 关键新特性

| 类别 | 亮点 |
|:-----|:------|
| **SQL** | VARIANT 类型、Pipe 语法（SQL）、Session Variables、UDTFs、字符串 Collation、XML 内建数据源 |
| **PySpark** | 原生 Plotting API、Python Data Source API、Python UDTFs、UDF 统一性能分析 |
| **Spark Connect** | 轻量 Python 客户端（1.5MB）、Spark ML on Connect、Swift 客户端 |
| **Structured Streaming** | Arbitrary State API v2、State Data Source Reader、TransformWithState Python 支持 |
| **基础设施** | Scala 2.13/JDK 17 默认、K8s Operator、ANSI SQL 默认、Hive 4.0 元数据 |
| **ML** | Target Encoding、嵌套输入列支持 |

### 重要架构变更
- ❌ 移除：Scala 2.12、JDK 8/11、Mesos、SparkR（将废弃）
- ❌ 移除：AWS SDK bundle（Hadoop AWS 替代）
- ⚠️ 默认变更：ANSI SQL、RocksDB Shuffle、Prometheus UI metrics、结构化日志框架
- ✅ 新增：Zstd 默认 ORC 压缩、AES-GCM RPC 加密、并行压缩（LZF/Zstd）

## 四、Apache Flink 2.0 发布 🚀

### 发布概览
- **Flink 2.0.0**：从 1.x 到 2.x 的重大升级
- 5 月 26 日博客发布

### 关键新特性

| 类别 | 亮点 |
|:-----|:------|
| **解耦状态管理** | 远程存储作为主存储、异步执行模型、K8s 云原生 |
| **Materialized Table** | 声明式流批一体、Schema 在线演进、Paimon 集成 |
| **DataStream V2 API** | 全新流处理 API（实验性）、底层构建块（State/Time/Watermark） |
| **自适应批处理** | 自适应 Broadcast Join、自动 Join 倾斜优化 |
| **SQL 增强** | QUALIFY 子句、C 风格转义字符串、Application Mode SQL Gateway |
| **基础设施** | Java 17 默认/21 支持、Kryo 5.6、config.yaml 替代 flink-conf.yaml |

### 重要 Breaking Changes ⚠️
- **彻底移除**: DataSet API、Scala DataStream/DataSet API、SourceFunction/SinkFunction/SinkV1
- **移除**: TableSource/TableSink/TableSchema、Per-job 模式、flink-conf.yaml
- **不保证**: 1.x ↔ 2.x 状态兼容性
- 连接器适配计划：Kafka/Paimon/JDBC/ElasticSearch 新版本随 2.0.0 同时发布

## 五、Text-to-SQL 最新研究（arXiv 爆发）⚠️【深度跟踪】

6 月 1-2 日 arXiv 上 Text-to-SQL 论文集中爆发（129 篇搜索结果）：

### 重要论文

| 论文 | 核心贡献 | 日期 |
|:-----|:---------|:-----|
| **EntSQL** | 企业级 Text-to-SQL 基准测试（1066 条，5 个业务领域），最佳仅 15.9% | Jun 2 |
| **CAPER** | Clause-PRM 从句级奖励建模，GPT-5.4 上 EX 相对提升 15.3% | Jun 2 |
| **SIRIUS-SQL** | 多候选 + 执行反馈锚定，BIRD 75.88% / Spider 91.20% | May 31 |
| **MERIT** | 多视角长时记忆（全局策略+局部决策），BIRD-Interact 和 Spider2-Snow 优于静态检索 | May 30 |
| **DivSkill-SQL** | 互补技能集成，Spider2-Lite 提升 +11.1pp（Snowflake）/ +8.3pp（BigQuery） | May 20 |
| **BADGER** | 企业级评估框架（Hybrid-EX）Cohen's κ=0.717 | Jun 1 |
| **EviLink** | 多假设 Schema Linking，Spider2-Snow 字段召回 90.15% | May 28 |
| **LLM-friendly DB Design** | Schema 变换优化 Text-to-SQL，提升 4.2% | Jun 2 |
| **LogRouter** | 两级成本感知路由，日志 QA 延迟降 55% | May 18 |
| **EnterpriseMem-Bench** | 多轮 Text-to-SQL 基准（300 会话/1400 轮），无记忆第 3 轮准确率归零 | May 25 |

### 关键趋势
1. **Text-to-SQL 评估重心转向企业级和 Agentic**：EntSQL / BADGER / EnterpriseMem-Bench
2. **多候选 + 执行反馈 + Clause-PRM 成为新范式**：SIRIUS-SQL / CAPER / DivSkill-SQL
3. **长时记忆在对话式交互中不可或缺**：MERIT / EnterpriseMem-Bench（无记忆第 3 轮崩盘）
4. **Schema Linking 仍是最关键瓶颈**：EviLink / LLM-friendly DB Design
5. **生产级多模型路由开始实用化**：LogRouter
