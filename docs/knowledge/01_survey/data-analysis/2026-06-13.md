### 1️⃣ ⭐ Zerobus Ingest 正式 GA：Petabyte 级 Serverless 流式写入 (Jun 11)

**来源**: [Databricks Blog - Ingesting the Milky Way](https://www.databricks.com/blog/ingesting-milky-way-petabyte-scale-zerobus-ingest)
**作者**: Aleksandar Tomić, Victoria Bukta 等 (Databricks Engineering)

Zerobus Ingest 是 Databricks 全托管 Serverless 流式写入服务，直接推送到 Delta Lake (Unity Catalog)。

**性能数据**:
| 指标 | 数值 |
|:-----|:-----|
| 吞吐量 | **12 GB/s** 单表持续写入 |
| 总行数 | **1.04 万亿行** / 24h |
| 并发流数 | **2,048** 个同时连接 |
| 数据源 | NASA NEOWISE 数据集 (1PB / 11年 / 2,000亿点) |
| 延迟 | 异步 ack 循环，接收确认即释放缓冲 |

**核心架构创新**:
- **动态分区 (Dynamic Partitioning)** — 流连接级别保证有序，而非分区级别。Hot routing 实现真正自动伸缩，pod 可增可减
- **ZeroParser (Rust 零拷贝 Protobuf 解码器)** — 开源 SDK 组件，单 CPU 核 ~1 GB/s protobuf 解析，超越 codegen 方案的动态灵活性
- **Write-Ahead Log (WAL)** — 低延迟持久的 gRPC 双向流 ack 机制
- **Delta Kernel Rust** — 底层 Delta 写入引擎

**意义**: 标志着 **"Kafka 不再是 Lakehouse 的必经管道"** — Zerobus 可直接替代 Kafka 作为数据入湖层，无 broker/partition 管理。

### 2️⃣ ⭐ Genie + Lakehouse Federation：自然语言查询所有数据，无需迁移 (Jun 12)

**来源**: [Databricks Blog - Talk to all your data, wherever it lives](https://www.databricks.com/blog/talk-all-your-data-wherever-it-lives)
**作者**: John Spencer

**核心内容**: Lakehouse Federation 连接 20+ 外部数据源，在 Unity Catalog 中统一治理，Genie 直接自然语言查询。

**关键能力**:
- **支持的源**: AWS Glue, Snowflake, Oracle, BigQuery, PostgreSQL, Redshift, MySQL
- **Foreign Table Comments**: 源表描述/列注释自动同步到 Unity Catalog，AI 即可理解 schema
- **Unity Catalog Semantics**: 在 Federated 表上直接定义 Metric Views（如 ROI=impressions/spend），Genie 查询时自动解析
- **SET MANAGED 一键升迁**: Foreign → Managed table 转换，**50%+ 成本节省，20× 查询加速**

**意义**: 解决企业"Big Bang 迁移"的死锁 — 无需迁移即可通过 AI 查询所有数据，零拷贝联邦治理。

## 📄 arXiv cs.DB (6月8-10日提交)

### TAHOE: Text-to-SQL 提示优化即数据管理 (2606.12387)

**作者**: Zhiyi Chen, Jie Song, Peng Li
**核心方法**: 将 Text-to-SQL 提示优化建模为动态数据管理问题，通过 Error-Hint Learning 构建 Hint Bank（Syntax Hints + Semantic Hints），Strategy Layer 处理冲突用户意图。

**关键性能数据**:
| 指标 | 提升效果 |
|:-----|:---------|
| Spider 2.0-Snow pass rate | **61.95% → 79.42%** (+17.47pp) |
| Pass-at-4 | **72.57% → 87.61%** |
| Snowflake 语法通过率 | 100% |
| 平均 critic 轮次 | 2.79 → **0.12** (23× 减少) |
| 跨模型迁移 (Doubao-2.0-lite) | +19.7pp pass-rate gain |

**意义**: Text-to-SQL 进入 "优化即数据管理" 新范式 — 把提示词看作可积累、可检索、可复用的数据资产。

### Bespoke-Card: Agent 按需合成基数估计器 (2606.09361)

**作者**: Johannes Wehrstein, Anton Winter, Timo Eckmann, Carsten Binnig (TU Darmstadt)
**投稿**: AIDB@VLDB'26

**核心方法**: LLM Agent 驱动 (planning agent → coding agent → validator) 按 workload 特征合成专用基数估计器代码。

**关键性能数据**:
| 指标 | 效果 |
|:-----|:-------|
| PostgreSQL JOB 总运行时间 | **-33%** |
| 所有子计划中位数 q-error | **-41%** |
| 合成成本 | **<$10**, <1h |
| 方法论 | 结构化 q-error 反馈, 回归分析, 异常子计划, 课程学习 (filter→join→full-subplan) |

**意义**: 打开基数估计的第三条道路 — 不是通用统计 (classical)，不是学习模型 (learned)，而是 **按需合成代码 (agent-synthesized)**。

---

### Hellerstein: Determination Provenance — 从歧义到代数 (2606.10270)

**作者**: Joseph M. Hellerstein (UC Berkeley)
**核心**: 经典数据起源 (provenance) 只能解释 "结果是如何推导的"，但不能回答 "为何是这个结果而非另一个"。Determination Provenance 引入 **support (支撑集)** → commutative semiring，**filtration (过滤)** → 查询相关深度。实例化到事务隔离和 Datalog¬ 两个场景。

**意义**: 为并发事务、逻辑程序等多结果系统的 **根源诊断** 提供代数基础。

---

### Flash-GMM: 高效软聚类 Triton Kernel (2606.10896)

**作者**: Gal Bloch, Ariel Gera, Matan Orbach, Ohad Eytan, Assaf Toledo (IBM Research)
**核心**: Fused Triton Kernel 消除 GPU 显存中全责任矩阵 (responsibility matrix) 的物化。

**性能数据**: **20×** 加速，支持 **100×** 更大数据集。集成到 IVF 近似最近邻搜索中，固定召回率下减少 **1.7×** 距离计算量。

## 🎯 关键趋势与洞察

### 趋势 1: Serverless Streaming 进入 Lakehouse 原生时代
Zerobus Ingest 12 GB/s + 2,048 并发推翻了 "Lakehouse 不适合高吞吐流式" 的旧认知。湖仓本身就是最佳流式目的地。

### 趋势 2: 语义层 = AI 数据平台的核心战场
Databricks (Unity Catalog Semantics + DAX transpiler + Metric Views) vs Snowflake (Jedify 投资 + CoWork Cortex Sense) 在语义层方向高度一致。语义治理决定 AI 回答质量。

### 趋势 3: Text-to-SQL 进入细化监督阶段
TAHOE (hint bank)、Robust Active Learning (异方差互信息) 等论文表明：Text-to-SQL 从 "模型能力比拼" 转向 **"训练数据/提示词/反馈数据的精细管理"**。

### 趋势 4: AI Agent 作为数据库研究中新的第一公民
InquiTree (Agent 科学推理评估)、Bespoke-Card (Agent 合成基数估计器)、Determination Provenance (Agent 多结果诊断) — Agent 不再只是用例，而是 DB 理论和系统设计的新的核心要素。
