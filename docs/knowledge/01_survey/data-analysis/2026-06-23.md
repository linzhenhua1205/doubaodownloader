## 🏢 厂商发布

### Databricks — Data + AI Summit 2026 会后聚焦

**① Lakehouse//RT: 实时湖仓数据仓库（⭐ 核心发布）**
- 基于全新 **Reyden 引擎**，在湖仓上直接提供毫秒级查询响应，无需数据搬迁
- **预览用户实测**: 相比实时 serving 层性能提升 **最高 16×**；小数据集响应低至 **10ms**，大数据集 **<100ms**
- **基准测试**：12,000 QPS 下 <100ms 延迟（TPC-H）；复杂查询（TPC-DS）比竞品快 **25×**
- **三大竞争壁垒**: (1) 负载下保持平坦延迟（竞品随 QPS 升高崩溃）(2) 大数据集不降速（2/3 竞品1TB级失败）(3) 复杂查询不退化（竞品 window function 失败）
- **AUTO sizing + 增量伸缩**：按单个节点增删，非传统 2×副本翻倍
- 客户Meta/Enverus/SES/Cisco/Bally's 等验证；**Beta 阶段**，30%折扣至2027年1月
- **趋势判断**: 湖仓一体从"批+流"进入实时层 —— "separate serving layer" 模式终结的开始

**② Genie Code: 全页命令中心 + ML 工程化（⭐ Agent 化数据分析）**
- **全页命令中心**: 多线程管理、状态跟踪、指令/skill/connector 一站式可见
- **Genie Code for ML**: 接入 MLflow(实验/模型/指标)、Model Serving(端点健康/诊断)、AI Runtime(GPU 感知)，90分钟完成从原始数据到生产 ML 工作流
- **Scheduled Tasks (即将上线)**: Genie Code 可定时自主运行任务，用户回来后审查结果
- **Genie ZeroOps**: 扩展到生产运维闭环——监视→调查→修复
- Genie 产品线 **90% 客户使用**、**10× 增长**

**③ Genie One/Agents/Ontology 产品家族**
- Genie One: 企业内部上下文感知的 AI 同事
- Genie Agents: 可被共享的 AI 体验
- Genie Ontology: 学习团队构建特征/训练模型/评估候选的模式

## 📄 arXiv 论文 (cs.DB · 6/23 + 6/19)

### Text-to-SQL / NL2SQL

**⑤ SQLConductor: Search-to-Policy Learning for Text-to-SQL** (2606.23537, 6/22)
- 将 Text-to-SQL 编排建模为 **逐步策略学习** 问题
- **MCTS 搜索候选工作流 → Stability-weighted SFT → Curriculum RL 强化**
- **BIRD-Dev 73.2% EX**，超越直接训练大模型的方案
- 核心创新: 将 offline 工作流搜索转化为 inference 时可部署的逐步编排策略

**⑥ CoeusBI: Baidu 的 LLM 驱动的 BI 系统** (2606.15384, VLDB 2026)
- **Dual-Agent Architecture**: offline View Generation Agent + online Routing Agent
- **Hierarchical Schema Linking**: 通过视图向量检索处理超宽 schema
- 离线 Agent 将复杂 JOIN 自动转为单视图查询，消除手动语义建模的需求
- 百度日活数千用户，多业务线部署

### Data Agent / Agentic Database

**⑦ AgenticDB: Agentic Performance Reconfiguration** (2606.20318, 6/18)
- 首个系统性 **Agent 驱动数据库重配置框架**
- DBMS 级 + OS 级双层动作空间(内存/调度/内核参数等)
- MySQL/PostgreSQL 上 YCSB/Sysbench/TPC-H 评测
- **平均超越最强基线 118.1%**，时间到最优减少 22.6%

### 实时 / 内存感知引擎

**⑧ REMOP: 远程内存感知算子优化** (2606.19576, 6/17)
- 针对 **远程/分解内存** 场景优化 DuckDB 算子
- 引入 transfer rounds 到延迟成本模型
- **transfer rounds 减少 97%**，spill-heavy 场景算子耗时降低 48%
- TPC-H spill 查询平均降低 22.7%，TPC-DS 降低 26.4%

### 数据智能与 AI 分析

**⑨ "The Table Says Otherwise"** (2606.23667, 6/23)
- 用 **反事实关系数据** 测试 LLM 的表格推理能力
- 评估 LLM 是否真正理解关系数据结构的约束

**⑩ DataMagic: 表格数据→数据洞察视频** (2606.20388, VLDB 2026)
- 将表格数据自动转化为数据洞察视频，覆盖可视化+叙事+排版

## 📈 趋势判断

1. **Lakehouse//RT 标志湖仓实时化进程关键拐点**: 不再需要独立 serving 层，一个引擎覆盖 10ms~TB 级分析+12K QPS 并发，将重构实时分析市场格局

2. **Agent 数据库双路径成型**: SQLConductor (Text-to-SQL 策略编排) + AgenticDB (Agent 驱动数据库配置)，分别覆盖"如何查询"和"如何调优"两个维度，数据库 Agent 化正从论文走向工程

3. **Snowflake Agentic Enterprise 全栈覆盖**: 从 Agent Discovery 规范→实时管道→Postgres AI→安全→治理，Snowflake 在 Data+AI Summit 后以"Agentic"统一叙事，展现平台级竞争姿态

4. **远程分解内存成为数据库优化新前线**: REMOP 在 DuckDB 上的 97% transfer round 减少，预示计算-内存分离架构下算子优化进入精细化运营阶段
