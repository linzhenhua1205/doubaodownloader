## 🏢 厂商发布

### ① Databricks Lakehouse//RT: Reyden 实时引擎（Jun 16, 2026）

> **重磅**: 湖仓统一实时分析的里程碑。不再需要单独的 serving layer。

- **Reyden 引擎**: 全新构建的实时数仓引擎，直接在 Delta/Iceberg 开放格式上提供毫秒级查询
- **核心能力**: 10ms（小数据集）、sub-100ms（大数据集）、**12,000 QPS** 时仍保持 sub-100ms 延迟
- **对比优势**: 预览用户看到 **16× 性能提升** vs. 传统实时 serving layer
- **消除架构碎片**: 不再需要将数据复制到专有的 serving layer，避免 3 倍成本（存储重复 + 治理副本 + 工程维护）
- **AUTO sizing + 增量自动伸缩**: 按节点增减而非整份副本扩缩，精确匹配负载
- **预览客户实测**:
  - **Meta Enterprise**: 典型查询返回 **10s of ms**，湖上数据无需独立系统
  - **SES (卫星通信)**: **20× 更快**，20 亿行遥测数据毫秒级响应
  - **Enverus (能源数据)**: 部分查询 **100× 更快**，可折叠独立分析栈
  - **Magnite (广告科技)**: 数百 QPS，**sub-200ms** 一致性
  - **Bally's (博彩)**: **7× 更快**，sub-second 性能
  - **Equilibrium Energy (能源交易)**: AI Agent+交易员实时协同，**3.6× 更快中位延迟**
- **价格**: Beta 阶段，2027 年 1 月前 **30% 折扣**
- **关键意义**: 统一实时分析与湖仓存储治理，是湖仓一体架构的**终极愿景兑现**

## 📝 arXiv 论文精选

### ⑦ Data Agents Under Attack: 安全漏洞系统研究（2606.08661, Jun 7）

> **首个系统化的 Data Agent 安全研究，10 条原则级别的洞察之一**

- **背景**: Data Agent 整合了 LLM 推理 + 关系数据库 + 可执行分析工具 + 多步工作流编排
- **三层漏洞框架**: 识别出 8 个 Data Agent 特有风险，分布在**解释层、执行层、策略层**
- **攻击分类**: 3 个攻击目标 × 7 种策略 × 14 种技术
  - 目标: 数据窃取 / 查询劫持 / Agent 行为操纵
  - 策略: Prompt 注入 / Schema 误导 / 工具滥用 / 结果篡改等
- **评测**: 6 个系统实测（4 个开源 Data Agent + 2 个商业云分析服务）
- **结论**: 当前系统普遍存在严重安全缺陷

### ⑧ SQLConductor: MCTS 搜索→策略学习（2606.23537, Jun 22）

- **核心思想**: 将 Text-to-SQL 建模为**步骤级编排问题**，而非固定 pipeline
- **方法**: Monte Carlo Tree Search 探索候选工作流 → 稳定性估计识别鲁棒监督信号 → Stability-weighted SFT → Curriculum RL
- **结果**: **73.2% EX** on BIRD-Dev，使用紧凑编排策略协调冻结的大模型，优于直接训练同等或更大模型的方法
- **泛化**: 在 OOD 数据集上也表现良好

### ⑨ TAHOE: Hint 优化的工程落地（2606.12387, Jun 10）

- **核心**: 将 Text-to-SQL prompt 优化视为**动态数据管理问题**
- **Hint Bank**: 开发阶段编译器反馈 → Syntax Hints，执行/用户反馈 → Semantic Hints
- **Strategy Layer**: 建模冲突的用户意图为竞争策略，附带 Recency + 后验统计数据
- **结果 (Spider 2.0-Snow, GPT-5.5)**:
  - pass rate: **61.95% → 79.42%**
  - pass@4: **72.57% → 87.61%**
  - Snowflake 语法通过率: **100%**
  - 编译器反馈回合: **2.79 → 0.12**
- **Hint 迁移**: 同一个 Hint Bank 迁移到弱模型（Doubao-2.0-lite），pass rate 提升 **19.7 pp**

### ⑩ TACO: 开放域跨数据库 Text-to-SQL 基准（2606.14201, Jun 12）

- **定位**: 填补现有基准在开放域场景的空白（模糊问题、未指定数据库、跨库查询）
- **规模**: 1,500 条真实 + 13,000 条合成，覆盖交通、医疗、金融
- **基线 TACO-SQL**: Question Rewriting + Table Linking + Query Planning
- **发现**: 当前方法离人工编写 SQL 仍有显著差距
