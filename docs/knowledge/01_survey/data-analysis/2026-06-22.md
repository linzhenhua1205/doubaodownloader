## 📄 arXiv cs.DB 最新论文 (6/19 新提交 7 篇)

### ⭐ [5] AgenticDB: Agentic Performance Reconfiguration for Database Workloads

> **arXiv**: 2606.20318 | 作者: Xinyue Yang, Chaozheng Wang 等

**核心**: Agent 框架实现数据库工作负载的**自适应重配置**，不限于 DBMS knob 调优，而是扩展到 DBMS + OS 两层变更。

**突破**:
- 现有自动调优工具大多局限于 DBMS 参数迭代搜索，成本高、空间窄
- AgenticDB 实现 context-grounded harness：诊断瓶颈 → 探索 DBMS+OS 变更 → 安全执行 → 观察反馈 → 积累经验
- OS 级别动作空间：内核参数、内存分配、I/O 调度

**实验**: MySQL 和 PostgreSQL 上，YCSB/Sysbench/TPC-H
- 平均改进 **118.1%** over 最强基线
- 时间-最佳比降低 **22.6%**
- OS 级别动作空间、鲁棒执行生命周期、记忆增强规划——三个组件均被消融验证有效

> **意义**: 数据库调优从"参数搜索"转向"Agent 驱动重配置"，同时支持 OS 级别优化，是个范式升级

### ⭐ [6] REMOP: REmote-Memory-aware OPerator Optimization

> **arXiv**: 2606.19576 | DuckDB 实现

**核心**: 当算子 spill 超过本地 DRAM，将页面移动到远程内存时，传统磁盘 spill 策略不适用于远程内存场景（因为传输轮次的开销比传输量更大）。

**创新**:
- 引入 **transfer rounds** 作为延迟成本模型的关键维度
- 为 blocked nested-loop join、external merge sort、external hash join 推导算子特定的 buffer 分区策略
- DuckDB 实现

**性能**:
| 指标 | 提升 |
|:-----|:-----|
| Transfer rounds 减少 | up to **97%** |
| 算子运行时下降 (spill-heavy) | up to **48%** |
| TPC-H 端到端平均值 | **22.7%** ↓ |
| TPC-DS 端到端平均值 | **26.4%** ↓ |

## 📄 arXiv cs.IR 最新论文 (6/19 新提交 11 篇)

### ⭐ [9] Closing the Calibration Gap in Semantic Caching

> **arXiv**: 2606.19719 | Aditeya Baral 等

**核心**: 语义缓存在 **PR-AUC 排名指标和实际部署之间存在系统性偏差**。

- PR-AUC 最高的模型在部署时往往最差
- 提出 **P-CHR AUC** (Precision-Cache Hit Ratio AUC)：跨缓存利用率的精度度量
- 提出 **CRR** (Calibration Retention Rate)：离线排名质量在部署中保留多少
- 实验证明：训练目标决定校准差距，而非数据规模；后处理校准只能部分弥补

> **对 RAG 系统的意义**: 语义缓存模型选择本质上是**校准问题**而非排名问题

### ⭐ [10] Multi-Agent Transactive Memory (MATM)

> **arXiv**: 2606.19911 | To Eun Kim 等

**核心**: 代理间共享 trajectory 记忆库 — 多 Agent 系统中，一个 Agent 解决的问题方案可通过检索被其他 Agent 重用。

- 生产者 Agent 贡献轨迹 → 消费者 Agent 检索改进任务执行
- 交互环境 ALFWorld + WebArena 评估
- 检索 MATM 轨迹可提升下游任务表现并减少交互步数
- 无需协调或联合训练

> **意义**: 建立多 Agent 系统的"群体记忆"模式，是 Agent 知识共享的基础设施设计模式
