## 2️⃣ OptCC — AllReduce 信息论下界首次闭合（arXiv:2606.01680）

**机构**: 华中科技大学 / 华为  
**接收**: arXiv 2026-06-01

- **核心贡献**: 首次推导了**非对称网络带宽下AllReduce完成时间的下界**
- **关键发现**: 当降速节点保留至少一半原始带宽时，相对于无故障最优的不可接受开销仅为 **O(1/p)**（p为GPU数）
- **算法**: OptCC四阶段流水线AllReduce算法，逼近理论下界
- **性能**:
  - 50%网络故障下：**2-6%** 开销（vs NCCL无故障ring）
  - SOTA容错方案：**57%** 开销
- **意义**: 首次为AllReduce容错提供**理论最优基准**，开启了"理论下界→逼近算法"的新范式

> **来源**: arXiv:2606.01680 | "Don't Let a Few Network Failures Slow the Entire AllReduce"

## 3️⃣ ReCoVer — 512卡失去256卡仍保持训练轨迹（arXiv:2605.11215）

**机构**: Argonne National Lab / UIUC  
**提交**: 2026-05-11（v2: 2026-05-22）

- **核心思想**: 保持"每迭代微批次数恒定"这一核心不变量，确保梯度随机等价于无故障运行
- **三层解耦协议**:
  1. **Fault-tolerant collectives** — 隔离故障传播
  2. **步内细粒度恢复** — 保持迭代内进度
  3. **可扩展工作负载策略** — 动态重分配幸存者微批配额
- **并行性无关**: 可直接插入3D并行和HSDP
- **性能**:
  - 512 GPU上失去 **256 GPU** 仍保持训练轨迹
  - vs checkpoint-restart: **2.23x** 有效吞吐
  - 234 GPU-hours多处理 **74.9%** 更多token
  - 差距随时间扩大

> **来源**: arXiv:2605.11215 | "ReCoVer: Resilient LLM Pre-Training System via Fault-Tolerant Collective and Versatile Workload"

## 4️⃣ FlexNPU — Ascend NPU透明虚拟化，动态PD共置（arXiv:2606.04415）

**机构**: 华为 / 多机构合作  
**提交**: 2026-06-02

- **核心贡献**: 首个 **透明用户态NPU虚拟化层**，拦截AscendCL API
- **原理**: 通过per-device daemon解耦应用与物理NPU，无需修改模型代码/AI框架/NPU驱动
- **动态PD co-location**: 根据Compute-bound prefill与Memory-bound decode的资源互补特性动态调度
- **性能**:
  - 384卡 Ascend 910C DeepSeek-R1 → 吞吐提升 **+26.33%**（vs静态PD分离）
  - Qwen2.5-7B → TTFT降低 **>92%**（vs静态PD共置），TPOT几乎不变
  - 零测量推理开销
- **意义**: 为NPU生态提供首个类GPU虚拟化方案，FPGA-like灵活调度能力

> **来源**: arXiv:2606.04415 | "FlexNPU: Transparent NPU Virtualization for Dynamic LLM Prefill-Decode Co-location"

## 5️⃣ Orth-Dion — 补齐Muon优化器的分布式低秩收敛缺口（arXiv:2605.16341）

**机构**: MILA / Université de Montréal / University of Tokyo  
**提交**: 2026-05-07

- **背景**: Dion用低秩因子近似Muon（谱优化器）的分布式训练，但列归一化不产生Muon隐含目标的rank-r polar factor，收敛差 **O(√r)** 因子
- **方法**: Orth-Dion用**QR正交化**替换列归一化
- **理论**: 非欧几里得光滑度下，速率 **O(√Lᵣ/T)**，匹配精确谱方法，Dion相同通信开销
- **证明创新**: 去除了误差反馈分析中常用的bounded-drift假设，使用**自洽不动点论证**+**时间平均压缩**
- **实验**: 大规模LLM预训练验证了√r缩放，**收敛缺口完全补齐**
- **意义**: DeepSeek-K2、GLM-5等模型使用Muon系列优化器，Orth-Dion使低秩版达到全秩精度

> **来源**: arXiv:2605.16341 | "Orth-Dion: Eliminating Geometric Mismatch in Distributed Low-Rank Spectral Optimization"

## 6️⃣ PD分离生态：KVServe（SIGCOMM'26）+ PPD（ICML'26）持续演进

### KVServe — 首个服务感知KV压缩（arXiv:2605.13734）
- **机构**: 多机构合作
- **接收**: **SIGCOMM 2026**
- **核心**: 统一KV压缩为模块化策略空间，贝叶斯Profile引擎搜索3D Pareto候选集，**50x**离线搜索开销降低
- **在线控制器**: 分析延迟模型+轻量级bandit选profile
- **性能**: PD分离下 **9.13x** JCT加速，KV分离下 **32.8x** TTFT降低
- **已集成vLLM**

### PPD — Prefill-capable Decode（arXiv:2603.13358, ICML'26）
- **核心洞察**: append-prefill对小规模解码节点干扰比full prefill小**一个数量级**
- **方法**: 第二轮+请求本地处理在decode节点，利用缓存的KV状态
- **性能**: Turn2+ TTFT降低 **~68%**，TPOT几乎不变

### ESS — DeepSeek V3.2 Latent-Cache卸载（arXiv:2512.10576）
- **思路**: 将Latent-Cache选择性卸载到CPU内存，解耦batch-size与GPU内存限制
- **性能**: 32K上下文 **69.4%** 吞吐提升，128K **123%** 吞吐提升

> **来源**: arXiv:2605.13734 / arXiv:2603.13358 / arXiv:2512.10576

## 7️⃣ LiveR — 实时重配置，14-23x vs checkpoint-restart（arXiv:2605.22014）

**机构**: 香港中文大学 / 华为  
**提交**: 2026-05-21

- **背景**: Spot实例和可回收资源需要秒级适应窗口，现有系统仍用checkpoint-restart
- **方法**: LiveR在**当前world继续训练**的同时，异步准备目标world，流式传输模型状态，在线重塑并行维度（TP/PP/DP）
- **实现**: 基于Megatron-LM + PyTorch
- **性能**:
  - 重配置延迟：分钟级 → **秒级**
  - vs checkpoint/restart: **14x-23x** 加速
  - 稳态开销极小，**99%** training goodput
- **意义**: 使低价波动性GPU（spot实例）的LLM训练变得切实可行

> **来源**: arXiv:2605.22014 | "LiveR: Fine-Grained Elasticity via Live Reconfiguration for Model Training"
