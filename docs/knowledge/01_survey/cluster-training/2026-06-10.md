## 1️⃣ 🏆 推理感知KV共享 + 非自回归联合解码（06/09最新提交）

### 1.1 RKSC: Reasoning-Aware KV Cache Sharing (arXiv:2606.09937, ICML'26 Workshop)

**核心创新**: 消除多分支推理流水线中的**两种结构性冗余**

- **ASKES (Attention-Similarity KV Sharing)**: 基于hidden-state余弦相似度，将prefix KV cache一次计算→广播到所有语义相似的分支。**严格泛化**vLLM/SGLang的token-exact prefix caching
- **CGEE (Confidence-Gated Early Exit)**: 两重退出机制：(1) 跨分支generation confidence一致时完全跳过verification (2) per-layer entropy稳定时在中间层终止verification
- **RSBCM (Reasoning-Selective Block Cache Manager)**: 注意力加权深度优先eviction防止无边界增长

**性能数据**: 5个模型家族(7B-10B)×4基准×1,000问题 → 平均 **3.008×** 加速（峰值 **3.990×**），相比vLLM prefix缓存领先**1.66×**，CGEE错误率仅 **0.37%**（1,616次verify调用中6次错误）

**意义**: 零训练、零架构改动，将KV共享从"精确前缀匹配"提升到"语义相似度匹配"层面

### 1.2 K-Forcing: Joint Next-K-Token Decoding (arXiv:2606.10820) ⭐🔥

**核心创新**: **完全替代AR解码**——将AR模型蒸馏为push-forward映射，单次forward生成k=4个token
- 将独立均匀噪声变量转换为多个未来token的联合样本
- Progressive Self-Forcing Distillation：逐步扩展预测窗口，同时保持序列分布匹配AR teacher
- 兼容标准AR serving基础设施

**性能数据**: LM1B/OpenWebText × k=4 → **2.4-3.5×** 加速（跨batch size），质量退化可接受

**意义**: 非自回归范式的实用化里程碑——用蒸馏替代了昂贵的AR逐步生成，且兼容现有serving栈

### 1.3 WhiFlash: Cross-Paradigm SD Token-Level Routing (arXiv:2606.07710)

**核心创新**: 首个跨范式SD——同一token-level控制器统一**自回归draft**（推理场景）和**扩散并行draft**（结构化输出）
- 观察：单个序列内draft准确率剧烈波动→静态范式显著次优
- 轻量entropy-based或learned neural policy控制器进行token级路由决策
- **缓存管理优化**: Lazy Catch-up + KV-only Prefill → 切换开销降至每轮延迟的**7%以下**

**性能数据**: 吞吐提升 over EAGLE-3 **69.6%**🔥，over DFlash **37.3%**

**意义**: SD从"范式之争"进入"范式融合"——验证了自回归和扩散两种draft路线在token级互补

## 2️⃣ 弹性训练与实时重配置：从分钟级到秒级

### 2.1 DynaTrain: Fast Online Parallelism Switching (arXiv:2605.18815)

**核心创新**: Virtual Parameter Space (VPS) 抽象
- VPS统一所有分布式训练状态为单一逻辑坐标空间→任意并行配置变为确定性映射
- 状态路由→过渡层执行rank-local transfer（memory-aware, deadlock-free）
- Elastic Device Manager：新世界构建与旧世界训练重叠，隐藏拓扑变化成本

**性能数据**: 70B dense模型 **< 2s** 重配置，235B MoE模型 **4.36s**，超越SOTA checkpoints方案**三个数量级**

### 2.2 LiveR: Live Reconfiguration Runtime (arXiv:2605.22014, Megatron-LM)

**核心创新**: 替换storage-backed restart为**live bounded-memory handoff**
- 当前world继续训练时异步准备target world, bootstrapping新worker在隔离中完成
- 模型状态直接通过高带宽互联streaming（online reshape across TP/PP/DP）
- 轻量commit → 无需stop-and-restart

**性能数据**: 重配置从分钟→秒级，加速**14×-23×** over checkpoint/restart，volatile资源下维持**99%** goodput

### 2.3 FlashCP: Load-Balanced Context Parallelism (arXiv:2606.08476)

**核心创新**: 长上下文CP训练的通信优化
- **Sharding-Aware通信机制**: 消除冗余KV通信
- **Whole-Doc sharding**: 最大化通信节省 + 保持负载均衡
- Heuristic搜索算法：最优组合Whole-Doc和Per-Doc sharding

**性能数据**: 跨数据集最高**1.63×** over SOTA CP框架

## 3️⃣ SD工程化：延迟模型 + 系统安全 + 端侧部署

### 3.1 Interpretable Latency Model for SD in LLM Serving (arXiv:2605.15051)

**首个SD生产延迟分析框架**
- 用Little's Law从请求率推导有效batch size
- 将per-request需求分解为load-independent + load-dependent组件（prefill/draft/verify）
- vLLM实测验证跨verifier/drafter规模、prefill/decode长度、请求率、draft长度、接受率
- **关键发现**: 解释了为什么SD加速在服务器负载增加时经常消失
- 扩展至MoE模型——稀疏专家激活改变负载区间下的有效服务成本

### 3.2 GRIEF: Fuzzing LLM Serving Systems (arXiv:2605.11202)

- 首个针对LLM推理引擎的灰盒fuzzer：timed multi-request traces为一等输入
- vLLM + SGLang早期campaign发现**15个漏洞**（10个确认，含2个CVE）
- 漏洞类型：KV-cache隔离失败、cross-request性能干扰、crash/liveness bugs
- **意义**: 并发、缓存、状态重用是LLM基础设施的一等安全边界

### 3.3 Lever: Speculative LLM Inference on Smartphones (arXiv:2605.16786)

- flash-backed推理 + SD → 小draft模型留DRAM + 大flash-resident target模型验证
- I/O/compute-aware token树构建 + 早期退出剪枝 + CPU-NPU协同映射
- 平均推理延迟降低**2.93×**，将flash-backed与memory-resident差距缩小

### 3.4 SSV: Sparse Speculative Verification (arXiv:2605.19893)

- 解决SD与动态稀疏注意力的结构性不匹配
- overlap-aware grouped-query + refresh/reuse NSA kernel融合 + profile-guided自适应
- H100上 **3.49×** end-to-end over AR NSA, **6.86×** kernel加速

## 4️⃣ 稀疏注意力 × SD协同突破

### 4.1 STS: Efficient Sparse Attention with Speculative Token Sparsity (arXiv:2605.15508)

**核心创新**: 小draft模型的注意力分数预测大目标模型的重要token
- 集成到SD框架：repurpose draft attention scores → token-and-head-wise sparsity mask
- 剪枝大模型90%的attention计算

**性能数据**: NarrativeQA上 **2.67×** 加速 @ 90% sparsity, 零显著精度退化。**新的sparsity-accuracy trade-off SOTA**

### 4.2 CLSA: Cross-Layer Sparse Attention with Shared Routing (arXiv:2606.06467)

**核心创新**: YOCO架构上扩展——不仅共享KVCache，**还共享routing index**
- 单个indexer计算token-level top-k选择 → 跨decoder层重用索引
- 保持token稀疏attention的细粒度选择性 → 摊销路由开销

**性能数据**: 128K上下文：解码加速**7.6×**，整体吞吐提升**17.1×**

### 4.3 VeriCache: Lossy KV Turns into Lossless Inference (arXiv:2605.17613)

**核心创新**: 用有损KV cache作为draft → 与全KV cache并行验证 → 保证输出一致性
- 关键是：compressed-KV解码（HBM带宽瓶颈）与full-KV swap（PCIe/网络瓶颈）可并行化
- 支持token-dropping和量化方法的统一compressor接口
- 与传统SD可组合

**性能数据**: 全KV推理的**4×**吞吐，**完全相同输出**

## 5️⃣ NVFP4训练的推理端对应：模型压缩

### IBP: Lossless Compression for GPU Memory (arXiv:2605.30728, EuroSys'26)

- **Invariant Bit Packing (IBP)**: 识别跨tensor组的不变bit→消除 → warp并行解压 + 异步PCIe传输
- GNN训练**74%**加速, DLRM embedding **180%**, LLM推理**24%**加速
- **意义**: NVFP4训练+IBP压缩推理→4-bit全链路

## 6️⃣ 趋势判断

1. **SD进入"范式融合"时代** — WhiFlash统一自回归+扩散，K-Forcing做掉AR解码本身，下一步是更灵活的token级范式切换
2. **KV共享从精确到语义** — RKSC的attention-similarity broadcast超越prefix caching，CLSA跨层共享routing index
3. **弹性训练实用化拐点** — DynaTrain sub-2s + LiveR 14-23×将重配置从"运维手段"变为"运行时操作"
4. **稀疏注意力+SD深度协同** — STS/SSV/VeriCache三篇从不同角度证明：稀疏attention和SD可互相增强，形成1+1>2
5. **SD安全不可忽视** — GRIEF发现2个CVE，KV-cache隔离问题是真实威胁
6. **NVFP4 4-bit全链路形成** — 训练端NVFP4 + JAX/MaxText + 推理端IBP无损压缩，端到端4-bit计算链路初步成型
