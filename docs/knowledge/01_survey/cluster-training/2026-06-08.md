## 1️⃣ vLLM冷启动首次系统性分析（MLSys'26里程碑）

### Breaking the Ice: Analyzing Cold Start Latency in vLLM (arXiv:2606.07362, MLSys'26)

| 维度 | 内容 |
|:-----|:------|
| **来源** | MLSys 2026 (Bellevue) · University of Amsterdam & UPB |
| **核心突破** | 随着vLLM V1 API和torch.compile引入重大架构变化，本文首次系统分析vLLM启动延迟——此前从未被系统研究过 |
| **方法** | (1) 拆解vLLM启动为6个基础步骤；(2) 量化每步的CPU/IO瓶颈；(3) 构建轻量分析模型预测冷启动延迟 |
| **六步骤** | ① Python环境初始化 → ② 模型加载 → ③ 权重分配 → ④ 图编译/缓存 → ⑤ 显存分配 → ⑥ Worker就绪 |
| **关键发现** | **启动过程以CPU为瓶颈**（不是GPU），每步呈现一致的可解释缩放趋势 vs 模型规模/系统参数 |
| **产出** | 轻量解析模型可在给定硬件配置下准确预测vLLM启动延迟，为大规模推理资源规划提供行动指南 |
| **开源** | github.com/upb-cn/vllm-startup-profiler |
| **意义** | 云推理服务中冷启动是SLO达成的关键障碍，此工作首次建立了从"黑盒启动"到"可预测启动"的基础 |


## 2️⃣ Speculative Decoding: 模型自由+扩散双阶段+树修正+脉冲管线四新范式

### AdaPLD — 自适应检索+重用的模型自由SD (arXiv:2606.05742)

| 维度 | 内容 |
|:-----|:------|
| **核心问题** | 模型自由SD依赖文本复用，但词汇锚定检索在字面变异下召回有限；确定性的片段复制在面对多义上下文时脆弱 | 
| **方法** | (1) 在保持高精度词汇复用的同时，当词汇匹配失败时用语义相似度回收额外复用机会；(2) 构造分支复用假设应对延续不确定性，而非依赖单一复制片段 |
| **加速比** | 最高**3.10x**解码加速；显著减少目标模型前向传递次数 |
| **意义** | 无需辅助draft模型、无需训练的纯检索-复用SD，破解了词汇vs语义的最佳平衡点 |

### D²SD — 双扩散草案的置信引导前缀树 (arXiv:2606.04446)

| 维度 | 内容 |
|:-----|:------|
| **核心问题** | 现有扩散草案器只生成单一草稿序列，首个token不匹配后续全部丢弃；批量增加候选序列引入冗余分支 |
| **方法** | (1) 首个扩散草案器生成block + 每位置信度 → 锁定最可能的拒绝边界 → 选择Top-K前缀范围恢复；(2) 第二可变前缀扩散草案器在每选中前缀上重新锚定，单batched pass生成替代延续；(3) 共享前缀候选通过级联注意力联合验证 |
| **加速比** | 明显超越基线和强自回归SD方案 |
| **意义** | 双阶段扩散—先用一个粗生成定位高置信前缀，再用第二个精修候选分支——类似"先粗定位、再精搜" |

### TreeFlash — 并行AR近似的树型SD (arXiv:2606.03819)

| 维度 | 内容 |
|:-----|:------|
| **核心问题** | 单过shot块drafters（非自回归导致分布发散）在树型草案中问题更严重：不同分支被迫共享后续token的边际分布 |
| **方法** | MLP层以drafther隐状态+前一token为条件近来自回归分布，保持O(1)解码时间复杂度 |
| **加速比** | 块效率+**12%** ，加速比+**9%** vs 边际树型方案 |
| **意义** | 首次在one-shot draft（非顺序生成）中近似自回归条件分布 |

### SPD — Speculative Pipeline Decoding (arXiv:2605.30852)

| 维度 | 内容 |
|:-----|:------|
| **核心思想** | 将目标LLM分区为n级管线，同一时间并行处理n个token；推测模块在不同深度聚合中间特征预测下一token，与目标模型管线步严格并行 |
| **方法** | 管线并行 → 无气泡推测验证（zero bubble + 有限难度 + 更高接受率） |
| **加速比** | 理论加速比显著超越主流基线 |
| **意义** | **管线并行** + **推测解码**首次联合——管线并行的气泡被视为"免费"的推测机会 |


## 3️⃣ Speculative Decoding进军RL训练生产化

### RL Post-Training Rollout加速 — NeMo-RL + vLLM + SD (arXiv:2604.26779)

| 维度 | 内容 |
|:-----|:------|
| **来源** | NVIDIA NeMo-RL团队 |
| **核心问题** | RL post-training越来越被自回归rollout生成瓶颈限制 |
| **方法** | 在NeMo-RL中使用vLLM后端实现speculative decoding，支持同步和异步管线，兼容MTP heads、小型外部草案模型、Eagle3 |
| **加速比** | 8B规模同步RL：**1.8x** rollout吞吐；235B规模异步RL（高保真模拟预测）：**2.5x**端到端训练加速 |
| **意义** | **首个在生产级RL框架(NVIDIA NeMo-RL)中集成SD的工作**，表明SD不仅是推理加速工具，更是RL训练基础设施组成部分 |

### Hybrid Verified Decoding — 缓存SD vs 模型草案的动态选择 (arXiv:2606.01019)

| 维度 | 内容 |
|:-----|:------|
| **方法** | 预测缓存草稿在验证前的接受长度，据此在缓存验证和模型基草案器间选择 |
| **测试** | 3种LLM × 16数据集 |
| **加速比** | Agent工作流**2.73x**平均加速，全面超越EAGLE3 |
| **洞察** | 高收益缓存草稿集中在草稿空间的一小部分；收益引导的选择显著减少顺序解码工作 |
| **意义** | "什么时候用缓存SD，什么时候用模型SD"——这篇给出了答案 |


## 4️⃣ PD分离生态：从静态解耦到拓扑感知KV路由

### NetKV — PD分离中网络感知的Decode实例选择 (arXiv:2606.03910)

| 维度 | 内容 |
|:-----|:------|
| **核心洞察** | 当前调度器仅考虑计算负载和prefix-cache局部性，完全忽略拓扑距离和动态拥塞 |
| **问题证明** | **忽略网络项使仅cache感知调度在上下文较长时任意次优** |
| **方法** | thin operator-to-scheduler接口 + 网络成本oracle；NetKV的O(\|D\|)贪心算法对陈旧遥测证明鲁棒 |
| **加速比** | Mooncake 64GPU四层fat-tree仿真：均值TTFT降**21.2%** vs round-robin，**17.6%** vs cache+load调度；SLO达成率升**20.1pp**；TBT始终<0.5ms |
| **意义** | 首次在PD分离调度中正式证明网络拓扑距离不可忽略，并为PD分离系统提供可落地的网络awareness方案 |

### Tangram — 非均匀KV缓存的系统级实用化 (arXiv:2606.06302)

| 维度 | 内容 |
|:-----|:------|
| **核心挑战** | 非均匀KV压缩保留更多信息但引发内存碎片、调度复杂度和kernel利用率下降 |
| **方法** | (1) 确定性预算分配：每head基于固有模式分配静态内存足迹，消除动态调度开销；(2) Head Group Page：聚类相似retention需求的head，独立向量化页表最大化回收；(3) AOT负载均衡 |
| **加速比** | 吞吐**2.6x** vs 现有基线，完整保留模型精度 |
| **意义** | 使非均匀KV压缩从学术概念走向系统落地 |


## 5️⃣ 跨地域Pipeline并行调度

### BACE-Pipe — 带宽感知+成本优化的跨地域LLM训练 (arXiv:2605.25375)

| 维度 | 内容 |
|:-----|:------|
| **核心挑战** | 跨地域Pipeline并行面临异构带宽和区域电价差异；现有方案非延迟优先（高电费）或成本优先（JCT长） |
| **方法** | (1) 动态作业优先级机制联合考虑作业特征和实时网络利用；(2) 带宽感知路径搜索器找出满足通信约束的可行管线路径；(3) 成本最小化分配器优先在低电价区域分配GPU |
| **效果** | 均值JCT降**27.9%-64.7%**，总电费降**12.6%-30.6%** |
| **意义** | 第一个同时优化JCT和电费的跨地域管线调度方案 |

### AMDP — 异步多方向Pipeline并行 (arXiv:2605.29664, ICML 2026)

| 维度 | 内容 |
|:-----|:------|
| **核心挑战** | 现有异步pipeline方案因前向后向参数不匹配→收敛退化 |
| **方法** | (1) 限制每管线首级最多处理2个minibatch再backprop，限定参数更新次数；(2) 启动多条并发pipeline，数量随管线深度自适应调整；(3) 跨minibatch累积梯度单次应用，确保参数不匹配仅限一个优化步内 |
| **效果** | GPT/BERT风格模型显著加速训练，收敛保持 |
| **意义** | 异步pipeline首次在不牺牲收敛的前提下保持高利用率 |
