## 1️⃣ 🏆 Bebop：MTP + 拒绝采样打破RL训练熵边界（arXiv:2606.12370, 06/10提交）

**核心问题**: MTP在RL训练中acceptance rate因模型熵波动而急剧下降，导致加速效果消失

**核心发现三连**:

1. **熵边界定律**: MTP acceptance rate与RL阶段模型熵上升之间存在**清晰负线性关系**——RL rollout token级别熵无序膨胀→draft预测置信度崩塌
2. **概率拒绝采样 > 贪婪draft**: Probabilistic Rejection Sampling 相比 Greedy Draft Sampling，能显著缓解熵引入的扰动
3. **端到端TV Loss替代KL/交叉熵**: 提出新的损失函数直接优化多步拒绝采样的acceptance rate，而非KL散度这一代理目标

**性能数据**:
- TV Loss优化 → ~**10%** acceptance rate提升，峰值达**95%**
- 额外吞吐增益 → 最高**25%**
- 端到端异步RL训练加速 → **1.8×**（Qwen3.5/Qwen3.6/Qwen3.7，数学推理/代码生成/Agent任务）
- 关键洞察：**Pre-RL**训练MTP（e2e TV Loss + 拒绝采样）即可在**整个RL过程中保持一致的acceptance rate和加速效果**，无需代价高昂的在线MTP更新

**意义**: 为MTP+RL训练对偶打开了实用化大门，证明RL rollout瓶颈（而非计算瓶颈）才是RL训练的真正瓶颈

## 2️⃣ VIA-SD：多级验证范式（arXiv:2606.12243, ICML'26, 06/10）

**核心创新**: **打破SD的"二元验证"假设**——拒绝的token不需要全部重新计算

传统SD中，draft token被拒绝后，target必须从头重新计算该位置。VIA-SD发现，**大量被拒绝的token可由target模型的子模型（slim submodel）正确验证**，根本无需完整target的参与。

**三级验证层次结构**:
1. 高置信度 → 直接接受（draft token成立）
2. **中置信度** → **slim-verifier再生**（仅激活target的部分参数，通过intra-model routing实现）
3. 低置信度 → 完整target验证

**性能数据**: 4任务×多模型家族 → rejection rate↓**0.10-0.22**，速度↑**10-20%** over 强SD基线，**2.5-3×** over 非SD解码

**意义**: 将SD从"接受/拒绝"二元决策升级为 **"接受/子模型验证/完整验证"三级决策**，兼容现有SD框架无需修改训练

## 3️⃣ Teaching Diffusion to Speculate Left-to-Right（arXiv:2606.11552, 06/09）

**核心问题**: 扩散draft模型双向生成token块，但AR验证器严格从左到右评估——训练目标(loss)与验证信号(reward)之间存在**根本性不对称**

**三大正交训练介入**:
1. **Token Positional Weighting**: 按位置加权loss，低位置（更可能被接受）高权重
2. **First-Error Focal Loss**: 针对块内打破已接受前缀的**第一个错误位置**施加焦点损失
3. **Chain Loss Term**: 用可微替代函数模拟期望接受长度，端到端优化

**性能数据**: 4个target模型×6个基准（推理/代码/对话）→ accepted draft length ↑**21-76%** 每基准，**零额外前向开销**，不改变推理流水线或拒绝采样精确性保证

**意义**: 三大介入沿正交方向（位置/块条件首位错误/联合前缀）叠加增强，与多draft自选择等测试时对齐机制正交可组合

## 4️⃣ SpectrumKV：单Token精度的PD分离KV传输（arXiv:2606.08635, 06/07）

**核心创新**: **将PD分离中的KV传输视为"精度分配问题"**，而非"裁剪问题"

现有PD-side KV reduction方法是二元的：选中token全精度传输，未选中不传。SpectrumKV对每个token**分配独立精度等级**:
- Attention sinks + 高重要性token → FP16
- 中等重要性token → INT8
- 低重要性token → INT4（模型可容忍时）
- 轻量部署时探针：3次aggressive NIAH测试判断模型是否支持INT4

**性能数据**:
| 模型 | 50% KV budget, WikiText-2 PPL变化 | vs PDTrim |
|:-----|:-----|:-----|
| Qwen2.5-7B | +1.97% | PDTrim +25.85% |
| Mistral-7B | **-0.06%** | PDTrim +22.07% |
| Gemma-2-9B | **-0.44%** | PDTrim +35.63% |
- NIAH 4K检索: Qwen b=0.3 → 52.6% vs PDTrim 26.3%
- 端到端GPU传输路径: b=0.5 → **50-62% TTFT ↓**

**意义**: PD分离的KV传输从"传哪些token"进化到"每个token传多少精度"

## 5️⃣ RTP-LLM：阿里巴巴生产推理引擎全解析（arXiv:2605.29639, 05/28）

**核心架构**: 首次完整披露阿里巴巴**服务1亿+用户**的工业生产推理引擎

**五大技术支柱**:
1. **模型加载**: File-Order-Driven I/O + 并行I/O-通信重叠 → **4.7-6.3×** 加载加速
2. **PD分离+分层KV缓存**: 解耦prefill与decode，多层KV cache层级管理，生产流量调度下cache复用↑**215%**
3. **模块化SD**: 支持多种SD算法
4. **自适应KV量化**: 动态精度调整
5. **多级并行解耦多模态**: Encode-Prefill-Decode (EPD) 分离架构

**完整性能数据**（8B-235B模型，对比vLLM/SGLang）:
| 指标 | 性能提升 |
|:-----|:---------|
| 模型加载速度 | **4.7-6.3×** |
| TTFT P95（生产流量） | **35-37%↓** |
| 生产流量cache复用 | **215%↑** |
| SD吞吐（多种算法） | **1.12-2.48×** |
| 多模态推理吞吐 | **1.86-2.52×** |
| 量化推理批延迟 | **35-40%↓** |
| 量化推理TTFT | **1.9-3.0×** |

**意义**: 迄今最完整披露的**10亿用户级**生产推理引擎技术细节，PD分离+SD+量化+多模态四合一已被验证在生产环境有效

## 6️⃣ NCCLZ：解耦量化+熵编码的GPU通信压缩（arXiv:2605.12396, 05/12）

**核心创新**: **解耦[量化→熵编码]在堆栈的不同层**

现有压缩通信库要么依赖MPI（不能充分利用NCCL），要么紧密耦合全量压缩器。NCCLZ的创新在于：
- **量化层**放在接口层（interface level）
- **熵编码层**嵌入NCCL原语内部
- 轻量device-side selector动态选择编码策略
- 压缩与通信重叠，减少暴露开销

**性能数据**: 科学数据+训练梯度+合成负载 → 相比NCCL加速 **<9.65×**，相比现有压缩辅助通信库加速**<3.34×**

**意义**: 为NCCL生态提供首款原生压缩加速方案，比之前05/05记录的ZipCCL（1.18×端到端速度提升）在特定负载上有量级提升

## 7️⃣ SPARe：100K+ GPU容错框架（arXiv:2603.00357, ICML'26）

**核心创新**: 通过**冗余分片堆叠+自适应重排序**在梯度同步期间掩盖节点故障

与传统复制（overhead线性膨胀）不同，SPARe:
- 跨并行组**堆叠冗余数据分片**
- 自适应重排序执行
- 推导了**可承受故障数的闭式表达式**和计算开销公式
- 联合优化冗余度和检查点策略，最小化训练完成时间

**性能数据**:
| 规模 | 计算开销 | 时间减少 |
|:-----|:---------|:---------|
| 100k+ GPU | 仅 **2-3×**（传统复制线性膨胀） | **40-50%** 训练完成时间↑效率 |
| 600k GPU | 保持不变 | 差距进一步扩大 |

**意义**: 首个面向**超大规模（100K+ GPU）LLM训练的容错框架**，将故障容忍从"事后检查点重启"升级为"运行时冗余分片掩盖"
