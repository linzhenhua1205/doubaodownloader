## 1️⃣ 🏆 NVIDIA NVFP4 + JAX/MaxText Blackwell预训练正式发布（06/08博客）

### Train Models Faster with JAX and MaxText Using NVFP4 on NVIDIA Blackwell

| 配置 | 硬件 | 精度对比 | 加速比 |
|:-----|:-----|:---------|:-------|
| Llama 3 8B | GB200 | NVFP4 vs FP8 | **1.35×** |
| Llama 3 8B | GB300 | NVFP4 vs FP8 | **1.31×** |
| Llama 3.1 405B | GB200 | NVFP4 vs FP8 | **1.44×** |
| Llama 3.1 405B | GB300 | NVFP4 vs FP8 | **1.73×** ⭐ |

**技术要点**:
- **五大核心技法**: (1) 16元素micro block scaling（vs MXFP4 32元素，outlier无扩散）(2) E4M3 block scale + per-tensor FP32 scale（vs MXFP4 E8M0，同loss需再练36% token）(3) Random Hadamard Transform仅在WGRAD输入（不破坏2D scale一致性）(4) 2D weight scaling：每16×16 weight block一个FP8 scale（FPROP/DGRAD共用）(5) 梯度端Stochastic Rounding（权重保留round-to-nearest-even）
- **作用范围**: 仅MLP层（FFN）量化至NVFP4；Attention GEMM保持高精度（softmax指数放大QK^T噪声，outlier密集）
- LLM预训练10K步loss曲线完全对齐FP8基线（差距仅+0.026 nats，在step噪声之内）
- **来源**: NVIDIA技术博客, 2026-06-08 | arXiv:2509.25149

## 2️⃣ Speculative Decoding：协作范式（CoSpec）+因果解耦（Domino）+ 攻击面（Mistletoe）

### 2.1 Domino: Decoupling Causal Modeling from Autoregressive Drafting (arXiv:2605.29707)

**核心创新**: 将SD中因果依赖建模与昂贵自回归执行**彻底解耦**
- 先用并行draft backbone生成整块初步分布 → 轻量Domino head用prefix-conditioned因果信息精炼
- base-anchored训练课程：先强化并行backbone → 逐渐转向因果校正分布
- **Qwen3实现**: Transformers后端 **5.49×** end-to-end加速, SGLang服务 **5.8×** 吞吐提升

### 2.2 CoSpec: Collaborative Speculative Decoding (arXiv:2605.24793)

**核心创新**: 颠覆SD范式——draft与target模型的**token冲突时，不一定target对**
- 观察到non-trivial比例的冲突中，draft的token导致正确答案
- 用**RL训练仲裁策略(arbitration policy)**决定是否采纳draft的token
- 从"imitation（模仿）"转向"collaboration（协作）"

### 2.3 Mistletoe: Stealthy Acceleration-Collapse Attacks on SD (arXiv:2605.14005)

**关键发现**: SD存在机制级漏洞——draft-target分布不匹配创造攻击面
- 联合优化degradation目标（降低draft-target一致率）+ 语义保持约束
- 引入零空间投影：将degradation梯度投影到局部语义保持方向之外
- 显著降低平均接受长度τ，使加速完全崩溃，但输出质量/困惑度不变
- **意义**: SD从纯加速技术上升到安全攻防领域——催生SD安全设计的紧迫需求

### 2.4 Attention Drift: What Autoregressive SD Models Learn (arXiv:2605.09992)

**根本性发现**: 首次识别**Attention Drift现象**——draft生成时注意力逐渐从prompt转移到自身新生成token
- 跨EAGLE3/MTP heads均出现，说明是draft架构的本质属性
- 根源：chain steps间未归一化的残差路径，隐藏状态幅度随深度单调增长
- **修复**: Post-norm + per-hidden-state RMSNorm → template扰动下接受长度**2×**提升，long-context **1.18×**
- Attention Drift被系统记录的论文，将驱动下一代draft架构设计

### 2.5 TTS: Test-Time Speculation (arXiv:2605.09329)

**核心创新**: 在线蒸馏——在inference时持续适配SD draft model
- 关键洞见：verification step已经调用了target model → 训练信号是免费的
- 利用已验证的draft token重新训练draft，每轮更新提高准确率
- 最长生成场景下接受长度提升**72%**（平均41%），Qwen-3/Qwen-3.5/Llama 3.1全系列验证
- **意义**: 解决了生成长度长时SD退化至无加速的经典难题

## 3️⃣ SD系统架构实现突破

### 3.1 Cassandra: Self-SD算法-硬件协同设计 (arXiv:2605.26558, ISCA'26)

- 细粒度数据选择 → 优化pruning + mantissa truncation构建无训练draft
- 轻量encoder-decoder硬件模块无缝集成商用GPU/NPU
- **RTX 4090上Llama 3 8B**: 2.41×加速, 同内存下比EAGLE-3多生成**1.81×** tokens
- **意义**: SD从数据中心走向消费级边缘设备

### 3.2 PARD-2: 双模SD框架 (arXiv:2605.08632)

- Confidence-Adaptive Token优化：从token预测准确率转向整体接受长度
- 单draft模型同时支持target-dependent和target-independent双模式
- **6.94×** lossless加速，超越EAGLE-3 **1.9×**，超越PARD **1.3×** (Llama3.1-8B)
- 代码开源: github.com/AMD-AGI/PARD

### 3.3 SpecBlock: Block-Iterative with Dynamic Tree Drafting (arXiv:2605.07243)

- 块内层移机制 + 块间隐状态继承 → 路径依赖+低成本drafting
- 协同训练的rank head替代固定top-k → 按位置分配推理时分叉
- validation-prefix mask仅训练draft实际产生的前缀
- 成本感知bandit在部署期动态更新draft
- 比EAGLE-3加速**8-13%**，仅用**44-52%**的drafting成本

### 3.4 SlimSpec: 低秩LM-Head加速SD (arXiv:2605.10453)

- 短板分析：draft LM-head在vocab投影上是大瓶颈
- 低秩参数化draft LM-head，压缩内表示而非截断输出词汇表
- **4-5×** LM-head加速 + 保持竞争性接受长度
- 端到端加速超越现有方法**8-9%**

### 3.5 D-PACE: Dynamic Position-Aware Cross-Entropy (arXiv:2605.18810)

- 从可微分的期望接受长度代理中推导每个位置的训练权重
- 当draft改进时，训练信号自动转移到当前限制接受的位置
- 一致改善clock time加速和平均发射长度，训练开销仅**2.3%**

### 3.6 SpecKV: 压缩感知的自适应γ选择 (arXiv:2605.02888)

- 问题：几乎所有SD系统使用固定γ（通常=4），但最优值随压缩等级变化
- 用draft model的confidence + entropy预测接受率（correlation ≈ 0.56）
- 小MLP每步动态选择γ → 超越固定γ=4基线**56%**
- 决策开销仅0.34ms（<0.5% step time）
- 开源: github.com/Amorfati123/SpecKV

## 4️⃣ 梯度压缩、检查点与RL训练优化

### 4.1 ZipCCL: Lossless Gradient Compression for LLM Training (arXiv:2604.27844)

- 观察到训练通信数据（激活/梯度/参数）呈**近高斯分布** → 指数字典编码理论
- GPU优化压缩kernel：通信感知的数据布局 + memory access pattern设计
- 自适应通信策略：根据负载特征动态切换集合操作
- **64-GPU集群**: 通信时间减少**1.35×**, 端到端训练加速**1.18×**, 零质量损失
- 支持MoE + Dense Transformer，直接集成NCCL

### 4.2 SparrowRL: Sparse Delta Checkpoints for WAN RL (arXiv:2602.11456)

- 关键洞见：RL微调每一步仅~**1%**参数元素变化
- 将所有RL步骤表示为稀疏delta checkpoint，流水线压缩+多流传输+rollout覆盖
- Qwen3-8B: per-step传输payload减少**79×**，WAN吞吐提升**2.4-9.5×**
- 跨地域RL训练差距缩小至理想RDMA单数据中心基线的**8.91%**以内
- tokens/dollar比保留RDMA集群高**1.21-1.59×**
- **意义**: RL后训练从HPC独占走向commodity网络可用

### 4.3 LLMTailor: 层裁剪检查点 (arXiv:2602.22158, PDSW'25)

- 实证发现：LLM各层更新高度不均匀 → 只检查点显著更新的层
- Llama3.1-8B: 检查点大小缩小**4.3×**，Qwen2.5-7B: 检查点时间加速**2.8×**
- 保持模型质量无下降

## 5️⃣ 趋势判断

1. **NVFP4进入生产预训练** — Blackwell本土4-bit支持 + 五大收敛保障技法 + 已验证10K步loss对齐，将推动主流LLM训练迁移至4-bit混合精度
2. **SD安全成为新课题** — Mistletoe攻击表明SD存在机制级漏洞，未来需要"SD防火墙"
3. **协作取代模仿** — CoSpec证明draft在冲突token上有时更优，仲裁策略正在取代严格target对齐
4. **SD系统集成成熟化** — PARD-2 (AMD), SlimSpec, SpecBlock, SpecKV 等正在从算法研究转向可部署组件（含开源代码）
5. **梯度压缩+检查点向RL赛道渗透** — SparrowRL (79x payload), ZipCCL (1.18x) 表明通信瓶颈正在从预训练转向post-training RL阶段
