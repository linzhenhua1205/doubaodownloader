## ① OpenBMC 3.0.0 发布（2026-05-19）

**来源**: [OpenBMC GitHub Tags](https://github.com/openbmc/openbmc/tags) | **版本**: 3.0.0

OpenBMC 历经约1年开发（自2.18.0于2025-05-30发布），正式进入 **3.x 时代**，累计 **2,483 次提交**、**38 位贡献者**。

### 🔑 关键变化

**新增支持平台（11个）**：

| 厂商 | 新增平台 | 说明 |
|:-----|:---------|:-----|
| **NVIDIA** | **nvl32-obmc** ⭐ | **新增NVIDIA NVL32平台支持**，与 gb200nvl-obmc 并存，覆盖更多NV AI服务器机型 |
| **Facebook/Meta** | anacapa, bletchley15, clemente, fbdarwin, rainiera6, sanmiguel, santabarbara, ventura2, yosemite5, yosemite5a7 | **10个新Meta自研服务器/交换机平台**，反映Meta在OpenBMC生态中的持续大力投入 |
| **Qualcomm** | qcom-bmc-ast2600 | 从 qcom-dc-scm-v1 **重构为** qcom-bmc-ast2600，明确基于ASPEED AST2600方案 |

**移除平台**：
- IBM `witherspoon`、`witherspoon-tacoma`（老旧Power平台，生命周期结束）

**开发活跃度**：
- **3.1.0-dev** 分支已启动（2026-05-14 标记），基于 `wrynose` Yocto分支
- 上游持续活跃，Master分支每日有提交

**对AI基础设施的影响**：
- NVIDIA `nvl32-obmc` 的加入表明NV正在扩展OpenBMC在其AI服务器中的覆盖面（从GB200 NVL72到更多NVL形态）
- Meta 10个新平台说明开源BMC路线依然是超大规模数据中心的首选

## ② DMTF Redfish Service Validator 3.1.4（2026-05-15）

**来源**: [DMTF/Redfish-Service-Validator Releases](https://github.com/DMTF/Redfish-Service-Validator/releases)

2026年上半年 Redfish 生态持续活跃，Service Validator 累计发布 **11个版本**（3.0.4 → 3.1.4）：

| 版本 | 日期 | 关键特性 |
|:-----|:-----|:---------|
| **3.1.4** | 2026-05-15 | 修复 `OriginOfConditionUnavailable` 检查，新增**属性名和枚举值大小写一致性检测** |
| 3.1.3 | 2026-05-01 | 新增 `OriginOfCondition` 链接检查 |
| 3.1.2 | 2026-04-17 | 新增 Message 注解使用检查 |
| 3.1.1 | 2026-04-10 | ✅ **`--timeout` 选项**、HTML报告UI增强（资源结果/载荷独立控制、配置详情）、**Markdown表格复制**、测试报告引入**耗时统计**、日志分时间戳子目录 |
| 3.1.0 | 2026-03-27 | ⭐ **HTML报告UI大改版**、`@odata.type`数据类型错误保护修复 |
| 3.0.9 | 2026-03-13 | 顶级Schema错误降级为Warning |
| 3.0.8 | 2026-03-06 | 修复展开资源时的Action对象检查 |
| 3.0.7 | 2026-03-04 | 修复Allow头检查崩溃（Schema未定义资源capabilities场景） |
| 3.0.6 | 2026-02-23 | GUI EXE打包重构，无功能变化 |
| 3.0.4 | 2026-02-20 | ⭐ **测试流程/追踪/报告大更新** |

### 关键趋势
- Redfish 合规性验证工具**高速迭代**（几乎每2周一个版本），表明Redfish标准本身正在快速演进
- HTML报告UI持续优化，说明DMTF在提升测试工具的用户体验
- `--timeout` 选项的加入针对大规模Redfish服务扫描场景
- 属性名/枚举值大小写一致性检测表明DMTF在推动更严格的模式合规
