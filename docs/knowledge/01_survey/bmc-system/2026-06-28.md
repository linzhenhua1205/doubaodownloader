## 1️⃣ 🔴 CVE-2026-50031: FreeIPMI ipmi-oem 响应消息缓冲区溢出漏洞

| 元数据 | 值 |
|:------|:----|
| **CVE ID** | [CVE-2026-50031](https://nvd.nist.gov/vuln/detail/CVE-2026-50031) |
| **CVSS 3.1** | **7.5 (HIGH)** — `AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:H` |
| **发现时间** | 2026-06-03 发布，2026-06-17 更新 |
| **影响组件** | FreeIPMI `ipmi-oem` 客户端命令（< 1.6.18） |
| **影响范围** | Dell `get-active-directory-config` + Fujitsu `get-sel-entry-long-text` 两个子命令 |

**漏洞描述**: `ipmi-oem` 是 FreeIPMI 套件中执行 IPMI OEM 厂商特定命令的客户端工具。在获取 Dell Active Directory 配置和 Fujitsu SEL 长文本条目时，**响应消息存在可被利用的缓冲区溢出**。

**攻击场景**: 攻击者构造恶意响应（中间人或恶意 BMC 模拟），发送给运行 `ipmi-oem dell get-active-directory-config` 或 `ipmi-oem fujitsu get-sel-entry-long-text` 的管理主机，触发栈缓冲区溢出导致 **拒绝服务（DoS）**。

**修复**: FreeIPMI 1.6.18 已修复。从发行公告获取：[info-gnu 2026-06](https://lists.gnu.org/archive/html/info-gnu/2026-06/msg00000.html)，相关 bug 跟踪：[#68363](https://savannah.gnu.org/bugs/index.php?68363)、[#68364](https://savannah.gnu.org/bugs/index.php?68364)。

**重要性**: 🔴 **高** — 不影响 OpenBMC 内置 IPMI 栈，但广泛使用的 FreeIPMI 管理工具存在 RCE≈DoS 级漏洞。IPMI 网络可达的环境下风险较高。提示：避免将 IPMI 管理接口暴露于不可信网络。

**评判**: 这个漏洞不属于 OpenBMC 自身，但 FreeIPMI 是服务器运维中极常用的 IPMI 客户端工具，值得所有使用 IPMI 管理的团队注意。

## 2️⃣ 🛠️ bmcweb: Redfish 错误响应合规性批量修复（2026-06-25）

| 元数据 | 值 |
|:------|:----|
| **仓库** | `openbmc/bmcweb` |
| **提交** | `8b0993927` (merged 2026-06-25) |
| **作者** | Myung Bae (IBM) |
| **标题** | `Change a few message responses as errors` |

**问题**: 8 类 Redfish 请求错误消息在以 Warning 级别（即仅带 `@Message.ExtendedInfo` 对象）返回，而非标准 Redfish 错误响应信封（即外层 `error` 对象 + `code` + `message` 字段）。

**涉及的响应类型**:
| 消息 ID | 含义 |
|:--------|:-----|
| `PropertyDuplicate` | 请求体属性重复 |
| `ResourceAlreadyExists` | 资源已存在 |
| `CreateFailedMissingReqProperties` | 缺少必需属性 |
| `PropertyValueFormatError` | 属性值格式错误 |
| `PropertyValueNotInList` | 属性值不在许可列表中 |
| `PropertyValueTypeError` | 属性值类型错误 |
| `PropertyValueError` | 属性值错误 |
| `PropertyNotWritable` | 属性不可写 |
| `PropertyValueModified` | 属性值被修改 |
| `PropertyMissing` | 属性缺失 |

**修复效果**:
- 修复前：`curl` 仅返回 `"Password@Message.ExtendedInfo": [...]`（Warning 级别）
- 修复后：返回标准 Redfish `{"error": {"code": "Base.1.19.PropertyValueFormatError", "message": "..."}}` 信封

**重要性**: 🟡 **中** — 合规性修复，直接关系到 Redfish Service Validator（RSV）的通过率和第三方 Redfish 客户端对错误响应的正确解析。IBM 主导的推进。

## 3️⃣ 🐛 bmcweb: Event Log 时间戳解析回归修复（2026-06-25）

| 元数据 | 值 |
|:------|:----|
| **仓库** | `openbmc/bmcweb` |
| **提交** | `ab4232093` (merged 2026-06-25) |
| **作者** | Haiyue Wang |
| **标题** | `Fix event log timestamp parse error` |

**问题描述**: 提交 `aa2dfd26a514` ("Remove static variables") 重构了 Event Log 参数解析逻辑，移除了从完整日志条目中提取时间戳的功能。`dateStringToEpoch()` 无法处理带有尾部文本的时间戳（如 `"2026-06-23T05:34:03+00:00 ,REDFISH_MESSAGE_ID"`），导致时间戳解析全部失败，Event Log 功能完全不可用。

**根因**: OpenBMC 主树中的 `rsyslog.conf` 使用 `RedfishTemplate` 格式：
```
template(name="RedfishTemplate" type="list") {
    property(name="timereported" dateFormat="rfc3339")
    constant(value=" ")
    property(name="$!REDFISH_MESSAGE_ID")
    constant(value=",")
    property(name="$!REDFISH_MESSAGE_ARGS")
}
```

日志文件 `"2026-06-23... ,REDFISH_MESSAGE_ID,arg1,arg2"` 格式中时间的尾部 `,REDFISH_MESSAGE_ID` 被 `dateStringToEpoch()` 拒绝。

**修复**: 恢复从完整日志条目中提取时间戳的逻辑。Haiyue Wang 为 Intel 的贡献。

**重要性**: 🔥 **高** — Event Log（`/redfish/v1/Managers/bmc/LogServices/EventLog`）是 BMC 带外管理的基本功能，此回归会导致 Redfish Event Log 完全不可用。

## 4️⃣ 🐛 bmcweb: OpenBMCManager v1_1_0 CSDL BaseType 继承错误修复（2026-06-25）

| 元数据 | 值 |
|:------|:----|
| **仓库** | `openbmc/bmcweb` |
| **提交** | `68123d480` (merged 2026-06-25) |
| **作者** | Gunnar Mills (IBM) |
| **标题** | `OpenBMCManager: fix v1_1_0.Manager BaseType` |

**问题**: `OpenBMCManager.v1_1_0.Manager` 的 CSDL `BaseType` 被错误设置为 `Resource.OemObject`，而非正确的 `OpenBMCManager.v1_0_0.Manager`。导致 **v1_0_0 的所有属性（包括 Certificates NavigationProperty）被静默丢弃**。由于 bmcweb 运行时会输出 `"@odata.type": "#OpenBMCManager.v1_1_0.Manager"`，RSV 会按该类型解析 OEM 属性，发现 Certificates 等属性不存在定义 → 抛出 **Unknown Property Error**。

**来源**: 问题由提交 `762a74183`（Gerrit `#85785`）引入。

**修复**: 将 `BaseType` 修正为 `OpenBMCManager.v1_0_0.Manager`，使用标准 CSDL 版本化模式。

**重要性**: 🟡 **中** — 影响 Oem/OpenBmc 下 Certificates 属性的 RSV 校验通过率，IBM 主导修复。
