## 2️⃣ phosphor-user-manager: 修复固件升级后预定义组丢失

- **来源**: OpenBMC `phosphor-user-manager` commit `19c08834`
- **作者**: Chandramohan Harkude · 合入日期: 2026-06-23
- **根因**:
  - `/etc/group` 位于 overlayfs 上层（persistent rwfs），一旦运行时被修改，overlayfs 将整个文件提升到上层，永久遮蔽 rootfs 中的新定义
  - 固件升级新增预定义组（如 `redfish`、`ipmi`、`ssh`、`hostconsole`）后，已部署设备上该组永远不可见，需 factory reset 才能恢复
- **修复**: `ensurePredefinedGroupsExist()` — 初始化时用 `getgrnam_r` 逐一检查，缺失即 `groupadd`
- **重要性**: 🔴 高 — 直接影响 BMC 固件 OTA 后用户权限分配的可用性

## 3️⃣ phosphor-host-ipmid: ARM OEM SSIF 能力支持

- **来源**: OpenBMC `phosphor-host-ipmid` commit `4032853c0b`
- **作者**: Manoj Kumar Prabakaran (ARM) · 合入日期: 2026-06-23
- **关键特性**:
  - 新增 in-tree ARM OEM IPMI provider 库，注册 `Get System Interface Capabilities` 命令（`0x06 0x57`）
  - 返回 SSIF capability: 支持 multipart transfer、PEC（packet error checking）、255B 输入 / 252B 输出消息
  - `ipmitool raw 0x06 0x57 0x00` → `00 88 ff fc` 验证通过
- **重要性**: 🟡 中 — ARM 服务器 BMC 生态（Ampere等）补齐 SSIF 协议能力
