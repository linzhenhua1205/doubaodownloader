## 1️⃣ 🔥 bmcweb: 账户禁用时立即失效所有活动Session（安全修复）

**来源**: [bmcweb commit `430b30bd5f`](https://github.com/openbmc/bmcweb/commit/430b30bd5f800b5c7dc0b2bd856f71f72b93c785) | 作者: Chandramohan Harkude | 日期: 2026-06-08

**技术详情**:
- **问题描述**: 通过 Redfish `PATCH /redfish/v1/AccountService/Accounts/` 设置 `{"Enabled":false}` 禁用用户时，D-Bus 上 `UserEnabled` 已翻转，但用户已有的所有 X-Auth-Token session **仍然可用**。后续携带该 Token 的请求仍返回 `200 OK`，直到 Token 自然过期。但 Basic auth 已正确返回 `401`。DELETE 用户无此问题（因为 `InterfacesRemoved` 信号触发 `removeSessionsByUsername`）。
- **修复方案**: 在 `include/user_monitor.hpp` 中新增 `onUserPropertiesChanged()` 处理器，监听 `User.Attributes.UserEnabled` 从 true→false 的 D-Bus 属性变更信号，调用 `SessionStore::removeSessionsByUsername()` 立即删除该用户的所有活动 session。
- **影响范围**: 同时覆盖 **IPMI 和 Redfish** 两种管理通道的用户禁用操作。该问题属于中等严重性的安全漏洞（权限提升/持久化）。
- **测试验证**: 创建用户→生成 Token→禁用用户→携带 Token 请求→返回 401

## 2️⃣ bmcweb: PCIeDeviceType Redfish 属性支持（NVIDIA GPU 场景）

**来源**: [bmcweb commit `af1b2b438f`](https://github.com/openbmc/bmcweb/commit/af1b2b438faccba5c7cef52f8f4325e6f2ef3027) | 作者: Eric Liu (NVIDIA) | 日期: 2026-06-04

**技术详情**:
- **功能**: 将 D-Bus 的 `Inventory.Item.PCIeDevice.DeviceType` 属性映射到 Redfish `PCIeDevice.v1_19_0.PCIeDevice` 的 `DeviceType` 字段。
- **映射关系**:
  - `SingleFunction` → `SingleFunction`
  - `MultiFunction` → `MultiFunction`
  - `Simulated` → `Simulated`
  - `Retimer` → `Retimer`
  - `Unknown`/空 → 字段省略（兼容 PDI 默认行为）
- **实现**: `pcie_util.hpp` 新增 `redfishPcieDeviceTypeFromDbus()` 映射函数；`pcie.hpp` 在 `addPCIeDeviceProperties()` 中注入 DeviceType 字段。
- **测试**: 在 NVIDIA nvl32-obmc (Alon8-GBT-20) 平台实测，GPU_0_0 正确返回 `"DeviceType": "SingleFunction"`，ConnectX 网卡路径字段省略（因 D-Bus 属性未注册）。Redfish Service Validator 全部通过。

## 3️⃣ bmcweb: zstd 压缩破坏 FileBody 的严重回归修复

**来源**: [bmcweb commit `5f15b5f014`](https://github.com/openbmc/bmcweb/commit/5f15b5f014bf778f22431935db4fa486e18efe96) | 作者: Tan Siewert (9elements) | 日期: 2026-06-01

**技术详情**:
- **问题**: commit `2d7dc991` 将 body 存储改为 `std::variant` 后，调用 `str()` 对 FileBody（如 Web UI 的 favicon.ico 等文件）会触发 `emplace()`，**原地销毁活跃的 FileBody** 并关闭 FD。后续任何操作访问已关闭的 FD（返回 -1），导致文件流崩溃。
- **根因**: `attemptZstdCompression()` 没有识别 body 是否为 file-backed 类型，尝试在已流式压缩的文件对象上再次进行操作。
- **修复**: 在 `attemptZstdCompression()` 中增加 early return：当 body 是 file-backed 类型时直接跳过，因为文件已由 writer 中的 `zstdCompressor` 处理。
- **影响**: 修复了启用 `Accept-Encoding: zstd` 时访问 Web UI 静态文件的回归。

## 4️⃣ 🔥 phosphor-dbus-interfaces: 固件更新错误体系对齐 Redfish Update 1.4.0 规范

**来源**: [phosphor-dbus-interfaces commit `f597b1c3`](https://github.com/openbmc/phosphor-dbus-interfaces/commit/f597b1c313aed3e5e78bff525f565680c4623071) + [`3f5a4451e1`](https://github.com/openbmc/phosphor-dbus-interfaces/commit/3f5a4451e13d5aaa3e27dd127222479575ec5e04) | 作者: Jagpal Singh Gill | 日期: 2026-06-04

**技术详情**:
- **`ApplyFailed` → `ActivateFailed` 重命名**: Redfish Update registry v1.4.0 已弃用 `ApplyFailed`，替换为 `ActivateFailed`。该变更对齐 DMTF 标准（[Redfish-Publications registries/Update.1.4.0.json#L299](https://github.com/DMTF/Redfish-Publications/blob/main/registries/Update.1.4.0.json#L299)）。
- **新增 `UpdateNotApplicable` 错误类型**: 表示固件更新镜像不适用于目标设备，对应 Redfish Update v1.4.0 注册表中的同名消息 ID（[#L524](https://github.com/DMTF/Redfish-Publications/blob/main/registries/Update.1.4.0.json#L524)）。
- **影响**: 两个 commit 共同完善了固件更新流程中的错误报告体系，使 OpenBMC 的 D-Bus 错误类型与 Redfish 标准完全对齐。

## 5️⃣ pldm: 从 placement new 迁移到 `start_lifetime_as`（GCC-16 兼容性）

**来源**: [pldm commit `842113b534`](https://github.com/openbmc/pldm/commit/842113b5349cd8dcb1bdbcc8d2845449ffe8317f) | 作者: Patrick Williams | 日期: 2026-05-27

**技术详情**:
- **问题**: GCC-16 新增 `-Wuninitialized` 和 `-Wmaybe-uninitialized` 警告，广泛使用的 placement new 模式（将指针转换到已有存储上）在这些检查下触发误报。
- **修复**: 使用 `std::start_lifetime_as` 替代 placement new，更准确表达"在已有存储上启动生命周期"的意图。提供 GCC-15 兼容的 polyfill（GCC-15 没有 `start_lifetime_as`）。
- **备注**: 仓库中还有大量 `reinterpret_cast` 调用需后续迁移，以避免 GCC 后续版本中的未定义行为。
