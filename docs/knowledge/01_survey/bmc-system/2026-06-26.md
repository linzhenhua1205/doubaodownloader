## 1️⃣ 🔒 phosphor-user-manager: 固件升级后预定义组自动恢复 🆕🔥

| 元数据 | 值 |
|:------|:----|
| **仓库** | `openbmc/phosphor-user-manager` |
| **提交** | `19c08834` (merged 2026-06-23) |
| **作者** | Chandramohan Harkude |
| **标题** | `user-mgr: create missing predefined groups on init` |

**背景痛点**: OpenBMC 的 `/etc/group` 位于 overlayfs 上。固件升级后，若 rootfs 新增预定义组（如 `hostconsole`），该组**永远不会在已部署设备上出现**——因为旧的 persistent `/etc/group` 持续遮蔽下层 rootfs。用户需执行工厂重置才能恢复。

**解决方案**: `initialize()` 中新增 `ensurePredefinedGroupsExist()`，启动时用 `getgrnam_r` 检查每个预定义组（redfish/ipmi/ssh/hostconsole），缺失时自动 `groupadd` 写入 overlay upper 层。

**关键设计**:
- 幂等性：仅检查已知的预定义组集合
- 容忍缺失错误，不阻塞启动流程
- 测试已验证：手动删除后重启，组被自动重建

**重要性**: 🔥 **高** — 直接影响已部署设备的账号管理安全性，解决了一个长期以来可能导致服务访问控制异常的 overlayfs 遮蔽问题。

## 2️⃣ 🎯 bmcweb: Processor EnvironmentMetrics + PowerLimitWatts 🆕🔥

| 元数据 | 值 |
|:------|:----|
| **仓库** | `openbmc/bmcweb` |
| **提交** | `45b86809` (committed 2026-06-25) |
| **作者** | Ender Hsieh (NVIDIA) |
| **标题** | `bmcweb: add EnvironmentMetrics support for Processors` |

**实现功能**: 为 Processor 资源添加 **Redfish EnvironmentMetrics** 端点，暴露 GPU 功耗限制信息：
- `/redfish/v1/Systems/{system}/Processors/{processor}/EnvironmentMetrics`
- `PowerLimitWatts.DefaultSetPoint` ← `DefaultPowerCap`
- `PowerLimitWatts.SetPoint` ← `PowerCap`
- `PowerLimitWatts.AllowableMax/Min` ← `Max/MinPowerCapValue`
- `PowerLimitWatts.ControlMode` ← `PowerCapEnable`

**实现机制**: 利用 D-Bus Association (`controlled_by`) 自动发现 inventory 对象关联的 `Control.Power.Cap` 接口。仅在关联存在时暴露 EnvironmentMetrics 链接。

**验证结果**: Redfish Service Validator 通过（Pass: 8,945, Fail: 0）

**重要性**: 🔥 **高** — NVIDIA GPU 功耗管理 Redfish 全链路（NSM → D-Bus → Redfish）贯通的重要一环，对智算集群的功耗封顶/调度管理有直接价值。
