## 五、AI PM 的 Agent 决策框架（Product School, 2026-03-03） ⭐

**来源**: Product School CEO Carlos Gonzalez de Villaumbrosia, "When Should Product Teams Use AI Agents?" 2026-03-03

### 📌 核心发现

**自主性预算（Autonomy Budget）** 是 AI 时代 PM 的核心概念。产品团队不应问"AI agent 好不好"，而应问"这个工作流能承受多少自主性"。

### 📊 关键数据

- **Gartner 预测**：40% 的 agentic AI 项目将在 **2027 年前被取消**
- **MIT NANDA 报告**：95% 的组织从 GenAI 倡议中获得 **零回报**
- Agent 的本质是**昂贵的自主权**——使用条件必须严格

### 🔬 "Should This Be an Agent?" 24 分决策矩阵

| 维度 | 0分(不用) | 1分(考虑) | 2分(适合) |
|:----|:---------|:---------|:---------|
| 规则自动化可行性 | 规则清晰 | 有边界情况 | 规则频繁打破 |
| 工作流复杂度 | 单步骤 | 少数步骤 | 多工具多步骤 |
| 错误成本 | 高+不可逆 | 中等 | 低+可逆 |
| 人过度信任风险 | 非常高 | 可能 | 不太可能 |
| 输入变异性 | 高度可预测 | 混合 | 混乱/非结构化 |

**总分解读**：0-8 不要用 agent → 用脚本/规则；9-16 推荐模式（agent建议，人决定）；17-20 agent+审批；21-24 强 agent 候选但需 guardrails

### 🎯 核心方法论：四阶自主性光谱

```
Reactive (零自主) → Tool-using (辅助自主) → Semi-autonomous (人在回路) → Fully autonomous (目标驱动)
   安全/即时价值        快速ROI/低治理      高杠杆/可控风险       最大价值/最高治理需求
```

**Product School 建议**：大多数团队应长期停留在 B 和 C 阶

### 💡 案例：Dow Jones（道琼斯）AI 摘要实验

- Ryan Daly Gallardo（SVP of Product, Consumer）在 ProductCon 分享
- **做法**：AI 生成新闻摘要，**每份都经人工编辑审核后上线**
- 核心方法论：AI 做70%重活，人类保留最终裁决权
- 不是问"AI 是否影响参与度"，而是 **结构化 A/B 测试**验证效果

### 🧠 三大观点阵营

| 阵营 | 主张 | 代表人物/数据 |
|:----|:-----|:------------|
| Move Faster | 自主性=速度，Shipping Velocity 成为护城河 | Elena Verna, Lovable Head of Growth; McKinsey 研究 |
| Trust First | 自主性扩大爆炸半径，安全是生产力的前提 | Jeetu Patel, CPO at Cisco; OWASP "Excessive Agency" 风险 |
| Productive Synthesis | 能承受多少自主性取决于团队成熟度 | Carlos 本人立场 |

### ⚠️ 四个危险信号（不该用 agent 的场景）

1. **不可逆操作**（定价变更、删除数据、发消息给客户）→ 必须人工审批
2. **agent 可读取不可信内容并执行操作** → Prompt Injection 风险（OWASP #1 风险）
3. **规则已经工作的很好** → agent 并非万能，确定性自动化更可靠
4. **"人在回路"变成时间黑洞** → Wharton 研究："几乎总对"的 AI 反而削弱人类警惕性

## 六、AI 产品实验方法论（Product School, 2026-03-18） ⭐

**来源**: Product School CEO Carlos Gonzalez de Villaumbrosia, "AI Experimentation: How AI PMs Test and Learn Faster" 2026-03-18

### 📌 核心发现

AI 实验与常规实验有本质差异——**非确定性输出**、模型漂移、信任度量——需要全新的实验框架。

### 📊 关键数据

- **McKinsey 数据**：多数公司仍停留在 AI 实验阶段，只有少数已规模化超越试点
- 实验设计决定结果可信度：**模型稳定性 + 足够样本量 + 预设成功标准** 缺一不可

### 🔬 五类 AI 实验方法选择框架

| 实验类型 | 何时使用 | 适用问题 |
|:--------|:---------|:--------|
| **A/B 测试** | 需要干净回答 | "这个变化是否影响了指标？" |
| **原型测试** | 验证理解和信任 | "用户是否理解 AI 能做什么？" |
| **假门测试 (Fake-door)** | 验证需求（无构建成本） | "用户想要这个功能吗？" |
| **金丝雀发布 (Canary)** | 真实功能安全上线 | "生产环境表现如何？" |
| **多变量/自适应** | 优化阶段 | "如何调优？" |

### 💡 案例：媒体公司测试 AI 摘要

- 团队不猜测 AI 摘要是否会影响用户参与度
- 提出单一假设问题：**"AI 摘要对用户参与度有何影响？"**
- 运行干净的 A/B 测试，得出可量化的答案
- 关键原则：**模型版本必须冻结**，测试期间不能更新

### 🧠 三层面指标框架

```
模型层面：Accuracy, Precision, Recall, F1, Latency
产品层面：采用率、满意度、留存率、任务成功率
业务层面：转化率、收入提升、成本节省、风险降低
```

**核心洞察**：AI 时代的实验不仅追踪"模型是否对"，更要追踪**用户是否觉得有用且可信**

### ⚠️ AI 实验特有挑战

1. **非确定性输出** → 冻结模型版本 + 足够样本量
2. **模型漂移** → 上线后持续监控 + 警报机制
3. **偏差与公平性** → 按用户分群切片分析 + 边缘测试
4. **信任度量** → 必须在模型指标之外纳入信任、安全与体验指标
