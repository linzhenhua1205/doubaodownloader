## ② 🚀 Google-SpaceX $9.2亿/月算力大单 — 超节点级别的算力租赁

**来源**：[TechCrunch](https://techcrunch.com/2026/06/05/google-will-pay-spacex-920m-per-month-for-compute/) Sean O'Kane, 2026-06-05 | Google SEC 监管文件

### 核心条款
| 指标 | 数据 |
|:-----|:------|
| **月度费用** | **$9.2亿/月**（$920M/month） |
| **协议期间** | 2026年10月 — 2029年6月（约 **33个月**） |
| **总合同金额** | 约 **$303.6亿** |
| **算力资源** | 约 **110,000 块 NVIDIA GPU** + CPU、内存及相关组件 |
| **取消费** | 双方均可在 **2026年12月31日后** 提前90天通知终止 |
| **爬坡期** | 2026年9月前以折扣价逐步接入 |
| **性质** | Google 自称「**桥接容量（bridge capacity）**」 |

### 超节点规模分析
**11万GPU** 是什么概念？对比同类交易：

| 交易 | 规模 | 单价 |
|:-----|:-----|:-----|
| Google × SpaceX（本交易） | 11万GPU, **$303.6亿总合同** | ~$9.2亿/月 |
| Anthropic × SpaceX（2026年5月） | 不详（Colossus 1数据中心全部算力） | **$12.5亿/月** |
| Google 自身 Capex（2026年） | $1,800亿+ | — |

### 超节点基建的深层信号
| 维度 | 解读 |
|:-----|:------|
| **GPU短缺** | 全球最大 AI 计算持有者 Google（拥有 TPU + GPU）仍需外租 **11万GPU**，说明顶级 CSP 的算力缺口仍然极大 |
| **NVIDIA护城河** | Google 拥有自家 TPU 但仍在采购 NVIDIA GPU，反映 **NVIDIA GPU 训练生态的不可替代性** |
| **算力即最大成本** | 该交易与 Anthropic 的 $12.5B/月 交易共同表明 **算力正成为 AI 公司的最大单一成本项和战略瓶颈** |
| **超节点规模** | 11万GPU = 约 **1,528 个 NVL72 机架**（每架72 GPU）或 **458 个 NVL576 超节点**（每架576 GPU），这是一个超大规模超节点集群 |

## ③ ⚡ Flex CTO：机架功耗5年内从10kW到1MW — 超节点供电架构演进路线图

**来源**：[DIGITIMES](https://www.digitimes.com) Flex CTO COMPUTEX 专访, 2026-06-06

### 演进路线
| 阶段 | 时间 | 单机架功耗 | 散热方案 | 供电架构 |
|:----|:-----|:----------|:---------|:---------|
| 传统 | 2021年 | **10kW** | 风冷 | 2N UPS |
| 现状 | 2026年 | **40~120kW** | 液冷已成标配 | HVDC+BBU |
| 近期 | 2027~2028年 | **200~500kW** | 液冷+冷板混合 | 800V直流+分布式储能 |
| 中期 | 2029~2030年 | **500kW~1MW** | 浸没式+两相冷却 | 全直流微电网 |
| Flex CTO断言 | **最难的一跳还在前面** | — | — | — |

### 对超节点供电架构的根本性挑战
| 挑战 | 说明 |
|:-----|:------|
| **热密度** | 1MW/rack 意味着单位体积热通量激增，二次流体侧需要更高的冷却能力 |
| **供电架构压缩** | 从5级变换（电网→UPS→PSU→机架→设备）压缩到3级（电网→直流母线→设备） |
| **配电密度** | 母线排/铜排截面需成倍增加，机架内走线和热管理矛盾加剧 |
| **运维安全** | 极高电压/电流下的运维安全标准尚未建立 |
| **成本非线性** | 100倍功耗对应的电气和冷却基础设施成本非线性增长 |

### 行业信号
> Flex 作为全球最大电子制造服务商之一，其 CTO 的公开论断具有强烈的 **产业信号效应**：超节点机架功耗的「**10kW→1MW 跨越**」是整个数据中心基础设施行业的终极课题。

### 当前应对路线（多线并进）
| 方案 | 进展 |
|:-----|:------|
| ODCC 冷板液冷超流体规范（ODCC2504004） | 已发布，合成油介电冷却 |
| 800V HVDC 直流供电 | 英伟达提交 OCP 标准，台达展示方案 |
| 分布式 BBU/储能 | 部分 CSP 已部署试点 |
| 液冷从可选→唯一 | 40-120kW/rack 物理极限下，液冷已成标配 |

## ④ 🏭 NVIDIA DSX OS v1.0 开源发布 — AI Factory 超节点操作系统

**来源**：[NVIDIA Technical Blog](https://developer.nvidia.com/blog/nvidia-dsx-os-delivers-open-modular-software-for-operating-ai-factories-at-scale/), 2026-05-31（本周持续在首页重点推）

### 超节点运维的「操作系统」级定义
NVIDIA DSX OS 是一套 **开源、模块化** 的软件组件集合，专门为 AI Factory 规模化运营设计。它的发布标志着 AI 超节点运维从「人工拼凑工具链」向「统一平台 OS」的范式转变。

### 八大组件矩阵
| 组件 | 功能 | 对超节点价值 |
|:-----|:------|:------------|
| **DSX Exchange** | IT/OT通信枢纽，连接计算/网络/电力/冷却 | 超节点四大子系统（GPU/网络/供电/冷却）的统一控制面 |
| **DSX MaxLPS** | 动态功率策略引擎，GPU→机架→冷却→负载级细分 | **固定功率预算内多运行 40% GPU**，推理性能影响极小 |
| **DSX Flex** | 电网服务连接库，需求响应/削峰/可再生能源适配 | 超节点与电网的协同降成本 |
| **NVIDIA Infra Controller (NICo)** | API驱动的裸金属生命周期管理 | BlueField DPU 硬件级租户隔离 |
| **AI Cluster Runtime (AICR)** | 版本锁定运行时配置 | 消除配置漂移 |
| **NVSentinel** | K8s原生GPU故障检测与自动修复 | 秒级封锁+排空 |
| **Fleet Intelligence** | 全局舰队可见性/完整性验证 | 超大规模集群的统一监控 |
| **KAI Scheduler + Run:ai** | GPU感知智能调度 | 拓扑感知+分片+层级配额 |

### 三大核心目标量化数据
| 目标 | 手段 | 量化指标 |
|:-----|:------|:---------|
| ⚡ **更快创收** | 开源组件复用 NVIDIA DGX Cloud 已验证运维栈 | 消除数月定制开发 |
| 💡 **更高能效** | MaxLPS 四级动态功率分配 | 固定功率内 **多跑40% GPU** |
| 🛡️ **更高可靠性** | 自动修复+版本一致性+Fleet Intelligence | 秒级故障发现与自愈 |

### 生态伙伴
CoreWeave、Lambda、Nscale、Red Hat、Supermicro、Mirantis、Rafay、Spectro Cloud、Vultr 等已集成在生产中使用。

## ⑤ 💎 IMEC MTP 铌酸锂 200mm SiPho 晶圆级异质集成量产突破

**来源**：[arXiv:2605.28971](https://arxiv.org/abs/2605.28971), IMEC/Ugent 24位作者, 2026-05-27

### 核心突破：超节点光互联的量产制造基础
IMEC 首次在 **4片整200mm SiPho 晶圆** 上实现薄膜铌酸锂(TFLN)微转移印刷(MTP)异质集成：

| 指标 | 数据 |
|:-----|:------|
| **晶圆尺寸** | 4 × **200mm** 整晶圆（非 die 级） |
| **放置精度** | 3σ **420nm**（晶圆级） |
| **打印良率** | **>95%**（600相位调制器+300幅度调制器） |
| **插入损耗** | <2 dB |
| **半波电压** | 4V（推挽配置） |
| **调制带宽** | **>70 GHz** |
| **工艺** | 微转移印刷（μTP），可扩展至更大晶圆 |

### 对超节点光互联的意义
| 维度 | 影响 |
|:-----|:------|
| **CPO量产可行性** | SiPho 异质集成从「实验室die级」到「晶圆级量产」的关键一步，为 **CPO收发器的规模化制造**铺路 |
| **超节点scale-up** | 200mm晶圆级>70 GHz调制器意味着超节点内光互联的带宽密度可以量产化实现 |
| **成本下降曲线** | 晶圆级良率>95%是光互联成本降至可与铜缆竞争的关键拐点 |
| **Marvell T100 CPO 协同** | 与 Marvell COMPUTEX发布的 T100 Teralink CPO 交换机形成 **器件+系统** 的上下层支撑 |

### 同步进展
| 成果 | 来源 | 核心数据 |
|:-----|:------|:---------|
| AI Agent自主设计硅光调制器 | [arXiv:2606.00915](https://arxiv.org/abs/2606.00915) | LLM Agent自主完成MRM/MZM/版图设计，无需人工干预 |
| GF Fotonix 0.08 pJ/bit单片接收前端 | [arXiv:2605.06808](https://arxiv.org/abs/2605.06808) | 28.9 GHz带宽，业界领先能效 |

## ⑥ 📊 超节点供应链/BOM 成本结构重塑 — 内存占比从15%升至35-40%

**来源**：[DIGITIMES](https://www.digitimes.com), 2026-06-06 | WSTS 2026行业预测

### WSTS 全球半导体市场预测
| 指标 | 数据 |
|:-----|:------|
| 2026全球半导体市场 | **$1.5 万亿** |
| 内存市场增长 | **+250% YoY** |
| AI芯片占比 | 预计超 **35%** |
| 供应缺口 | 内存缺口延至 **2028+** |

### 对超节点BOM成本的冲击
| 部件 | 2023年BOM占比 | 2026年BOM占比 | 趋势 |
|:-----|:-------------|:-------------|:-----|
| **HBM** | ~15% | **~25-30%** | 🚀 暴涨 |
| **GPU** | ~35% | **~20-25%** | 占比相对下降（HBM占比提升更快） |
| **DDR5** | ~3% | **~5-8%** | 稳定上涨 |
| **PCB+CCL** | ~5% | **~8-12%** | 供应紧张，配额制 |
| **其他** | ~42% | **~25-30%** | 被压缩 |

### 超节点成本重构
> **核心判断**：AI超节点 BOM 成本结构正在从「GPU主导」转向「**GPU+HBM双轮驱动**」，HBM 正成为超节点成本中最昂贵的单品部类。这对超节点的定价、部署策略、以及替代方案（如CXL内存池）的采用率产生深远影响。
