## ① 🔥 **Qualcomm Investor Day 2026：超节点级 CPU + HBC + UALink 路线图 + Modular 收购对标 CUDA**（6/24 纽约）

**来源**: ServeTheHome 现场报道（Patrick Kennedy），2026-06-24
**标题**: 「Qualcomm Investor Day 2026 Data Center Announcements CPUs, AI Accelerators, and More」
**链接**: https://www.servethehome.com/qualcomm-investor-day-2026-data-center-announcements-cpus-ai-accelerators-and-more/

### 核心内容

Qualcomm 在纽约 Investor Day 上全面重启数据中心战略，CEO Christiano Amon 将 **机架级数据中心基础设施**（rack-scale data center infrastructure）列为公司未来三大方向的第一个。

#### Qualcomm Dragonfly C1000 CPU 技术参数

| 维度 | 规格 | 超节点相关性 |
|:----|:-----|:------------|
| **核心数** | **250+ cores** (chiplet 设计) | 对标 Intel Xeon 6+ / AMD Venice |
| **主频** | **~5GHz** | 远高于竞争对手密集核版本（Turin Dense / Clearwater Forest 主频偏低），宣称 2× 更好 per/watt |
| **PCIe** | **PCIe Gen7** | 面向下一代超节点互联需要 |
| **内存** | LPDDR + 可选 **HBC 高带宽计算** attach | 详见下文 |
| **互联** | CXL + UALink 集成规划 | UALink 直接接入 |
| **RAS** | 企业级 RAS 特性 | 面向数据中心场景 |

> 💡 **关键判断**：Dragonfly C1000 瞄准的不是传统企业市场，而是 **大型系统集成商/超大规模客户**。ServeTheHome 指出这与之前的 Centriq 2400 路线完全不同。

#### High Bandwidth Compute (HBC) — 内存堆叠技术

Qualcomm 发布的最具创新性的技术：

| 维度 | 描述 | 行业影响 |
|:-----|:-----|:---------|
| **架构** | 将内存直接堆叠在计算 tile 之上 | 无需 CoWoS 中介层 → HBM 内存 |
| **优势** | 消除在 interposer 上的数据移动功耗 | vs HBM: 同容量但接近 SRAM 性能 |
| **能效** | **最佳性能/瓦**（宣称） | 对超节点级功耗密度至关重要 |
| **路线图** | **AI250 代（2027）** 推出 HBC Gen1 → 后期 **HBC Gen2 集成 UALink/ESUN** 互联 | **2027 年超节点互联格局将生变** |

> **超节点意义**：HBC 从物理架构层面绕过了 HBM 的 interposer 功耗墙，对超节点整机柜设计中的供电/散热约束可能有重大影响。

#### 生态与客户

| 维度 | 内容 | 意义 |
|:-----|:-----|:------|
| **Meta 合作** | Mark Zuckerberg 宣布 **多代协议**使用 Qualcomm 处理器 | 继 Arm AGI CPU 后又一重大数据中心 CPU 客户 |
| **Microsoft** | Satya Nadella 视频站台，合作从 PC 到 AI Agent 再到数据中心 | 云厂商背书 |
| **Modular 收购** | 收购 Modular（AI 编程框架公司）对标 CUDA/Dynamo/Triton | 解决 Qualcomm AI100 系列长期软件栈短板 |
| **Alphawave 合作** | 已验证基于 Alphawave 的互联方案 | 高端 SerDes/光互联能力 |

#### Qualcomm 数据中心财务目标

| 指标 | 原目标（18个月前） | 更新后目标 | 核心驱动 |
|:----|:------------------|:----------|:---------|
| **FY29 总营收目标** | $22B | **$40B** | 数据中心为主要增长引擎 |
| **27年单客户规模** | — | 2个超大规模客户 >$1B/客户 | AI 加速器 + CPU 产品 |
| **TAM 份额** | — | 5-7年内 >5% 的 $1T+ TAM | CPU + AI 加速器 + 定制芯片 + 互联四条线 |

> 💡 **决策洞察**：Qualcomm 数据中心的四条产品线（CPU/AI加速器/定制芯片/互联）与 **超节点所需的计算-互联-加速全栈** 高度吻合，2027-2028 将成为超节点生态的关键变数。

## ② 🔍 **LineShine LX2 芯片架构深度解剖**（ISC 2026 后首发）

**来源**: ServeTheHome 深度分析（Patrick Kennedy），2026-06-23
**标题**: 「Arm CPUs Take Number 1 in Latest Top500 List with Chinese LineShine」
**链接**: https://www.servethehome.com/arm-cpus-take-number-1-in-latest-top500-list-with-chinese-lineshine/

### LX2 CPU 架构细节（此前未完整披露）

#### 芯片架构

```
┌─────────────────────────────────────────────┐
│              LX2 CPU (304核)                 │
├──────────────────────┬──────────────────────┤
│   Die 0 (152核)      │   Die 1 (152核)      │
│  ┌───┬───┬───┬───┐  │  ┌───┬───┬───┬───┐  │
│  │NUMA│NUMA│NUMA│NUMA│  │NUMA│NUMA│NUMA│NUMA│
│  │ 0  │ 1  │ 2  │ 3  │  │ 4  │ 5  │ 6  │ 7  │
│  │38核 │38核 │38核 │38核 │  │38核 │38核 │38核 │38核 │
│  └───┴───┴───┴───┘  │  └───┴───┴───┴───┘  │
│ EACH NUMA DOMAIN:                         │
│   4GB HBM + 32GB DDR5 (32GB→256GB总容)    │
└─────────────────────────────────────────────┘
```

| 维度 | 规格 |
|:-----|:-----|
| **架构** | 双 die 设计，每 die 152 核 Armv9，支持 **SVE + SME**（矩阵加速） |
| **核心总数** | 304 核 / CPU |
| **主频** | 1.55 GHz |
| **NUMA 域** | 8 域（每 die 4 域 × 38 核） |
| **HBM** | 32GB / CPU（每 NUMA 域 4GB） |
| **DDR5** | 256GB / CPU（每 NUMA 域 32GB） |
| **内存模式** | 可配为 flat 拓扑（HBM + DDR 容量相加）或 cache 模式（HBM 为 DDR 缓存） |
| **TDP** | **690W / CPU**（GPU 级别的功耗） |
| **集成网络** | **800Gbps NIC 片内集成**（异常少见，自 Intel Xeon+OmniPath 后首次） |

#### 系统级架构

| 层级 | 组成 | 规模 |
|:-----|:-----|:----|
| **Node** | 2×LX2 CPU / node | 608 核/node |
| **Blade** | **8 节点**/blade (16 CPU) | 4,864 核/blade |
| **Compute Frame** | 16 blade/frame (256 CPU) | 77,824 核/frame |
| **Cabinet** | 2 frame/cabinet (512 CPU) | 155,648 核/cabinet |
| **全系统** | ~89-90 cabinets | ~**13.79M 核心** |

#### 网络互联：LingQi 1.6Tbps

- **每节点** 1.6Tbps 全连接
- **拓扑**：4 层 Fat-Tree，含一层光互联（optical layer）
- **效率**：HPL 实测 2.198 EF / 理论峰值 2.74 EF = **80% 效率**，远超美国三大 exascale 系统的 50-65%
- **42MW 功耗**，52 GFLOPs/watt（低于 El Capitan 60 GFLOPs/watt）

> 💡 **关键洞察**：LineShine 证明了 **纯 CPU 超大规模互联可以超越 GPU 系统的 HPL 效率**，且 Arm 生态已具备从芯片→OS（Kylin）→互联（LingQi）→系统的全栈能力。虽然 AI 混合精度性能弱于 GPU 系统（HPL-MxP 仅 #4），但对超节点架构中 **CPU 节点作为控制面/存储面的角色定位具有参考价值**。

### 关键信号

- **超节点战局已从 NVIDIA 单核驱动 → 多极竞争**：Qualcomm 明确以机架级数据中心为目标，NVIDIA ISC 发布 Vera Rubin 超节点，中国 LineShine 证明纯 CPU 超大规模可行
