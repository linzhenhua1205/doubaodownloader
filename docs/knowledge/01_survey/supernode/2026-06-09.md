## ① 🔥 NVIDIA Vera Rubin 散热架构重大调整 — 取消双片冷却设计

**来源**：[DIGITIMES Asia](https://www.digitimes.com) Exclusive, 2026-06-08
> 标题: "Exclusive: Nvidia drops dual-piece cooling architecture for Vera Rubin platform"

### 核心事件

NVIDIA 在 COMPUTEX 2026 闭幕后，被 DIGITIMES 独家爆料已对 **Vera Rubin 平台**的散热架构进行了 **重大重新设计**：**取消了此前计划的双片冷却架构（dual-piece cooling architecture）**。

| 维度 | 此前方案（Blackwell/早期规划） | 新方案（Vera Rubin 正式版） |
|:-----|:-----------------------------|:---------------------------|
| **冷却架构** | 双片冷却（dual-piece） | 全新整体式散热设计 |
| **散热器设计** | GPU+内存分区冷却 | 一体化散热方案 |
| **机柜集成** | 需适配双路冷却通路 | 简化为单路冷却架构 |
| **装配复杂度** | 较高（双冷却组件对齐） | 降低（单组件集成） |
| **热管理策略** | 分区独立控温 | 整体热平衡管理 |

### 对超节点的影响

| 维度 | 分析 |
|:-----|:------|
| **超节点机架设计** | NVL72/NVL576 的冷却管路布局需重新适配，原有双冷却通路设计转为单路；对 **Supermicro/Foxconn/Quanta** 等整机柜集成商意味着设计迭代 |
| **液冷方案供应商** | 冷却液类型、管路走向、CDU配置均可能调整；**Frore LiquidJet Nexus** 等新方案获得机会 |
| **供应链切换成本** | COMPUTEX 期间（6月初）刚展示 Vera Rubin 量产机架，一周内即确认冷却架构变更，说明 **散热是 Vera Rubin 交付的最后一块拼图** |
| **与 Blackwell 的差异** | Blackwell 平台（B200/B300）的空冷/液冷方案已成熟定型，Vera Rubin 的散热重设计表明 **Vera 的计算密度和热密度显著高于 Blackwell**，需要彻底重新设计散热架构 |
| **发布节奏信号** | 散热设计在量产阶段仍在调整，可能暗示 **Vera Rubin NVL72 量产爬坡时间表存在短期不确定性** |

### 多方验证

- 6月6日 Jensen Seoul 行期间已有业界观察人士注意到 Vera Rubin 散热方案的未最终确认
- Supermicro 在 COMPUTEX 展示的 Vera Rubin NVL72 机架采用了 **新型 1,000× 电绝缘冷却液**（Tom's Hardware, 6月1日披露），可能与散热架构变更有关
- 散热架构重构 + 新型冷却液 = **Vera Rubin 散热方案的整体升级**

> ⚡ **核心判断**: Vera Rubin 的散热架构调整是 NVIDIA 从 Blackwell 向 Vera 代际跨越中的**必然阵痛**。单GPU 50 PFLOPS FP4 的散热挑战巨大，双片冷却被取消可能意味着 NVIDIA 找到了更优的全集成散热方案。

## ② 🚀 Samsung 已开始为 Vera Rubin 出货 HBM4，正在谈判 HBM5 及下一代代工合作

**来源**：[DIGITIMES Asia](https://www.digitimes.com) Realtime Headlines, 2026-06-09 05:11
> 标题: "Samsung is already shipping HBM4 for Vera Rubin — and is now negotiating HBM5 and next-gen foundry with Nvidia"

### 核心数据

| 指标 | 数据 |
|:-----|:------|
| **产品** | **HBM4** 内存 |
| **目标平台** | **NVIDIA Vera Rubin** |
| **供应状态** | ✅ **已开始出货** |
| **下一阶段** | HBM5 供应谈判中 |
| **同时谈判** | **下一代代工合作**（Samsung Foundry 争取 NVIDIA 先进节点订单） |

### 超节点供应链里程碑

| 维度 | 分析 |
|:-----|:------|
| **HBM4 量产进度** | Samsung 率先在 **6月上旬** 向 NVIDIA 出货 HBM4，比此前市场预期的 Q3 2026 有所提前 | 
| **多源供应** | SK Hynix 同期也加速龙仁 HBM 工厂建设 + NVIDIA × SK Hynix 多年期全产品线合作（见下文）→ **HBM4 的双源供应格局已确立** |
| **Vera Rubin HBM 规格** | Vera Rubin 单 GPU 预计搭配 **12~16 层堆叠 HBM4**，单 GPU HBM 容量有望达 **~288-384 GB**，总带宽 **8+ TB/s** |
| **Samsung Foundry 战略意图** | Samsung 正在将 HBM4 供应作为敲门砖，争取 NVIDIA **2nm GAA 代工订单**，这是 Samsung Foundry 在 2nm GAA 良率受挫后的关键战略动作 |
| **对超节点的影响** | HBM4 提前出货意味着 **Vera Rubin NVL72 超节点的核心存储瓶颈已解除**，整机柜量产装配有望按原计划推进 |

### 与 COMPUTEX Jensen Seoul 行的关联

在 Jensen 首尔行（6月5-6日）中，与 Samsung/SK Hynix 高层会晤的核心议题就是 **HBM4 供应路线图锁定**。Samsung 仅 3 天后就宣布已出货，说明 **Jensen 行在供应链锁定上取得了实质进展**。

## ③ 💎 NVIDIA × SK Hynix 多年期全产品线内存合作 — 超节点 HBM + 全栈存储锁定

**来源**：[DIGITIMES Asia](https://www.digitimes.com) Realtime Headlines, 2026-06-08
> 标题: "Nvidia and SK Hynix announce multiyear memory partnership covering AI servers, PCs, and robotics"

### 合作协议核心架构

| 产品线 | 覆盖范围 | 说明 |
|:-------|:---------|:------|
| **HBM（AI服务器）** | HBM4/HBM5 | 超节点核心内存，延续独家供应合作 |
| **DDR5（AI服务器）** | 服务器主存 | 覆盖 AI 训练/推理服务器平台 |
| **GDDR7（PC/工作站）** | 消费级 AI 显卡 | RTX Spark / DGX Spark 等 SFF 设备 |
| **LPDDR（机器人/边缘）** | 嵌入式计算 | 面向机器人/自动驾驶等长尾 AI |
| **NAND（企业存储）** | SSD/企业级存储 | AI 数据湖/检查点存储 |

### 对超节点的影响深度

| 维度 | 分析 |
|:-----|:------|
| **HBM 供应锁定** | SK Hynix 作为 NVIDIA HBM 主供应商的地位从 **HBM 单一产品** 升级为 **全产品线战略伙伴**，HBM 供应优先级进一步提升 |
| **超节点 BOM 成本控制** | 多产品线批量采购协议→可能获得更优定价→对超节点 BOM 中的内存成本（占比 35-40%）有稳定作用 |
| **内存供应链稳定性** | 在 DRAM 超级周期（+250% YoY）背景下，多年期全产品线合作确保 NVIDIA 超节点产线的内存供应不断链 |
| **DDR5 和 GDDR7 联动** | AI 服务器（DDR5）+ AI PC/工作站（GDDR7）+ 机器人（LPDDR）的**全产品线覆盖**意义深远——NVIDIA AI 生态从数据中心扩展至终端设备 |
| **竞争格局** | 与 Samsung HBM4 出货形成 **HBM 双源 + 细分产品线分工**：SK Hynix 主供 HBM + 全产品线，Samsung 主攻 HBM4 率先量产 + Foundry 合作谈判 |

### 组合判断

> **SK Hynix × 全产品线 + Samsung × HBM4 首批出货 + Jensen Seoul 行锁单** = NVIDIA 在 2026 年 6 月第一周完成了 **HBM 供应链的史上最大规模锁定**。超节点不再有任何存储供应上的战略不确定性。

## ④ 🏗️ AMD 超节点路线图全面揭晓 — Helios NVL72 对标 + MegaPod 256-GPU

**来源**: [Tom's Hardware - Anton Shilov, March 23, 2026 (6月深度分析再发酵)](https://www.tomshardware.com/)
**相关**: COMPUTEX 2026 AMD EPYC Venice + Instinct MI450 系列展示

### AMD 超节点双产品线

#### Helios 机架系统（72 × MI455X，2026 H2）

| 指标 | 数值 |
|:-----|:------|
| **形态** | **AMD 首款 AI 机架系统**（对标 NVIDIA NVL72） |
| **GPU** | **72 × MI455X**（CDNA 5，FP4 40 PFLOPS/GPU） |
| **互联** | UALink（或 UALink-over-Ethernet） |
| **HBM4 总容量** | **31 TB** |
| **总带宽** | **1,400 TB/s** |
| **FP4 算力** | **2,900 PFLOPS（密集）** |
| **网络** | Pensando Vulcano **800 GbE** NIC（首批 Ultra Ethernet 兼容） |
| **主 CPU** | EPYC Venice 系列（256核 Zen 6） |
| **对标 NVL72** | 算力略低（2.9 vs ~3.6 EFLOPS）但 **HBM 容量更大**，内存密集型负载可能有优势 |
| **交付时间** | 官方 **2026 H2**，传言可能延期至 2027 Q2（AMD 否认） |

#### MegaPod 系统（256 × MI500，2027 末）

| 指标 | 数值 |
|:-----|:------|
| **GPU** | **256 × MI500**（CDNA 6 架构） |
| **HBM** | HBM4E 或 HBM5 |
| **系统形态** | **3 机架系统**：外 2 架各 32 计算托盘（1 CPU+4 GPU），中间 1 架 18 托盘 UALink 交换机 |
| **CPU** | 64 × EPYC Verano（Zen 7 / Zen 6+，A16 工艺，BSPDN 背面供电） |
| **对标 NVL576** | 比 Kyber VR300 多 **78% GPU 封装数**（256 vs 144） |
| **冷却** | **全液冷**（含网络硬件） |
| **时间线** | **2027 年末**，2028 年规模量产 |

### 对超节点竞争格局的影响

| 维度 | 分析 |
|:-----|:------|
| **NVL72 迎来首位直接对手** | AMD Helios 是 **业界首个直接对标 NVIDIA NVL72 的开放生态超节点方案**，基于 UALink 开放标准互联 |
| **UALink 生态成熟度是瓶颈** | Helios 和 MegaPod 都依赖 UALink 交换机（当前生态未成熟），这可能是 AMD 超节点交付的最大风险 |
| **内存容量优势** | Helios 31TB HBM4 总容量超过 NVL72（~20TB），对于 **长上下文 LLM 推理、MoE 模型等内存密集型** 场景有差异化竞争力 |
| **HBM 供应竞争** | AMD 需在 NVIDIA 已锁定大量 HBM 产能的背景下争取 HBM 分配，MI455X 的 HBM4 供应可能受限 |
| **超节点市场从单一→双极** | 2027 年将是首个 **NVIDIA vs AMD 超节点正面竞争** 的年份 |
