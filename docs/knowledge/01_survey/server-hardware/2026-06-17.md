## 1️⃣ 🧮 Tensordyne Napier — 对数数学 AI 处理器 (最有架构影响力的新玩家)

- **来源**: [ServeTheHome — Vic A (Jun 15)](https://www.servethehome.com/tensordyne-napier-ai-processor-announced-with-logarithmic-math/)
- **对比参考**: NVIDIA Blackwell, Groq, Cerebras

### 架构核心——对数数学 (Logarithmic Math)

Tensordyne 的颠覆性在于**改变了数学本身**而非堆叠算力：
- 用 **对数变换将乘法运算转化为加法运算** → 加法器面积/功耗远小于乘法器
- 释放的硅面积重新分配给 **SRAM** → 实现 **5× Blackwell 的片上 SRAM (256MB)**
- 对 AI 推理负载而言，**更大 SRAM = 更多数据在片上 = 更少数据搬运 = 更低延迟**

> **关键判断**: 这是第一家以 "改变数学而非缩放规模" 为差异化路线的 AI 芯片公司。是否有效取决于实际模型部署验证。

### TDN72 整机柜规格

| 参数 | 数值 | 备注 |
|:-----|:-----|:-----|
| 制程 | TSMC 3nm | 经 Broadcom 流片 |
| 晶体管 | **1380亿 (138B)** | 单芯片 |
| 算力 (单芯片) | **2.1 PFLOPS** | @1.33GHz 加速核 |
| CPU | 1.5GHz, 整合于芯片 | x86 兼容 |
| 片上 SRAM | **256MB** | **5× Blackwell** |
| 显存 | 144GB HBM3E / 芯片 | |
| TDN72 整机柜总计 | **72 节点, 68 PFLOPS, 42TB HBM** | |
| 散热 | **120kW 空冷** | 非液冷！关键差异化 |
| 互联 (TDN Link) | **<1μs, 1TB/s** 跨 72 芯片 | |
| 托盘层 | 9 芯片/1RU + 1.3TB HBM + 8TB 存储 + Intel Xeon host + 双 200GbE | |
| 拓扑 | **任意芯片任意分组** — 拓扑无关故障切换 | |

### 性能对标 (官方宣称，待独立验证)

> **2 万亿参数 GPT MoE 模型推理**:

| 方案 | 机架数 | 总功耗 |
|:-----|:------:|:------:|
| **Tensordyne TDN72** | **1 机架** | **120kW** |
| NVIDIA + Groq | 9 机架 | 1.5MW |
| AWS + Cerebras | 14 机架 | 800kW |

**目标**: 最高 **1,300 tokens/s/user**，支持 **10-20 万亿参数** 模型。

### 生态与时间线

| 时间 | 里程碑 |
|:-----|:-------|
| 现在 | 芯片已流片，架构公开 |
| Q1 2027 | Beta 计划启动 |
| Q2 2027 | 系统出货目标 |
| 合作伙伴 | **HPE + Juniper** (机箱/基础设施) · TSMC via Broadcom (流片) |

### 🎯 技术洞察

- **软件是最大风险**: 需要 PyTorch/Triton 直接编译 + 自定义 Python eDSL (tensordyne.nn) + HuggingFace 模型 Hub — 相对 CUDA 生态的差距巨大
- **空冷是重要差异化**: 120kW 空冷 vs 普遍液冷，降低部署门槛，但如果功耗密度提升则可能受限
- **TDN Link 亚微秒延迟**: 对 MoE 和 Agentic AI 推理很重要 — 专家路由和激活迁移高度依赖互联延迟

## 3️⃣ 🧵 AMD Mustang Peak (Zen 6 Threadripper) — 首曝官方规格

- **来源**: [Tom's Hardware — Zak Killian (Jun 17)](https://www.tomshardware.com/pc-components/cpus/first-official-details-of-amds-next-gen-mustang-peak-threadripper-cpus-come-into-view-chips-feature-ddr5-pcie-6-0-and-a-new-socket)
- **来源类型**: AMD 技术文档门户泄露 (InstLatX64 发现)

### 确认规格

| 项 | 数据 |
|:---|:------|
| 内部代号 | **Mustang Peak** |
| 核心架构 | **Zen 6** |
| CCD 制程 | **TSMC 2nm** (与 Olympic Ridge / Venice 共享 CCD) |
| 内存 | **DDR5** |
| 互联 | **PCI Express 6.0** |
| 插槽 | **TR6** (新平台) |
| CCD 名称 | "Powderhorn" — **每 CCD 12 核** (vs Zen 5 的 8 核) |

### 推算规格

| 推算项 | 推测值 | 推算依据 |
|:-------|:-------|:---------|
| 最大核心数 | **144 核 / 288 线程** | 12 核 × 12 CCDs (Threadripper Pro) |
| 内存通道数 | 8 通道 (TR Pro) / 4 通道 (TR) | 源于 EPYC Venice 架构 |
| 内存带宽 | 可能支持 **MRDIMM** | EPYC Venice 采用 MRDIMM 可达 ~1.6 TB/s |
| 时钟频率 | **显著高于 6 GHz** | Zen 6 设计目标 |

### 🎯 洞察

> **推理**: Mustang Peak 共享 EPYC Venice 的 CCD 设计，但 IO Die 可能是定制化版本。PCIe 6.0 (x16 单向 128GB/s) + 144 核心 + DDR5/MRDIMM 的组合，将使 Mustang Peak 成为面向 AI 推理工作站/小型集群的"旗舰工作站 CPU"— 远超"个人电脑"范畴。

## 5️⃣ 🖧 Senao SX906 — Intel Xeon 6 SoC (Granite Rapids-D) DPU 卡

- **来源**: [ServeTheHome — Patrick Kennedy (Jun 12, Computex 2026)](https://www.servethehome.com/this-is-an-intel-xeon-6-soc-dpu-on-a-pcie-card-from-senao-at-computex-2026/)
- **定位**: 服务器级别的 DPU 卡 — 一台"在 PCIe 插槽上跑的服务器"

### 规格矩阵

| 型号 | 核心数 | 基频 | 带宽 | 功耗 | 特色 |
|:-----|:------:|:----:|:----:|:----:|:-----|
| Xeon 6523P-B | **24C** | 2.5GHz | 100G | **295W** | 双 QAT |
| Xeon 6553P-B | **36C** | 2.6GHz | 200G | **355W** | 双 QAT + 媒体转码加速 |
| Xeon 6563P-B | **38C** | 2.4GHz | 200G | **355W** | 双 QAT + 媒体转码加速 |

### 关键特性

| 特性 | 详情 |
|:-----|:------|
| 平台 | Intel Xeon 6 SoC (Granite Rapids-D) — **P-核设计** |
| 网络 | 双 100G QSFP28 (通过 Intel E830 控制器连接 Host CPU) |
| 管理 | **ASPEED AST2600 BMC** + AST1060 PFR |
| 固件 | **OpenBMC**, Intel Platform Firmware Resilience (PFR) |
| 内存 | 4 通道 DDR5 (板载焊接) |
| 扩展 | 双 PCIe Gen5 x8 MCIO (共 24 条 PCIe 5.0 通道) + 金手指 x8 |
| 安全 | **TPM 2.0**, Secure Boot |
| 散热 | **双槽主动风冷** |
| 形态 | FH10.5L+ (266mm) |

### 🎯 洞察

> **这是 x86 阵营对 NVIDIA BlueField-3/4 最直接的 DPU 回应。** 用 24-38 个 P-核来替代传统 DPU 的 Arm 核方案，意味着可以运行完整的 x86 软件栈，无需跨架构移植。但 **295-355W 功耗** 显著高于 BlueField-3 等 Arm DPU — 这是性能 vs 功耗的取舍。OpenBMC 集成使管理面与主流数据中心 BMC 生态兼容。
