## 2️⃣ 🔄 AMD Versal Premium Gen 2：HBM供应枯竭迫使FPGA转向LPDDR5X

**来源**: [ServeTheHome - AMD Pivots From HBM to LPDDR5X For New Versal Premium Gen 2 Memory on Package Chips](https://www.servethehome.com/amd-pivots-from-hbm-to-lpddr5x-for-new-versal-premium-gen-2-memory-on-package-chips/) (Ryan Smith, June 30, 2026)

### 🔬 事件脉络

| 时间 | 事件 |
|:-----|:------|
| **2018** | Xilinx 推出 Virtex UltraScale+ HBM（首款封装内HBM FPGA） |
| **~2023** | AMD Versal HBM 系列（32GB HBM2e）量产 |
| **2025.09** | AMD 宣布 Versal HBM 停产 — **原因：HBM2e供应中断** |
| **2026.06.30** | **Versal HBM 最后下单日** |
| **2026.06.30** | AMD 宣布 Versal Premium Gen 2 Memory on Package — **改用LPDDR5X** |

### 🧠 关键技术转折

- **HBM供应链悖论**：HBM迭代太快（HBM→HBM2→HBM2e→HBM3→HBM3e→HBM4），三大原厂快速切产线，老一代HBM2e迅速无货
- **AI boom 挤占**：Instinct MI系列HBM需求爆发，直接把Versal FPGA的HBM配额挤光
- **LPDDR5X 优势**：供应稳定、生命周期长（DDR级长尾）、封装内布线难度低于HBM堆叠
- **代价**：带宽远低于HBM（但FPGA客户更看重供应稳定性和可用性，而非绝对峰值带宽）

### 💡 关键判断

- 这是**AI供应链溢出效应**的典型案例：HBM供不应求不仅影响GPU，还波及其他需要HBM的芯片（FPGA/ASIC）
- AMD Versal 产品线寿命长达2045年（UltraScale+），HBM切型必然 → **FPGA产品的长期供货承诺与HBM的短生命周期存在根本冲突**
- 对数据中心PCB/系统设计的影响：Versal Gen 2 的功耗/管脚重新设计将影响配套主板和散热方案

## 4️⃣ 🏗️ Arm CPU 登顶Top500 — 中国LineShine LX2纯CPU超算2.2EF

**来源**: [ServeTheHome - Arm CPUs Take Number 1 in Latest Top500 List with Chinese LineShine](https://www.servethehome.com/arm-cpus-take-number-1-in-latest-top500-list-with-chinese-lineshine/) (Patrick Kennedy, June 23, 2026)

### 🔬 系统架构

| 层级 | 规格 |
|:-----|:-----|
| **CPU** | LX2 (华为设计)— 152核/Die × 2 Die = **304核/CPU** |
| **架构** | Armv9 + SVE + SME |
| **TDP** | **690W/CPU** |
| **内存** | 32GB HBM + 256GB DDR5/CPU（HBM可配为cache模式） |
| **互联** | 片内集成 **800Gbps NIC** + LingQi 1.6Tbps fabric（4层fat-tree，光层） |
| **节点** | 2 CPU/节点 → 8节点/刀片(16 CPU) → 16刀片/计算框(256 CPU) → 2框/柜(512 CPU) |
| **全系统** | ~89-90机柜 ≈ **1,379万核** |

### 📊 性能与能效

| 指标 | 数值 |
|:-----|:------|
| **Linpack Rmax** | **2.2 EFLOPS** |
| **理论峰值** | 2.74 EFLOPS (效率**~80%**) |
| **功耗** | **42 MW** |
| **能效** | 52 GFLOPs/watt |
| **对比El Capitan** | 快~22%，但能效略低(El Capitan ~60 GFLOPs/watt) |

### 💡 Top10格局变化

| 排名 | 系统 | 算力 | 核心架构 |
|:-----|:-----|:-----|:---------|
| #1 | LineShine (中国) | 2.2 EF | **Arm CPU only** |
| #2 | El Capitan (AMD) | ~1.7 EF | MI300A |
| #3 | Frontier (AMD) | ~1.7 EF | EPYC + MI250X |
| #4 | Aurora (Intel) | — | Xeon + Intel GPU |
| #5 | JUPITER (欧洲) | ~1 EF | — |
| #9 | Fugaku (日本) | — | Armv8 CPU only |

### 💡 关键判断

- **纯CPU系统首次超越GPU系统登顶Top500** — 但代价是42MW功耗（约为Frontier的2×）
- 内置800Gbps NIC是重要趋势信号：**CPU+网络+内存全集成到一个封装**
- LineShine的80% Linpack效率远高于GPU系统的50-65%，说明CPU系统在特定负载下仍有利用率优势
- 这是4年来AMD首次失去Top500榜首位置
