## 1️⃣ 🔬 IBM Nanostack — 后GAAFET时代晶体管架构：向上堆叠实现 sub-1nm

**来源**: [ServeTheHome - IBM Outlines Sub-1nm Nanostack Transistor Technology: Building the Next Gen By Going Up](https://www.servethehome.com/ibm-outlines-sub-1nm-nanostack-transistor-technology/) (Ryan Smith, June 25, 2026)

### 背景：GAAFET即将触及天花板
- 当前三大代工厂（TSMC N2、Intel 18A、Samsung SF3E）正从 FinFET 切换至 **GAAFET（Gate-All-Around）**
- 但据 Imec 预测，GAAFET 将在 **2030年代中期** 耗尽可扩展性
- IBM Research 发布 **Nanostack** — 一种 CFET（Complementary FET）架构的后GAAFET方案

### 核心技术创新
| 特性 | 参数 |
|:-----|:-----|
| **结构** | nFET 晶体管上垂直堆叠 pFET 晶体管（CMOS 互补对），形成 **3D 堆叠晶体管** |
| **Nanosheet 厚度** | 仅 **15个硅原子** |
| **Sheet 间距** | ~9nm（约27个原子） |
| **工艺** | 双晶圆直接键合（wafer-to-wafer bonding），nFET 和 pFET 分别在不同晶圆上制造 |
| **键合介质** | 超薄低介电常数（low-k）介电层，最小化寄生电容 |

### 性能提升（vs IBM 2nm GAAFET）
| 指标 | 收益 |
|:-----|:-----|
| **晶体管密度** | **~2×** → 约 **4.5亿晶体管/mm²+** |
| **性能（等功率）** | **+50%** |
| **功耗（等性能）** | **-70%** |
| **SRAM 面积** | **-40%**（SRAM 密度升级难度远高于逻辑门，此数据意义重大） |
| **比喻** | 约 **1000亿晶体管** 集成在指甲盖大小区域内 |

### 制程依赖
- **必须**配合 **背面供电网络（BSPDN）**（Intel 18A 已商用化）
- **必须**配合 **High-NA EUV 光刻**（IBM 即将接收 ASML 高数值孔径扫描仪）
- 堆叠可以进一步扩展为多层（multi-stack），持续超越单层 GAAFET 密度

### 商用时间线
- IBM 目标：**5年内商业化**，适配 **7Å（0.7nm）节点**
- 潜在客户：Rapidus（已授权IBM 2nm）、TSMC/Intel/Samsung（有自有研发团队）
- **未解问题**：散热管理（两个热晶体管堆叠）、良率和成本（双晶圆制造成本可能 > 单片2×但密度2×）

### 📌 产业意义
- IBM 不再拥有晶圆厂（2015年出售给 GlobalFoundries），但通过技术授权影响代工厂路线图
- SRAM 面积缩减 40% 对 **AI 加速器设计** 尤其关键 — SRAM 是当前 AI 芯片的瓶颈单元

### Arm 收入份额结构分析
- Arm 占非x86收入的 **>95%**（其余为 IBM Z/ Power）
- NVIDIA NVL72 单台最高 **$650万**，含 36× Grace CPU + 72× Blackwell GPU
- 一台 NVL72 的收入 ≈ **928台** 入门级 $7K 1P 服务器 或 **433台** $15K 2P 服务器
- NVIDIA 2026年预计出货 **400万颗** Grace/Vera CPU

- AMD + Intel 2025年出货 **~2000万颗** EPYC/Xeon SP
- x86 在 **出货量** 上仍占据绝对主导
- 但 CPU/存储供应短缺持续限制 x86 收入增长

### 📌 产业洞察
- Arm 市场份额可能在 **2026下半年或2027年超过50%**（Vera Rubin NVL72 价格 > Blackwell）
- NVIDIA 计划推出 **纯 Vera CPU 机架** 用于 Agentic AI 推理，进一步推高 Arm CPU 出货
- Dell 244% 增长反映了 **品牌厂商在AI部署中的份额回升**（之前 ODM Direct 占 64%+）

## 3️⃣ 🔥 Qualcomm Dragonfly C1000 + HBC 高带宽计算 — 内存革命来了

**来源**: [ServeTheHome - Qualcomm Investor Day 2026 Data Center Announcements CPUs, AI Accelerators, and More](https://www.servethehome.com/qualcomm-investor-day-2026-data-center-announcements-cpus-ai-accelerators-and-more/) (Patrick Kennedy, June 24, 2026)

### Qualcomm Dragonfly C1000 CPU
| 特性 | 规格 |
|:-----|:-----|
| **核心数** | **250+ 核**（Chiplet 设计） |
| **主频** | **~5GHz**（与 Clearwater Forest / Turin Dense 的低频高核策略不同） |
| **互联** | PCIe Gen7, CXL, LPDDR 内存 |
| **内存选项** | 可选 **HBC（High Bandwidth Compute）** 直连 |
| **RAS** | 企业级可靠性特性 |
| **TAM** | Qualcomm 目标 **$200B CPU TAM** |

### HBC（High Bandwidth Compute）— 核心创新
| 传统 HBM 方案 | Qualcomm HBC 方案 |
|:--------------|:------------------|
| HBM 通过 CoWoS 中介层连接 | LPDDR **堆叠在计算Die上方** |
| 远距离互连，功耗大 | 超短距离，**消除 HBM 互连功耗税** |
| 带宽受限于中介层走线 | 带宽接近 SRAM 级别延迟 |
| 容量受 HBM 堆叠层数限制 | 单 Die 可达 **~768GB** |

- **AI250 代**（2027年）首款 HBC 产品
- **HBC Gen2**（AI300 代）将集成 **UALink 和 ESUN 互联** 协议
- 对标 CUDA/Dynamo/Triton → Qualcomm 收购 **Modular** 构建统一软件栈

### 数据中心战略
- **Modular 收购**：解决 AI100 系列长期以来的软件栈难题
- **4条产品线**：CPU（Dragonfly）+ AI加速器 + 定制芯片 + 互联产品
- **连接性**：从 Die-to-Die → CPO（共封装光学）→ PAM4 电/光 SerDes → QAM16 相干光全覆盖
- **FY29 收入目标**：从 18个月前 $22B 上调至 **$40B**，数据中心为核心驱动
- **FY27**: 2家超大规模客户各贡献 **>$10亿**
- **合作伙伴验证**: Meta（Zuckerberg 亲口确认多代合作协议）、Humain、Microsoft（Satya Nadella 站台）

### 📌 产业意义
- HBC 可能改变 AI 加速器的内存层级结构：LPDDR 堆叠代替 HBM，有望降低成本和功耗
- 250+ 核 @5GHz 在密度和单核性能间走不同路线（vs Intel/AMD 堆核降频策略）
- Qualcomm 的 "从手机SoC经验→内存堆叠" 路径具有独特优势

## 4️⃣ ⚡ 800V DC 液冷 Busbar + 分离式电源机架 — 下一代AI机架供电架构

**来源**: [ServeTheHome - Liquid-Cooling a TE Connectivity 800V DC Busbar and More from the Wiwynn Booth](https://www.servethehome.com/liquid-cooling-a-te-connectivity-800v-dc-busbar-and-more-from-the-wiwynn-booth/) (Patrick Kennedy, June 26, 2026)

### 液冷 Busbar 关键参数
| 指标 | 气冷 Busbar | 液冷 Busbar |
|:-----|:----------:|:----------:|
| **最大功率** | **140kW** | **750kW** |
| **工作电压** | 800VDC | 800VDC |
| **冷却方式** | 自然对流 | 冷却液循环 |
| **供应商** | TE Connectivity（已有标准件号） | TE Connectivity |

### 供电架构演进三阶段
| 阶段 | 状态 | 描述 |
|:----|:----|:------|
| **Phase 1** | ✅ 已发生 | 风扇移出计算节点 → 液冷全覆盖 |
| **Phase 2** | ✅ 进行中 | **电源移出计算机架** → 独立 **Power Rack** |
| **Phase 3** | 🚧 前沿 | Busbar 液冷化, 800VDC 整机架供电 |

### Wiwynn 800VDC 展示细节
- **TE Connectivity HVDC IT GEAR TO ELCON MICRO+ Power Cable Assembly**
  - 输入侧：800V/75A HVDC
  - 输出侧：400V ELCON MICRO+
  - 整体组件可 **机械臂自动化组装**（适配产线）
- **后端电源机架**：整组 PSU + 电池 + 管理模块，不占用计算节点空间
- **Busbar 盲插**：节点滑入即可供电，无需人工插拔电源线
- **互联空间释放**：电源移出后，机架后方可容纳更多 **Scale-Up 铜缆域** 连接

### 📌 产业意义
- **800V DC** 正成为下一代 AI 机架标配供电电压（vs 传统 48V/240V）
- 液冷 Busbar 到 **750kW** 意味着单机架功率将突破 MW 级
- 供电架构变革是机架级（Rack-Scale）设计的核心使能技术

## 5️⃣ 🖥️ Supermicro GB300 DGX Station (Super AI Station) — 拆解详析

**来源**: [ServeTheHome - Taking an Up-Close Look at the Supermicro GB300 Super AI Station](https://www.servethehome.com/taking-an-up-close-look-at-the-supermicro-gb300-super-ai-station/) (Ryan Smith, June 27, 2026)

### 核心规格
| 组件 | 规格 |
|:-----|:-----|
| **SoC** | Grace Blackwell GB300（72核 Grace CPU + Blackwell GPU） |
| **TDP** | **1600W**（全液冷） |
| **GPU 显存** | **252GB HBM3e**（7路/8路启用，7.1TB/s 带宽） |
| **CPU 内存** | **496GB LPDDR5X**（4× SOCAMM，液冷覆盖） |
| **网络** | 2× **400GbE** QSFP（ConnectX-8） |
| **存储** | 4× PCIe Gen5 M.2 2280（展示用 Micron 7450 Gen4×4 480GB） |
| **显卡** | 可选 NVIDIA RTX PRO（GB300 无图形输出能力） |
| **价格** | ~**$125,000** |

### 散热设计
- **全液冷**：GPU、CPU、内存 SOCAMM 均覆盖冷板
- 前置大尺寸散热排（radiator），3风扇前→后抽风
- **5U 机架安装** 可选（不仅是桌面工作站）

### 主要限制
- **GB300 无图形输出**（纯计算 GPU），必须配独立 RTX PRO 显卡才能输出桌面
- 无 USB-C 端口
- 1.6kW 需要 **专用供电回路**

### 📌 产业意义
- DGX Station 填补了单一开发者与大规模集群之间的空缺
- ~$125K 定价 ≈ 一台高端跑车，对于 AI 团队来说是 **开发调试成本可控的本地平台**
- **DGX Station for Windows** 即将推出（OEM 版本，Q4 2026）
