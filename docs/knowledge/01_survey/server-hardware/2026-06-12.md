## 1️⃣ AMD EPYC Venice (Zen 6) 首次公开对标 NVIDIA Vera 🎯

- **来源**: Tom's Hardware, Jake Roach, June 10, 2026 — "AMD fires back at Nvidia, claiming 256-core Zen 6 'Venice' CPU beats Vera by 3.3x in rack-level performance"
- **来源**: 快科技, June 10, 2026 — AMD 2nm Zen 6 EPYC Venice 256核512线程
- **技术详情**:
  - AMD 首次公布 **256核 Zen 6 Venice** 官方对标数据，宣称在 **整机架级别** 性能可达 NVIDIA Vera 的 **3.3 倍**
  - Venice 基于 **TSMC N2 (2nm)** 工艺，支持 **PCIe 6.0**，内存带宽约 **1.6 TB/s**
  - 这是 AMD 首次在公开场合将服务器 CPU 对标 NVIDIA 的 Grace CPU（而非对标 Intel Xeon），标志着**CPU vs CPU 竞争向 CPU vs GPU SoC 竞争延伸**
  - 256 核意味着相比当前 EPYC Turin (Zen 5, 192 核) 增加 **33% 核心数**
  - 超线程 (SMT) 策略不明：若保留 SMT 则可达 **512 线程**，但 Intel 已在 Diamond Rapids 中取消 HT
- **性能数据**: 宣称在机架部署场景下 **3.3× over NVIDIA Vera**（AMD 官方数据，待第三方验证）
- **战略意义**: 这是 CPU 厂商首次将 NVIDIA 的产品线（Vera Grace CPU）作为直接竞争对手，而非仅仅对标 Intel — 反映出 AI 集群中 CPU 的"控制平面 + 数据预处理"关键角色被放大

## 2️⃣ Intel Xeon 7 Diamond Rapids 技术规格进一步确认：延迟至2027，取消HT 🔍

- **来源**: ServeTheHome, Vic A, June 2, 2026 — "Intel Xeon 7 Diamond Rapids Now Slated for 2027"
- **技术详情**:
  - Diamond Rapids 确认基于 **Intel 18A-P** 工艺，定位 **P-core 架构** 的 Xeon 7 系列
  - **关键演进**:
    - 内存通道翻倍至 **16 通道 DDR5**，带宽大幅提升
    - **PCIe Gen 6** 支持
    - 核心数比 Xeon 6 (Granite Rapids) 增加约 **50%**
    - **不支持超线程 (Hyper-Threading)** — 延续 Clearwater Forest 的架构决策
    - Intel 表示 Hyper-Threading 将在下一代 **Coral Rapids** 平台恢复
  - 取消 **8 通道版本** 的 Diamond Rapids 平台 — 仅保留了高端 16 通道配置
  - 单核性能将是关键看点：随着 Intel 在桌面端放弃 HT 后核心微架构的改进，Diamond Rapids 的单核性能值得期待
- **发布时间**: **2027 年**（ServeTheHome 确认）
- **竞争环境**: Diamond Rapids 将在 Xeon 6+ (Clearwater Forest, 2026 H1) 之后一年发布，期间 AMD Venice 和 NVIDIA Vera 已全面在生产中部署
- **战略优势**: 使用 **Intel Foundry 18A** 生产，不受 TSMC 产能瓶颈制约，这对超大规模客户是重要加分项

## 3️⃣ AI 需求驱动数据中心 CPU 采购激增，代理商称 CPU 成新瓶颈 🔥

- **来源**: Tom's Hardware, Chris Stokel-Walker, June 8, 2026 (Premium) — "Demand for data center CPUs has surged, and AI agents are responsible"
- **技术详情**:
  - 行业专家指出：AI 代理 (Agent) 的大规模部署正在推动数据中心 **CPU 采购量急剧上升**
  - 核心逻辑：AI 推理集群中 **CPU 承担的控制平面、数据预处理、编排调度** 工作负载远高于训练阶段
  - 随着 AI 从训练向推理转移，每个 GPU 需要配比更多的 CPU 核资源和内存带宽
  - 这不是传统的"服务器替换"需求，而是 AI 新基建带来的 **新增 CPU 采购**
- **行业影响**: AI Agent 驱动的 CPU 需求正在形成新的供应链压力点，继 GPU/HBM 之后 CPU 可能成为下一个瓶颈

## 4️⃣ Samsung Foundry 转向 5nm/8nm 订单，2nm 反攻尚需时间🇰🇷

- **来源**: DIGITIMES Most-Read (June 11) — "Samsung Foundry turns to 5nm, 8nm orders as 2nm comeback takes shape"
- **技术详情**:
  - Samsung Foundry 短期内战略调整为承接 **5nm/8nm** 成熟节点订单，而非激进冲击先进节点
  - 同时其 **2nm (SF2)** 技术路线仍在推进，但大规模反攻市场份额尚需时间
  - 这意味着：AMD (Venice/N2) 和 NVIDIA 对 **TSMC 的依赖将进一步加深**
  - 对服务器行业的影响：在 TSMC 产能持续紧张背景下，Intel Foundry 18A 成为除 TSMC 外的唯一高性能选项，**双轨化趋势加速**
- **硬件设计启示**: 服务器 CPU 和 GPU 的工艺选择空间在收缩，设计者需提前锁定代工厂产能窗口

## 6️⃣ Minisforum S5 无风扇全闪 NAS：Wildcat Lake 平台新应用 🖥️

- **来源**: ServeTheHome, Ryan Smith, June 9, 2026 — "Minisforum S5 All-Flash NAS Shown Based on Intel's Wildcat Lake Platform"
- **技术详情**:
  - 基于 **Intel Core Series 3 (Wildcat Lake)** 平台的全闪 NAS
  - **5× M.2 2280** 插槽（PCIe Gen4 x1 每条），侧重容量而非速度
  - **12GB LPDDR5X-7500** 内存，**64GB UFS 2.2** 独立存储 OS
  - 突破性设计：**被动散热 (fanless)** — 全铝机身+鳍片散热
  - I/O：**10GbE** (Realtek RTL8127) + **2.5GbE** + 双 **40Gbps USB4** + HDMI 2.1
  - 预装 MinisCloud OS + OpenClaw 代理 AI 软件
  - 每个 M.2 提供约 **2GB/s** 带宽，10GbE 网络可完全饱和
- **目标市场**: 中端全闪 NAS 市场，填补 Alder Lake-N 入门级与高端 NAS 之间的空白
- **行业信号**: Wildcat Lake 平台的 **6 条 PCIe Gen4 通道** 被精妙分配到 5×SSD + 10GbE，展示了新一代低功耗平台的 I/O 灵活性
