## 1️⃣ 🔌 Senao SX906: Intel Xeon 6 SoC DPU — 24~38 P-cores 的 PCIe 卡

- **来源**: [ServeTheHome — Patrick Kennedy (Jun 12)](https://www.servethehome.com/this-is-an-intel-xeon-6-soc-dpu-on-a-pcie-card/)
- **首发**: Computex 2026, Senao 展台
- **关键词**: Intel Xeon 6 SoC, Granite Rapids-D, DPU, SmartNIC, BMC

### 核心规格

Senao SX906 SmartNIC/DPU 将 **完整的 Intel Xeon 6 SoC (Granite Rapids-D)** 放在一张 PCIe 卡上：

| 项目 | 规格 |
|:-----|:-----|
| CPU | Intel Xeon 6523P-B (24C) / 6553P-B (36C) / 6563P-B (38C) |
| 基频 | 2.5GHz / 2.6GHz / 2.4GHz |
| 网络 | **双 100G QSFP28** (36C/38C 型号支持 200Gbps 总吞吐) |
| 主机互联 | Intel Ethernet Controller E830 → 200GbE PCIe 到 Host |
| PCIe 接口 | PCIe Gen5 x8 (边缘连接器) + 2× PCIe Gen5 x8 MCIO |
| 内存 | 4× DDR5 通道 |
| BMC | **ASPEED AST2600** 板载 |
| 安全 | PFR ASPEED AST1060 + TPM 2.0 |
| 固件 | **OpenBMC**, Intel Platform Firmware Resilience |
| 功耗 | 295W (24C) / 355W (36C/38C) |
| 形态 | 双槽, FH10.5L+ (266mm) |
| 特色加速 | 36C/38C 含 **QAT + 媒体转码加速器** |

### 架构设计要点

- **这是真正的"卡上服务器"**: 板载 AST2600 BMC + OpenBMC + 独立管理端口 (RJ45 + USB-C Console)
- **24 lanes PCIe Gen5 扩展**: 两个 MCIO 连接器 + 边缘连接器提供 24 条 PCIe Gen5
- **功耗对比**: 24C 295W / 36C 355W — 跟双槽 GPU 同一量级
- STH 编辑评论: **"这不再是 SmartNIC，这是真正的 DPU"**

> 💡 **解读**: 这是 DPU 市场的重大事件。NVIDIA BlueField 3/4 是 ARM 核方案，Intel 路径是用**完整 x86 P-core 架构 (Granite Rapids-D)** 做 DPU。好处是 x86 生态兼容性（无需为 ARM 重编译），BMC 可独立管理，双 MCIO 可挂存储/加速器。这可能改变 DPU 市场竞争格局——如果价格合适的话。

## 2️⃣ 🧮 Tensordyne Napier: 3nm 对数数学 AI 推理芯片，风冷 TDN72

- **来源**: [ServeTheHome — Vic A (Jun 15)](https://www.servethehome.com/tensordyne-napier-ai-processor-announced-with-logarithmic-math/)
- **关键词**: Tensordyne, Napier, 对数数学, TDN72, 3nm, 推理

### 核心架构

Tensordyne Napier 是**试图从数学层面改变 AI 加速器设计**的激进方案：

| 指标 | 数值 |
|:-----|:-----|
| 制程 | **TSMC 3nm** |
| 晶体管数 | **1380 亿** (138B) |
| 单芯片算力 | **2.1 PFLOPS** |
| 加速器主频 | 1.33 GHz |
| CPU 主频 | 1.5 GHz |
| 片上 SRAM | **256 MB** (宣称 5× NVIDIA Blackwell 的 SRAM) |
| HBM3E | **144 GB/芯片** |
| 互联 | TDN Link: <1μs 片间延迟, 1TB/s 带宽 |

### 革命性: 对数数学 (Logarithmic Math)

| 传统方式 | Napier 方式 |
|:---------|:------------|
| 乘法器 (Multiply) | **加法器 (Add)** — 对数域中乘法变加法 |
| 乘法器面积大 | 加法器小 → 省下的面积做 SRAM |
| 标准 FP 数值 | 对数定义下的数值表示 |

**直接结果**: 减少乘法器面积 → 更多片上 SRAM → 减少数据搬运 → 更好的系统级能效

### TDN72 整机柜规格

| 层级 | 构成 | 规格 |
|:-----|:-----|:-----|
| **芯片** | Napier × 1 | 2.1 PFLOPS / 144GB HBM |
| **Tray 层 (1RU)** | 9× Napier | 1.3TB HBM3E, 8TB 存储, Intel Xeon Host, 双 200GbE |
| **Pod 层** | 4× Tray = 36 芯片 | — |
| **TDN72 整机柜 (52RU)** | 2× Pod = **72 芯片** | **68 PFLOPS**, **42TB HBM**, 120kW **风冷** |

### 竞争定位 — 推理场景的两万亿参数模型

Tensordyne 对标推理场景给出了对比：

| 方案 | 机柜数 | 功耗 | 吞吐 (tokens/s/user) |
|:-----|:-------|:-----|:---------------------|
| **TDN72 (1 rack)** | **1** | **120kW** | **~1,300** |
| NVIDIA + Groq | 9 | 1.5MW | 同等 |
| AWS + Cerebras | 14 | 800kW | 同等 |

⚠️ **注意**: 这是厂商自报数据，需要第三方验证。Napier 已 tape-out，但 Q1 2027 才开始 beta 计划。

> 💡 **解读**: Tensordyne 的对数数学路径是**真有技术含量**的差异化尝试。加法器面积比乘法器小得多→更多面积做 SRAM→减少 DRAM 访问→降低功耗。但软件生态是最大挑战———要在 PyTorch/Triton 生态上跑对数数学，改造量不小。

## 3️⃣ 🏛️ AMD 2026-2027 企业路线图全览

## 4️⃣ 💾 AMD 收购 MEXT: 内存分层引擎, Flash 当 DRAM 用

- **来源**: [Tom's Hardware — Anton Shilov (Jun 15)](https://www.tomshardware.com/tech-industry/amd-takes-over-mext-for-memory-tiering-tech-that-enables-flash-to-appear-as-dram-to-applications)
- **关键词**: AMD, MEXT, 内存分层, Predictive Memory Engine, CXL, 数据中心

### 核心价值

AMD 收购 **MEXT (Memory Expansion Technology)**，获得 **Predictive Memory Engine (PME)**：

- **功能**: 将不常访问的数据从 DRAM **自动卸载到 NAND 闪存**
- **对应用透明**: 应用感知不到数据在 DRAM 还是 Flash 上
- **算法**: 机器学习预测哪些数据页被访问概率低 → 卸载到 Flash
- **收益**: 等效 DRAM 容量 **翻倍或更多**，数据中心内存成本大幅降低

### 行业意义

- AMD 正在为自己搭建 **数据中心内存分层体系**: DRAM (近端) → CXL 内存池 → Flash (远端)
- 这是解决 "AI 训练/推理场景内存瓶颈" 的硬件+软件结合方案
- 与 Intel 的 CXL 内存池化方案形成竞争

> 💡 **解读**: 服务器行业的内存成本正因 HBM 短缺和 DDR5 涨价而飙升。AMD MEXT 的"Flash 当 DRAM"能力，对**大内存场景**（数据库、推理 KV Cache、内存计算）有直接价值。注意这也暗示 AMD 在内存分层方面加大布局，可能会影响 CXL 内存池的标准竞争。

## 5️⃣ 🔬 Intel + AMD ACE CPU 扩展指令集: AI 导向的 x86 新指令

- **来源**: [Tom's Hardware — Bruno Ferreira (Jun 20)](https://www.tomshardware.com/pc-components/cpus/intel-and-amds-new-ace-cpu-extensions-bring-an-efficient-ai-oriented-instruction-set-to-x86)
- **关键词**: ACE, x86, AI 指令集, Intel, AMD, x86 联盟

### ACE 是什么

**ACE (Advanced Compute Extensions)** 是 Intel 和 AMD 联合为 x86 生态开发的 **AI 导向新指令集**，是 x86 生态系统咨询组成立一周年的重要成果：

| 维度 | 说明 |
|:-----|:------|
| 目标 | **矩阵乘法的功率与密度优化** |
| 方法 | 新指令设计让矩阵乘法更节能、面积更高效 |
| 范围 | x86 服务器 CPU 和客户端 CPU 均受益 |
| 对比 | 类似 x86 历史上 AVX-512 的里程碑，但专为 AI 设计 |
| 合作方 | Intel + AMD 联合开发，共同推进 |

### 行业意义

- 这是 Intel 和 AMD **首次联合定义 CPU 指令集扩展**
- 表明服务器 CPU 厂商正从"通用计算"转向 "AI-first 计算"
- 与 NVIDIA Grace ARM CPU 竞争——x86 阵营需要联合提升 AI 计算效率
- ACE 指令将使 x86 CPU 在**小规模推理、边缘 AI、Agent 工作负载**中更有竞争力

> 💡 **解读**: ACE 是 x86 联盟迄今为止最重要的技术成果。如果 ACE 能让 x86 CPU 在 AI 推理任务上接近专用加速器的能效比(哪怕是少数场景)，它将改变 "CPU 只做控制面、AI 必须靠 GPU/加速器" 的既有分工。

## 6️⃣ 📅 Intel Diamond Rapids 延迟至 2027 — Venice 先行

- **来源**: [Tom's Hardware — Hassam Nasir (Apr 25/Jun 1)](https://www.tomshardware.com/pc-components/cpus/intels-upcoming-xeon-7-diamond-rapids-server-cpus-reportedly-delayed-to-2027)
- **关键词**: Intel, Diamond Rapids, Xeon 7, Venice, 延迟

### 时间线变化

| 事件 | 原计划 | 新计划 |
|:-----|:-------|:-------|
| **Diamond Rapids (Xeon 7)** | 2026 H2 | **2027** (Intel 18A-P) |
| **Venice (AMD Zen 6)** | 2026 H2 | **2026 H2** (按计划) |

### Diamond Rapids 规格 (不变部分)

| 指标 | 数值 |
|:-----|:-----|
| 核心架构 | **Panther Cove P-core** |
| 最大核心数 | **192 核** |
| 内存通道 | **16 通道 DDR5** (仅保留双 8 通道型号) |
| 内存类型 | **2nd Gen MRDIMM** |
| PCIe | **PCIe 6.0** |
| 封装 | **LGA9324** (9,324 针脚) |
| 制程 | **Intel 18A-P** |
| TDP | 最高 ~500W |

> 💡 **解读**: Diamond Rapids 的延迟意味着 Intel 在 2026 H2~2027 的**高端 CPU 市场将出现空窗**——AMD Venice (256c, 2nm) 将独占"最高核心密度"头衔近一年。Diamond Rapids 的 192c + PCIe 6.0 + 16ch MRDIMM 确实激进，但 Intel 18A 制程的良率爬坡可能是延迟主因。
