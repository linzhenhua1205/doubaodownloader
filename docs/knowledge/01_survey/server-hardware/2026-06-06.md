## ③ SambaNova 在 COMPUTEX 挑战 GPU 推理统治地位

**来源**: DIGITIMES（6/6） | **日期**: 2026-06-06

SambaNova 在 COMPUTEX 2026 上展示其分解式推理架构，挑战传统 GPU 在 AI 推理市场的主导地位：

| 项目 | 详情 |
|:-----|:------|
| 架构 | **分解式推理**：GPU 负责 Prefill → RDU（Reconfigurable Dataflow Unit）负责 Decode → CPU 编排 |

> 意义：推理市场正在从「通用 GPU 一刀切」向「分解式专用架构」分化。SambaNova 的 RDU+GPU 混合方案开辟了第三条路径

## ④ Flex CTO：5年内机架功耗从10kW到1MW — 最难的一跳还在前面

**来源**: DIGITIMES（6/6） | **日期**: 2026-06-06

Flex（伟创力）CTO 在 COMPUTEX 受访，描绘机架功耗演进路线图：

| 阶段 | 时间 | 单机架功耗 | 散热方案 |
|:----|:----|:----------|:---------|
| 现状 | 2026 | 40~120kW | 液冷已成标配 |
| 近期 | 2027~2028 | 200~500kW | 液冷+冷板混合 |
| 中期 | 2029~2030 | 500kW~1MW | 浸没式+两相冷却 |
| **最难一跳** | 10kW→1MW | 跨度100倍 | 从风冷→液冷→浸没全链条重构 |

> CTO 强调：「从10kW到1MW不是线性增长，而是**基础架构范式转换**。今天的液冷方案到500kW+会遇到可靠性、泵效率、连接器密度等全新瓶颈」

## ⑥ ASUS XA NB3I-E12 深度拆解 — 9U 8×B300 空冷 GPU 服务器

**来源**: ServeTheHome（Patrick Kennedy） | **文章日期**: 2026-05-30

STH 对 ASUS 这款 COMPUTEX 展出的旗舰 GPU 服务器进行了完整硬件拆解评测。

### 硬件规格

| 项目 | 详情 |
|:-----|:------|
| 尺寸 | **9U** 机架，半高以上空间被 GPU 散热器占据 |
| GPU | **8× NVIDIA B300 Blackwell Ultra**（HGX B300 基板） |
| CPU | Intel Xeon 6 系列 |
| GPU 互联 | 每 GPU 配一颗 **ConnectX-8 800Gbps XDR InfiniBand**（板载，非 PCIe 插卡） |
| 存储 | U.2 NVMe，每 GPU 配独立 SSD |
| 额外网口 | Intel X710-AT2 双 10GbE 管理口 |
| 散热 | **纯空冷**设计，大面积鳍片散热器 |
| 重量 | **需4人抬起安装**（Patrick 原话：旧8卡机80-100磅可单人，这台4个人才抬得动） |

### 架构亮点

1. **ConnectX-8 板载集成** — HGX B300 基板中 NIC 直接内建，走线通过线缆连接到前面板 OSFP 笼。B200 时代必须插 PCIe 形态，B300 变成基板标准特征
2. **OSFP 端口编号混乱** — 前面板 8 个 OSFP 笼的编号顺序并非 1→8 而是 2,3,1,4,7,6,8,5。ASUS 确认这是「正确接线方式」
3. **模块化 IO 托盘** — 前端 IO/SSD/网口区域可整体滑出检修，通过高密度连接器连接 CPU 主板
4. **POST 码显示屏** — ASUS 多年代码屏设计，方便机架级别快速故障定位
5. **纯空冷定位** — 刻意选择空冷以适应现有机架基础设施，避免液冷改造成本

> 一台 9U 的配置实际上宣告了「**空冷 GPU 服务器的物理极限**」——更密集的 GPU 配置（如 NVL72）必须走向液冷

## ⑦ Intel Xeon 6+ Clearwater Forest 架构深度详解 — 已出货

**来源**: ServeTheHome（Patrick Kennedy） | **文章日期**: 2026-06-01

这是 Intel 首款 **18A 工艺** 的服务器芯片，已正式出货给客户。

### 核心规格

| 参数 | 详情 |
|:-----|:------|
| 型号 | **Xeon 6900E+ 系列（Clearwater Forest）** |
| 核心数 | 最高 **288 E-core**（12 个 Compute Tile，每 tile 24 核） |
| 架构 | 非 SMT（单线程每核心） |
| L3 Cache | **576 MB**（12×Compute Tile 共享） |
| 内存 | **DDR5-8000**，12 通道，单插槽最高 1.5TB（12×128GB RDIMM） |
| PCIe | 兼容 Xeon 6900P 平台，仅需 **BIOS 更新** |
| 封装 | Compute Tile + I/O Tile + Base Tile + EMIB + Foveros Direct 3D |
| TDP | 450W（288 核，<2W/核），144 核版本更低 |
| 工艺 | **Intel 18A**（RibbonFET + PowerVia） |
| 安全 | Intel SGX + TDX 机密计算 |

### Intel vs AMD 性能对比（官方数据，需验证）

| 指标 | Xeon 6990E+ vs EPYC 9965 |
|:----|:------------------------|
| 每线程性能 | +1.3× |
| 每线程能效 | +1.3× |
| 核心数对比 | 288 核（无 HT） vs 192 核（384 线程） |
| 功耗对比 | 450W vs 500W |

### 关键设计决策

| 特性 | 选择原因 |
|:----|:---------|
| 无 SMT | 云原生/容器工作负载不需要 SMT 带来的线程干扰 |
| 576MB LLC | 针对 AI 推理时的 KV Cache 本地化和 VMs 密度优化 |
| DDR5-8000 | 大内存带宽满足数据密集推理负载 |
| Intel 18A | 提升能效比，同时作为 Intel Foundry 的旗舰案例 |
| **Application Energy Telemetry** | 每核心/每应用级功耗遥测，CSP 可做精确成本分配 |

> Intel 声称：「一台 Xeon 6990E+ 可以取代 **960 台第二代 Xeon 6258R 服务器**，48 机架合并为 1 机架」。虽然数字激进，但反映出 E-core 密度路线的核心理念

|:-----|:------|
