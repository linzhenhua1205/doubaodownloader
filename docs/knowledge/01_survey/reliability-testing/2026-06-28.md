## 1️⃣ 🎯 AdaGC: 自适应梯度裁剪消除 LLM 预训练 Loss Spikes

**来源**: arXiv:2502.11034 · **2025-02-16 提交（v3: 2026-06-09）** · cs.LG · **已接收: ICML 2026**

**作者**: Guoxia Wang, Shuai Li, Congliang Chen, Jinle Zeng, Jiabin Yang, Dianhai Yu, Yanjun Ma, Li Shen

**背景**: Loss spikes 是大规模 LLM 预训练中**最顽固的训练稳定性障碍**之一。现有研究多归因于单一因素（数据异常值/硬件故障/数值精度），但本文通过实证发现——loss spikes 通常是**异构因素的合流效应**（data outliers × hardware faults × precision × hyperparameters），单一因素理论无法解释实际场景。

**核心方法 — AdaGC**:

| 组件 | 说明 |
|:-----|:------|
| **自适应 per-tensor 梯度裁剪** | 不是用全局阈值或固定阈值，而是对每个 tensor 维护一个**指数滑动平均（EMA）** 的历史裁剪值序列，基于其统计特性动态确定裁剪边界 |
| **优化器无关** | 可直接与 AdamW、Muon、Lion 等任意优化器配合使用 |
| **通信优化** | 相比 GlobalGC（全局梯度裁剪需要 all-reduce 全局 norm），AdaGC 的 per-tensor 裁剪可直接在本地完成，**减少混合并行训练中的通信开销** |
| **内存开销** | 仅需为每个 tensor 维护一个 EMA 状态，开销可忽略 |

**实验数据**:

| 模型 | 规模 | 对比基线 | 关键指标 |
|:-----|:-----|:---------|:---------|
| **Llama-2 7B** | 7B 参数 | GlobalGC | **Spike score → 0**（完全消除）; 下游准确率提升 **1.32%** |
| **Mixtral 8×1B** | ~8B (MoE) | GlobalGC | **Spike score → 0**; 下游准确率提升 **1.27%** |
| **ERNIE 10B-A1.4B** | 10B (MoE) | GlobalGC | **Spike score → 0**; 下游准确率提升 **2.48%** |
| **+ Muon 优化器** | — | Muon 基线 | 零 spike + 更高平均准确率 |
| **+ Lion 优化器** | — | Lion 基线 | 零 spike + 更高平均准确率 |

**意义**:

1. **Loss spike 的合流成因论**：本文首次明确指出 loss spike 不是单一因素导致，而是多个因素的**合流**——这与容错研究中「故障来源多样且交互」的认知一致
2. **梯度裁剪从「经验调参」进入「自适应」时代**：GlobalGC 需要调全局阈值，AdaGC 自动适应每个 tensor 的梯度动态分布
3. **与容错训练的直接关联**：loss spikes 触发 checkpoint restore/数据 batch skip，AdaGC 从源头上减少这类「软故障」的触发频率，是**检查点频率优化**的互补方案

**📌 交叉引用**:
- 与 [DiffCp](../bmc-system/2026-06-26.md)（差分检查点，减少检查点时间 89.2%）互补——DiffCp 解决「出问题后恢复更快」，AdaGC 解决「不出问题的比例更高」
- 与 [CCGrid2026 SDC](../bmc-system/2026-06-06.md)（指令级 SDC 注入研究）互补——CCGrid 表明 SDC 可引发 loss spikes，AdaGC 提供下游防护
- 与 [ZClip](https://arxiv.org/abs/2504.02507)（同期 z-score 自适应裁剪）形成竞争路线

## 2️⃣ 🔍 理解 LLM 训练中的静默数据损坏：生产节点实证研究

**来源**: arXiv:2502.12340 · **2025-02-17 提交** · cs.LG/cs.DC

**作者**: Jeffrey Ma, Hengzhi Pei, Leonard Lausen, George Karypis（AWS AI / 明尼苏达大学）

**背景**: SDC 研究长期面临一个根本困境——大多数论文使用**人工故障注入**（bit flip、stuck-at fault）而非真实 SDC 硬件来研究影响。本文获得了一个极难得的机会：通过云平台自动运维系统「扫出」的生产中检测到的**真实 SDC 故障节点**，并对比健康节点进行同条件训练。

**方法 — 三层次 SDC 影响分析**:

| 层次 | 方法 | 目标 |
|:-----|:-----|:-----|
| **子模块级** | XLA 编译器确定性执行，逐子模块对比健康 vs. SDC 节点的计算结果 | 判断哪些算子对 SDC 最敏感 |
| **单步优化器级** | 同步机制确保梯度计算完全一致后对比优化器输出 | 单步 SDC 扰动幅度的分布 |
| **训练周期级** | 长时间训练后对比 loss 曲线和收敛点 | SDC 的累积效应 |

**核心发现**:

| 发现 | 详细描述 | 工程含义 |
|:-----|:---------|:---------|
| **SDC 影响高度异构** | 不同 SDC 节点的扰动幅度差异显著，有些接近不可见，有些会导致梯度偏差放大 | 统一的「SDC 幅度假设」不成立，容错策略需考虑 SDC 的个体差异 |
| **大多数 SDC 扰动「看似温和」** | 在子模块和单步级别，大多数 SDC 引起的数值偏差相对较小，易于被误判为「无害」 | 不能因单步扰动小就忽略——累积效应才是关键 |
| **SDC 可导致收敛点漂移** | 即使训练未发散，SDC 节点训练的模型可能收敛到**完全不同的权重空间**，具有不同的参数分布 | 训练轨迹等价性假设被打破——这正是 ReCoVer 所强调的问题 |
| **SDC 可引发 loss spikes** | 某些 SDC 场景下 loss 曲线出现明显尖峰，与硬件故障相关的 loss spike 现象被直接观测到 | SDC 是 loss spike 的已知「异构成因」之一，与 AdaGC 的合流成因论呼应 |

**意义**:

这是**首个使用真实生产 SDC 硬件**而非人工故障注入研究 LLM 训练中 SDC 影响的工作。其核心贡献不在于提出新方案，而在于提供了「真实 SDC 到底长什么样」的**稀缺观测数据**：

- 此前 DSN'26 的 GPU SDC 解剖（arXiv:2605.04213）使用门级 stuck-at 故障注入 → 微观机制
- LLM-PRISM（arXiv:2604.10390）使用 RTL 级仿真 + 7,664 次训练运行 → 仿真尺度
- 本文 == **生产级真实 SDC 硬件验证** → 现实世界的最终验证

三条路线形成完整的 SDC 研究「三验证体系」：**门级 → RTL 级 → 生产级**，这在可靠性工程中是极罕见的完善度。

**📌 交叉引用**:
- 与 [DSN'26 GPU SDC Anatomy](../cloud-native/2026-06-22.md)（arXiv:2605.04213，门级 stuck-at 注入）互补——微观机制 vs 生产宏观效应
- 与 [LLM-PRISM](../bmc-system/2026-06-25.md)（arXiv:2604.10390，RTL 级仿真，7,664 次运行）互补——仿真预测 vs 生产验证
- 与 [CCGrid2026 SDC](../bmc-system/2026-06-06.md)（arXiv:2604.00726，指令级注入，轻量检测）互补——多种故障注入方法的交叉验证
- 与 [AdaGC](#1️⃣--adagc-自适应梯度裁剪消除-llm-预训练-loss-spikes)（本日条目 1）——SDC 引发的 loss spikes 可通过自适应梯度裁剪缓解

## 3️⃣ ⚡ SPAM: 尖峰感知动量重置——面向训练稳定性的优化器

**来源**: arXiv:2501.06842 · **2025-01-12 提交（v2: 2025-02-28）** · cs.LG

**作者**: Tianjin Huang, Ziquan Zhu, Gaojie Jin, Lu Liu, Zhangyang Wang, Shiwei Liu

**背景**: 与 AdaGC（梯度裁剪方案）不同，SPAM 从**优化器设计**角度出发解决 loss spikes 问题。核心观察：梯度尖峰可达到正常梯度的 **1,000× 以上**，这些异常梯度不仅影响当前步，还会污染 Adam 优化器的**动量状态**（一阶矩和二阶矩），造成持续数步的扰动。

**核心方法 — SPAM**:

| 组件 | 说明 |
|:-----|:------|
| **动量重置（Momentum Reset）** | 检测到梯度异常时，将 Adam 的一阶/二阶动量重置为近期正常值，而非回滚整个训练 |
| **尖峰感知梯度裁剪** | 与 AdaGC 的自适应 clip 不同，SPAM 结合了**动量状态健康度**来决策何时触发 clipping |
| **稀疏动量（Sparse Momentum）** | 仅维护和更新部分动量项，降低内存占用——在记忆约束下优于 GaLore 和 Adam-Mini |
| **覆盖范围** | 预训练（60M→1B）、4-bit 低精度预训练、RL fine-tuning、时间序列预测 |

**关键量化数据**:

| 指标 | 数值 |
|:-----|:-----|
| 梯度尖峰幅度 | 正常梯度的 **1,000× 以上** |
| 预训练模型范围 | 60M → 1B 参数 |
| 适用任务 | 预训练 + 4-bit PTQ + RL + 时间序列 |
| 内存效率对比 | 优于 GaLore 和 Adam-Mini |

**与 AdaGC 的对比**:

| 维度 | AdaGC | SPAM |
|:-----|:------|:-----|
| **设计理念** | 梯度裁剪（限制异常梯度） | 动量重置（修复被污染的优化器状态） |
| **作用位置** | 梯度→裁剪→优化器 | 优化器检测到异常→重置动量 |
| **会议接收** | **ICML 2026** | 待定（2025-01 提交） |
| **互补性** | 两者可叠加使用——先裁剪梯度限制异常，再重置动量消除累积污染 |

**意义**:

SPAM 的核心发现——**梯度尖峰可达 1,000× 正常幅度**——量化解释了为什么 loss spikes 如此难以预防：异常梯度不仅导致当前步发散，还会污染优化器的动量状态，造成「连锁反应」。SPAM 的动量重置机制从优化器角度提供了 AdaGC 裁剪之外的第二道防线。

- 与 [ZClip](https://arxiv.org/abs/2504.02507) —— z-score 自适应裁剪，三条路线并行
