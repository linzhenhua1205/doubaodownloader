## 🆕 1. TierCheck — 三层级检查点：面向故障异构性的分层存储

| 元数据 | 内容 |
|:-------|:-----|
| **来源** | arXiv:2605.17821, 2026-05-17 |
| **作者** | Shujie Han, Feng Jiang, Patrick P.C. Lee, Xiao Zhang 等 (CUHK + vivo) |
| **领域** | 检查点分层存储 / LLM 训练容错 |

### 🎯 核心贡献

现有检查点系统使用**单层存储后端**，强制在保存开销与恢复速度之间二选一。TierCheck 提出了**集群感知三层级检查点**架构，将存储放置与故障异构性对齐：

**三层设计：**
1. **Tier-1 — 本地 GPU/HBM 内存**：轻量差分检查点（differential checkpoint），仅保存与上一轮快照不同的参数变更 → **微秒级保存，毫秒级恢复**
2. **Tier-2 — 对等节点内存（peer memory）**：跨节点复制差分快照 → 单节点故障无需访问远端存储
3. **Tier-3 — 远端持久存储（NFS/并行文件系统）**：异步迁移全量基础检查点（base checkpoint）→ 应对集群级灾难

**关键技术：**
- **跨层级严格全局一致性**：使用检查点屏障（checkpoint barrier）+ 层级间因果序保证，无需暂停训练
- **集群感知恢复**：故障时根据存活节点位置选择最优层级恢复路径
- **高频检查点**：因 Tier-1/2 开销极低，可支持每步检查点（per-iteration checkpointing）

### 📊 实验数据

| 指标 | 值 |
|:-----|:----|
| 最大验证模型 | **40B 参数** |
| 端到端检查点时间 | **< 10s**（含所有三层） |
| 训练额外开销 | **低**（具体数值未报告，但声称"low training overhead"） |
| 恢复速度 | 本地故障：秒级；全集群故障：分钟级（基础检查点） |
| 设计目标 | **低开销持久化 + 快速恢复**的最优平衡 |

### 💡 为何重要

- 与 LLMTailor（按层裁剪检查点内容，下文详述）**正交互补**：LLMTailor 优化**检查什么**，TierCheck 优化**存哪里**
- 故障异构性认知：单节点 GPU crash 与全集群掉电需要不同的恢复策略，三层设计恰好对齐了这个频谱
- 与 SKT 504 GPU 生产报告（06-07 已跟踪）的另一交叉验证：检查点瓶颈在软件栈，TierCheck 通过内存级快照绕过了存储软件栈的瓶颈
- 每步检查点的可行性——如果引入开销低到可以每步执行，则**容错粒度从「分钟级回滚」推进到「步级恢复」**

## 🆕 2. V-ABFT — 基于方差的算法级容错自适应阈值（6-48× 改进，零误报）

| 元数据 | 内容 |
|:-------|:-----|
| **来源** | arXiv:2602.08043, 2026-02-08 |
| **作者** | Yiheng Gao, Qin Hua, Zizhong Chen (UC Riverside) |
| **领域** | ABFT / SDC 检测 / 混合精度容错 |

### 🎯 核心贡献

Algorithm-Based Fault Tolerance (ABFT) 是检测矩阵乘法中 SDC 的主流方法，但现有阈值确定方法存在严重缺陷：
- **分析界限法**过于保守（阈值天文数字，几乎检测不到任何故障）
- **A-ABFT 概率法**阈值为实际舍入误差的 **160-4200×**，检测粒度过粗

**V-ABFT 方法：** 通过直接建模验证差值的统计方差，推导出更紧的误差界限：
1. **方差估计**：从验证差值中估计其分布方差，自适应设置检测阈值
2. **O(n) 复杂度**：仅需 max/min/mean 统计量，相比 A-ABFT 的 O(pn) 大大降低
3. **平台无关**：已在 NPU 和 GPU 上完成集成验证

### 📊 性能数据

| 指标 | V-ABFT | A-ABFT (SOTA) | 改进 |
|:-----|:------|:--------------|:----|
| FP32/FP64 阈值-实际误差比 | **7-20×** | 160-4200× | **6-48× 更紧** |
| BF16 阈值-实际误差比 | **48-158×** | 160-4200× | **显著改善** |
| 误报率 | **0** (所有精度) | — | — |
| 时间复杂度 | **O(n)** | O(pn) | 更低 |
| 细粒度检测（fused-kernel） | ~**1000× 更细** | — | 利用验证前量化输出 |

**关键洞察：** 对于 fused-kernel ABFT（验证在输出量化之前），低精度 GEMM 可使用 FP32 级阈值（e_max ≈ 10⁻⁶），相比离线验证（e_max ≈ 10⁻³）检测粒度提升约 **1000×**。

**验证平台：** 合成数据 + 真实模型权重（LLaMA-7B, GPT-2, ViT），覆盖 BF16/FP16/FP32/FP64 四种精度。

### 💡 为何重要

- ABFT 是 SDC 检测的核心基线——V-ABFT 将阈值精度提升 **1-2 个数量级**，意味着**更早、更准确地检测到 GPU 计算错误**
- 与 CCGrid 2026 SDC 论文（06-06 已跟踪）形成**互补**：那篇用训练信号检测 SDC 影响（loss/gradient），V-ABFT 从**计算层直接检测 SDC 本身**——两线合围
- 零误报率（zero false positive）在实际生产中极为关键——误报导致不必要的训练中断和检查点回滚
- 已在 NPU 和 GPU 上完成集成，具备**跨平台实用性**

## 🆕 3. LLMTailor — 按层裁剪检查点：利用更新非均匀性减少 4.3× 体积（PDSW'25）

| 元数据 | 内容 |
|:-------|:-----|
| **来源** | arXiv:2602.22158, 2026-02-25 | **PDSW'25 接收** 🏆 |
| **作者** | Minqiu Sun, Xin Huang, Luanzheng Guo, Nathan R. Tallent, Kento Sato, Dong Dai (UNC Charlotte + PNNL + RIKEN) |
| **领域** | 选择性检查点 / 模型训练效率 |

### 🎯 核心贡献

**关键观察：** LLM 训练中，各层的参数更新量**高度非均匀**——某些层变化剧烈，某些层几乎不变。这意味着**选择性检查点**是可行的。

**LLMTailor 方法：**
1. **按层差异追踪**：追踪各层权重在训练步间的更新幅度
2. **选择性快照**：仅对更新幅度超过阈值的层做检查点
3. **复合检查点组装**：从不同时间点的部分快照中**过滤+组装**形成完整一致的检查点
4. **优化器状态处理**：不仅裁剪权重，还裁剪对应的优化器状态（如 Adam 动量和方差）

**与现有方案的本质区别：**
- 传统方案：全量保存 → I/O 开销与模型大小线性增长
- LLMTailor：按需裁剪 → 开销与**活跃层数**成正比

### 📊 实验数据

| 模型 | 检查点体积缩减 | 检查点时间加速 | 模型质量 |
|:-----|:-------------:|:-------------:|:--------:|
| **Llama3.1-8B** | **4.3× 缩小** | — | **维持** |
| **Qwen2.5-7B** | — | **2.8× 加速** | **维持** |
| 其他模型 | 显著缩减 | 显著加速 | 维持 |

### 💡 为何重要

- **简单但有效**的核心洞察：LLM 训练中参数更新非均匀性——这一观察有多篇独立论文交叉验证（包括 06-06 的 CCGrid SDC 论文发现梯度分布不均等）
- 与 MoEvement（NSDI'26，MoE 专家子集检查点，06-06 已跟踪）形成**二维互补**：

  | 方案 | 裁剪维度 | 适用范围 |
  |:----|:---------|:---------|
  | **LLMTailor** | 沿模型深度（按层裁剪） | Dense 模型 |
  | **MoEvement** | 沿模型宽度（按 expert 子集） | MoE 模型 |

  两者组合使用理论上可实现**更极致的检查点缩减**

- PDSW（Parallel Data Systems Workshop）接收，说明该工作在 HPC 存储社区有认可度
