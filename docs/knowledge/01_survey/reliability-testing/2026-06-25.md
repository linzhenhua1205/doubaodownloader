## 1️⃣ ⚡ LLM-PRISM：RTL 级 GPU 永久故障 SDC 建模（7,664 次训练运行实证）

**arXiv:2604.10390** — 提交 2026-04-11 | cs.AR

**核心贡献**：首个基于硬件底层（RTL）的 GPU 故障仿真 + 随机注入引擎，对 LLM 预训练的 SDC 韧性进行系统性刻画。

**方法论创新**：
- **RTL 级 GPU 故障仿真** — 在寄存器传输级模拟硅制造缺陷和老化导致的永久性故障，而非抽象的高层位翻转
- **随机注入引擎嵌入 Megatron-LM** — 在真实训练框架内实现细粒度故障注入
- **三精度格式覆盖** — FP16 / BF16 / FP8 三种数值格式的全维度比较

**关键发现**（7,664 次训练运行）：
- LLM **对低频故障具有固有韧性**，但影响高度非均匀
- **关键数据路径**和特定精度格式下，即使中等故障率也可引发**灾难性发散**
- FP8 训练对 SDC 最敏感，BF16 最具韧性

**意义**：首次提供了基于硬件底层（非抽象模型）的 LLM SDC 韧性基准，为硬件冗余设计和软件容错策略提供了量化依据。

## 2️⃣ 🔍 CCGrid 2026：LLM 训练中 SDC 轻量检测与重算恢复

**arXiv:2604.00726** — 提交 2026-04-01 | 接收: CCGrid 2026

**核心贡献**：受控实验框架分析间歇性 SDC 对 LLM 预训练的影响，提出**轻量检测 + 重算恢复**方案。

**实验设计**：
- **GPU 矩阵乘法指令级注入** — 非抽象位翻转，精确注入 GEMM 指令的特定 bit 位置
- **覆盖范围**：不同 bit 位置、Kernel 函数、执行阶段（前向/后向传播）

**关键发现**：
- 局部故障可产生**全局影响**：NaN 传播、loss 尖峰、梯度范数暴涨、注意力 logits 异常
- **持续参数发散**：即使单步故障也可导致不可逆的训练轨迹偏离
- **轻量检测方案**：监测参数更新的异常特征（梯度范数突变、loss 不连续性），检测到后**重算最近一步**
- 在 LLaMA 60M / 350M / 1.3B 三个规模上验证有效性

**对比 LLM-PRISM**：本工作侧重**间歇性故障**（transient）而非永久性制造缺陷，检测方法更轻量（无需硬件冗余）。

## 3️⃣ 📦 TierCheck：LLM 训练三层分级检查点（<10s 端到端，40B 参数验证）

**arXiv:2605.17821** — 提交 2026-05-17 | cs.DC

**核心贡献**：面向 LLM 训练中**异构故障谱**（从单 GPU 崩溃到全集群故障）的分级检查点系统。

**三层设计**：
```
Tier 1 (本地内存) → Tier 2 (对等节点内存) → Tier 3 (远端持久存储)
   ↑ 轻量差异检查点          ↑ 异步迁移          ↑ 重量基础检查点
   ↑ 秒级本地恢复            ↑ 分钟级组恢复       ↑ 小时级全集群恢复
```

**关键创新**：
- **跨层全局一致性** — 不阻塞训练的前提下保证各层状态一致
- **集群感知恢复** — 恢复时根据故障范围选择最优恢复层

**实验结果**（40B 参数模型）：
- 端到端检查点时间 **<10 秒**
- 支持**高频检查点**（频率可从原来数十分钟→数十秒）
- 在低开销持久化和快速恢复之间取得最优平衡

## 4️⃣ ✂️ LLMTailor：逐层选择性检查点（4.3× 容量压缩，2.8× 速度提升）

**arXiv:2602.22158** — 提交 2026-02-25 | 接收: PDSW'25

**核心发现**：LLM 各层在训练中的更新量高度**非均匀**——部分层变化剧烈，部分层几乎不变。

**方案**：
- **检查点合并框架**：从不同检查点过滤和组装层，形成复合检查点
- **可配置选择性策略**：仅持久化发生显著更新的层
- 无需修改训练代码或模型架构

**实验结果**：
| 模型 | 检查点尺寸压缩 | 检查点加速 |
|:-----|:--------------|:-----------|
| Llama3.1-8B | **4.3×** 减小 | — |
| Qwen2.5-7B | — | **2.8×** 加速 |

**意义**：打破了\"全模型全状态检查点\"的隐含假设，揭示了 LLM 训练中参数更新稀疏性可作为 CR 优化的新维度。与 TierCheck 分层思路互补。

## 5️⃣ 🔲 Grid-ECC：矩阵乘法纠错编码（确定性纠正 2 个错误符号，24% GPU 开销）

**arXiv:2508.04355** — 提交 2025-08-06（持续追踪）

**问题**：分布式训练中矩阵乘法的 SDC 逃逸检测，错误累积导致模型退化。

**方案**：基于网格结构编码的纠错编码框架：
- **网格结构编码** — 在所有参与矩阵之间分布冗余，增强定位和纠错能力
- 针对矩阵乘法运算定制的纠错机制

**实验结果**：
- **100% 确定性校正** — 跨三个矩阵分布的 **最多 2 个错误符号**
- GPU 架构上计算时间开销仅 **24%**
- 提供严格的理论纠错性质证明

**意义**：为矩阵乘法核心运算提供了一种硬件层之上的数学容错方案，与算法级 SDC 检测互补。
