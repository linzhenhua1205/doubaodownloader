## ① 🛠️ Cursor SDK 3.7 — Custom Tools + Auto-Review + JSONL Stores（6月4日）

**来源**: [Cursor Changelog](https://changelog.cursor.com/) · **GA**

SDK 从"可用的 Agent SDK"进化为**可编程的 Agent 基础设施**，四个核心新增能力：

| 能力 | 技术详情 |
|:-----|:---------|
| **Custom Tools** | 通过 `local.customTools` 直接将函数定义暴露给 Agent，内置 `custom-user-tools` MCP Server。**不再需要独立搭建 MCP Server 来暴露自定义能力**。Custom Tools 自动对子 Agent 可见 |
| **Auto-Review** | 本地 SDK Headless 运行默认不审批工具调用。通过 `local.autoReview` + `permissions.json` 中的 `autoRun.allow_instructions` / `block_instructions`，用自然语言定义哪些调用自动放行、哪些暂停审批。分类器（classifier）决策 | 
| **JSONL Stores** | 新增 `JsonlLocalAgentStore` 替代默认 SQLite。JSONL 是纯文本、append-only 文件，**可 diff、可纳入版本控制**。也支持自定义 `LocalAgentStore` 接口，存到 Postgres 或 in-memory |
| **Nested Subagents** | 子 Agent 可递归创建子 Agent（无限深度），每层独立 prompt + 模型。**无需配置，自动生效** |

**配套改进**:
- `requestId` 跨所有存储持久化，可将脚本/CI 运行关联到后端日志
- 导入 `@cursor/sdk` 不再惰性加载全 Local Agent 栈，Cloud-only/type-only 场景跳过本地运行时
- Composer 2 → Composer 2.5 自动路由
- Python SDK 0.1.6 发布，`list_runs` 支持 workspace-scoped

> **趋势研判**: SDK 化是 AI Coding Agent 走向工程化基础设施的关键一步。Custom Tools + Auto-Review 让 Agent 不再是"黑盒聊天"，而是可编程、可审计的自动化组件

## ② 🎤 Cursor Design Mode — Voice Input + Multi-Select（6月5日）

**来源**: [Cursor Changelog](https://changelog.cursor.com/) · **GA**

- **Multi-Select Elements**: 在浏览器中同时选中多个元素。Cursor 看到选中元素的代码、周围布局和视觉关系 → 支持批量操作（"让这个和那个一致""删除重复内容""调整一组组件"）
- **Voice Input**: Mic 在 Agent 运行期间持续可用。用户可在当前编辑未完成时**排队下一个语音指令** — 非阻塞并行交互模式
- **Canvas Design Mode**: 6月4日同批发布，Canvas 创建的交互式 artifact 也支持 Design Mode 直接标注

**Context Usage Report（6月4日同期）**:
- 可交互 Token 分布报告：system prompt、tool definitions、rules、skills 等分别占多少 Token
- **"Debug with Agent"按钮**：可让 Agent 分析上下文使用并提出优化建议
- Canvas 形式呈现，Agent 可定制报告回答具体问题

## ③ 🛡️ GitHub 安全验证覆盖第三方 AI Coding Agent — GA（6月9日）

**来源**: [GitHub Changelog](https://github.blog/changelog/2026-06-09-security-validation-for-third-party-coding-agents/) · **GA**

GitHub 将自动安全验证从 Copilot Cloud Agent 扩展至所有第三方 AI Coding Agent：

- **覆盖范围**: Claude Code、OpenAI Codex 等第三方 Agent 生成的代码
- **三合一安全栈**:
  - **CodeQL**: 漏洞扫描
  - **GitHub Advisory DB**: 新增依赖检查
  - **Secret Scanning**: API 密钥、Token 等敏感信息检测
- **工作流**: 发现问题后 Agent **自动尝试修复**再最终化 PR
- **效果**: 自 2025年10月上线以来已**主动阻止数百次安全泄露**
- **默认开启**: 遵循仓库 Copilot 设置，**无需 Advanced Security 许可**
- 已启用 Copilot Cloud Agent 安全验证的仓库自动获得第三方 Agent 保护

> **趋势研判**: 跨 AI Agent 的"安全层标准化"——不管用哪个 Agent，代码统一走安全门禁。这是 AI Coding Agent 从"工具"进化为"基础设施"的必经之路

## ④ 🔍 GitHub Copilot CLI — 专用安全审查命令 /security-review（6月10日）

**来源**: [GitHub Changelog](https://github.blog/changelog/2026-06-10-dedicated-security-review-command-now-available-in-copilot-cli/) · **Experimental Public Preview**

- **命令**: `/security-review` — 在 Copilot CLI 中对当前代码变更运行安全审查
- **扫描范围**: 注入漏洞、XSS、不安全数据处理、路径遍历、弱加密等常见高危漏洞类型
- **输出**: 按严重性和置信度评分的安全发现 + 无需离开终端的可操作建议
- **独立性**: 不依赖 GitHub Code Scanning、Dependabot 或 Secret Scanning，是**互补的轻量按需扫描**
- **定位**: 提交前快速审查

## ⑦ 🎯 HBR 六月刊 — AI 重塑决策权 + 虚假对齐陷阱 + AI 使用全景（6月初）

**来源**: [HBR](https://hbr.org/)

最新 HBR 六月刊聚焦三个核心主题：

### 决策权的四种常见错误
- **来源**: Lindy Greer, Jennifer Jordan, Maxim Sytch《What Companies Get Wrong About Decision Rights》
- 企业普遍存在四种决策权错误：决策边界模糊、授权但不赋权、集体决策无流程、权责分离
- AI Agent 引入后：**谁为 Agent 的决策负责？** — 决策权问题被进一步放大

### 虚假对齐陷阱（The False Alignment Trap）
- **来源**: Julia Dhar, Kristy R. Ellmer, Philip Jameson
- 表面上团队对齐了一致目标，实际上只是**"虚假对齐"**——每个人都以为自己理解一致
- 在 AI 驱动的组织中问题更突出：Agent 的"理解"和人类的"期望"之间的对齐需要可验证的反馈机制

### AI 使用全景（12,637 用例分析）
- **来源**: 《How People Are Really Using AI in 2026》
- 新发现的关键风险：**"Letting AI think for you"**——过度依赖 AI 替代思考过程
- 组织需建立 AI 使用边界和批判性思维训练
- ChatGPT 9 亿 MAU 的背景下，Vibe Coding + Agentic Workflows 成为主流

## ⑧ 🧩 Atlassian — 多项 AI Agent 工程化实践更新

**来源**: [Atlassian Blog](https://www.atlassian.com/blog)

### AI Agent in Jira 削减 80% 工程"杂务"
- 使用 AI Agent 自动处理**Flaky Test 修复和 Feature Flag 清理**
- 这两个类别合占工程团队碎片化时间的很大比例
- 来源: 《How We Cut up to 80% of Engineering "Chores" Using AI Agents in Jira》

### 设计系统 → MCP Schema — 降低 Agent Token 消耗
- 将 Atlassian Design System 文档翻译为机器可读的 Schema 文件
- 通过 **ADS MCP + Skills** 向 Agent 提供精确的设计上下文
- 效果: **提高准确性 + 生成速度 + 降低 Token 成本**
- 来源: 《Teaching AI to speak our design language》

### Agent Context Pruning — 长会话记忆管理
- Rovo Dev 在长会话中实施上下文剪枝
- 长任务中 Agent 积累大量上下文，通过智能丢弃不关键信息保持会话效率
- 来源: 《Agent Context Pruning: How Rovo Dev keeps long sessions useful》

### 新发现：新员工 AI 使用率高 1.5×
- Atlassian 研究发现新员工（1-3 年）比老员工（10+年）**每天使用 AI 的可能性高 1.5 倍**
- 趋势：削减新招聘反而切断了组织最快的 AI 采用路径
- 来源: 《New grads are 1.5x more likely to use AI daily》
