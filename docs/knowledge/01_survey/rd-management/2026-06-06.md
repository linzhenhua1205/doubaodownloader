## ① Cursor Design Mode Improvements — 多选+语音+绘图三重交互升级

| 属性 | 内容 |
|:-----|:------|
| **来源** | [Cursor Blog - Direct agents with visual prompts in Design Mode](https://www.cursor.com/blog/design-mode)，2026-06-05 |
| **类型** | 🛠️ 产品功能更新 |
| **版本** | Cursor 3.7 |

**核心更新**: Design Mode 新增三种视觉交互方式，让用户无需离开运行中的应用即可直接、精准地指导 Agent 编辑 UI：

1. **Multi-select（多选）** — 在浏览器中同时选中多个元素，Cursor 捕获选中元素的 identity（xpath、组件、属性、computed styles、fiber tree props）+ 截图（空间上下文、布局、页面状态）。支持：
   - 让一个元素匹配另一个的样式
   - 批量删除重复内容
   - 一组组件统一调整
2. **Voice Input（语音输入）** — 通过 Design Mode 叠加层用语音描述变更，Agent 运行时 mic 保持可用，可排队下一个更改
3. **Drawing（绘图标注）** — 在冻结的页面帧上圈选/框选区域，Agent 看到精确的页面状态

**方法论**: 从「文本描述→Agent理解」到「空间交互→Agent理解」——**元素身份（xpath/组件/样式）** 与 **截图空间上下文** 双信号互补

**关键数据**:
- 选中一个元素向上下文注入 2 种互补信号：元素标识（xpath/组件/attrs/styles/fiber props）+ 空间截图（layout/周围元素/精确页面状态）
- 特别适配 **Composer 2.5** 模型（快速+擅长界面工作）
- Agent 完成后自动 **hot reload**

## ② Cursor SDK v3.7 — Custom Tools + Auto-Review + Nested Subagents + JSONL Store

| 属性 | 内容 |
|:-----|:------|
| **来源** | [Cursor Changelog](https://www.cursor.com/changelog) - Jun 4, 2026 |
| **类型** | 🛠️ 产品功能更新 |
| **版本** | Cursor SDK 3.7 (TypeScript + Python) |

**四大核心能力：**

### Custom Tools（自定义工具）
- 通过 `local.customTools` 将函数定义直接暴露给 Agent（本质是内置 MCP server `custom-user-tools`）
- 无需搭建 stdio/HTTP MCP server，一个函数定义即可
- 自定义工具对父 Agent 的所有 subagent 可见

### Auto-Review for SDK（SDK 版自动审查）
- 默认 headless SDK 运行绕过审批 → `local.autoReview` 启用分类器审查
- 使用 `permissions.json` 的 `autoRun.allow_instructions` / `block_instructions` 通过**自然语言**描述哪些调用可自动放行
- **关键数据**: 可区分「只读检查 build artifacts → 允许」「删除操作 → 暂停审批」

### Nested Subagents（嵌套子Agent）
- Subagent 可递归 spawn 自己的 subagent，任意深度
- Reviewer agent → test-writer agent → deeper，每层独立 prompt + model
- **自动启用**：subagent 会话注册所需 executor，nesting 自动工作

### JSONL Store（JSONL 持久化）
- 新增 `JsonlLocalAgentStore` 替代默认 SQLite，纯文本 append-only、可 diff、可版本控制
- 实现 `LocalAgentStore` 接口可自定义：in-memory（CI 场景）或 Postgres

**方法论**: Agent 从「黑盒自动化」走向「可编程、可审查、可嵌套、可审计」的基础设施

**关键数据**:
- 轻量 import：`@cursor/sdk` 不再预加载完整本地 Agent 栈
- Composer 2 → Composer 2.5 自动路由
- 每个 `send()` 携带平台生成的 `requestId`，全链路追踪
- Python SDK 0.1.6 发布，支持 Buildkite 路径

## ④ GitHub Copilot: Budget and Usage Management APIs GA

| 属性 | 内容 |
|:-----|:------|
| **来源** | [GitHub Changelog](https://github.blog/changelog/2026-06-04-budget-and-usage-management-apis-now-generally-available/)，2026-06-04 |
| **类型** | 🏢 Enterprise 治理 |
| **版本** | GA |

**核心内容**: GitHub 的预算和用量管理 API 正式 GA，从此**预算全生命周期可通过编程管理**。

**关键能力**:
- **Budget API**: 编程式创建/更新/删除预算 + 调整金额和告警通知（此前仅 UI 可操作）。**限额：每账户 50 个预算**
- **Usage Summary API**: 按 account/org/repo/cost center/product/SKU 粒度查询，支持年/月/日维度
- **Cost Center API 增强**: 新增 `state` 参数（`?state=active`），只获取活跃成本中心
- **Usage Report API 调整**: 去掉了 `hour` 参数，`day` 参数返回日汇总而非小时级

**适用对象**: Enterprise 计划的所有者和账单管理员

**方法论**: AI 编码工具的**成本可观测性从"被动看账单"升级到"主动编程预算管理"**，是企业级规模化部署 AI 编码工具的前置条件。

## ⑤ Claude Code 2.1.160-167: Fallback Model + Parallel Tool Calls + Version Pinning

| 属性 | 内容 |
|:-----|:------|
| **来源** | [Claude Code Changelog](https://docs.anthropic.com/en/docs/claude-code/changelog)，2026-06-02 ~ 2026-06-06 |
| **类型** | 🛠️ 产品功能更新 |
| **版本** | 2.1.160 → 2.1.167 |

**关键更新（方法论导向）：**

### Fallback Model（2.1.166，Jun 6）
- `fallbackModel` 设置可配置最多 **3 个 fallback 模型**，在主模型过载/不可用时按顺序尝试
- `--fallback-model` 现在也适用于交互式会话
- **非可重试错误**（auth/rate-limit/request-size/transport）直接报错，其他错误自动在 fallback 上重试一次

### Parallel Tool Calls – 独立失败（2.1.161，Jun 2）
- **关键改进**: 并行工具调用中，**某个 Bash 命令失败不再取消同一批次的其他调用**
- 每个 tool 独立返回结果，互不影响
- 显著提升多任务场景的鲁棒性

### Version Pinning Managed Settings（2.1.163，Jun 4）
- `requiredMinimumVersion` / `requiredMaximumVersion` — 企业可强制 Claude Code 版本范围
- 版本不符合时拒绝启动，引导用户安装批准的版本

### /plugin list 命令（2.1.163，Jun 4）
- 列出已安装的 plugins，支持 `--enabled`/`--disabled` 过滤
- 与 Cursor 插件生态形成呼应的治理能力

### Hooks 改进（2.1.163，Jun 4）
- Stop/SubagentStop hooks 可返回 `hookSpecificOutput.additionalContext`，keep turn going 且不被标记为 hook error
- Skills 新增 `\$` 转义语法，在 command body 中含实际 `$` 前跟数字的场景

### Glob Pattern in Deny Rules（2.1.166，Jun 6）
- deny rule 的 tool-name 位置支持 glob 模式（`"*"` 拒绝所有工具）
- allow rules 拒绝非 MCP 的 glob

**关键数据**:
- 并行工具调用失败隔离 → 显著提升大规模自动化任务的成功率
- Fallback 最多 3 个模型链
- MAX_THINKING_TOKENS=0 或 `--thinking disabled` 可关闭模型的默认思考模式
