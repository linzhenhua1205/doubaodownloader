## ① ⚙️ **GitHub Copilot Code Review — 组织级 Runner 控制 + 内容排除 + 自定义指令不限长度**（6月12日）

**来源**: [GitHub Changelog](https://github.blog/changelog/2026-06-12-copilot-code-review-new-configurations-and-controls/)

**技术详情**:

Copilot Code Review 迎来三项重要企业级增强：

**a) 组织级 Runner 控制**
- Copilot Code Review 基于 GitHub Actions 运行。管理员现可在**组织级别**设置默认 Runner 类型（自托管/Large Runner），一键应用于所有仓库
- 支持**锁定设置**：组织默认 Runner 可覆盖所有仓库的独立配置（锁死后仓库无法更改）
- 配置入口：Organization → Copilot → Runner type

**b) Content Exclusion 内容排除**
- Copilot Code Review 现支持**仓库/组织/企业级内容排除设置**
- 管理员可配置路径规则，阻止 Copilot 在审查时访问指定文件或目录
- 用途：排除非审查相关上下文的干扰，或对齐企业数据边界

**c) 自定义指令字符限制移除**
- 此前 `copilot-instructions.md` 和 `*.instructions.md` 文件超过 **4,000 字符**即停止读取，此限制现已完全移除
- 允许更丰富、更灵活的审查指令定制

**意义**: 三项控制分别覆盖**基础设施治理（Runner）**、**数据边界治理（Content Exclusion）**、**指令灵活性（无限制自定义指令）**，表明 GitHub Copilot Code Review 的**企业治理体系三层补齐**。

## ② 🔒 **Claude Code v2.1.175 + v2.1.176 — enforceAvailableModels 治理闭环 + 多项安全修复**（6月12日）

**来源**: [Claude Code Changelog](https://docs.anthropic.com/en/docs/claude-code/changelog)

**v2.1.175 — `enforceAvailableModels` 托管设置**（6月12日发布）:
- 启用后，`availableModels` 白名单**也约束 Default 模型**（Default 若解析到被禁止的模型→自动回退到第一个允许的模型）
- 用户设置和项目设置**不能拓宽**由托管设置的 `availableModels` 列表 → 企业管理员获得最高级模型管控锁定权限

**v2.1.176 — 关键修复与增强**（6月12日发布）:
- **Session 标题语言本地化**：标题现在按对话语言生成（可设置 language 固定语言）
- **Bedrock 凭据缓存改进**：`awsCredentialExport` 的凭据缓存到 `Expiration` 时间，而非固定1小时 → 减少不必要的 AWS 凭据刷新
- **Fixed availableModels 强制执行漏洞**：此前通过 `ANTHROPIC_DEFAULT_*_MODEL` 环境变量的别名校验可**绕过** availableModels 白名单跳转到被禁止的模型——已修复
- **`/fast` 切换时校验**：如果 `/fast` 会切到 allowlist 之外的模型，现在会拒绝切换
- **Fixed auto mode 在 Fable 5 上对无 Opus 4.8 组织失败**：分类器改为回退到最佳可用 Opus 模型
- **Fixed hook `if` 条件对 Read/Edit/Write 工具路径**：`Edit(src/**)`、`Read(~/.ssh/**)`、`Read(.env)` 等文档中的模式现在正确匹配
- **Fixed Linux 沙箱在 `.claude/settings.json` 为绝对路径符号链接时无法启动**
- **Fixed Remote Control 从 web/mobile 连接时静默切换会话模型**
- **Fixed `/cd` 和工作树移动后会话仍报告之前目录的 git branch**

**意义**: Claude Code 企业治理体系实现**完整闭环**——从「设置白名单」（allowed models）→「锁定白名单不可被环境变量绕过」（v2.1.176）→「Default 模型也被约束」（v2.1.175 enforceAvailableModels）→「用户/项目设置不可拓宽」（v2.1.175）。三次迭代、三层锁死，企业管理员获得对 AI Agent 的**绝对模型管控**。

## ③ 📊 **HBR: "How Gen AI is Disrupting B2B Buying Decisions" — AI 中介重构 B2B 决策**（6月12日）

**来源**: HBR · Amit Joshi, Ivy Buche & Caroline Schwaer

**核心观点**:
- Generative AI 正在彻底改变 B2B 购买行为，将**发现、评估、推荐**迁移到 AI 中介环境中
- 公司既**不拥有、也不完全理解**这些 AI 中介的决策逻辑
- 案例：两家竞品药企 A 的新药获 FDA 批准，但 B 的竞品因临床讨论更多、治疗指南引用更多而在 AI 中介排名中占优 → **AI 中介的可见性成为新的竞争维度**

**对 R&D 管理的影响**:
- B2B SaaS 产品的购买决策越来越多由 AI Agent 做出，产品策略需要从"面向人类购买者"转向"面向 AI 评估者"
- 结构化数据、一致性、引用量比品牌营销更重要

## ④ 🤝 **Atlassian 最新研究：诚实披露 AI 使用被判定为 10× 更懒**（6月13日）

**来源**: [Atlassian Blog - Teamwork Lab](https://www.atlassian.com/blog/)

**核心发现**:
- Atlassian Teamwork Lab 最新研究发现，**诚实披露使用 AI 的工人被同事和管理者判定为比做同样工作的同行懒惰 10 倍**
- **但**：如果公司建立了**庆祝 AI 使用**的文化，这种负面判断消失
- Conor Donegan (作者)："公司削减新员工招聘→正在削减 AI 采用的最快路径"（新员工日 AI 使用率高 1.5×）

**对 R&D 管理的启示**:
- 组织**信任文化**是 AI 采用的最大瓶颈，比技术选型更重要
- 企业需要明确的 AI 使用政策 + 开放的 AI 分享文化，否则员工会**秘密使用 AI**（配合 HBR Anicich & Brouwers 6月研究）
- 新员工招聘的削减正在**反向影响**组织的 AI 能力建设

## ⑤ 📊 **Atlassian: Adam Grant — AI 时代敏捷 > 能力**（6月13日）

**来源**: [Atlassian Blog - Leadership](https://www.atlassian.com/blog/)

**核心观点 (Adam Grant, Wharton)**:
- AI 时代**敏捷比能力更重要**——因为 AI 快速补偿了能力的不足，但无法补偿组织的僵化
- 领导者重新设计团队的关键原则：**实验文化 > 规划文化**、**快速试错 > 完美执行**
- 与 AI 高效协作的团队特征是**高心理安全 + 快速反馈循环**

## ⑥ 📊 **HBR: "For Hope to Inspire, It Has to Be Grounded in Organizational Reality"**（6月11日）

**来源**: HBR · Alyson Meister, Nele Dael & David Bach (IMD)

**核心观点**:
- 希望不是乐观或一厢情愿——它是一种**以目标为导向的激励力量**，将当前行动与期望的未来联系起来
- 在不确定性和颠覆条件下，愿景本身很少能激发持续努力
- 领导者的差距在于：提出振奋人心的 AI 愿景，但**组织现实（流程、工具、技能）不支持那个愿景**→ 员工感到**被空口许诺**

**对研发管理的启示**:
- AI 转型中，领导者不能只画饼（"我们是 AI-first 的组织"），必须**同步建设组织基础设施和文化**
- 希望需要三条腿：**清晰的路径 + 可靠的资源 + 可信的执行力**
