## ① Cursor Design Mode 重大更新 — 视觉/语音/绘图三重交互（6月5日）

> **来源**: [Cursor Blog — Direct agents with visual prompts in Design Mode](https://www.cursor.com/blog/design-mode) | Erik Nilsson, Ian Huang & Ryo Lu

**核心能力**: 从「文本描述→Agent理解」升级为 **空间交互→Agent理解**，三种自然交互方式：

1. **点选**: 点击运行中App的UI元素，自动注入「**元素身份**（xpath/组件/属性/computed styles/fiber tree props）」+「**空间截图**（布局/周围元素/精确页面状态）」双信号
2. **多选**: 同时选中多个元素，描述它们之间的关系（如「让这个组件匹配另一个」→ 批量调整）
3. **绘图**: 在冻结页面帧上圈选/框选，Agent看到精确的**页面锁定状态**
4. **语音**: 运行时mic保持可用，可**排队发送指令**无需等待当前编辑完成

**关键创新**: 点选一个元素后即可**立即发送下一个编辑**，多个子Agent并行工作，Composer 2.5 hot reload自动展示结果。

**方法论**: 空间交互四通道（点/选/画/说）+ 双信号注入（元素身份+空间截图）→ 缩小「所见」与「Agent理解」之间的差距。这是人类空间思维模式与AI理解之间的原生交互范式。

## ② Cursor Enterprise Organizations — 三层企业治理架构（6月3日）

> **来源**: [Cursor Blog — Introducing organizations for Cursor Enterprise](https://www.cursor.com/blog/organizations) | Cursor Team

**三层结构**:

| 层级 | 作用 | 配置粒度 |
|:-----|:------|:----------|
| **Organization** | 顶层容器，统一身份/管理/成员 | 整个公司 |
| **Team** | 部门/子公司的运营单元，独立安全/预算/功能设置 | 业务单元 |
| **Group** | 轻量级用户集合，跨Team或其中，单独模型/预算/权限 | 用户队列 |

**企业实践案例（NVIDIA）**:
- Engineering/Product Team → 最宽松的网络访问 + Agent自动运行权限
- Sales/Marketing/Finance → 更紧的安全控制，Agent操作需手动审批
- **Sandbox团队**：单独staging team测新功能预览，工程师可在production + staging双团队切换无需第二个账号

**方法论**: 组织级AI治理的核心是**最小权限 + 按功能分层 + 沙盒先行**。模型访问、Agent自动执行、预算消耗、功能预览四个维度需独立管控。

## ③ GitHub Copilot Agent Tasks REST API — Agent即基础设施（6月4日 Public Preview）

> **来源**: [GitHub Changelog — Agent tasks REST API now available](https://github.blog/changelog/2026-06-04-agent-tasks-rest-api-now-available-for-copilot-pro-pro-and-max/)

**核心能力**: Copilot Cloud Agent通过REST API编程式启动和跟踪：
- 代码变更 → 验证 → 自动开PR，全流程自动化
- 支持自定义开发环境中运行，不占用本地资源

**典型场景**:
1. **批量跨仓库重构/迁移**：一个脚本遍历所有仓库启动Agent任务
2. **开发者门户一键建仓**：从内部开发者门户触发新仓库初始化
3. **每周自动发版**：Agent自动准备发版+release notes

**关键数据**: 支持 PAT（classic + fine-grained）和 OAuth tokens 认证

**方法论**: Agent从「交互式编码工具」进化为**可编程基础设施**——这是与Cursor SDK平行的关键跨越。Agent不再是IDE里的聊天窗口，而是通过API编排的基础设施组件。

## ④ GitHub 企业治理三层齐发（6月4-5日）

> **来源**: GitHub Changelog (4条)

### 4.1 Enterprise Managed Plugins in VS Code（6月5日 Public Preview）

`.github-private/.github/copilot/settings.json` 统一分发插件，覆盖 Copilot CLI + VS Code 双客户端。

**管控能力**: 自动安装插件、全局强制钩子/Hooks、MCP配置统一分发。

**方法论**: AI编码工具的企业管控从「模型选择控制」扩展到 **插件生态的全面治理**——自定义Agent/Skills自动分发。

### 4.2 Enterprise Teams GA（6月4日）

跨50+组织的统一团队管理：
- 生产规模: 2,500 teams/enterprise, 5,000 members/team
- SCIM IDP 驱动成员同步，实时传递到各级组织
- `@mention` 跨组织通知、统一 Reviewers 分配、审计日志全覆盖

### 4.3 Budget & Usage Management APIs GA（6月4日）

**从被动看账单到主动编程预算管理**：
- 编程式创建/更新/删除预算（限50个/账户）
- Usage Summary 按 org/repo/cost center/SKU/产品查询
- 支持按年/月/日粒度

**方法论**: 三管齐下构建企业级AI治理闭环：**Identity（Enterprise Teams）+ Policy（Managed Plugins/Hooks）+ Cost（Budget APIs）**

## ⑤ Claude Code v2.1.160-168 密集更新（6月2-6日）

> **来源**: [Claude Code Changelog](https://docs.anthropic.com/en/docs/claude-code/changelog)

### 企业级特性
- **Version Pinning Managed Settings** (2.1.163): `requiredMinimumVersion` / `requiredMaximumVersion` → 版本合规管控
- **Fallback Model** (2.1.166): 最多配置3个fallback模型链，主模型不可用时自动切换
- **Glob Pattern Deny Rules** (2.1.166): `deny rules` 支持 `*` 通配匹配所有工具

### Agent 可靠性
- **Parallel Tool Calls 独立失败** (2.1.161): 某个Bash失败不再取消整批工具调用，每个工具独立返回结果
- **Auto Mode 优化** (2.1.161): 降低日常动作的推理延迟，减少「无法评估此操作」阻塞
- **Cross-Session Messaging 安全加固** (2.1.166): `SendMessage` 中继消息不再携带用户权限

### 交互优化
- **/plugin list** (2.1.163): 列出已安装插件，支持 `--enabled/--disabled` 过滤
- **Edit读前检查优化** (2.1.160): `grep/egrep/fgrep` 查看单文件后直接Edit，无需单独 Read
- **/effort ultracode → xhigh** (2.1.160): 动态工作流触发关键词从 `workflow` 改为 `ultracode`
