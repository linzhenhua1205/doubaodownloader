## 🏆 TOP 要闻

### ① Anthropic「Claude Tag」正式上线：AI 从「聊天窗口」进化为「团队队友」

Anthropic 推出「Claude Tag」——一个**始终在线**的 Claude，作为 Slack 中的"AI 队友"。用户只需在频道中 @Claude 即可激活。

**核心突破**：

| 维度 | 细节 |
|:-----|:------|
| **持久上下文** | Claude 跟随频道对话持续学习，积累对企业业务的深入理解 |
| **跨渠道感知** | 经授权可自动从组织内其他频道获取信息 |
| **单身份共享** | 整个频道共享一个 Claude 身份，任何人可查看 Claude 的历史工作和承接对话 |
| **权限隔离** | 法律团队的 Claude 不会"记住"工程频道的对话——按频道严格隔离 |
| **环境模式(Ambient Mode)** | 不等待被 @——Claude 可**主动**跳入对话提醒团队、标记被遗忘的任务、跨组织汇报进展 |

> 💡 **进化节点**：此前 Claude 在 Slack 中只能被动响应@，Claude Code 也只处理编程任务。Claude Tag 是 **Agent 从工具进化到"数字同事"** 的关键一步——它不再是一个你指挥的界面，而是一个**有记忆、能主动、可交接的团队成员**。

**竞争格局对比**：
- **Microsoft Graph → Copilot/Work IQ**：微软同样在构建跨应用组织知识层
- **Snowflake/Databricks**：定位为「后台」——Agent 可调用的组织知识库
- **Glean**：构建模型与数据之间的企业智能层

（TechCrunch, Jun 23）

### ② 特朗普政府批准 Mythos 5 开放——100+ 美国公司/机构可使用

美国政府在经历了数周的对峙和 Anthropic 模型禁令后，**批准了 100 多家美国公司和政府机构使用 Anthropic Mythos 5**。禁令最初因担心 Mythos 5 / Fable 5 被「越狱」造成国家安全威胁而发布。

> 💡 这标志着 **AI 模型出口管制从「一刀切禁令」转向「白名单制度」**。政策的细节和受惠实体名单尚不清楚，但明确释放了一个信号：**美国政府不希望本土企业完全失去前沿模型能力**。

但同时，Anthropic 的纠纷远未结束——白宫仍在施压 Meta 签署类似的 AI 模型审核协议，而 Meta "尚未同意"。（The Verge / TechCrunch, Jun 26-28）

### ③ SpaceX 以 $600 亿收购 AI 编程平台 Cursor——编程助手赛道最大整合事件

**Ars Technica 独家报道**：SpaceX 将以 **$600 亿**估值收购 AI 编程平台 **Cursor**。文章标题一语道破意义：*"Separately, neither could compete. Now they hope they can."*

> 💡 **解读**：
> - **$600 亿**的估值远超绝大多数人对"AI 编程助手"市场的认知——说明 AI 编程能力**已被视为下一代战略基础设施**
> - SpaceX 意图将 Cursor 整合进其火箭制造和卫星生产的**物理系统编程**中（结合另条新闻——Nvidia 用 AI 编程 Agent 教会机器人安装 GPU）
> - 如果成功，这将成为 **AI 编程+物理世界融合**的标志性案例
> - 对 Cursor 而言，独立竞争空间收窄，被 SpaceX 收购意味着 AI 编程赛道进入**巨头整合阶段**

（Ars Technica, Jun 16）

### ④ Anthropic 暂停 Claude Agent SDK Token 计费——Agent 定价模型仍在探索

Anthropic 原计划对 **Claude Agent SDK** 按 Token 计费，但**在最后一刻暂停了**该计划。该变动原本会**大幅增加重度用户的使用成本**（Agent 多步骤任务消耗大量 Token）。

> 💡 **关键信号**：
> - Agent 场景的 Token 消耗比简单对话高 1-2 个数量级——现有的 Token 计价模型对 Agent 不友好
> - Anthropic 暂停表明 **Agent 的定价模式尚不成熟**——按 Token 计费还是按月费/任务/固定价格，行业仍在实验
> - 这与「企业 AI 预算管理」趋势一致——Accenture 等企业已开始审查和控制 AI Token 支出

（Ars Technica, Jun 16）

### ⑤ 企业 AI 预算管理新问题：员工用小任务刷爆 AI 预算

TechCrunch 报道：**越来越多的公司正在紧急寻找方法，阻止员工用大量微小任务消耗 AI 预算**。

问题核心：
- AI 订阅/Token 预算通常按用户或团队分配
- 员工发现 AI 可以做很多小任务（比如"帮我改这封邮件的语气""总结这个会议记录"……）
- 这些微任务累计起来**迅速耗尽预算**，而真正的高价值工程用法反而被挤压

> 💡 **行业启示**：这暴露了当前 AI 工具的**使用模式与成本模型的不匹配**。当 AI 从"偶尔用的工具"变成"日常助理"时，任务数量暴增但任务价值递减，导致**总量爆预算但产出不如预期**。这正是"Token 高效使用"成为热门话题的驱动力之一。

（TechCrunch, Jun 24）
