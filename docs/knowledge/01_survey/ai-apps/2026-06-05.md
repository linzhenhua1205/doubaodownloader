## 1. Agent OS：五种驯服不确定性的范式（深度文章归档）

- **来源**: [博客园 - 罗西的思考](https://www.cnblogs.com/rossiXYZ/p/20296163) | 2026-06-04
- **核心论点**: Agent OS Engineering = 在"执行者本身是概率性的"这一新约束下，重新组合计算机 70 年来五种驯服不确定性的成熟范式

### 1.1 问题空间：不确定性的六个来源

| # | 来源 | 性质 | 可消除？ |
|:-:|:-----|:-----|:---------|
| ① | LLM 输出概率性 | 认知不确定性 | 部分 |
| ② | Tool 调用可能失败 | 偶然不确定性 | 部分 |
| ③ | 环境状态变化 | 外部扰动 | 不可消除 |
| ④ | Context Window 有限 | 观测约束 | 不可消除 |
| ⑤ | 多 Agent 并发 | 竞争条件 | 可管理 |
| ⑥ | 模型升级行为漂移（假设腐化） | 平台演变 | 不可消除 |

**三个独有问题**（传统系统较少遇见）:
- **概率性执行主体** — 传统程序确定性执行，LLM 不是
- **观测窗口硬约束** — Context Window 是物理上限
- **假设腐化** — 模型升级后 prompt/阈值/参数可能全部失效

### 1.2 五种范式总览

| 范式 | 原理 | 经典领域类比 | Agent OS 映射 |
|:-----|:-----|:------------|:-------------|
| ① **冗余 + 投票** | 多副本/采样对冲个体失败 | TMR/纠错码/RAID | Best-of-N + Verifier、多模型 Ensemble |
| ② **闭环反馈** | 执行→观测→比对→修正 | PID/Kalman/TCP ACK | Verify-then-Act Loop、Kalman 式 Context 管理 |
| ③ **约束空间** | 让错误不可能发生 | 类型系统/Sandbox/IOMMU | JSON Schema、Capability 白名单、Sandbox |
| ④ **确定性优先路由** | 优先走确定性路径 | CLI > GUI、AOT > JIT | Rule > API > CLI > MCP > GUI > Free-form LLM |
| ⑤ **不可逆隔离** | 限制损害范围 | 事务+Rollback/CoW/Namespace | Grant 门控、Checkpoint、有界委托 |

### 1.3 ETCLOVG 七层架构

源自 CMU/Yale/Amazon 综述论文《Agent Harness Engineering: A Survey》，对 Agent Harness 生态 138+ 项目的归纳：

| 层 | 核心职责 | 主力范式 | 关键指标 |
|:---|:---------|:---------|:---------|
| **E** Execution | 隔离执行、资源限制 | 约束 + 隔离 | 逃逸率、冷启动延迟 |
| **T** Tool | 统一接口、确定性路由、幂等 | 确定性优先 | 确定性路径覆盖率、调用成功率 |
| **C** Context | 事实存储、投影、记忆管理 | 闭环反馈 | Context 利用率、漂移检测率 |
| **L** Lifecycle | 推理循环、编排、Feature Gate | 反馈 + 隔离 | 步数/任务、Feature pass 率 |
| **O** Observability | Trace、成本、健康 | 闭环反馈 | Trace 覆盖率 |
| **V** Verification | 副作用验证、评估 | 冗余 + 投票 | Feature pass 准确率 |
| **G** Governance | 权限、审计、安全 | 隔离 + 约束 | 越权拦截率 |

**关键洞察**:
- **L 层**最复杂：5 阶段执行流水线 (Admission→Prepare→Execute→Finalize→Persist) + Trace→Skill 进化闭环
- **Trace→Skill 闭环**：系统通过 Trace 分析自动发现可固化为规则的模式，确定性路径覆盖率可从初始 ~30% 提升到 ~60-70%
- **冗余 ROI 最低**（成本高）、**确定性优先 ROI 最高**（永久消除不确定性）

### 1.4 状态基石

- **5 条全局不变量**: INV-S (Session=append-only fact log)、INV-C (Context=有损投影)、INV-D (约束层级)、INV-R (路径排序)、INV-P (派生文件可重建)
- **4 域 10 对象**: 执行域(Agent/Task/Step/Artifact)、能力域(Capability/Context)、治理域(Grant/Audit)、记忆域(Episode/Observation)
- **6 维世界模型**: 环境状态、任务状态、认知状态（含不确定度）、用户状态、时间状态、社会状态
- **认知不确定度显式表示**是当前 Agent 系统最大的缺失

### 1.5 认知演进四阶段

| 阶段 | 时间 | 核心问题 |
|:-----|:-----|:---------|
| Prompt Engineering | 2022-2024 | 给模型什么输入？ |
| Context Engineering | 2025 | 模型每步看到什么？ |
| Harness Engineering | 2025-2026 | 整个执行环境如何工程化？ |
| **Agent OS** | 2026- | **如何构建概率性执行者的操作系统？** |

**关键转折点**: "模型能力不再是瓶颈 → Harness/OS 成为约束瓶颈"

### 1.6 分布式系统对标

| 可直接复用 | 需要重新发明 |
|:-----------|:-------------|
| Event Sourcing (Session=fact log) | 确定性 Replay (模型概率性→Fact Log+投影) |
| Idempotency Key | 自动 Gossip (单向 Context Window→主动 Retrieval) |
| Trace ID propagation | 固定超时重试 (语义错误不因重试消失→反馈+换策略) |
| Circuit Breaker/Sidecar Pattern | 静态配置 (假设会腐化→Feature Gate+Adaptive) |
| Control/Data Plane 分离 | |

### 1.7 工程实践

- **15 条工程原则**（5 范式驱动 + 15 架构原则）
- **7 条操作化阈值**（触发条件 + 默认动作 + 例外/升级）
- **6 维设计自检清单**（范式应用/概率性执行/观测窗口/腐化/不可逆/世界模型）

### 1.8 终极公式

> **Agent OS Engineering = 五种驯服不确定性范式的组合应用**
> 在"执行者概率性 + 观测有限 + 假设腐化"约束下的特化实现

### 2.1 🏆 ChatGPT 月活突破 10 亿，创历史纪录

- **来源**: [The Verge 报道](https://www.theverge.com/ai-artificial-intelligence) (citing [Reuters/Sensor Tower](https://www.reuters.com/technology/artificial-intelligence/chatgpt-app-hits-1-billion-monthly-active-users-record-time-data-shows-2026-06-03/)) | 2026-06-03
- **关键数据**:
  - ChatGPT 于 2026 年 5 月达成 **10 亿 MAU**（月活跃用户），距发布约 3 年
  - 超越 TikTok、Instagram、Google Maps、YouTube 等，成为史上**最快达到该里程碑的应用**
  - 标志着 AI 助手正式跻身超级应用行列
- **行业影响**: 扩大用户基数的同时，推理成本优化和基础设施扩展成为新挑战


### 2.2 🚀 OpenAI Codex 周活 500 万 + 白领市场爆发

- **来源**: [The Verge](https://www.theverge.com/ai-artificial-intelligence) & [TechCrunch](https://techcrunch.com/2026/06/02/openai-launches-new-codex-tools-for-white-collar-work/) | 2026-06-02
- **关键数据**:
  - Codex 周活跃用户 **500 万**，自 2 月桌面版发布增长 **6 倍+**
  - 知识工作者（非开发者）占 **20%**，增速是开发者的 **3 倍以上**
  - 新发布 **6 个垂直岗位插件**：数据分析、销售、产品设计、投资银行、客户支持、内容创作
  - **Sites 功能**上线：将 AI 产出一键发布为托管网站/应用，支持数据持续更新
  - 合作伙伴生态：Wix、Replit、Lovable、Figma
  - 企业/商业客户可获得 Preview 权限，构建"交互式托管网站和应用"
- **战略意义**: AI Coding 从"开发者工具"向"知识工作者生产力平台"演进


### 2.3 💸 GitHub Copilot 改用量计费，开发者强烈反弹

- **来源**: [Ars Technica](https://arstechnica.com/ai/2026/06/01/ai-costs-how-much-github-copilot-users-react-to-new-usage-based-pricing-system/) | 2026-06-01
  - 另见 TechCrunch: [GitHub Copilot’s new token-based billing spurs consternation](https://techcrunch.com/2026/06/02/github-copilot-token-billing-backlash/)
- **定价变革**: 从请求数计费转为 **信用点制**（1 credit = $0.01），2026 年 6 月 1 日生效
- **关键数据**:
  - 重度用户实测：**一个复杂 prompt 烧 171 credits**（$1.71）
  - 有团队反馈 **一天用完 8,000 credits**（全月配额），月成本从约 $19 飙至数千美元
  - 用户普遍反映新计费"透明但不合理"
- **替代方案**: 部分开发者转向 DeepSeek 自建方案（**7 美分/1500 万 tokens** = 同等能力成本差距 100 倍+）
- **行业转折**: AI Coding 从"补贴获客"走向"商业化收割"，token 效率成为核心竞争维度


### 2.4 🏗️ Microsoft 双线 Agent 布局：Scout + Project Solara

#### 🅰️ Microsoft Scout — OpenClaw 驱动的个人 AI 助手
- **来源**: [The Verge](https://www.theverge.com/2026/6/2/...microsoft-scout-openclaw-personal-assistant) & [TechCrunch](https://techcrunch.com/2026/06/02/microsoft-launches-scout-an-openclaw-inspired-personal-assistant/) | 2026-06-02
- **产品定位**: 基于 OpenClaw 框架的个人 AI 助手，**持续学习用户行为模式**
- **核心能力**: 记忆用户偏好、自动化日常任务、内嵌 Microsoft 365 生态
- **安全措施**: Microsoft 同步推出 **Execution Containers**，策略驱动执行沙箱，防止 Agent 删除文件等越权操作。OpenClaw 创始人 Peter Steinberger 表示："现在可以在企业内部安全运行 OpenClaw 了"

#### 🅱️ Project Solara — 专为 AI Agent 打造的 Android OS
- **来源**: [The Verge](https://www.theverge.com/2026/6/2/microsoft-project-solara-ai-agent-android-os) & [Ars Technica](https://arstechnica.com/ai/2026/06/02/microsofts-project-solara-is-an-android-os-designed-for-agents-instead-of-apps/) | 2026-06-02
- **核心定位**: **Agent 优先（而非 App 优先）** 的 Android 变体操作系统
- **设计哲学**: Microsoft 认为在移动端错失 App 生态先机，Agent 时代是"重新开局"的机会 — Solara 让 AI Agent 成为操作系统的一等公民，而非套壳 App
- **关键意义**: 从 OS 层面重构人机交互方式，可能成为 Agent 硬件的原生操作系统层

### 2.7 📊 趋势判断

> **三大趋势并行**：
> 1. **AI Coding 商业化拐点** — Copilot 计价改革效应持续发酵，token 效率成核心竞争维度；Codex 白领插件验证了"非开发者 AI 使用"是比"开发者 AI 辅助"更大的市场
> 2. **Agent OS 从概念走向产品** — Microsoft 同时押注 Scout（个人助手）和 Solara（操作系统层），结合微信/WhatsApp 的 Agent 化尝试暗示 Agent 成为"超级应用"的下一个形态
> 3. **基础设施投入加速** — Alphabet $850 亿、Anthropic IPO $965 亿估值、Lovable × Google Cloud 5x 增长，资本仍在加速涌入而非退潮
