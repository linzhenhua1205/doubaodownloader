## 2️⃣ MiniPIC — vLLM 中 <100LOC 实现 Position-Independent Caching

- **来源**: arXiv:2606.13126, June 11, 2026
- **作者**: Nathan Ordonez, Thomas Parnell (IBM Research)
- **代码**: 已开源

### 核心贡献

**痛点**：vLLM 的 prefix caching 仅能复用**完全匹配前缀**的 KV 条目，对 RAG/Agentic 工作负载中频繁出现的非前缀重复内容（文档段、代码文件等）无效。

**MiniPIC 解法**：仅需 **<100 行**核心引擎修改 + 自定义 attention backend，实现三大原语：
1. **Block-aligned padding** — 块对齐填充
2. **Span Separator (SSep)** — 段分隔符，修改 hash 行为
3. **Prompt Depend (PDep)** — 提示依赖，修改块级因果注意力结构

### 📊 性能数据

| 指标 | 数值 |
|:----|:-----|
| 预填充吞吐提升（2WikiMultihopQA） | **+49%** over baseline vLLM |
| 缓存段 TTFT 降低 | **最多 2 个数量级** |
| 未缓存段线性扩展 | ✅ 保持 |
| 最坏情况开销 | 仅 **5.7%** |
| 引擎改动量 | **<100 LOC** |

**意义**：在 100LOC 以内为 vLLM 添加了完整的 PIC 能力，支持 Block-Attention、EPIC、Prompt Cache 等多种方法在同一 vLLM 实例中共存，且原生集成 CPU offload。

## 3️⃣ ReSET — NVFP4 推理的 Step-Aware Temperature Scaling

- **来源**: arXiv:2606.13233, June 11, 2026
- **作者**: Sihwa Lee et al. (AIHA Lab)
- **代码**: [github.com/aiha-lab/ReSET](https://github.com/aiha-lab/ReSET)

### 核心贡献

**痛点**：NVFP4 推理在大推理模型（LRM）上会因量化导致**推理准确率下降**，且现有 NVFP4 kernel 在 small-batch 自回归解码中延迟提升有限。

**ReSET 解法**：两步走——
1. **ReSET 温度缩放**：基于推理步骤熵的在线温度自适应，利用 token-level + step-level 熵信号修正解码温度
2. **小 M CUDA Core Kernel**：为延迟关键的 small-batch 自回归解码设计专用 kernel

### 📊 性能数据

| 指标 | 数值 |
|:----|:-----|
| NVFP4 推理准确率恢复 | **约 +2pp** over NVFP4 baseline |
| 小 M 核 vs NVFP4 vLLM | **~2.5× kernel-level** speedup |
| 端到端解码 vs BF16 | **~2× end-to-end** speedup |
| CUDA Core kernel 适用场景 | 延迟关键的 small-batch 自回归解码 |

**意义**：解决了 NVFP4 在推理模型上"量化降准确率"的核心矛盾——通过软件级别的温度自适应恢复准确率损失，同时专用 CUDA Core kernel 充分发挥 NVFP4 的吞吐优势。
