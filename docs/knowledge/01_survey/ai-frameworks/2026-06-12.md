## 1️⃣ MPK (Mirage Persistent Kernel): 首次实现多GPU推理自动 Mega-Kernel 化编译器 🚀

- **来源**: [arXiv:2512.22219](https://arxiv.org/abs/2512.22219) v2 (June 9, 2026 更新)
- **作者**: Xinhao Cheng, Zhihao Zhang, Yu Zhou 等 (CMU/Tianqi Chen/Zhihao Jia 团队 — Mirage Project)
- **摘要**: 首个自动将**多GPU模型推理**转换为单个高性能 mega-kernel 的编译器+运行时系统

### 核心创新
1. **SM级图表示** (SM-level graph representation) — 在单个流多处理器粒度上捕获数据依赖，实现跨算子软件流水线
2. **MPK 编译器** — 将张量程序降级为优化后的 SM 级任务图，并为每个任务生成 CUDA 实现
3. **MPK 运行时** — 使用**去中心化 SM 间调度**在单个持久 mega-kernel 内执行所有任务

### 性能数据
| 指标 | 提升 | 对比对象 |
|:-----|:-----|:---------|
| 端到端推理延迟 | **1.7× 降低** 🔥 | 传统 Kernel-per-operator LLM 服务系统 |
| 额外开发工作 | **极小** | 自动完成，无需手写 |

### 关键意义
- 首次将 **mega-kernel 从单 GPU 扩展到多 GPU** 场景
- 突破传统 kernel-per-operator 执行模型的根本性局限
- MPK 源代码公开: https://github.com/mirage-project/mirage

### 影响判断
从 MPC (Mirage Project 之前的单 GPU mega-kernel 编译器) 到 MPK (多 GPU)，这是 MegaKernel 范式的关键跃迁。**1.7× 端到端延迟降低**且几乎无需开发投入，如果与 vLLM/SGLang 集成将极大改变推理框架的性能竞争格局。

## 2️⃣ FastKernels: Snowflake 揭示 LLM Kernel Agent 在生产中的真实差距 ⚠️

- **来源**: [arXiv:2605.23215](https://arxiv.org/abs/2605.23215) (May 22, 2026)
- **作者**: Gabriele Oliaro, Yichao Fu, May Jiang 等 (Snowflake AI Research)
- **摘要**: 首个面向**生产级推理框架**的 Kernel Benchmark，揭示现有评测与真实部署间的系统性偏差

### 核心发现
1. **FastKernels 覆盖范围**: 46 个代表性架构 × 8 类，其内核子集覆盖 HuggingFace Transformers 的 **96.2% (409/425)** 架构
2. **作为推理框架的性能**: 与 vLLM 和 SGLang 在主流 LLM 服务上**完全持平**
3. **震撼结论 — Kernel Agent 生产差距**:
   - 最强 kernel agent (KForge 等) 在 FastKernels 上仅达到 **0.94×** 聚合加速 vs 生产基线 😱
   - 弱一些的 agent 仅 **0.78×** 和 **0.53×**
4. 现有基准的问题：单 GPU + 合成输入、忽略编译栈上下文、奖励复制已知优化而非发现新优化

### 性能数据
| Kernel Agent 等级 | 聚合加速 vs 生产基线 (FastKernels) |
|:------------------|:-----------------------------------|
| 最强 Agent | **0.94×** (反而更慢) |
| 中等 Agent | **0.78×** |
| 弱 Agent | **0.53×** |

### 影响判断
这是一篇非常有价值的**自省性质论文**。它证明：当前 kernel agent 的技术虽然在 KernelBench 等评测上**看起来**取得了 2-5× 的加速，但在生产框架中**实际不可用**。FastKernels 的目标是消除这个 gap，让 benchmark 优势直接转化为生产吞吐提升。代码已开源: https://github.com/Snowflake-AI-Research/fastkernels

## 3️⃣ DITRON: 分布式多层级 Tiling 编译器 — 已企业级部署 🏭

- **来源**: [arXiv:2605.02953](https://arxiv.org/abs/2605.02953) (May 2, 2026)
- **作者**: Size Zheng 等 (北京大学/阿里巴巴/微软)
- **摘要**: 首个可扩展的 tile 级分布式编译器，弥合高性能 CUDA 库与编译器灵活性之间的鸿沟

### 核心创新
1. **三层编程抽象** (Core/Device/Task) — 将张量程序高效映射到异构分布式硬件
2. 支持多样并行策略，同时隐藏节点内和节点间通信复杂度
3. 已在企业级部署用于**训练和推理**

### 性能数据
| 场景 | 加速比 | 详细 |
|:-----|:-------|:-----|
| 独立 Kernel | **6-30%** | vs 专家调优 CUDA 库 |
| vLLM 端到端推理 | **5-30%** 🔥 | 多模型多场景 |
| 训练 MFU | **+10%** | 节省约 **50万 GPU 小时/月** |
| 推理端到端总增益 | **>20%** | 已应用于云服务和边缘推理场景 |

### 跨平台可移植性
- **NVIDIA 平台**: 显著加速
- **AMD 平台**: 同样实现显著加速，证明可移植性

### 影响判断
这是目前看到的**第一个同时在训练和推理两端企业级部署的 tile 级编译器**。在 vLLM 上的 5-30% 端到端加速在工业应用中意义重大。训练侧每月节省 50 万 GPU 小时是非常可观的 Cots 节省。DITRON 证明了 high-level tensor compiler 可以在分布式场景下追平甚至超越手写 CUDA。
