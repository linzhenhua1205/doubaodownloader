## 1️⃣ Fused INT8 Triton GEMM：揭示现有 DiT "INT8" 推理的虚假加速真相 🔴

- **来源**: [arXiv:2606.14598](https://arxiv.org/abs/2606.14598) (June 12, 2026)
- **作者**: Ali Asaria, Tony Salomone, Deep Gandhi (Ideogram 4.0)
- **摘要**: 发现现有 DiT "INT8" 推理实际上**先量化再反量化回 bf16**，从未真正使用 GPU INT8 Tensor Core。通过单个 fused Triton INT8 GEMM kernel，在 **RTX 3090 上实现 2.8-4.2× per-GEMM 加速** 🔥

### 核心发现 — "假 INT8" 问题
- 现有 W8A8 量化实现：weights/activations 经 INT8 量化后**立即反量化到 bf16**，用 bf16 GEMM 执行
- 这意味着 Ampere GPU 上的 INT8 Tensor Core **根本没被利用** 🤯
- 结果是：INT8 反而比 FP8/NF4 更慢（纯软件开销，无硬件收益）

### 创新方案
- **单个 fused Triton kernel**: int8×int8→int32，在 Ampere Tensor Core 上执行
- Per-token × per-channel 反量化 + bias 折叠到 epilogue
- 每个 GEMM shape 自动调优
- bit-exact 对 torch._int_mm，cosine similarity 1.0，完全无 NaN

### 性能数据

| 指标 | 性能 | 对比基线 |
|:-----|:-----|:---------|
| Per-GEMM 加速 | **2.8-4.2×** | vs bf16 GEMM |
| 768px 端到端 | **~1.1× (9-10%)** | vs bf16 |
| 1024px 端到端 | **156.5s** (单 RTX 3090) | NF4: 164.5s / FP8: 172.9s |
| 量化质量 | **无感知损失** | PickScore/CLIPScore 无退化 |

### 关键意义
- INT8 从"最慢变体"变成**最快变体**
- **1024px 推理在单张消费级 GPU 上首次可行**（以前需要双卡或数据中心级 GPU）
- 诚实局限：该 kernel 优势仅在消费级 Ampere 上成立，A100/B200 的 bf16/FP8 原生路径更快

## 2️⃣ Express: 新注意力近似算法 — Triton 实现超越 FlashAttention 2 ⭐

- **来源**: [arXiv:2606.10944](https://arxiv.org/abs/2606.10944) (June 9, 2026)
- **作者**: Albert Gong, Annabelle Michael Carrell, Raaz Dwivedi, Lester Mackey (Microsoft Research)
- **摘要**: 将非因果注意力近似转换为因果注意力的通用工具，配备高效 I/O-aware Triton 实现，**显著超越 FlashAttention 2**

### 核心创新
1. **Express 转换器**: 将 Thinformer 等非因果近似 → 因果近似，保持一致的近似保证
2. **理论保证**: $\log^{3/2}(n)/s$ 近似误差，只需 $O(s)$ 内存和 $O(s^2 \log^2(n))$ 压缩开销（n=序列长度）
3. **I/O-aware Triton 实现**: 优于 FlashAttention 2 的亲测加速

### 解决的四大推理瓶颈
| 瓶颈 | 描述 |
|:-----|:------|
| 长上下文 Prefill | 长序列预填充阶段注意力计算 |
| KV Cache 压缩 | 减少 KV cache 内存占用 |
| 长序列内存受限解码 | 受 GPU 内存限制的长序列生成 |
| 长序列计算受限解码 | 受 GPU 算力限制的长序列生成 |

### 影响判断
Express 提供了**理论上有保证 + 实践上高效**的注意力加速方案。结合 Triton kernel 实现比 FlashAttention 2 更快，且同时覆盖 4 个推理关键瓶颈，有潜力成为长上下文 LLM 推理的重要基础设施组件。

## 3️⃣ NVIDIA 团队：多 GPU 高斯泼溅的 PyTorch 原生抽象 — 25× 超越 SOTA 🏭

- **来源**: [arXiv:2606.11390](https://arxiv.org/abs/2606.11390) (June 9, 2026, NVIDIA)
- **作者**: Matthew Cong, Francis Williams, Jonathan Swartz, **Mark Harris**, Sanja Fidler, Ken Museth (NVIDIA)
- **摘要**: 通过 CUDA Unified Memory + NVLink，将多 GPU 暴露为**聚合 PyTorch Device**，模型代码无需任何跨设备通信逻辑

### 核心创新
1. **Operator-level 分布**: Gaussian 参数和渲染算子通过 CUDA Unified Memory + NVLink 自动跨 GPU 分布
2. **零代码修改**: 模型代码无需任何跨设备通信显式调用 — distribution 发生在 operator 层
3. **通用 PyTorch backend**: 支持任意 PyTorch operator，不仅仅是 Gaussian splatting

### 性能数据
| 指标 | 数值 | 对比 |
|:-----|:-----|:-----|
| 最大场景规模 | **>10 亿 Gaussian** | **25× 超越此前 SOTA** |
| 典型场景 | 城市级重建 | 街景细节完整保留 |

### 影响判断
这代表了一种**极简多 GPU 编程范式**：CUDA Unified Memory + NVLink 让多 GPU 对用户透明。虽然目前聚焦于 3D 渲染/重建（Gaussian Splatting），但 "multi-GPU as aggregate device" 的 Pattern 对未来推理框架的设计有参考价值。NVIDIA 核心团队的背书（Mark Harris, Sanja Fidler）也提升了此工作的信噪比。
