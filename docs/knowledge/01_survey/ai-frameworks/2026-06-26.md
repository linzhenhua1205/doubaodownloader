## 三、KV Cache 优化（最新论文）

### 3.1 HyperQuant — Rate-Distortion-Optimal 统一量化管线（arXiv:2606.23406, Jun 22）

**核心思路**: 将 Hadamard 变换 + 低维最优格点量化（E8/D4/A2/Z）+ Rice 编码 + 偏差校正融合为统一管线，同时覆盖权重和 KV cache。

| 指标 | 数值 |
|:-----|:-----|
| KV cache 压缩比 | **3.79×** (near-lossless) |
| 权重压缩比 | **3.9×** (4 bps linear weights) |
| 支持精度 | 3–5 bps 权重 · 1.7–4 bps KV |
| 硬件路径 | FP8/INT8/NVFP4/MXFP4 Tensor Core |
| 对比竞品 | 各操作点 **全面超越** HIGGS ≥ TurboQuant ≥ OCTOPUS |
| 扩散模型 | LTX-2 19B DiT 量化 **无可见伪影** |

**关键洞察**: int8 在 RHT (Randomized Hadamard Transform) 后的格点输出上**优于** fp8 — 即 Hadamard 变换使数据分布更适合整数而非浮点表示。

### 3.2 OScaR — INT2 KV Cache Token-Norm Imbalance 解决（arXiv:2605.19660, May 19）

> **2026-06-26 更新**: 社区实现已发布，作为当前 INT2 KV 量化的 Pareto 前沿参考。

| 指标 | 数值 |
|:-----|:-----|
| KV cache 压缩 | INT2 (vs BF16, **5.3×** 内存缩减) |
| Decode 加速 | **3.0×** (vs FlashDecoding-v2 BF16) |
| Throughput 提升 | **4.1×** (大 batch) |
| 核心机制 | Canalized Rotation + Omni-Token Scaling 消除 TNI |
| 对比 | 一致超越 TurboQuant，定义 Pareto Frontier |

> **意义**: TNI (Token Norm Imbalance) 机制性分析证明 per-channel 量化的根本限制不是 outlier channel，而是 token-group 间 norm 差异。

### 3.3 TriAxialKV — Agentic 任务感知混合精度 KV（arXiv:2605.17170, May 16）

| 指标 | 数值 |
|:-----|:-----|
| KV cache 缩减 | **4.5×** (vs BF16) |
| Throughput 提升 | **+30%** E2E |
| 任务场景 | OSWorld computer-use agent (Qwen3-VL-32B) |
| 精度 | 匹配 SGLang BF16 基线 |
| 三轴标签 | 时间近因 · 模态（text/image）· 语义角色（query/tool/obs/reason） |

### 3.4 STAR-KV — 低秩 + 量化联合压缩（arXiv:2606.08382, Jun 6）

| 指标 | 数值 |
|:-----|:-----|
| KV 低秩压缩 | **75%** (单独) |
| 联合量化总压缩 | 最高 **20×** |
| Attention 加速 | **6.9×** (Triton 自定义 kernel) |
| E2E 吞吐 | **3.1×** |
| 核心机制 | 可微分阈值 rank 选择 + 头级/块级粒度 + 低秩感知混合精度 |

### 3.5 SpectrumKV — PD 分离场景每 token 混合精度传输（arXiv:2606.08635, Jun 7）

| 指标 | 数值 |
|:-----|:-----|

## 四、编译器优化

### 4.1 Triton 3.7.1（6月18日）

**两个回归修复，均涉及正确性**:

1. **FenceAsync 缺少依赖** (#9610): `st.shared`（共享内存写）与 `async_copy_local_to_global`（异步拷贝）之间缺少 fence，导致异步拷贝可能在共享内存写入完成前读取旧值。FenceAsync 现在插入 required fence。

2. **LLVM InstCombine miscompilation** (llvm/llvm-project#174380): `add` 简化操作仅使用 RHS 的 known-zero bits，但 LHS 有已知零位且低位未使用时忽略对称情况。该错误通过 Triton 的 pinned LLVM 被继承。

> **影响**: 这两个 bug 直接导致了 PyTorch 2.12.1 中 B200 Flash Attention 非确定性问题（已通过升级至 Triton 3.7.1 修复）。

### 4.2 ROCm 7.2.3 支持（vLLM v0.23.0）(#43136)
- AITER v0.1.13.post1 (#44265)
- Native W4A16 HIP kernels for RDNA3 (gfx1100) (#41394, #44075)
- RISC-V RVV WNA16 helpers (#42730)
- Intel XPU: block-scaled W8A8 FP8 path (#39968), DeepSeek-V4 attention decode (#42953)
