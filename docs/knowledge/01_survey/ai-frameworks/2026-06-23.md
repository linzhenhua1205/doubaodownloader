## 一、推理框架

### 🔷 SGLang v0.5.13（6月13日发布，本周最重磅更新）

**概况**: 重大版本发布，DeepSeek V4 全面强化，Spec V2 正式成为默认路径。

#### 核心更新

1. **Spec V2 成为默认推测解码路径**
   - Tree drafting with topk > 1 在全后端（triton/FA3/MLA/aiter）生产就绪，page_size > 1 和 Mamba/hybrid-linear 模型均支持
   - Spec V1 被弃用，EAGLE/MTP 迁移到统一的 V2 worker
   - topk=1 场景提速（skip full-vocab softmax + 冗余 cat/topk/sort/gather 操作）

2. **DeepSeek V4 全面强化**
   - Context Parallel + MTP（#24934）
   - Sparse FlashMLA 通过 flash_mla_sparse_fwd 集成（#25418）
   - FP4 indexer 支持（#26209）
   - Breakable CUDA Graph 支持（#25195）
   - DeepEP waterfill load balancing（#25391）
   - MHC 内核 warmup：启动时预热 token-count buckets，消除 **20-40 秒冷桶前向阻塞**

3. **MoRI on AMD MI355X — 成本竞争力标杆**
   - DeepSeek-R1 在 AMD MI355X 上通过 MoRI 通信库进行分解推理
   - **$0.169/百万 token @ 129 tok/s/user** ← 极具竞争力的推理成本

4. **性能亮点**
   - 低 scheduler 开销：FutureMap + prefill input transfer 到 forward stream，减少 per-step launch 开销
   - Piecewise & Breakable CUDA Graph 扩展至 DSA 模型、Kimi-K2.5、DeepSeek V4
   - Qwen 3.5 on Blackwell：新 FlashInfer GDN 内核 + CuTeDSL GDN prefill kernel
   - HiCache 默认启用 hybrid model KV offloading（SWA/Mamba）
   - **异质 CPU + GPU EPD 分解**（Intel）：VLM vision encoding offload 到 Xeon CPU，**~1.3x P99 TTFT 和吞吐提升**

5. **新模型支持**: Nemotron 3 Ultra（Day-0）、Step-3.7-Flash、Command A+、Cosmos3、Ernie-Image、FLUX.2-Klein、Ideogram 4

## 三、KV Cache 优化（最新 arXiv 论文）

### 1️⃣ SAC: CXL 分解式 KV Cache 系统（arXiv:2606.19746）

**核心思路**: 利用 CXL cache-line 粒度按需取数，替代 RDMA 全量预取，专门优化稀疏注意力模型（DeepSeek-V3.2）。

**性能数据**（vs RDMA baseline on SGLang）:
- **吞吐 2.1x** 🏆
- **TTFT 降低 9.7x** 🏆
- **TBT 降低 1.8x**

**结论**: CXL-based disaggregation 是稀疏注意力模型的最优基础设施方案。

### 2️⃣ UltraQuant: 4-bit KV Cache for Agentic 负载（arXiv:2606.20474）

**核心思路**: 面向多轮 Agent 场景的 4-bit KV cache，TurboQuant+codebook + AMD CDNA4 原生 FP4 path。

**性能数据**（vs FP8 KV baseline on AMD GPU）:
- **P50 TTFT 降低 3.47x**（cache-pressured late rounds）
- **输出吞吐提升 1.63x**
- 所有 rounds 平均 TTFT 降低 2.3x

### 3️⃣ Albireo: 超越 Amdahl 定律的 LLM 推理系统（arXiv:2606.01927）

**核心思路**: 找到最优 TP 度 t_e，通过 overlap scheduling、I/O-compute overlap、sequence-parallel sampling 消除不可扩展开销。

**性能数据**（vs vLLM）:
- **吞吐最高 1.9x** 🏆
- **延迟降低 48%**
- **GPU 利用率提升 28%**
- **能耗降低 54%**
- 生产环境吞吐最高 2x

### 4️⃣ STAR-KV: 自适应低秩 KV Cache 压缩（arXiv:2606.08382）

**核心思路**: 可微分阈值机制 + 混合分解策略（key/value 不同分解）+ 低秩感知混合精度量化。

**性能数据**:
- **KV cache 压缩最高 75%**
- **结合量化最高 20x 总体 KV cache 缩减**
- Attention 模块 **6.9x 加速**
- **端到端生成吞吐 3.1x**

### 5️⃣ HyperQuant: 率失真最优量化 Pipeline（arXiv:2606.23406）

**核心思路**: 随机 Hadamard 变换 + 最优格量化（E8/D4/A2/Z）+ Rice 编码 + 偏差校正，统一量化 weights 和 KV cache。

**性能数据**:
- 4 bps 时，linear weights **~3.9x 压缩**, KV cache **~3.79x 压缩**（近无损）
- 1.7 bps 时仍优于 TurboQuant 和 OCTOPUS
- LTX-2 19B DiT 视频模型无可见帧伪影

### 6️⃣ PolyKV: 异质逐层 KV Cache 策略（arXiv:2606.15157）

**核心思路**: 不同 Transformer layer 使用不同的 eviction 策略和缓存预算。

**性能数据**: 512-token 平均 KV 预算下，对 FullKV 差距恢复 **40.0%-54.5%**；128-1024 预算 sweep 中平均提升 1.7%-6.4%。
