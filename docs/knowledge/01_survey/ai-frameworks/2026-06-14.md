## 1️⃣ GF-DiT — Diffusion Transformer 弹性并行调度框架（vLLM-Omni）

- **来源**: arXiv:2606.13501, June 11, 2026
- **作者**: Xinwei Qiang, Yifan Hu, Shixuan Sun et al. (上海交大)
- **关键词**: vLLM-Omni、Diffusion Transformer Serving、弹性并行

### 核心贡献

**痛点**：现有 DiT 推理系统对请求使用**固定并行配置**，但 DiT 工作负载在请求之间、执行阶段和系统条件上存在高度异质性，导致 GPU 利用率低下。

**GF-DiT 解法**：将 GPU 并行度作为**一级可调度资源**，运行时动态调整正在运行的请求的并行度：
1. **异步执行抽象**：将请求分解为独立可调度的轨迹任务，支持在线 GPU 重分配
2. **Group-Free Collectives**：轻量通信抽象，支持任意执行组的低开销在线形成和重配置

### 📊 性能数据（在 vLLM-Omni 上实现）

| 指标 | 数值 |
|:----|:-----|
| 吞吐提升 | **最高 6.01×** over 静态并行 |
| 平均延迟降低 | **最高 95%** |
| SLO 违反率降低 | **最高 90%** |
| 通信组建立开销 | 778ms → **≈60μs**（4 个数量级降低） |

**意义**：首个将 DiT 推理弹性并行化的系统级方案，通信组建立开销降低 10,000+ 倍使在线重配置成为实用选择。

## 2️⃣ M* (M-Star) — 多模态模型的通用模块化推理系统

- **来源**: arXiv:2606.12688, June 10, 2026
- **作者**: Atindra Jha, Naomi Sagan, Keisuke Kamahori et al. (Stanford/CMU/Meta)
- **关键词**: 多模态推理、Composite AI Models、数据流图

### 核心贡献

**痛点**：现有推理框架（vLLM、Triton 等）建立在狭窄的模型结构假设上——假设单一语言模型或简单编码器-解码器结构。面对**复合模型架构**（视觉编码器 + 语言骨干 + 扩散头 + 音频编解码器 + 动作生成器 + 世界模型）时力不从心。

**M\* 解法**：将模型表示为**数据流图（Walk Graph）**，请求处理作为图遍历：
- 模块化抽象支持任意模型组件组合
- 灵活的物理集群放置
- 模型无关的分布式运行时优化

### 📊 性能数据

| 指标 | 对比对象 | 数值 |
|:----|:---------|:-----|
| 端到端延迟（text-to-image） | vLLM-Omni (BAGEL) | **↓20%** avg |
| 实时因子（text-to-speech） | vLLM-Omni (Qwen3-Omni) | **低至 2.9×** |
| 吞吐（text-to-speech） | vLLM-Omni (Qwen3-Omni) | **高达 2.7×** |
| 机器人规划加速 | V-JEPA 2-AC 基线 | **最高 12.5×** |

**意义**：随着 GPT-5/Fable 等模型走向多模态融合，M\* 提供了一种通用化的多模态推理基础设施新范式。

## 3️⃣ STAR-KV — 自适应低秩 KV Cache 压缩（Triton 实现）

- **来源**: arXiv:2606.08382, June 6, 2026
- **作者**: Priyansh Bhatnagar, Ashkan Moradifirouzabadi et al.
- **关键词**: KV Cache 压缩、低秩分解、Triton Kernel、自适应
- **代码**: github.com/PriyanshBhatnagar/STAR-KV

### 核心贡献

**痛点**：低秩投影（Low-rank projection）是 KV cache 压缩的前沿方向，但现有方法依赖固定/启发式秩选择，无法实现激进压缩下的低精度退化。

**STAR-KV 四层设计**：
1. **可微分阈值机制** → 在 attention-head 和 block 级别自动选择最优秩
2. **混合分解策略** → 按 K/V 投影的敏感性采用不同的低秩分解
3. **低秩感知混合精度量化** → 利用数据统计实现近乎无损的低位量化
4. **自定义 Triton GPU 内核** → 实现计算加速

### 📊 性能数据

| 指标 | 数值 |
|:----|:-----|
| KV Cache 压缩率 | **最高 75%**（单独低秩） |
| 结合量化后的总体压缩 | **最高 20×** |
| Attention 模块加速 | **最高 6.9×**（Triton 内核） |
| 端到端生成吞吐 | **最高 3.1×** |
| 精度损失 | ✅ 近乎无损（低秩感知量化） |

**意义**：STAR-KV 通过自适应秩控制和 Triton 内核，首次将低秩 KV cache 压缩推到 **75% + 量化 = 20× 总体压缩** 的激进水平，且精度损失可忽略，是目前最强的 KV cache 低秩压缩方案。
