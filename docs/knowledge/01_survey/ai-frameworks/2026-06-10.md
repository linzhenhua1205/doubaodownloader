## 1️⃣ RKSC: 推理感知 KV Cache 共享 + 提前退出 — 1.66× 超 vLLM 前缀缓存 ⭐

- **来源**: arXiv:2606.09937 (cs.LG, June 7 提交)
- **论文**: 《RKSC: Reasoning-Aware KV Cache Sharing and Confident Early Exit for Multi-Step LLM Inference》
- **录用**: ICML 2026 Workshop on Statistical Frameworks for Uncertainty in Agentic Systems
- **核心创新**: 3 个无训练组件，解决 multi-branch 推理中的结构冗余：
  1. **ASKS (Attention-Similarity KV Sharing)**: 计算一次前缀 KV cache，通过 hidden-state cosine similarity 广播给所有语义相似分支——**严格推广**了 vLLM/SGLang 的 token-exact 前缀缓存
  2. **CGEE (Confidence-Gated Early Exit)**: 两层提前退出机制——分支间置信度足够高时跳过验证 forward pass；逐层熵稳定时提前终止验证
  3. **RSBCM (Reasoning-Selective Block Cache Manager)**: 通过 attention-weighted depth-priority eviction 防止无界缓存增长
- **性能数据** (5 模型家族 7B-10B, 4 基准, 1,000 问题):
  - **3.008× 平均加速** vs No-KV baseline（峰值 **3.990×**）
  - **1.66× 平均改进** vs vLLM 等效前缀缓存 🔥
  - CGEE 误差率仅 **0.37%**（1,616 次 verify 调用中仅 6 个错误）
- **无需微调或架构更改**，代码已开源: https://github.com/AnirudhSekar/RKSC
- **意义**: 首次在推理分支共享缓存中引入语义相似度（而非 token 精确匹配），显著扩大了缓存可共享范围

## 2️⃣ Compiler World Model: 潜空间张量程序搜索 — TVM 上 4.61× 超 PyTorch ⭐

- **来源**: arXiv:2606.09312 (cs.LG/cs.PL, June 8 提交)
- **论文**: 《Toward Compiler World Models: Learning Latent Dynamics for Efficient Tensor Program Search》
- **核心创新**: "世界模型"启发的张量程序评估器——将调度评估建模为程序状态上的**动作条件潜动力学**。不是评估静态代码快照，而是用轻量过渡模型在连续潜空间中 rollout 调度轨迹
- **性能数据** (TVM AutoScheduler 实现):
  - GPU 上: **1.37×** 优于 Ansor (相同 64-trial budget)
  - CPU 上: **1.54×** 优于 Ansor
  - 用 **10× 更少测量** 匹配 Ansor-10K (几何均值差距 <2.2%)
  - **全模型推理加速**: vs PyTorch **4.61×** 🚀 / vs PyTorch-opt(cuDNN) **3.67×**
- **关键洞察**: 传统 cost model 把每个候选当独立静态快照，忽略了调度轨迹中的动作依赖性和代码表层变化——world model 通过潜状态编码解决了这个问题
- **意义**: 将强化学习的 world model 思路引入编译器搜索，大幅降低测量成本

## 3️⃣ KV Cache 量化导致 Alignment Collapse — vLLM FP8 生产环境确认 ⚠️

- **来源**: arXiv:2606.09864 (cs.LG/cs.AI, May 31 提交)
- **论文**: 《Alignment Collapse Under KV Cache Quantization: Diagnosis and Mitigation》
- **核心发现**: 低比特 KV cache 量化会**无声地摧毁安全对齐**——标准评测指标 (perplexity/accuracy) 完全无法检测
- **关键数据**:
  - Mistral-7B: perplexity 仅 **1.03×** 时，拒绝率已降低 **15.2%**
  - **不存在通用安全位宽**——每个模型有尖锐的相变边界，标准指标完全不可见
- **根因**: 安全特征占据低维激活子空间（10²-10³× 更易受量化噪声影响），perplexity 平均在被压缩的完整空间上，掩盖了问题
- **诊断方法 (PCR — Per-Channel Reduction)**: 将模型分为 3 种故障模式:
  1. **outlier-crushes-safety**: 安全在非离群通道，被离群驱动的 scale factor 连带摧毁
  2. **outlier-as-safety**: 安全重叠在离群通道上，更细粒度也无法挽救
  3. **multi-layer dilution**: 安全分布在多层，单层修复无效
- **修复**: 无训练协议，~35 GPU 分钟，**恢复 97% 的对齐损失**（包括 KIVI quantizer）
- **已确认**: 生产 vLLM 中 FP8 KV cache 存在此漏洞 🔴
- **意义**: 首次揭示 KV cache 量化的安全影响——之前所有研究只看 perplexity，这是一个全新的安全攻击面
