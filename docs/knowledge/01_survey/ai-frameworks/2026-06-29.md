## 🧠 KV Cache 优化 — 本期重点：六线并行突破

6 月 arXiv 涌现一批高质量 KV cache 优化论文，覆盖从硬件(CXL)到算法(量化/压缩/调度)全栈：

### 1. SAC — CXL 分解式稀疏注意力 KV Cache

> **arXiv:2606.19746** | SAC: Disaggregated KV Cache System for Sparse Attention LLMs with CXL

| 指标 | vs RDMA 基线 |
|:----|:------------|
| Throughput | **2.1×** |
| TTFT | **9.7× 更低** |
| TBT | **1.8× 更低** |

**核心思路**：利用 CXL 缓存行粒度的 load/store 语义，为稀疏注意力模型按需取 top-k KV 条目，避免 RDMA 粗粒度全量预取的低效。

### 2. AsymCache — Multi-Segment Attention

> **arXiv:2606.02964** | Multi-Segment Attention: Enabling Efficient KV-Cache Management

| 指标 | vs 最新基线 |
|:----|:-----------|
| TTFT 加速 | **1.90-2.03×** |
| TPOT 加速 | **1.62-1.71×** |

**核心思路**：计算-延迟感知的 KV cache 管理，将 GPU 注意力内核性能对齐到缓存驻留决策。Agent serving 集成后延迟再降 **18.1%**。

### 3. CacheTune — 非前缀 KV Cache 复用

> **arXiv:2605.24022** | Adaptive KV Cache Reuse for Fast Long-Context LLM Serving

| 指标 | 性能 |
|:----|:----|
| TTFT 加速 | **3.72-4.86×** |
| Throughput 提升 | **3.93-6.21×** |
| SSD/HDD offload TTFT | **2.34-2.36×** |

**核心思路**：频域分析识别语义关键 KV 对，选择性重计算 + 稀疏 KV 传输 + 多流异步重叠。

### 4. OCTOPUS — 八面体参数化 KV 量化

> **arXiv:2605.21226** | OCTOPUS: Optimized KV Cache via Octahedral Parametrization

- 联合量化旋转坐标三元组，每个三元组方向映射到方形的八面体参数化
- 比所有此前旋转编解码器更优，差距在极低比特率下更大
- **Fused Triton 实现，decode 零额外延迟**

### 5. Still — 单轮前向摊还 KV 压缩

> **arXiv:2606.07878** | Still: Amortized KV Cache Compaction in a Single Forward Pass

| 指标 | 性能 |
|:----|:----|
| 压缩比 | **8× 到 200×** |
| RULER 基准 | **超最强基线 8-22 分** |
| 上下文长度 | 8k 到 128k |

- 小型 per-layer Perceiver，单次前向产生紧凑 K/V
- 可迭代应用实现长期缓存，超出 per-context 方法能力范围

### 6. ObjectCache — S3 对象存储 KV Cache

> **arXiv:2605.22850** | ObjectCache: Layerwise Object-Storage Retrieval for KV Cache Reuse

| 配置 | 额外延迟 |
|:----|:--------|
| 64K context | **仅 +5.6%** vs local DRAM |
| 4K context | +56-75 ms |
| 共享带宽限流 | **1.2-1.8×** TTFT 降低 |

- S3 兼容对象存储，co-design 传输协议和调度
- 100 Gbps RoCE + Ceph RGW + DAOS 原型验证

### 7. PolyKV — 异构层间 KV 预算分配

> **arXiv:2606.15157** | PolyKV: Heterogeneous Retention and Allocation for KV Cache Compression

| 指标 | 性能 |
|:----|:----|
| FullKV 差距恢复 | **40.0%-54.5%** |
| vs 最强单策略基线 | **+1.7%-6.4%** |

- 按层信号选择合适的压缩策略，非均匀预算分配

### 8. ITME — CXL-Hybrid 内存 KV 扩展

> **arXiv:2606.12556** | ITME: Inference Tiered Memory Expansion

- TB 级字节可寻址远程内存扩展
- 确定性访问模式使系统可主动管理数据移动
- **Throughput +35.7%**

### 9. Albireo — 突破 Amdahl 定律

> **arXiv:2606.01927** | Scaling LLM Inference Beyond Amdahl's Limits

| 指标 | vs vLLM |
|:----|:--------|
| Throughput | **1.9×** |
| Latency | **-48%** |
| GPU 利用率 | **+28%** |
| 能耗 | **-54%** |

### 10. RTP-LLM — 阿里巴巴工业级推理引擎

> **arXiv:2605.29639** | RTP-LLM: High-Performance Alibaba LLM Inference Engine

| 指标 | vs vLLM/SGLang |
|:----|:--------------|
| 模型加载 | **4.7-6.3×** |
| TTFT P95 | **-35~37%** |
| Cache 复用 | **+215%** |
| Spec decode throughput | **1.12-2.48×** |
| 多模态 throughput | **1.86-2.52×** |
| 量化推理延迟 | **-35-40%**, TTFT **1.9-3.0×** |

- PD 分离 + 多级 KV cache + 自适应量化 + 模块化 spec decode
- 生产环境 1 亿+ 用户, Alibaba 全集团部署

## 📊 底层趋势判断

1. **KV Cache 进入系统级创新爆发期** — CXL 硬件、对象存储、非前缀复用、摊还压缩、统一量化五条路线同时突破，各自对准不同约束条件：显存/带宽/延迟/硬件成本。后续部署策略需根据模型架构(稀疏/稠密)和应用场景(长上下文/实时交互)做分层选择。

2. **Blackwell 生态成熟是 6 月主线** — TRT-LLM TrtllmGenAttention 成为默认 decode 后端、SGLang CuteDSL 内核全面登陆 SM100/SM120、vLLM FlashInfer b12x FP4 GEMM、Triton 3.7 2CTA+Multicast — 所有框架在 Blackwell 上已从"可运行"进入"优化默认"阶段。

3. **DeepSeek-V4 成为推理框架试金石** — vLLM/SGLang/TRT-LLM 全部做了深度适配，验证了稀疏注意力(CSA/HCA)、Mega-MoE、FP4/MXFP4 量化等新技术栈的成熟度。

4. **工业级 vs 学术级推理系统差距缩小** — Alibaba RTP-LLM 开源(arXiv)与 vLLM/SGLang 在指标上直接对标，结束了此前工业系统闭源领先的局面。
