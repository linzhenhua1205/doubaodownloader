## 三、KV Cache 优化论文（2026年6月新发）

### 3.1 HyperQuant: 率失真最优量化管线（arXiv:2606.23406, 6月22日）

**作者**: Yuval Domb, Hadar Sackstein, Tomer Solberg (moonmath.ai)

**核心方法**:
- per-tile Randomized Hadamard Transform (RHT) 使 weight/activation 近似高斯分布
- 低维最优格点量化（E8, D4, A2, Z）
- 无损 bit-stripping + Rice 编码
- KV cache 偏置校正

**性能数据**:
- **KV cache ~3.79x 压缩**（4 bps 时近无损）
- **线性 weight ~3.9x 压缩**（4 bps）
- 在所有 operating point 上 beats **TurboQuant** 和 **OCTOPUS**
- 在 19B LTX-2 DiT 视频模型上量化无可见伪影
- 后端支持：fp8-e4m3 / int8 / nvfp4 / mxfp4
- **int8 beats fp8** on post-RHT lattice output — 反直觉但实验验证

### 3.2 OScaR: Omni-Scaled Canalized Rotation（arXiv:2605.19660, 5月19日）

**核心方法**: 识别 Token Norm Imbalance (TNI) 为 KV cache 量化 fidelity 的主要瓶颈 → Canalized Rotation + Omni-Token Scaling

**性能数据** (vs BF16 FlashDecoding-v2 baseline):
- **解码 3.0x 加速**
- **内存 5.3x 缩减**
- **吞吐 4.1x 提升**
- 在 INT2 量化下接近无损
- 适用于 text-only/multi-modal/omni-modal LLM

### 3.3 ReQAT: 推理模型 FP4 量化感知训练（arXiv:2606.15682, ICML 2026）

**核心发现**: FP4 失败集中在低熵 token（数字、运算符等符号性承诺），量化噪声在推理轨迹中级联放大

**方法**: Trace-Aligned QAT + Selective Entropy Minimization + Q-FIT 初始化

**性能数据**:
- **DGX Spark 上 3.9x 吞吐加速**
- **B200 上 3.1x 吞吐加速**
- 同一训练预算下超越 BF16 fine-tuning 准确率
- 实现 W4A4KV4 全量化推理（weight/activation/KV cache 全 4-bit）

### 3.4 UltraQuant: Agent 负载 4-bit KV Cache（arXiv:2606.20474, 6月18日）

**性能数据** (vs FP8 KV baseline, AMD CDNA4):
- **P50 TTFT 降低 3.47x**（cache 压力大的后几轮）
- **输出吞吐提升 1.63x**
- 使用 UE8M0 group scales + native scaled-MFMA on CDNA4

### 3.5 Models Take Notes at Prefill（arXiv:2606.17107, 6月14日）

**关键洞察**: KV cache 可以被编辑和组合——模型在 prefill 阶段将字段条件结论写入"笔记"；KV 内容可编辑（erratum correction）+ 可组合（skill splicing）

**性能数据**:
- **prefix cache 保持 98.5% hit-rate**（erratum 为 append-only）
- **p90 TTFT 降低 53-398x**
- 与 production vLLM 兼容，已验证跨 scale/quantization/MoE/multimodal

### 3.6 Alignment Collapse Under KV Cache Quantization（arXiv:2606.09864, 5月31日）

**关键发现**: KV cache 量化会静默破坏安全对齐，标准指标（perplexity/accuracy）无法检测

**性能数据**:
- **Mistral-7B 失去 15.2% refusal**，perplexity 仅劣化 1.03x
- 无 universal safe bit-width——每个模型有尖锐的相变点
- **PCR 方法可恢复高达 97% 的安全对齐**，仅需 35 GPU 分钟（20 calibration prompts）
- 在 vLLM 生产环境 FP8 KV cache 上验证

### 3.7 SpectrumKV: PD 分解式 KV cache 传输（arXiv:2606.08635, 6月7日）

**性能数据**:
- **50-62% TTFT 降低**（50% KV 传输预算）
- 三级精度：FP16 (attention sink) + INT8 + INT4（模型容忍时）
- 在 50% budget 下 perplexity 变化 +1.97%/-0.06%/-0.44% vs PDTrim 的 +25.85%/+22.07%/+35.63%

### 3.8 STAR-KV: 自适应低秩 KV Cache 压缩（arXiv:2606.08382, 6月6日）

**性能数据**:
- **75% KV cache 压缩率**
- 与量化结合可达 **20x 整体 KV cache 缩减**
- **Attention 模块 6.9x 加速**，**端到端 3.1x 生成吞吐**

## 五、关键趋势与洞察

### 5.1 KV Cache 量化安全风险浮出水面

**Alignment Collapse (2606.09864)** 揭示了 KV cache 量化的隐蔽安全风险：低比特量化可以静默破坏安全对齐而不被标准指标察觉。这标志着 KV cache 量化的评估标准需要从"perplexity/accuracy"扩展到"安全性"。

推荐做法：在生产部署中加入 PCR 诊断检查（35 GPU 分钟）以验证安全对齐。

### 5.2 FP4 成为 KV Cache 量化新标准

本周多篇论文涉及 FP4 KV cache 量化：
| 论文 | 方法 | 关键结果 |
|:-----|:-----|:---------|
| **ReQAT** | FP4 QAT for reasoning | 3.9x DGX Spark, 3.1x B200 |
| **UltraQuant** | FP4 for agent workloads | 3.47x TTFT reduction |
| **HyperQuant** | Lattice + Rice coding | 3.79x KV compression |

结合 TensorRT-LLM v1.3.0rc19 对 NVFP4 的持续强化，FP4 正在从实验走向生产。

### 5.3 KV Cache 可编辑性新范式

**Models Take Notes (2606.17107)** 提供了一个全新的 KV cache 使用范式：将 KV cache 视为可编辑、可组合的"笔记"。与 production prefix caching 兼容（98.5% hit-rate），p90 TTFT 最多降低 398x。可能改变 RAG 和 Agent 系统的 KV cache 利用策略。

### 5.4 Triton 3.7.1 是 Blackwell 推理的关键依赖

PyTorch 2.12.1 和 Triton 3.7.1 的同步发布（FenceAsync 修复 + InstCombine miscompilation 修复）表明，Triton 在 Blackwell (SM100/SM120+) 上的正确性仍在持续稳定化中。这是所有使用 Blackwell + torch.compile 用户必须关注的升级。
