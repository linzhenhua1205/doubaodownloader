## 3️⃣ DeepSpeed + Muon Optimizer 官方支持 (6月3日) ⭐

- **来源**: [PyTorch 官方博客](https://pytorch.org/blog/using-muon-optimizer-with-deepspeed/) — Zhipeng Wang, Guokai Ma et al. (DeepSpeed 团队)
- **版本**: DeepSpeed ZeRO Stage 2/3 Muon 合并
- **生产采用者**:
  - **DeepSeek-V4** (1.6T 参数) — 用于加速收敛和训练稳定性
  - **Kimi-K2** (1T 参数, Moonshot AI) — 使用 MuonClip 变体
  - **GLM-5** (744B 参数, Zhipu AI) — GLM-4.5 和 GLM-5 预训练均使用，并发明 "Muon Split" 技术
- **性能数据 (Moonlight-16B-A3B 微调)**:

| 指标 | AdamW | Muon | 差异 |
|:-----|:------|:-----|:-----|
| MMLU | 0.660 | **0.678** | **+1.8pp** |
| GSM8K | 0.805 | **0.810** | +0.5pp |
| MBPP+ | 0.534 | **0.548** | +1.4pp |
| 峰值显存 | 34.5 GiB | **31.4 GiB** | **节省 9%** |

- **原理**: Muon 只维护一个动量缓冲 (vs Adam 的两个)，对 2D 隐藏权重做 Newton-Schulz 正交化，等化奇异值，放大稀有但重要的更新方向。NanoGPT speedrunning 训练速度提升 35%。
- **路线图**: Gram-Schmidt Newton-Schulz kernel (review中) → CPU Offloading → MuonClip

## 4️⃣ NVIDIA CompileIQ — AI 进化编译器自动调优 (5月26日) ⭐

- **来源**: [NVIDIA 技术博客](https://developer.nvidia.com/blog/extract-more-kernel-performance-with-nvidia-compileiq-auto-tuning/)
- **版本**: CUDA 13.3 内置 (pip install compileiq)
- **工作原理**:
  - 进化/遗传算法搜索 PTXAS/NVCC 内部编译器参数 (寄存器分配、指令调度、循环展开等)
  - 输出 Advanced Controls File (ACF)，通过 `--apply-controls` 编译器标志应用
  - 支持多目标优化: 运行时 × 编译时间 × 功耗 → 输出 Pareto 前沿
- **性能数据**:
  - **Meta 生产验证**: TritonBench + Helion kernels **最高 15% 提升** (GTC 演讲)
  - Reduction kernel 示例: 0.777ms → 0.770ms (1.01x)
  - 已在领先 AI 实验室生产使用
- **安全性**: 用户 workload 不离开本地环境，ACF 可安全提交到版本控制

## 6️⃣ arXiv 框架论文精选 (6月1-4日) ⭐

### KForge: LLM 驱动的跨平台 Kernel 生成
- **来源**: [arXiv:2606.02963](https://arxiv.org/abs/2606.02963) (ISCA 2026 Workshop MLArchSys 接收)
- **双 Agent 迭代优化**: 生成 Agent + 性能分析 Agent
- **NVIDIA B200**: vs TensorRT-LLM 端到端吞吐 +**2.12%** (gpt-oss-20b)
- **Intel Arc B580**: 5.13× 几何平均加速 (vs PyTorch eager/torch.compile, 37 GEMM workloads)
- 主要通过算子融合和混合精度执行实现

### Albireo: 突破 Amdahl 极限的推理系统
- **来源**: [arXiv:2606.01927](https://arxiv.org/abs/2606.01927)
- 通过重叠调度/I/O 与计算 + 序列并行采样，突破 TP 度的非可扩展部分
- vs vLLM: 吞吐 **1.9×**, 延迟 **-48%**, GPU 利用率 **+28%**, 能耗 **-54%**
- 生产环境中可达 2× 吞吐

### KVarN: 2-bit KV Cache 量化 SOTA
- **来源**: [arXiv:2606.03458](https://arxiv.org/abs/2606.03458)
- Hadamard 旋转 + 双轴方差归一化，解决自回归解码下的误差累积
- **2-bit 精度**: MATH500, AIME24, HumanEval SOTA
- **vLLM 实现已开源**: github.com/huawei-csl/KVarN

### Lodestar: 在线学习推理路由
- **来源**: [arXiv:2606.00946](https://arxiv.org/abs/2606.00946)
- 持续收集集群快照，训练在线奖励预测器进行请求路由
- vs SOTA 启发式: **平均 TTFT 降低 1.41×**, P99 TTFT 降低 1.47×
- 异构集群上可达 **4.38×/4.42×** 改善
- 约 **5 分钟** 内学习到高效路由策略

### DriftSched: 自适应 QoS 推理调度
- **来源**: [arXiv:2606.02982](https://arxiv.org/abs/2606.02982)
- 运行时 Token Drift 自适应补偿: MAE **-38.8%**, RMSE **-40.5%**
- SJF 调度策略: 中位延迟 **-42%**, P99 延迟 **-16%** (vs FIFO)
