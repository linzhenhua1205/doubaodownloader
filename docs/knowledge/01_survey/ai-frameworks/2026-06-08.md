## 1️⃣ vLLM Cold Start 延迟首次系统性分析 — 启动拆分6阶段，CPU为瓶颈 ⭐

- **来源**: [arXiv:2606.07362](https://arxiv.org/abs/2606.07362) (MLSys 2026) | **日期**: 2026-06-05
- **开源**: 含 benchmark 数据集、分析工具和预测脚本

### 核心贡献

vLLM 已进化为 LLM 推理的 de facto 引擎，但其**启动延迟**从未被系统性研究过。本文首次将 vLLM 启动过程拆分为 **6 个基础阶段**：

1. 模型加载（模型文件 I/O）
2. 权重初始化与内存分配
3. CUDA Graph 捕获与预热
4. KV Cache 预分配
5. Scheduling 基础设施初始化
6. 预热推理（warmup runs）

### 关键发现

- **启动过程主要为 CPU bound**，GPU 利用率极低
- **每个阶段随模型参数和系统配置呈现一致的、可解释的扩展趋势**
- 提出了轻量级**分析模型**，给定硬件配置即可**准确预测 vLLM 启动延迟**

### 评价
随着 LLM 推理服务走向大规模弹性伸缩（尤其是 cold-start 场景），启动延迟直接决定了弹性效率。本文填补了 vLLM 领域的一个重要空白——此前社区更多关注运行态性能，忽略了启动态优化。

## 2️⃣ DriftSched — Runtime Token Drift 感知调度 | vLLM 多租户 P99 -16%

- **来源**: [arXiv:2606.02982](https://arxiv.org/abs/2606.02982) | **日期**: 2026-06-02
- **架构**: 基于 vLLM 的 continuous batching 多租户调度

### 核心创新

LLM 推理服务中，请求的实际输出长度常常偏离接入时的估计值，产生 **Runtime Token Drift**，导致：
- 工作负载误分类
- 请求队列不平衡
- 尾延迟恶化 → QoS 下降

DriftSched 提出**自适应偏差校正机制**：

| 指标 | 改进 |
|:-----|:-----|
| MAE (平均绝对误差) | **-38.8%** |
| RMSE (均方根误差) | **-40.5%** |

在 NVIDIA L4 GPU 上对比多种调度策略：

| 调度策略 | 中位数延迟 (vs FIFO) | P99 延迟 (vs FIFO) |
|:---------|:--------------------|:--------------------|
| FIFO | baseline | baseline |
| Priority | 中等改进 | 中等改进 |
| Weighted | 有限 | 有限 |
| **SJF (最短作业优先)** | **-42%** 🏆 | **-16%** 🏆 |
| Aging Priority | 接近 SJF | 接近 SJF |

### 评价
一个非常实践性的工作：vLLM 现有的调度策略（FIFO、round-robin）忽略了 token drift 的普遍性。DriftSched 的偏差校正 + SJF 调度组合在生产环境中可以直接落地，无需修改模型或运行时架构。

## 3️⃣ TensorRT-LLM Blackwell — NVFP4 vs BF16 实测：1.6× 吞吐，但启动延迟是代价

- **来源**: [arXiv:2605.00519](https://arxiv.org/abs/2605.00519) (Silicon Showdown) | **日期**: 2026-05-01

### 核心发现

对 NVIDIA Blackwell 架构下的 TensorRT-LLM 推理栈进行了**系统性实证分析**，发现关键 trade-off：

| 量化格式 | 吞吐 | vs BF16 | 代价 |
|:---------|:-----|:--------|:-----|
| **NVFP4** | **151 tok/s** | **1.6×** 🏆 | 启动延迟显著增加 |
| BF16 优化 | 92 tok/s | baseline | 快速启动 |

### 关键洞察 — "后端二分法"（Backend Dichotomy）

NVFP4 虽然在生成吞吐上显著领先 BF16，但存在以下约束：
- **启动延迟 vs 生成速度的权衡**：NVFP4 的优化需要更多编译和预热时间
- **复杂的运行时约束**：要获得 NVFP4 的加速，需要特定的运行时配置
- **VRAM Wall**: 70B+ 模型面临"二选一"困境：激进量化（如 Q2）降低模型智能，或 PCIe CPU offloading 使吞吐下降 90%+

作为对比，Apple Unified Memory Architecture (UMA) 在 80B 4-bit 模型上可线性扩展，能效优势高达 **23×（tokens/joule）**。

### 评价
这是首个在 Blackwell 消费者级硬件上对比 TensorRT-LLM NVFP4 vs BF16 的独立实证。核心启示：**NVFP4 的加速不是"免费午餐"**，带来显著的启动延迟代价和运行时复杂度。
