## 1️⃣ APEX4: 纯 W4A4 推理 — 无需修改的 vLLM 替换，最高 2.09× 加速 ⭐

- **来源**: arXiv:2606.08761 (cs.LG, June 7 提交)
- **论文**: 《APEX4: Efficient Pure W4A4 LLM Inference via Intra-SM Compute Rebalancing》
- **核心贡献**: 首次系统研究 **Intra-SM 计算平衡** 瓶颈，识别 Tensor Cores 与 CUDA Cores 吞吐比 (ρ) 为关键硬件指标
- **关键发现**:
  - W4A4-g128 kernel 在 **RTX 3090** (ρ=16) 上 **2.0-2.5×** 加速
  - 但在 **A100** (ρ=64) 上 **退化至 0.43-0.47×** — W4A4 并非普遍有效，而是平台依赖
- **性能数据** (作为 vLLM 无修改替换):
  - L40S (ρ=8): **1.66×** 端到端加速
  - RTX 3090 (ρ=16): **1.78×**
  - A40 (ρ=16): **2.09×**
  - A100 (ρ=64): **1.20-1.40×** (混合粒度模式恢复)
- **精度**: LLaMA-2-70B 上 perplexity 在 FP16 的 0.63 以内，zero-shot 准确率超 W4Ax Atom-g128 达 **4.0-4.4%**
- **意义**: 第一个证明纯 W4A4 可在消费级 GPU (如 RTX 3090) 上"无痛"部署，无需混合精度回退

## 2️⃣ NVFP4 + JAX/MaxText: Blackwell 上 4-bit 混合精度预训练正式可用，1.73× 加速 ⭐

- **来源**: NVIDIA Developer Blog, June 8 2026
- **作者**: Max Xu, Haixin Liu, Abhinav Goel, Tejash Shah (NVIDIA)
- **链接**: https://developer.nvidia.com/blog/train-models-faster-with-jax-and-maxtext-using-nvfp4-on-nvidia-blackwell/
- **核心**: NVFP4 训练配方在 **JAX/MaxText (TransformerEngine)** 中正式可用，5 项关键技术：
  1. **16-element 微块缩放** (vs MXFP4 的 32-block，单离群点影响更小)
  2. **E4M3 块缩放因子** (MXFP4 的 E8M0 需多 36% token 才能匹配 NVFP4 的最终 loss)
  3. **Random Hadamard Transform** (仅 WGRAD，不破坏 2D 权重缩放一致性)
  4. **2D 权重缩放** (每 16×16 权重块一个 FP8 scale，FPROP/DGRAD 共享)
  5. **Stochastic Rounding** (仅梯度，权重/激活用 round-to-nearest-even)
- **性能数据** (MaxText pretraining, Llama 3, seq_len=8192):
  | 模型 | 硬件 | FP8 TFLOP/s/GPU | NVFP4 TFLOP/s/GPU | 加速比 |
  |:----|:----|:--------------:|:----------------:|:-----:|
  | Llama 3 8B | GB200 | 1,497 | 2,017 | **1.35×** |
  | Llama 3 8B | GB300 | 1,759 | 2,301 | **1.31×** |
  | Llama 3.1 405B | GB200 | 1,557 | 2,241 | **1.44×** |
  | Llama 3.1 405B | GB300 | 2,103 | 3,633 | **1.73×** |
- **精度验证**: 10,000 步训练后，NVFP4 与 FP8 基线的 **loss 差距仅 +0.026 nats**，在步间噪声范围内
- **注意**: NVFP4 仅应用于 MLP 层 GEMM，Attention 层保留更高精度（softmax 指数放大量化噪声）

## 3️⃣ AgentCompile: LLM 引导的 CUDA 推理编译器 — 5.66× 超 PyTorch Eager ⭐

- **来源**: arXiv:2606.07665 (cs.PL, June 3 提交)
- **论文**: 《AgentCompile: An LLM-Guided Compiler for Direct CUDA Inference》
- **核心架构**: LLM 仅作为 **咨询搜索元数据引擎** — 提供语义标签、候选优先级、参数提示、风险标注；编译器完成 CUDA candidate 物化、约束检查、实测选择、未优化时自动回退
- **性能数据** (端到端自回归生成 vs PyTorch eager):
  - Qwen3-1.7B: **5.66×** 加速
  - Qwen3-4B: **4.05×** 加速
  - Llama-3.2-1B-Instruct: **4.26×** 加速
- **设计理念**: 不是让 LLM 写代码，而是让 LLM 做**编译器决策的语义导航**——标记哪些区域值得特殊化、哪些 CUDA 实现族适用、参数配置建议
- **回退机制**: 当特殊化不支持或收益不大时，自动回退到通用路径
- **意义**: 解决了 AI compiler 领域"LLM 生成的代码可靠性差"的核心矛盾——LLM 提建议，编译器执行+验证
