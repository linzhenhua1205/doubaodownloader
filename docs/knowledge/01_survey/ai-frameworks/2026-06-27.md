## 1️⃣ 推理框架

### 1.1 vLLM — v0.23.0 (6月15日，408 commits, 200 contributors)

**核心变化**：

- **DeepSeek-V4 深度成熟**：稀疏 MLA 元数据与 DeepSeek-V3.2 解耦 (#44699)、TRTLLM-Gen attention 内核 (#43827)、EPLB 支持 Mega-MoE (#43339)、选择性 prefix-cache retention for SWA KV cache (#43447)
- **Model Runner V2 扩展**：默认用于 Llama 和 Mistral dense 模型 (#43458)；新增 FlashInfer 采样器 (#42472)、breakable CUDA graphs (#44050)、pipeline-parallel bubble elimination (**消除 PP 气泡**, #42187)
- **Rust 前端**：streaming generate、dynamic LoRA 端点、版本/服务信息端点
- **多级 KV Cache 卸载**：新增对象存储二级缓存 (#41968)、HMA 默认启用 (#41847)、per-request offloading policy (#43205)
- **Transformers v5** 兼容

**性能数据**：
| 优化 | 加速比 | 条件 |
|:-----|:------:|:-----|
| CUTLASS FP8 scaled-mm padding bypass | **+20%** | batch-invariant |
| MoE-permute buffer pre-allocation | **+9–14%** | H200 |
| CutlassFP8 padding pre-processing | **+13.5%** TTFT | — |
| padded NVFP4 quant kernel | **+2.4–5.7%** E2E | — |
| GPU↔CPU sync elimination (4/n) | 减少同步开销 | — |

**新模型支持**：Step-3.7-Flash、Cosmos3 Reasoner、Gemma 4 Unified、Granite Speech Plus、Cohere Mini Code

### 1.2 SGLang — v0.5.14 (6月26日)

**核心变化**：

- **DeepSeek-V4 on GB300**: 在相同交互性下达到 5× 更高吞吐量（对比 Day-0 方案）
- **Waterfill & LPLB MoE 负载均衡**：Waterfill 用于 shared-expert dispatch，LPLB（线性规划）用于 redundant expert replicas
- **KDA CuteDSL Prefill kernel (SM100)**：**1.08–1.52×** 快于 Triton 路径
- **Linear-attention prefix-cache 内存节省**：int8 checkpoint pool 存储 recurrent states，大幅提升 prefix-cache 容量
- **MSCCL++ 集成**：自动调优 collectives for TP=8 single-node / TP=16 two-node
- **NVFP4 MoE for DeepSeek-V4** on Blackwell
- **NPU (Ascend)** 支持 DeepSeek-V4 (#25144)
- **Spec V2 成为默认 speculative-decode 路径** (v0.5.13)

**新模型**：GLM-5.2、LiquidAI LFM2.5、Kimi-K2.7-Code、DiffusionGemma、Zyphra ZAYA1

### 1.3 TensorRT-LLM — v1.3.0rc19 (6月23日)

**核心变化**：

- **TrtllmGenAttention 成为 Blackwell (SM12x) 默认 decode 后端** (#14618)
- Async Ulysses pipeline for diffusion (LTX-2, WAN) (#13978)
- MegaMoECuteDsl NVFP4 MoE backend (#14608)
- EAGLE3 dynamic trees on Blackwell (#12958)
- Per-expert LoRA in Cutlass backend (#14881)
- **KV cache prefetch**: `prefetch_reuse_blocks` + 可配置 prefetch count (#15149)
- Prometheus metrics for prompt cache、speculative decoding、perplexity (#12636)
- Qwen 原生 image support (#13449)
- Step-3.7-Flash 支持 (#14711)

## 2️⃣ 训练框架

### 2.1 PyTorch 2.12 (5月13日, 2,926 commits, 457 contributors)

**核心变化**：

- **`torch.accelerator.Graph`**：设备无关的 graph capture/replay API，统一 CUDA/XPU/out-of-tree 后端
- **Microscaling (MX) 量化导出**：`torch.export.save` 支持 float8_e8m0fnu dtype，解锁 MXFP4/MXFP6/MXFP8 完整导出
- **`torch.cond` CUDA Graph**：利用 CUDA 12.4 条件 IF 节点，GPU 内完成分支评估
- **Inductor FMA addcdiv**：bitwise numerical parity with eager + Triton fusion
- **ROCm**：expandable memory segments、rocSHMEM 对称内存 collectives、hipSPARSELt FP8 semi-structured sparsity、FlexAttention pipelining (**5–26%** speedup on MI350X)
- **batched linalg.eigh**: **up to 100×** faster (cuSolver syevj_batched)

**Deprecations**：
- CUDA 12.8 wheel 废弃，默认 CUDA 13.0
- TorchScript 正式废弃，推荐 torch.export + ExecuTorch
- 未来 torchcomms 将默认用于分布式

### 2.2 DeepSpeed — v0.19.2 (6月16日) / v0.19.0 (5月6日)

**核心变化**：

- **AutoEP**: 自动专家并行配置 (#7938)
- **Muon Optimizer ZeRO-3**: 大型模型优化器支持 (#7919)
- **AutoSP (Sequence Parallelism)**: 支持 Multimodal Models (ViT + LLM) (#7984)
- **Zero3 defragment utility**: 碎片整理工具 (#7940)
- **SDMA allgather** via mori (#7999)
- **优化 singleton MoE collectives** (#7997)
- **DeepCompile + ZeRO-3**: 多版本 PyTorch 兼容修复 (#8024, #8033)
- **Biren SUPA accelerator 支持** (#8054)

## 4️⃣ KV Cache 优化

### 4.1 vLLM 多级卸载框架

- **对象存储二级缓存** (#41968)：KV cache 溢出到对象存储
- **HMA 默认启用** (#41847)：Hybrid Memory Allocator 整合
- **Per-request 卸载策略** (#43205)：通过 lifecycle hook 允许按请求定制卸载行为
- **Triton small CPU→GPU swap fast-path** (#42212)

### 4.2 SGLang 线性 attention prefix-cache

- **int8 checkpoint pool** (#28185)：存储 recurrent states 在 Mamba radix cache 中，大幅增加 prefix-cache 容量
- **speculative conv-window 去重** (#28302)：滑动窗口布局减半中间缓存占用
- **Decode-side radix cache** (v0.5.13)：PD 分离部署中恢复 radix-cache 命中率

### 4.3 TurboQuant 2-bit KV Cache (vLLM v0.20.0)

- 新的 attention backend 实现 **2-bit KV cache 压缩**，**4× 容量**
- 支持 FA3/FA4 prefill (#40092)

## 5️⃣ 趋势观察

### 5.1 DeepSeek-V4 成为推理框架的"试金石"

在 6 月，所有三大推理框架（vLLM、SGLang、TensorRT-LLM）均完成了对 DeepSeek-V4 的深度优化：
- **SGLang** 率先在 GB300 上达到 **5× throughput**
- **vLLM** 完成了稀疏 MLA 解耦、EPLB、多 backend 支持
- **TRT-LLM** 添加了 NVFP4 MoE backend、EAGLE3 dynamic trees

### 5.2 Blackwell 生态快速成形

- Triton 3.7 大幅增强 SM100/SM120/SM121 支持
- TRT-LLM TrtllmGenAttention 成为 Blackwell 默认 decode 后端
- SGLang KDA CuteDSL Prefill Kernel 解锁 **1.52×** 加速
- vLLM FlashInfer Blackwell GDN prefill + FP4 GEMM

### 5.3 KV Cache 优化的"三线并进"

1. **容量扩展**：多级卸载 (object store, HMA) + 2-bit TurboQuant (4×)
2. **调度精细化**：PersistentKV 页面感知调度 (1.3×)
3. **CPU 路径**：Cache-Resident LLM 证明权重驻留 CPU LLC 可行 (11.5×)

### 5.4 训练框架的"统一化"趋势

- PyTorch 2.12: `torch.accelerator.Graph` 统一跨后端 graphs、`torch.export` 统一 MX 量化导出
- DeepSpeed: AutoEP/AutoSP 自动化并行配置
- torchcomms 即将默认用于分布式
