### 🔍 Inspektor Gadget 完成首次安全审计（June 3）

**来源**: [CNCF Blog](https://www.cncf.io/blog/2026/06/03/inspektor-gadget-results-from-the-first-security-audit/)
**项目**: Inspektor Gadget（CNCF Sandbox，eBPF-based K8s 可观测性工具包）

Inspektor Gadget 完成了由 OSTIF 协调、Shielder 执行的首次独立安全审计。对于运行 eBPF 工具进行 K8s 集群可观测性的团队，这是重要的信任里程碑。

**审计范围**:
- 协作威胁建模 + 手动源码审计 + 动态测试 + Semgrep/GoSec 静态分析 + AI 辅助代码审查
- 三个测试环境：本地 Linux 主机、远程 Daemon 部署、minikube K8s 部署

**发现 3 个 CVE（无 Critical/High）**:

| 编号 | 严重程度 | 问题 | 影响场景 | 修复版本 |
|:-----|:---------|:-----|:---------|:---------|
| CVE-2026-24905 | **Medium** | `ig image build` 命令注入：Makefiles 嵌入用户输入未正确转义 | CI/CD 中构建不可信 gadget | v0.48.1 |
| — | **Medium** | eBPF ring buffer 事件淹没（硬编码 256KB），恶意容器可噪声隐藏活动 | 安全监控流水线中的逃逸 | v0.50.1 |
| CVE-2026-25996 | **Low** | columns 输出模式 ANSI 转义序列未清理 | 受陷容器注入终端逃逸码 | v0.49.1 |

**6 条强化建议（非主动漏洞）**:
1. TCP 监听器默认启用 TLS（当前仅日志警告）
2. CI/CD 中 pin 和验证外部依赖哈希/签名
3. 实现 K8s namespace blocklist，阻止对 kube-system 等敏感命名空间的追踪
4. 限制远程客户端通过 Daemon 启用主机级追踪
5. 自动化第三方依赖漏洞扫描
6. 降低 DaemonSet pod 的 RBAC 权限（特别是 `nodes/proxy` GET）

**Gadget Bypass 测试**：研究人员发现 6 种绕过场景（如使用 `openat2` 替代 `openat`、通过 `io_uring` 逃避、静态链接库），反映了 eBPF 追踪的猫鼠博弈本质。

**➡ 行动项**: 更新至 v0.50.1 或更高版本

### 🖥️ K8s Dashboard → Headlamp 迁移实战（June 1）

**来源**: [K8s Blog](https://kubernetes.io/blog/2026/06/01/from-kubernetes-dashboard-to-headlamp-understanding-the-transition/)
**项目**: Headlamp（CNCF 孵化中，K8s 社区推荐的 Dashboard 替代品）

Kubernetes Dashboard 是许多人的第一个 K8s 窗口，但随着 K8s 生态演进，项目维护挑战使社区将重点转向 **Headlamp**。

**关键迁移事实**:
- Kubernetes Dashboard 仍是官方项目，但**维护节奏已显著放缓**
- **Headlamp** 被社区视为实际上的后继者，已集成 Cluster API、Karpenter Plugin 等新能力
- 迁移路径：Headlamp 可**并行运行**，无需立即下线旧 Dashboard
- Headlamp 的 Plugin 架构支持**自定义资源视图**，对运行 Operator 的团队特别有用
- **AI Assistant**（2025 年引入）可在 UI 中自然语言查询集群状态

**Ops 影响**: 仍在使用 K8s Dashboard 的团队应评估迁移计划。Headlamp 的活跃开发和 CNCF 社区支持使其成为长期选项。

### 🤖 CNCF AI-native 圆桌补充细节（June 2）

**来源**: [CNCF Blog](https://www.cncf.io/blog/2026/06/02/cloud-native-is-now-ai-native-engineering-production-ready-ai/)

此前 06-09 已覆盖，获取完整原文后有新细节：

**K8s 为 AI 重构的三项关键举措**:
1. **Pod Groups (Workload API)** — 将 Pod 组视为单一故障域，保证大规模 AI 矩阵初始化的邻近性
2. **DRA** — 将 GPU 等专用芯片纳入 K8s 调度器
3. **Inference Gateway** — 利用 Gateway API 标准构建 AI 网关，管理 Prompt 和生成式模型响应

**工程师角色转变**:
- Prototyping 已取代 PRD — PM 先用 AI 生成原型再测试，然后才写正式文档
- 新瓶颈：**AI 生成的海量代码需要人工审查**
- 解决方向：Agentic SRE，AI 辅助根因分析 + 修复，但关键决策由人掌控

**AI 供应链安全两方向**:
- 一致的 Evals 框架 + guardrails（部署前）
- `llms.txt` 开放标准确保模型引用可信源
