### 4️⃣ BPF 程序协程化（Suspend/Resume BPF）
- Kumar Kartikeya Dwivedi 在 **LSFMM+BPF 2026** 提出将 BPF 程序表达为 **coroutine（协程）**
- 核心目标：允许 BPF 程序暂停后恢复执行，解决当前"必须一次运行到底"的限制
- 虽然仍处于实验阶段，但有望大幅简化长运行 BPF 任务的编写
- 社区讨论活跃 (9 comments) ([LWN](https://lwn.net/Articles/991105/))

### 5️⃣ overlayfs 进展：嵌套层 + composefs 成熟
- Amir Goldstein 在 LSFMM+BPF 2026 fs track 汇报 overlayfs 最新进展
- **嵌套 overlayfs 层**（nesting）支持已趋于成熟
- **composefs 用例**（2023峰会后开始推动）已带来重要变化，部分功能已合入主线
- 与容器镜像挂载场景高度相关 ([LWN](https://lwn.net/Articles/990041/))

### 6️⃣ buffer-heads 清理工作
- Jan Kara 持续推进 **buffer_head 从内核文件系统中的清理**
- LSFMM+BPF 2026 专题汇报：元数据的通用 buffer head 追踪基础设施、Amiga 文件系统清理、锁修复
- metadata 路径的 buffer head 使用量下降是清理成功关键指标 ([LWN](https://lwn.net/Articles/990769/))

### 7️⃣ 7.1 内核开发统计深度分析

### 8️⃣ RMR/BRMR — RDMA 块复制与多播
- 基于内核 RTRS（RDMA Transport Library）的两个新组件：
  - **RMR (Reliable Multicast over RTRS)**: 可靠多播

- 目标：为云厂商提供低开销的持久虚拟块设备 ([LWN](https://lwn.net/))

## 🔧 systemd v261 正式发布 (6/19)

> [systemd v261](https://github.com/systemd/systemd/releases/tag/v261) — Luca Boccassi 发布

### 关键新特性

| 特性 | 说明 |
|:-----|:------|
| **☁️ 云 IMDS 子系统** | 新增 Instance Metadata Service 支持，方便在云环境中通过 systemd 直接访问元数据 |
| **🔁 LUO/KHO kexec 持久化** | PID1 支持内核 Live Update Orchestration / Kexec Handover，FD Store 在 kexec 重启间保持（支持 memfd） |
| **👤 用户会话 FD 持久化** | 用户 session manager 支持通过 notify socket 接收并重启时通过 `$LISTEN_FDS` 传递，联合 LUO 实现用户单位跨 kexec 状态持久化 |
| **🔒 RestrictFileSystemAccess=** | 基于 **BPF LSM** 的新安全设置，限制仅执行存储在 dm-verity 文件系统上的已签名二进制文件 |
| **⚙️ CPUSetPartition=** | 新 unit 设置，配置 cpuset cgroup 分区类型（root/isolated/member） |
| **📊 CPUPressureWatch=/IOPressureWatch=** | 新 per-unit 设置，使服务可获取 CPU/IO 压力事件的通用通知 |
| **🔌 Varlink shutdown 方法** | io.systemd.Manager 新增 PowerOff/Reboot/SoftReboot/Halt/Kexec Varlink 方法，补充现有 D-Bus 接口 |
| **⏱️ EventLoopRateLimit** | PID1 事件循环速率限制配置，防止 busy-loop 导致系统卡死 |
| **🛡️ 跨 TPM 测量加固** | systemd-pcrosseparator.service 新增 PCR 0-7,9,12-14 隔离测量，保护 firmware 测量 |
| **🔌 DHCP Relay** | sd-dhcp-relay 新增 DHCP relay agent 支持，替换旧的 [DHCPServer] 设置 |

### 不兼容变更
- `systemd-nspawn --user=` → `--uid=`（`--user` 不再接受参数，仅作为 standalone switch）
- Varlink io.systemd.Unit 枚举字段改为 underscore 格式（`tty-force` → `tty_force`）
- 支持 udev 旧数据库版本 0 已移除（从 v247 之前升级不再支持）
- libsystemd 不再保证链接 libm（影响 rsyslog 等下游）
