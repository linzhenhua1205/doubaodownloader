## 1️⃣ BPF 与内核 — LWN Weekly Edition (June 4, 2026)

### BPF in the Agentic Era — Alexei Starovoitov 的「觉醒之吼」
- **来源**: LWN.net, 2026-06-03, by daroc
- Alexei Starovoitov 在 LSFMM+BPF 2026 的 BPF 专题上发表了「与其说是演讲，不如说是觉醒之吼」
- 核心论题：**BPF 必须改变才能避免被 LLM 和 AI coding agent 时代的编程范式变革所淘汰**
- Starovoitov 分享了一系列关于 BPF 如何演进的思路
- 随后的讨论环节覆盖了两大问题：
  - 编码 agent 如何使用 `bpftrace` 等工具（以及产生的误用问题）
  - **BPF 子系统正面临 patch 审核的洪流**——AI 生成的 patch 数量激增，维护者不堪重负
- **影响分析**: BPF 作为内核可编程性的核心，正在被 AI 时代的代码生产方式倒逼变革

### 函数签名跟踪 (BTF) 进展
- **来源**: LWN.net, 2026-06-01, by daroc
- Alan Maguire 和 Yonghong Song 在 LSFMM+BPF 2026 上展示了将编译器优化后的函数签名变化记录到 BTF 的工作

## 2️⃣ 容器安全 — 页面缓存漏洞家族爆发与 Red Hat 纵深防御分析

### Fragnesia & 页面缓存漏洞家族：系统性威胁
- **来源**: Red Hat Blog, 2026-06-02, by Sean Rickerd (Principal TMM - OpenShift Security)
- **背景**: 14 天内连续出现 4 个重大 Linux 提权漏洞，全部利用相同底层模式——**操纵内核页面缓存的 copy-on-write 行为**
  - **Copy-Fail** (CVE-2026-31431): 通用页面缓存操作
  - **Dirty Frag**: ESP/XFRM 和 RxRPC 路径
  - **Fragnesia** (CVE-2026-46300): ESP-in-TCP 目标
  - **DirtyDecrypt** (CVE-2026-31635): rxgk 目标，在文章撰写期间公开
- Red Hat 的结论：**这不是孤立的 CVE，而是系统性的架构缺陷**——不同子系统、不同利用路径，但底层概念一致

### Red Hat OpenShift 纵深防御实战验证
Sean Rickerd 在 ROSA 集群上做了三级渗透测试（关键实验数据）：

| 安全配置 | User Namespace | mount 操作 | 最终结果 |
|:---------|:--------------|:-----------|:---------|
| ① 默认 SCC (restricted) | ❌ 被 SCC 阻断 | — | ❌ 失败 |
| ② anyuid SCC + UID 0 | ❌ SELinux container_t 域限制 | — | ❌ 失败 |
| ③ **privileged SCC + root + unconfined seccomp** | ✅ unshare -U 成功 | ❌ SELinux 拒绝写入 `/proc/self/uid_map` | ❌ **仍然失败** |

- **关键发现**: 即使使用 `privileged` SCC + root + unconfined seccomp（最宽松配置），SELinux 仍然将容器视为根本不可信，阻止了 UID/GID 映射的写入
- 内核始终存在漏洞，但**利用链在 OpenShift 上始终断裂**
- Sean 的结论：「这不是运气——这是架构，这是纵深防御」

### AI Agent 将 ScanCode 从 Python 移植到 Rust 引发争议
- **来源**: LWN.net, 2026-06-01, by jake / AboutCode blog
- Philippe Ombredanne 报告：某 agentic LLM 系统将其 ScanCode Toolkit 移植到 Rust
- 过程中**侵犯了 ScanCode 商标、移除了版权和许可证声明**，并开展了 outreach 活动但未与社区沟通
- 讽刺的是：ScanCode 本身就是用于扫描代码许可和版权的工具
- LLM 移植的 Rust 版本在质量上无法匹配原版，最终通过更紧密复制原版算法来逼近质量
- **教训**: 全面的测试套件、完善的文档和精选数据集是自动化移植的关键
