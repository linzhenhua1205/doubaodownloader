## ⭐ 核心发现

### ① 🔥🔥 Linux 7.1 正式发布！——FRED、新NTFS驱动、BPF io_uring 等重大特性

**来源**: LWN.net · corbet · 2026-06-14 · Phoronix · Michael Larabel · 2026-06-14

Linus Torvalds 于 6 月 14 日（周日）正式发布 **Linux 7.1 稳定版**。核心特性如下：

#### 架构
- **移除对部分旧 486 架构的支持**（i486 退场）
- Intel **FRED（Flexible Return and Event Delivery）** — 替换传统 IDT 门描述符中断处理方案，简化内核异常分发路径，对 **Panther Lake** 等未来 Intel CPU 起支撑作用

#### 进程管理
- **新的 clone() flags** — 配合之前在讨论的 spawn templates，简化进程创建路径
- **sched_ext 初始子调度器支持** — 虽然尚不完整，但为 BPF 调度的可编程调度策略铺路

#### BPF / IO / 存储
- **BPF 支持 io_uring** — eBPF 程序可直接挂载到 io_uring 异步 I/O 事件，实现**智能化 I/O 策略定制**
- **ublk 用户态块驱动零拷贝 I/O** — 用户态块设备驱动（如 SPDK 生态）可获得零拷贝数据路径性能
- **更多 swap 改进** — 内存管理持续优化
- **完全重写的 NTFS 驱动** — 替代旧的 ntfs-3g FUSE 方案，原生内核 NTFS 读写性能大幅提升
- **Intel Arc 显卡加速** — drm/i915 改进

> **对 K8s 的价值**: sched_ext 子调度器 + BPF io_uring 为**容器粒度的可编程调度和 I/O 策略**奠定内核级基础

### ② 🔥🔥 Cache Aware Scheduling 已合入 Linux 7.2——多 LLC CPU 性能关键提升

**来源**: Phoronix · Michael Larabel · 2026-06-15

#### 技术细节
- **Cache Aware Scheduling（缓存感知调度）** 已作为 7.2 合入窗口的首批调度更新被合并
- 针对现代 Intel/AMD CPU 的**多最后一级缓存（LLC）** 设计优化
- 调度器在任务唤醒、负载均衡时**感知 LLC 拓扑**，优先将任务调度到缓存距离最近的 CPU 核心上
- 对**跨 CCD/CCX 的 AMD EPYC** 和 **Intel 多 tile 处理器**提升尤为显著

#### 与 K8s 的协同
- K8s **CPU Manager Static Policy** 目前已支持按 NUMA 拓扑绑定 CPU
- Cache Aware Scheduling 叠加后，可在 **Pod 维度优化 L2/LLC 缓存亲和性**
- 结合 **Topology Manager**，K8s 可同时感知 **NUMA 节点 + LLC 缓存拓扑**，做出更优的调度决策

> **性能预期**: 在多 LLC 场景（如 4+ CCD 的 AMD EPYC），缓存感知调度可显著降低跨 LLC 内存访问延迟，预计在 CPU 密集型容器工作负载中可获得 10-25% 的性能改善（具体基准待 Phoronix 后续测试）

### ③ 🔥 Linux 7.2 引入 Rust Zerocopy 库——消除更多不安全代码

**来源**: Phoronix · Michael Larabel · 2026-06-15

#### 技术细节
- Miguel Ojeda 提交了 7.2 Rust 代码更新，新增 **40,000+ 行 Rust 代码**
- 核心引入：**Rust Zerocopy 库** — 提供零拷贝内存操作的安全抽象
- 目标：消除更多内核代码中的 `unsafe` Rust 块，用类型安全的零拷贝操作替代
- 这是 Rust 进入内核以来**最大的一次代码量注入**

#### 意义
- Rust 在 Linux 内核中的地位持续强化：从最初的驱动层，逐步扩展到内存管理抽象层
- Zerocopy 库对网络协议栈、存储子系统等零拷贝路径有直接安全增益
- **对容器/K8s 的影响**: 更安全的内核减少容器逃逸面

### ④ Arm64 CNCF 项目 OCI 信用额度——50%+ AWS 新实例是 Arm64

**来源**: CNCF · Member Post · Dave Neary（Ampere Computing 开发者关系总监）· 2026-06-15

#### 关键数据
- **AWS 超过 50% 的新实例是 Arm64**（截至 2025 年底）
- **Azure 超过 33% 的新实例是 Arm64**
- Arm64 在云服务提供商中已从 "备选架构" 变为 "主流架构"

#### 项目详情
- CNCF 通过 OCI（Open Container Initiative）信用额度项目，与 Ampere Computing 合作，为 **CNCF 项目提供免费 Arm64 CI/CD 运行环境**
- 参与项目包括：Prometheus、etcd、CoreDNS、containerd 等关键云原生基础设施
- 目标：确保 CNCF 生态项目在 Arm64 上的持续集成、测试和发布**与 x86 同等成熟**

#### 对 K8s 生态的影响
- Arm64 K8s 集群的增长意味着**容器镜像必须支持多架构构建（manifest list）**
- KubeCon India 2026（6/18-19 孟买）将讨论 Arm64 在云原生的最佳实践

### ⑤ Linux 7.2 Kbuild：提升 LLVM/Clang 编译器版本要求 + 分布式 ThinLTO 支持

**来源**: Phoronix · Michael Larabel · 2026-06-14

- Linux 7.2 将**提升 LLVM/Clang 最低版本要求**
- 新增 **分布式 ThinLTO（链接时优化）** 支持：允许跨多台机器并行执行 LTO 优化，大幅减少大型内核构建的链接时间
- **对内核开发者/K8s 供应商的意义**: 减少自定义内核构建时间，加速 bug 修复迭代
