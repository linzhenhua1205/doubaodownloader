### ① 🔧 Overlayfs 最新进展：嵌套层 + composefs 驱动创新

**来源**: LWN.net · jake · 2026-06-12 · LSFMM+BPF Summit 2026 文件系统分会场

Amir Goldstein 在 LSFMM+BPF Summit 的文件系统分会场（缩短议程）中做了 overlayfs 联合文件系统的更新报告：

- **新特性回顾**：过去几年 overlayfs 新增了若干功能，Goldstein 重点介绍了 **nesting overlayfs layers（嵌套层）** 的现状
- **composefs 驱动的变更**：自 2023 年 LSFMM 峰会上讨论 composefs 用例以来，已经衍生出一些**有趣的变化**
  - composefs 是一种只读镜像格式，允许 overlayfs 将多个只读层联合挂载为单个文件系统视图
  - 这一用例直接推动了 overlayfs 在元数据管理、层合并策略上的改进
- **嵌套层的挑战**：overlayfs 支持递归嵌套（一个 overlay mount 之上再叠 overlay），但涉及 inode 编号管理、目录合并策略等复杂问题
- **对容器/K8s 的意义**：
  - overlayfs 是 Docker/containerd 默认的容器镜像层叠文件系统
  - composefs 驱动的改进将**直接影响容器镜像的层合并性能**和**启动延迟**
  - 嵌套层支持对 K8s sidecar + init container 的多层镜像加载场景有性能优化空间

> **评价**: overlayfs 作为容器生态最底层的文件系统，其嵌套层和 composefs 改进直接影响容器启动速度和镜像管理开销，值得关注后续合入 7.2 主线的可能性

### ② 🔑 External Secrets Operator — 多账号 K8s Secret 治理实战方案

**来源**: CNCF · Member Post · Viktoria Bisova (Itigix) · 2026-06-09

#### 问题背景
- 现代企业将 Dev/Staging/Prod 环境分布在**不同 AWS 账号/EKS 集群**中
- 共享凭证（第三方 API Key 等）分散在各账号的 AWS Secrets Manager 中，**轮转时需逐账号手动更新**
- 核心矛盾：环境隔离（安全最佳实践）vs 秘密共享（运营效率）

#### 架构方案：ESO + Bitwarden Secrets Manager

```
┌─────────────────┐     ┌──────────────────┐     ┌──────────────┐
│  Bitwarden SM   │────→│  ExternalSecrets  │────→│  K8s Secret   │
│  (单一权威源)     │     │  Operator (每个集群) │     │  (应用消费)     │
└─────────────────┘     └──────────────────┘     └──────────────┘
```

- **ClusterSecretStore**（全局范围）：开发者可在任何 namespace 引用中央存储，无需重复配置认证
- **ExternalSecret** 声明式资源：告诉 ESO "从 Bitwarden 取 `stripe-api-key`，在 K8s 中创建 `payment-creds`"
- **15 分钟轮转刷新间隔**：凭证在 Bitwarden 更新后，15 分钟内自动同步到所有集群

#### 关键实现细节
1. **Cert Manager** 自签 CA 保障 Bitwarden SDK 与 ESO 之间的 HTTPS 通信
2. **Machine Account Access Token**（只读+最小权限）存储为 K8s Secret
3. **15 分钟 refreshInterval**：`refreshInterval: 15m` 自动轮转，更新一次 Bitwarden → 所有集群自动接收

#### 技术价值
- **Provider 无关设计**：同样模式适用于 HashiCorp Vault、AWS Secrets Manager、Azure Key Vault、Google Secret Manager 等
- **对多集群管理的启示**：新集群仅需安装 ESO + ClusterSecretStore，**零手动秘密配置**
- **多账号 K8s 的标准化模式**：秘密存储与消费解耦，容器安全基线
