## 🐧 Linux 内核

### 1️⃣ Linux 7.2 merge window 进展与关键合入
- 7.1 于 6/14 发布后，7.2 merge window 已合入 **~7,000 non-merge changesets**，多数核心子系统已完成
- **BPF 协程化（Coroutine）** 🆕: Kumar Kartikeya Dwivedi 在 LSF/MM/BPF 2026 峰会上提出将 BPF 程序表达为 **coroutine**（协程）([LWN](https://lwn.net/Articles/991002/))
  - 核心突破：BPF 程序将支持**暂停和恢复执行**，无需一次运行完成
  - 意义：可简化长时间运行的 BPF 任务（如网络包追踪、观测数据流处理等），突破现有"必须一次执行完"的限制
  - 当前状态：仍为实验性，但社区反响积极（9条评论讨论）
- **overlayfs** 推进: Gao Xiang 等持续推进嵌套层 + composefs，容器镜像挂载场景关键 ([LWN merge window 总结](https://lwn.net/Articles/990769/))
- **buffer-heads 清理**: Jan Kara 持续推进 metadata 路径清理，Amiga 文件系统清理 + 锁定修复 ([LWN](https://lwn.net/Articles/991016/))

### 2️⃣ OSPM 2026 峰会报告发布 (6/22 完整报道)
- 4月中在剑桥举行的 Power Management and Scheduling in the Linux Kernel Summit
- 关键议题：
  - **sched_ext 用户态调度器** — 最新设计与用例
  - **idle-state selection** — idle 状态选择优化
  - **lock-holder preemption** — 锁持有者抢占机制
- 完整会议总结已由 LWN 分日发布 ([LWN](https://lwn.net/Articles/991025/))

### 3️⃣ RDMA 块设备复制: RMR / BRMR
- 基于 RTRS (RDMA Transport Library) 的两项新组件：
  - **RMR**: Reliable Multicast over RTRS — 可靠多播
  - **BRMR**: Block device over RMR — 块设备复制
- 目标：为云厂商提供低开销的持久化虚拟块设备方案 ([LWN](https://lwn.net/Articles/990400/))

## 🔧 Systemd v261 发布 (6/19)

最大亮点变化（LWN 专题报道 + GitHub Release Notes 完整分析）：

### ☁️ IMDS 子系统（全新）
- 新增 **systemd-imdsd** 守护进程，通过本地 Varlink IPC 提供云实例元数据访问
- 内置 hwdb (`40-imds.hwdb`) 识别主流云厂商：AWS EC2、Azure、GCE、Hetzner、Oracle Cloud、Scaleway、腾讯云、阿里云、Vultr
- `systemd-imds` 工具可将 IMDS 字段导入本地系统凭据，供后续服务消费
- 网络锁定：可限制对 IMDS 的访问（传统 cloud-init 客户端绕过需解锁）
- 意义：支持创建**真正通用的云镜像**，与具体云厂商解耦

### 🔁 LUO/KHO — kexec 持久化
- PID1 支持内核 **Live Update Orchestration (LUO)** / **Kexec Handover (KHO)** 系统
- 单元 FD Store 在 kexec 重启后**保持**，单元在重启后重新获取已保存的命名 fd
- 当前仅支持 memfd 类型；含 LUO Sessions 支持
- 用户会话管理器同样支持 FD Store 持久化（$LISTEN_FDS 传递）

### 🔒 RestrictFileSystemAccess= — BPF LSM 应用
- 新增单元设置 `RestrictFileSystemAccess=`，利用 **BPF LSM 程序**限制执行仅允许位于签名且已验证的 **dm-verity** 文件系统上的二进制
- 这是 systemd 利用 eBPF 实现安全加固的最新实践

### 其他关键变化
- **CPUPressureWatch=** / **IOPressureWatch=** — 服务级 CPU/IO 压力事件通知
- **OOMRule=** — systemd-oomd 支持 OOM 规则集（`/etc/systemd/oomd/rules.d/`）
- **systemd-sysinstall** — 新组件，提供现代文本化 OS 安装器（封装 repart/bootctl link/creds）
- 所有外部库依赖已全面切换到 **dlopen()** 动态加载（libc 除外），详见 ELF dlopen metadata note
- **ConditionFraction=** — 按机器 ID 哈希的百分比条件，支持**分阶段灰度上线**
- **Confidential Computing** 支持：`systemd-vmspawn --coco=`（目前 AMD SEV-SNP）
- **systemd-sysupdate** 不再视为实验性，已移至 `/usr/bin/`
- 贡献者列表包含 **Claude Opus 4.6** 署名 — AI 辅助开发的里程碑

## ☸️ Kubernetes v1.36 特性汇总

K8s v1.36（"Haru" 春）从 5 月起逐篇发布博客，以下是截至 6/23 的核心 GA/Beta：

### ✅ 毕业到 GA
| 特性 | 说明 |
|:-----|:------|
| **PSI Metrics GA** | 内核 PSI（Pressure Stall Information）指标，kubelet 检测 OS 支持后才暴露，性能证明开销为 0.037–0.125 核(80 pod 密度) |
| **Volume Group Snapshots GA** | 跨多个卷的组级别快照，v1.27 alpha → v1.32 beta → v1.34 v1beta2 → v1.36 GA |
| **User Namespaces GA** | 容器级用户命名空间隔离，终于正式 GA（v1.25 开始 alpha） |
| **Declarative Validation GA** | 声明式验证 |
| **Fine-Grained Kubelet API Authorization GA** | 细粒度 kubelet API 鉴权 |

### 🔬 Beta / Alpha
| 特性 | 进展 |
|:-----|:------|
| **Mixed Version Proxy Beta** | 集群升级时通过代理安全处理多版本 API Server |
| **Workload-Aware Scheduling** | v1.35 引入 Workload API，v1.36 持续推进 |
| **Pod-Level Resource Managers (Alpha)** | 精细化单 Pod 级别资源管理 |
| **In-Place Vertical Scaling (Beta)** | Pod 级资源原地纵向伸缩 |
| **Tiered Memory Protection (Memory QoS)** | 分级内存保护 |
| **Deprecation of Service ExternalIPs** | 开始废弃 Service ExternalIPs 字段（安全隐患） |
| **PSI GA 证明了可忽略的性能开销** | Kernel PSI 开启开销仅 0.037–0.125 核; Kubelet 读取开销 ≤0.1 核 |
