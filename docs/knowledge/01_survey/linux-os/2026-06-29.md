### 2️⃣ 内核加固：allocation tokens + bootpatch-SLR 🛡️
- **来源**: LWN.net 2026-06-25, corbet
- **核心内容**:
  - **7.2 将合入**: 动态分配结构体的内存布局改变，使其更难被覆盖利用
  - **bootpatch-SLR (Structure Layout Randomization)**: 启动时随机化内核结构体布局，使漏洞利用更困难（时间线更长，仍在开发中）
  - 内核加固思路：即使有 bug，也要让 exploitation 尽可能困难
- **意义**: 消除所有内核 bug 不现实，主动防御（ASLR-like 的结构体布局随机化）是内核安全的长线方向
- **来源**: [LWN: Hardening the kernel with allocation tokens and bootpatch-SLR](https://lwn.net/Articles/1078699/)

### 5️⃣ Headlamp 插件生态大扩展 — Cluster API / Volcano / Knative 🎨
- **来源**: Kubernetes Blog 2026-06-25
- **三个新 Headlamp 插件同日发布**:

## 🛡️ 安全 & 供应链

### 7️⃣ Akrites 漏洞缓解项目启动 — Linux Foundation 🏢
- **来源**: LWN.net 2026-06-26, corbet / Linux Foundation
- **使命**: 将漏洞修复从"发布"加速到"部署"的全链路加速
- **核心设计**:
  - 机密性不可协商：未披露的漏洞在被修复前不公开
  - 修复流回每个项目自己的仓库
  - 当关键包无人维护时，Akrites 担任**最后维护者**
  - 与政府防御力量协调行动
- **背景**: AI 使漏洞反向工程和利用开发的速度大幅加快，"成功与否以部署速度衡量，而非发布速度"
- **来源**: [LWN: The Akrites vulnerability-mitigation project launches](https://lwn.net/Articles/1078803/)
