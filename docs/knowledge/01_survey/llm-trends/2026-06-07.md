### 1️⃣ OpenAI Lockdown Mode：抗提示注入的"锁定模式"

**来源**: [TechCrunch Jun 6](https://techcrunch.com/2026/06/06/openai-unveils-lockdown-mode-to-protect-sensitive-data-from-prompt-injection-attacks/)

OpenAI 宣布推出 **Lockdown Mode**（锁定模式），专门应对日益严重的 **提示注入攻击**（攻击者将恶意指令隐藏在网页/文档中，诱导模型泄露数据）。

**核心机制**：
- 禁用实时网页浏览（仅允许访问缓存内容）
- 禁止从网络检索和展示图片（仍可生成图片）
- 禁用 Deep Research 和 Agent Mode（代理模式）
- 保留基础对话和文件上传功能

**局限性**：OpenAI 坦诚即使开启 Lockdown Mode，缓存内容和上传文件中仍可能存在提示注入，无法100%防御。

**目标用户**："Lockdown Mode 不适用于所有人。它专为处理敏感数据、需要对数据外泄风险加强保护的个人和组织设计。"

**推出范围**：自服务 ChatGPT Business 账户 + 符合条件的个人账户，正在灰度中。

> 💡 这是**大模型商业部署安全性**的重要信号——随着Agent深度嵌入企业流程，提示注入已成为实际威胁，OpenAI 首次以产品化方式应对。

## 二、arXiv 论文亮点（6月4-5日提交）

### 5️⃣ CLSA（微软）：跨层稀疏注意力，128K 上下文 7.6× 解码加速

**论文**: [You Only Index Once: Cross-Layer Sparse Attention with Shared Routing](https://arxiv.org/abs/2606.06467)  
**作者**: Yutao Sun, Yanqi Zhang, Li Dong, Jianyong Wang, Furu Wei (Microsoft)

**核心贡献**：
- 基于 **KV 共享架构**（如 YOCO），不仅跨层共享 KV Cache，还**共享路由索引**
- 单个 indexer 一次性计算 token-level top-k 选择，将其结果**跨层复用**，大幅摊销路由开销
- 联合优化 **prefill + KV-cache 存储 + 长上下文解码** 三大瓶颈

**能力数据**：
- **7.6× 解码加速**（128K 上下文）
- **17.1× 整体吞吐提升**
- 短/长上下文评测均保持精度

> 💡 CLSA 将 token 级稀疏注意力的"路由成本"从 O(layers) 降到了 O(1)，是 KV 共享架构的重要演进方向。

### 6️⃣ Vortex：可编程稀疏注意力系统，4.7× MLA 吞吐

**论文**: [Vortex: Efficient and Programmable Sparse Attention Serving for AI Agents](https://arxiv.org/abs/2606.06453)  
**作者**: Zhuoming Chen, Xinrui Zhong, Qilong Feng, Ranajoy Sadhukhan, Yang Zhou, Michael Qizhe Shieh, Zhihao Jia, Beidi Chen (CMU)

**核心贡献**：
- Python 嵌入式前端 + page-centric tensor 抽象，**可编程定义各类稀疏注意力算法**
- 后端紧密集成 LLM Serving 栈，将理论效率转化为实际吞吐提升
- 支持 **AI Agent 自动生成和迭代**稀疏注意力算法

**能力数据**：
- MLA 架构（GLM-4.7-Flash）：**4.7× 吞吐提升**
- MiniMax-M2.7（229B 参数）：**1.37× 吞吐**（NVIDIA B200 GPU）
- AI Agent 自动生成算法：**3.46× 优于全注意力基线**

- 感知退化解耦降低 **35%**（vs 均匀 baseline）

> 💡 这是国产昇腾生态上**首个将 MLA 架构落地到实际金融业务**的论文，验证了 MLA 在端侧/行业场景的部署优势。

### 8️⃣ TARPO：Token 级自适应混合推理——离散 CoT ↔ 连续潜空间

**论文**: [TARPO: Token-Wise Latent-Explicit Reasoning via Action-Routing Policy Optimization](https://arxiv.org/abs/2606.05859)

**核心贡献**：
- 纯 RL 框架，**每步自适应切换**离散 token 生成 ↔ 连续潜空间推理
- 轻量 action head router 观察当前隐状态，采样路由决策
- LLM 主干 + router **端到端联合优化**（共享 group-relative advantage）

**能力数据**：
- 在 Qwen2.5（1.5B-7B）和 Llama-3.1-8B 上 **一致优于** 现有显式推理和潜推理 RL 基线
- 训练稳定性良好，学习到自适应 token 级切换策略

### 9️⃣ NF-CoT：归一化流潜空间推理——保留 KV Cache + 因果解码

**论文**: [Latent Reasoning with Normalizing Flows](https://arxiv.org/abs/2606.06447)

**核心贡献**：
- 在 LLM backbone 内实例化 **TARFlow-style 归一化流**，对蒸馏自显式 CoT 的紧凑连续思维建模
- 连续思维位置由 NF head 生成，文本位置由标准 LM head 生成，**同一因果流内**
- 保留 KV Cache 兼容性、概率采样、可解释似然估计三大优势
- 代码生成任务中准确率超越显式 CoT 和已有潜推理方法，**大幅降低中间推理成本**
