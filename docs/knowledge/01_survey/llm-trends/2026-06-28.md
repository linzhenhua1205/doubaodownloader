## 一、NVIDIA 开源 Nemotron-TwoTower：扩散语言模型架构突破

**来源**：arXiv:2606.26493 · NVIDIA Research · [代码和权重](https://github.com/NVIDIA/NeMo) 已开源

**核心贡献**：首次将扩散语言模型中"上下文表示"与"迭代去噪"两个角色解耦为独立双塔架构：

| 维度 | 细节 |
|:-----|:------|
| **架构** | 冻结的 AR 上下文塔（causal 处理 clean tokens）+ 可训练的扩散去噪器塔（双向 block attention，通过 cross-attention 读取上下文） |
| **基模** | Nemotron-3-Nano-30B-A3B（30B 参数，hybrid Mamba-Transformer MoE） |
| **训练数据** | ~2.1T tokens |
| **质量保留** | 98.7% AR 基线质量 |
| **生成吞吐** | **2.42×** wall-clock 加速 |
| **开源** | ✅ 代码 + 模型权重完全开放 |

**意义**：传统扩散LM用一个网络同时做上下文表示和去噪→双塔解耦后每个网络专注自身角色。这是扩散LM走向实用化的重要一步——吞吐接近AR模型的同时保留扩散模型的迭代生成优势。

## 二、SOLAR：跨语言推理的软Token对齐新方法

**来源**：arXiv:2606.26466 · Jiayi He, Jungsoo Park, Wei Xu, Alan Ritter

**问题**：多语言LLM对语义相同的prompt在不同语言下产生不一致的推理结果。原因是模型commit到离散输出token后，推理路径在不同语言间发散。

**SOLAR 方法**：在SFT阶段添加辅助目标，将各语言的**软token表示**（概率加权token embedding混合）在共享嵌入空间中对齐到英文表示。

**效果**：

| 指标 | 提升幅度 |
|:-----|:---------|
| 相对基模准确率 | **最高 +17.7 点** |
| 相对标准 SFT | **最高 +3.8 点** |
| 低资源语言 | 获益最大 |
| 语言集群可分性 | 显著降低 |
| 跨语言相似度 | 最终层显著增强 |

**方法启示**：软token是连续的、language-agnostic的信号；硬token是离散的、language-specific的信号。在对齐阶段保持"软"态，避免过早commit到特定语言的表达方式。

## 三、MemStrata：消除RAG过时事实错误——~0%错误率

**来源**：arXiv:2606.26511 · Neeraj Yadav

**关键发现**：RAG在知识演化的场景下，过时事实和最新事实的cosine相似度几乎无法区分（AUROC 0.59，接近随机）。RAG被迫服务过时值的频率为 **15-40%**。

**MemStrata 方案**：存储方式类似RAG，但当一个事实被新值"矛盾"时，通过确定性 (subject, relation, object) 超驰规则将旧值退休到一个**双时态分类账**中——**无需相似度阈值、不调用LLM**。

| 指标 | RAG | MemStrata |
|:-----|:---|:----------|
| 静态知识准确率 | 基线 | 持平 |
| 演化知识准确率 | 0.20-0.47 | **0.95-1.00** |
| 过时事实服务率 | 15-40% | **~0%** |
| 检索延迟 | — | **~2.1s** |

## 四、ConflictScore：量化LLM处理矛盾证据的能力

**来源**：arXiv:2606.26437 · Siyi Liu, Aaron Halfaker, Dan Roth, Patrick Xia

**核心指标**：将响应拆解为原子声明，逐条标注证据冲突情况，聚合为两个互补测量：
- **CS-C (Count)**：存在冲突的声明比例
- **CS-R (Ratio)**：支持与矛盾证据的平衡度

**贡献**：现有事实性指标只判断"被支持还是被矛盾"，无法捕捉**同时存在支持和矛盾证据**的场景。ConflictBench覆盖了ambiguity/contradiction/divergent opinions等多种冲突形式。可作为**纠正反馈机制**改善TruthfulQA的诚实性。
