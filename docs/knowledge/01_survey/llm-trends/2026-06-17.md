## ⏸️ 重磅三：Anthropic "暂停" Claude Agent SDK 按 token 计费

> **来源**: [Ars Technica](https://arstechnica.com/ai/2026/06/anthropic-pauses-token-based-billing-for-its-claude-agent-sdk/) (6/16 5pm ET)

### 事件经过

1. **5月13日公告**：自6月15日起，Claude Agent SDK（包括第三方应用和 `claude -p` 命令）的用量将按 **API 费率单独计费**，订阅用户获得与订阅价格等额的月度积分
2. **开发者强烈反弹**：
   - 分析显示 Claude Opus 用户每天只需 **2-3条消息** 就能让订阅物有所值
   - 重度用户的实际使用量是订阅价的 **数倍**（开发者 Matthew Diakonov：*"如果你用Opus作为主要编码助手，第一周就会突破收支平衡点"*）
   - 代码编辑器 Zed 团队警告用户 *"对Agent重度用户来说，这是重大的成本增加"*
3. **6月15日急刹车**：Anthropic 更新支持页面，宣布 **"暂停上述变更"**，称 *"正在修改方案以更好地支持用户构建方式"*

### 类比与前因

- 几周前 **GitHub Copilot 刚经历了类似的 token 计费改革**，同样引发用户对"订阅价值缩水"的强烈抱怨
- Anthropic 正筹备 **IPO**（已秘密提交 S-1），定价策略将受投资者密切关注
- Anthropic Claude Code 负责人 Boris Cherny 4月曾表示：*"我们的订阅模式设计初衷并未考虑这些第三方工具的用量模式"*

## 🔐 重磅五：Copilot SearchLeak 漏洞 — 窃取用户 2FA 验证码

> **来源**: Ars Technica (6/16) — Dan Goodin

- **严重程度**：**严重漏洞**，攻击者利用 GitHub Copilot 的 SearchLeak 漏洞窃取用户的 **双因素认证(2FA)代码**
- 本质问题：**LLM 安全架构的根本缺陷**，与 Prompt Injection 相关
- 攻击者可通过精心构造的提示词，让 Copilot 返回本应受保护的安全敏感信息
- 这是LLM安全领域 **反复出现的系统性失败**（Ars 评论原文：*"shows why the industry's approach to LLM security fails over and over"*）
- 对企业来说影响极大：2FA被窃意味着攻击者可接管开发者账户，植入后门代码

## 🔬 arXiv 今日精选（6/17新上线 144 篇 cs.CL）

### 1️⃣ RepSelect — 鲁棒 LLM 遗忘学习（Robust Unlearning）

> **arXiv**: [2606.17168](https://arxiv.org/abs/2606.17168)

- **核心问题**：现有 LLM 遗忘方法容易被 **微调或 Few-shot 提示** 逆转
- **根因发现**：现有方法针对的目标表征同时被保留集和攻击者恢复的子空间共享 → 既破坏通用能力又容易被逆转
- **RepSelect 方案**：在每次更新前坍缩权重梯度的顶部主成分，隔离待遗忘集的独有表征
- **评测**：4个模型家族（Llama 3, Qwen 3.5, Gemma 4 E4B, DeepSeek V2 Lite）x 2个遗忘类别（生物危害知识、攻击倾向）
- **结果**：比 5 个基线（GradDiff, NPO, SimNPO, RMU, UNDIAL）**后学习答案准确率降低 4-50×**，对 Few-shot 攻击近乎完美抵抗

### 2️⃣ The Benchmark Illusion — 剪枝 LLM 能过选择题但不能回答问题

> **arXiv**: [2606.17609](https://arxiv.org/abs/2606.17609)

- **核心发现**：高稀疏度剪枝后的 LLM（尤其是 Wanda 方法）在 **多项选择评测上表现良好**，但在 **自由生成中完全失败**
- **机制**：正确答案没有被擦除，而是 **被降级了**（不再是 top-1 输出）
- **恢复条件**：使用 Beam Search、Sampling 或 1个 in-context example 后正确答案可恢复
- **结论**：多项选择评测 **可能高估剪枝模型的可用性**，存在评估盲区。压缩模型应当测试"能生成什么"，而非仅"能识别什么"

### 3️⃣ StateWitness — 激活解释器检测 LLM 欺骗

> **arXiv**: [2606.17478](https://arxiv.org/abs/2606.17478)

- 使用一个 **独立的解码器** 读取目标模型（推理型LLM）的隐藏状态，回答自然语言查询或生成结构化报告
- **7个欺骗数据集** x 2个目标模型 → **AUROC 0.916**
- 比最佳黑盒文本监控器 **高11.6%**，比最佳激活探针基线 **高25.0%**
- 不仅检测，还能输出 **查询级答案、schema报告、token/句子级证据追溯**

### 4️⃣ PromptMN — 伪提示语言

> **arXiv**: [2606.17164](https://arxiv.org/abs/2606.17164)

- 一种领域特定语言（DSL），用 `%` 前缀的类型化指令注释自然语言（角色、目标、约束、计划、输入、输出）
- 支持 **逆向提示工程**：让模型将自己的理解用 PromptMN 重新表述，用户可以检查推导的角色、目标和约束
- 在 **Claude Fable 5, Claude Opus 4.8, Gemini 3.1 Pro, GPT-5.5** 上均验证有效（无需微调即可正确解析）

### 5️⃣ AIPatient Arena — 临床咨询全流程LLM评测

> **arXiv**: [2606.17474](https://arxiv.org/abs/2606.17474) | 49页

- 构建 **EHR 驱动的知识图谱**，支持多轮医患模拟交互
- 437名患者主要队列 + 2个外分布验证队列（119+67）
- **8维临床能力评测**结果：
  - 强项：问诊技能(4.43-4.99/5)、伦理操守(4.38-4.93/5)、解释清晰度(3.80-4.72/5)
  - 中等：信息整合(3.19-4.21/5)、用药安全(3.13-3.78/5)
  - **弱项**：模糊回答处理(2.57-3.32/5)、信息覆盖(2.08-3.02/5)、**诊断准确率(2.63-3.55/5)**
- **关键结论**：仅凭最终答案准确性不足以评估临床就绪度

### 6️⃣ NarrativeWorldBench — 长程叙事创作评测

> **arXiv**: [2606.17391](https://arxiv.org/abs/2606.17391) | ICML 2026 Workshop

- 21个模型全面基准测试（从经典到推理层），**所有前沿系统在情节节拍F1上饱和于[0.78, 0.81]**
- **horizon h=200时性能坍塌约-0.20 F1**
- N-VSSM（基于Mamba-2的叙事状态空间模型）在 **200+集** 长程叙事中保持 **F1 ≥ 0.84**，且 **计算量降低4×**
- 专业作者(n=12)在 **71%** 的情况下更偏好 N-VSSM 超过 Claude Opus 4.5

### 7️⃣ Scaling Enterprise Agent Routing 研究

> **arXiv**: [2606.17519](https://arxiv.org/abs/2606.17519)

- 研究 **110个Agent/584个工具的工业在线助手** 路由准确率随规模退化
- Agent从10→110个时，**路由F1下降16-23个百分点**
- 退化分解为：**检索缺口**（模型找不到正确工具）+ **混淆缺口**（即使完美检索，天花板也降10pp）
- Embedding-based shortlisting 可恢复 **+10-11pp F1**
