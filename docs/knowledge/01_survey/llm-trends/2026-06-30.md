### 1. Agents-A1：35B MoE Agent 模型达到万亿参数级性能——"扩展视野而非参数"

**来源**: arXiv:2606.30616 · 上海 AI Lab/南洋理工大学等 50 位作者 · [代码+权重已开源](https://github.com/Agents-A1)

| 维度 | 细节 |
|:-----|:------|
| **模型** | Agents-A1, 35B MoE (Mixture-of-Experts) Agentic Model |
| **对比对象** | 1T 参数模型：Kimi-K2.6, DeepSeek-V4-pro |
| **关键洞察** | **Scaling the Horizon, Not the Parameters**——通过扩展 Agent 的"视野"(horizon)而非参数规模来提升性能 |

**核心方法论——三阶段训练配方**:

| 阶段 | 方法 | 目的 |
|:---:|:-----|:-----|
| 1️⃣ | **全领域 SFT** | 对齐基础模型以覆盖广泛的 Agent 行为 |
| 2️⃣ | **领域级 Teacher 模型** | 在每个专业领域训练专家 Teacher，捕获领域专长 |
| 3️⃣ | **多Teacher领域路由在线蒸馏**（含显著词汇对齐） | 将6个异构领域统一为1个可部署 Student 模型，提高知识迁移效率 |

**关键基础设施**: 构建了长视野**知识-行动基础设施**，连接外部知识、行动、观察和验证器结果，生成平均长度 **45K tokens** 的 Agent 轨迹。

**性能对比 (Agents-A1 vs 1T 模型)**:

| 基准 | Agents-A1 (35B) | Kimi-K2.6 / DeepSeek-V4-pro (1T) | 对比 |
|:-----|:---------------:|:---------------------------------:|:----:|
| **SEAL-0** | **56.4** | ~1T 基线 | ✅ 领先 |
| **IFBench** | **80.6** | ~1T 基线 | ✅ 领先 |
| **HiPhO** | **46.4** | ~1T 基线 | ✅ 领先 |
| **FrontierScience-Olympiad** | **79.0** | ~1T 基线 | ✅ 领先 |
| **MolBench-Bind** | **56.8** | ~1T 基线 | ✅ 领先 |
| **SciCode** | 44.3 | ~1T 基线 | ⚡ 有竞争力 |
| **HLE** | 47.6 | ~1T 基线 | ⚡ 有竞争力 |
| **BrowseComp** | 75.5 | ~1T 基线 | ⚡ 有竞争力 |

**意义**: 这是"参数规模竞赛"路线的有力反驳——**用 1/30 的参数达到同等甚至更优的 Agent 任务性能**。证明了 Agent 化的数据构建策略（长轨迹 + 多领域蒸馏）可以改变 scaling 格局。完全开源进一步降低了研究者参与门槛。

### 2. Google 限制 Meta 的 Gemini 用量——AI 基础设施瓶颈暴露

**来源**: Financial Times · The Verge (Jun 28-29)

**事件**: Google 因无法满足计算需求，开始限制 Meta 对其 Gemini 模型的使用量。

**核心细节**:
- Meta 越来越依赖 Gemini 用于多种需求
- Google 告诉 Meta 及其他客户——**无法提供他们想要的计算容量**
- 原因：尽管已投入**数百亿美元**建设芯片、数据中心和电力设施，仍无法满足激增的 AI 模型和服务需求
- 标志着 AI 行业的基础设施压力已从"创业公司缺卡"蔓延到**大型科技公司之间也缺卡**

**行业影响**:
- GPU 算力成为**战略性稀缺资源**，影响范围远超芯片采购层面
- 云厂商面临"服务自己 vs 服务客户"的权衡
- 长期来看，可能加速自研芯片（如 OpenAI Jalapeño、Google TPU）和去 Google 化进程

## 📄 arXiv 论文精选 (Jun 29-30)

### 🎯 新模型与方法

#### 1. MOPD: 多Teacher在线蒸馏——能力整合新范式
- **arXiv:2606.30406** · Wenhan Ma et al.
- **核心**: Multi-Teacher On-Policy Distillation——在 LLM 后训练阶段，从多个 Teacher 模型蒸馏能力到一个 Student
- **关键创新**: 不是静态蒸馏，而是**在线策略蒸馏**，让 Student 在生成时从 Teacher 获取动态指导
- **意义**: 为"能力整合"提供了系统化方法——不再需要单一超大模型覆盖所有能力

#### 2. CORTEX: 本体语料图实现 Web 级语料结构化组织
- **arXiv:2606.30175** · Chengtao Gan et al.
- **核心**: 三层异构图（质量精炼内容层 + LLM驱动本体层 + 跨域对齐层）
- **关键数据**: 24.14B token 精炼语料 + 跨域搜索推理基准 CortexBench
- **意义**: 从"平面文档过滤"升级到"结构化知识组织"——对预训练数据处理有潜在重大影响

### 🧪 能力评测

#### 3. LLM-as-a-Judge 在 Agent 场景下的可靠性
- **arXiv:2606.29920** · Yangda Peng et al. (THU-KEG)
- **RuVerBench**: 首个评估 LaaJ 在 Agent 场景中校验能力的数据集 — 2,458 实例覆盖 deep research 和 agentic coding
- **关键发现**: 
  - 最强模型仍有**显著噪声**
  - 弱模型对 Prompt 设计更敏感
  - Batch 校验存在准确率-效率权衡
  - Majority voting 边际效益递减

#### 4. 临床推理图：LLM 诊断推理有"能力"无"一致性"
- **arXiv:2606.29876** · Nisarg A. Patel (UCSF) · ICML 2026 Spotlight
- **核心**: LLM 在复杂临床案例上达 **60-70% 诊断准确率**，但图谱分析显示——同一模型对相似病例的推理结构几乎不相似
- **关键数据**: 750 条追踪记录，5 个模型，50 个 NEJM 病例
  - 正确同组 vs 错误同组图谱相似度几乎相同 (0.488 vs 0.484)
  - 结构化反思提示虽增加特征分析 (+33%)，但**不增加跨案例一致性**
- **意义**: 准确率仅覆盖能力维度——**推理一致性和稳定性是一个独立维度**，需要额外的评估框架

#### 5. On-Prem Open LLMs 的 Text-to-SQL 表现
- **arXiv:2606.29733** · Vladimir Beskorovainyi
- **BIRD 开发集 (n=1534, Execution Accuracy)** 上的系统评测
- **四大发现**:
  - ① 代际差异 > 参数规模：Qwen2.5-Coder 7B (39.1) 碾压 CodeLlama 7B (20.9)
  - ② **自我纠错 (Self-correction)** 是稳健的免费午餐——所有模型族均显著受益
  - ③ **Schema Linking 无效**——96.5% 召回率的链路器与无链路器无统计差异
  - ④ Self-consistency 性价比极差——0.13pp 提升需 ~5× token 开销
