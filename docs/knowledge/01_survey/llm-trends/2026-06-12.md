### 3️⃣ MiniMax 发布稀疏注意力（MSA）— 109B 模型 28.4× 计算降低

**来源**: arXiv:2606.13392 | MiniMax | 2026-06-11

MiniMax Sparse Attention (MSA) — **生产级稀疏注意力架构**，核心亮点：

| 指标 | 数据 |
|:-----|:-----|
| 模型规模 | **109B 参数**，原生多模态训练 |
| 1M 上下文 | 注意力计算降低 **28.4×** vs GQA |
| H800 实测加速 | Prefill **14.2×**，Decoding **7.6×** |
| 设计 | 基于 GQA 的分块稀疏注意力，Index Branch 独立选择 Top-k KV 块 |
| 推理内核 | **已开源**（GitHub），模型已公开发布 |

> 💡 **意义**：这是**首个在 100B+ 级生产模型上验证的稀疏注意力方案**，证明「稀疏注意力 = 真的快」在工业级规模成立。与之前大多停留在学术实验的稀疏注意力研究形成鲜明对比。

### 4️⃣ 新型论文：Recursive Agent Harnesses — 递归 Agent 架构的实证突破

**来源**: arXiv:2606.13643 | 2026-06-12（当天）

研究团队命名了 **递归 Agent 框架（Recursive Agent Harness, RAH）** 范式：
- 父 Agent 生成子 Agent 脚本，子 Agent 拥有完整文件系统+代码执行+规划能力
- 核心贡献：**框架递归**（harness recursion）延伸了模型递归（model recursion）思路

**评测数据**（Oolong-Synthetic，最高 4M token）：

| 配置 | 准确率 |
|:-----|:------:|
| GPT-5 (Codex baseline) | 71.75% |
| GPT-5 + RAH | **81.36%** (+9.61pp) |
| Claude Sonnet 4.5 + RAH | **89.77%** |

> 💡 **关键**: RAH 的增益来自**框架设计本身而非模型**（Control 实验：同模型 +RAH vs 无 RAH），印证了 Anthropic Dynamic Workflows 的工程方向。约 10pp 的提升在 Agent 推理中是显著跨越。

### 5️⃣ 「多Agent优势」是幻觉？— 最新 arXiv 论文引发反思

**来源**: arXiv:2606.13003 | 2026-06-12

**核心发现**（打脸行业共识）：

- **自动生成的 MAS（多Agent系统）在标准推理和交互式多步任务上持续不如 CoT-SC（思维链+自一致性）**，尽管成本高出 **10×**
- 专家设计的 MAS 在专用任务上确实优于自动生成架构，但成本效益分析揭示：**自动 MAS 存在系统性的架构臃肿**，表面复杂性与功能实用性脱节
- 诊断性合成数据集证明：现有评估框架未能考虑**增量计算成本的边际效用**

> 💡 **启示**: 多Agent ≠ 更好。当 Agent 系统变得流行时，这篇论文为「是否需要多Agent」提供了关键修正视角。设计原则：**从简单开始（CoT-SC），证明需要复杂化再升级**。
