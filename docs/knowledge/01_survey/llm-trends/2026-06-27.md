### 1. OpenAI GPT-5.6 正式发布：Sol / Terra / Luna 三款模型，美国政府要求限制发布范围

- **来源**: TechCrunch (2026-06-26), The Verge (2026-06-26), OpenAI Blog
- **概述**: OpenAI 于 6 月 26 日正式发布 GPT-5.6 系列，包含三个子模型：

| 模型 | 定位 | 输入价格 (per M tokens) | 输出价格 (per M tokens) |
|:-----|:-----|:----------------------:|:----------------------:|
| **Sol** | 旗舰最强 | $5 | $30 |
| **Terra** | 日常平衡 | $2.5 | $15 |
| **Luna** | 快速低成本 | $1 | $6 |

- **GPT-5.6 Sol 能力亮点**:
  - 与 Anthropic Claude Mythos 5 竞争，**使用约 1/3 的输出 tokens 即达到可比质量**
  - 在编码工作流评测中**略优于** Mythos 5
  - 引入 **"max" 推理努力模式**和 **"ultra" 模式**（协调子 Agent 解决高度复杂任务）
  - Agentic 能力大幅提升，覆盖 coding / biology / cybersecurity 领域
- **安全性设计**:
  - **安全 guardrails 直接内建于核心模型行为**中，而非依赖外部过滤器
  - 刻意规避了 Anthropic Fable 5 的问题（模型过度谨慎导致请求被降级到旧模型）
  - 针对对抗攻击进行了加固，优先守卫防御性而非进攻性安全任务
- **美国政府限制发布**:

  - OpenAI 公开表态："我们不认为这种政府访问流程应成为长期默认做法"
  - 称这是"短期步骤"，预计"未来数周"内更广泛开放
  - 正在与政府合作制定新的行政令框架和"可重复的未来模型发布流程"
- **背景**:

## 📄 arXiv 论文精选（Jun 25-26）

### 🎯 新模型发布

#### 1. Nemotron-TwoTower：NVIDIA 开源扩散语言模型——2.42× 吞吐，保留 98.7% 质量

- **作者**: Reda, Kamalu, Waleffe, Patwary, Shoeybi, Catanzaro (NVIDIA)
- **时间**: 2026-06-25
- **核心创新**: **解耦式双塔架构**——冻结的 AR 上下文塔（处理干净 token）+ 可训练的扩散去噪器塔（双向块注意力 + 交叉注意力到上下文）
- **基础模型**: Nemotron-3-Nano-30B-A3B（30B 参数，Hybrid Mamba-Transformer MoE）
- **训练数据**: ~2.1T tokens
- **效果**:
  - **质量保留**: 达到自回归基线 **98.7% 的质量**
  - **吞吐提升**: **2.42×** 更高的 wall-clock 生成吞吐
  - **完全开源**：代码 + 模型权重
- **意义**: 证明了扩散 LM + 预训练 AR 模型组合的可行性，为生成速度与质量平衡提供了新路径

#### 2. Dynamic-dLLM：无需训练的扩散 LLM 加速——3× 推理加速

- **作者**: Wu et al. (2026-06-25)
- **核心**: Dynamic Cache Updating (DCU，自适应分配缓存更新预算) + Adaptive Parallel Decoding (APD，动态校准解码阈值)
- **效果**: 在 LLaDA-8B-Instruct / Dream-v0-7B-Instruct 等模型上，**平均 3× 加速**，在 MMLU / GSM8K / HumanEval 上不牺牲性能
- **注意**: 插拔式方案，无需重新训练

### 🧪 能力评测与理解

#### 3. Where Larger Models Excel: Constraint-Guided Reasoning

- **作者**: Lin & Huang (2026-06-25)
- **核心**: 系统性地量化大模型推理优势——Qwen3-32B 比 8B 平均高 **6.43%**，GPT-OSS-120B 比 20B 高 **7.38%**
- **核心发现**: 大模型的关键优势是**约束引导推理(Constraint-Guided Reasoning)**——能更好地识别显式和隐式约束、组织结构化推理、用约束排除不可行路径并验证中间步骤
- **方法论**: 提出 AdvCluster 框架，自动识别大模型稳定优势的问题并提取细粒度优势描述

#### 4. Think Like a Scientist? LLM 研究方法推荐的系统偏差

- **作者**: Carlon et al. (2026-06-25)
- **核心**: 对 1,000 篇 arXiv CS 论文，比较 GPT-5.1 / Gemini 3 Pro / DeepSeek-V3.2 推荐的方法与论文实际使用的方法
- **关键发现**:
  - LLM 推荐的方法多样性**严重缩水**——有效模型数从论文的 1,232 个压缩到 59-96 个
  - **提供者选择偏差最大**：Jensen-Shannon 散度比其他维度高 3-5×
  - LLM 间排名相关性 (0.55-0.68) 高于 LLM-论文相关性 (0.33-0.56) ——意味着偏差在模型间是共享的
- **实践意义**: 依赖 LLM 做研究方法建议而不交叉验证，会显著收窄方法论探索空间

#### 5. ConflictScore: 衡量 LLM 处理冲突证据的能力

- **作者**: Liu et al. (2026-06-25)
- **核心**: 现有正确性指标无法捕获"证据同时支持与反对"的场景。ConflictScore 分解为 CS-C（冲突比例）和 CS-R（支持/反对平衡）
- **效果**: 可作为纠正反馈机制，提升 TruthfulQA 上的真实性

### 🛠️ Agent 与系统

#### 6. ContextForge: 长对话上下文回收——降低推理 token 开销

- **作者**: Thomas (2026-06-25)
- **核心**: 结构化查询生成 + 外部记忆检索 + 受控合成，**无需完整上下文重放**
- **效果**: 15 轮医疗对话测试中，token 消耗降低的同时保持回答质量

#### 7. MemStrata: 消除 RAG Agent 的过期事实错误

- **作者**: Yadav (2026-06-25)
- **核心**: RAG 没有时间模型，过时事实与新事实的 embedding 相似度 AUROC 仅 0.59（接近随机）。MemStrata 引入确定性主-谓-宾接替规则 + 双时态账本
- **效果**: 在演进知识上准确率 0.95-1.00（RAG 仅 0.20-0.47），过时事实错误率降到 ~0%（RAG 15-40%），延迟仅 ~2.1s（vs LLM 重排序 16-18s）

#### 8. ProfileFoundry: 10 万合成人对象数据集——用于 Agent 记忆/隐私/工具使用评测

- **作者**: Selvam & Ghosh (2026-06-25)
- **核心**: 确定性生成器，100,000 合成 Person Objects，包含 709,228 事件、40,338 家庭、52,491 雇主，跨 8 个地区
- **意义**: 为 LLM Agent 的记忆、隐私、文档理解、记录链接等评测提供可控的合成数据层
