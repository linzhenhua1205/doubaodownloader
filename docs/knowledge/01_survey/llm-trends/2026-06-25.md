## 🔬 arXiv 论文精选

### ⭐ JetSpec: 并行草稿树突破推测解码天花板

- **链接**: [arXiv:2606.18394](https://arxiv.org/abs/2606.18394) (cs.CL)
- **核心**: 提出 **JetSpec**，一种 head-based 推测解码新框架，将单向前向推导效率与分支级因果条件化结合
- **创新**: 在冻结的目标模型上训练因果并行草稿头，使候选树与目标模型的**自回归因子分解对齐**
- **效果**: 在 **Qwen3** (稠密 + MoE) 上，MATH-500 达 **9.64×** 加速，对话负载达 **4.58×** 加速
- **意义**: 突破了推测解码的 scaling ceiling—更大草稿预算转化为更长接受前缀和更高端到端加速

### ⭐ PhoneBuddy: 开放模型手机 Agent 训练的实战配方

- **链接**: [arXiv:2606.23049](https://arxiv.org/abs/2606.23049) (cs.CL)
- **核心**: 构建 PhoneBuddy 训练管线，结合真实 App 环境 + 模拟 App 环境 PhoneWorld
- **关键发现**: 混合 RL（真实+模拟） > 纯真实 RL > 纯 SFT
  - 150 任务真实手机评测: 45.33% (混合 RL) vs 40.67% (纯真实 RL) vs 36.67% (SFT)
  - AndroidWorld: **83.2%** (混合 RL)
- **意义**: 证明模拟环境不是真实 RL 的替代品，而是**可扩展、可重置、可自动检查**的互补来源

### ⭐ ConPress: 多问题上下文压力诱导 LLM 自我压缩

- **链接**: [arXiv:2602.01472](https://arxiv.org/abs/2602.01472) (cs.CL)
- **核心发现**: 当一个 prompt 中有**多个可回答的独立问题**时，LRM 自发产生更短的推理链——作者称之为 **Self-Compression**
- **方法**: ConPress 构建多问题 prompt → 采样 → 过滤正确且简洁的轨迹 → SFT 内化
- **结果**: 仅 8k fine-tuning 样本，**推理 token 减少 59% (MATH500) 和 33% (AIME25)**，精度持平
- **意义**: 无需外部教师、无需人工剪枝、无需 RL 的轻量级推理压缩方法

### 📊 Scale or Reason? 推理蒸馏的计算等价分析

- **链接**: [arXiv:2509.22193](https://arxiv.org/abs/2509.22193) (cs.CL)
- **核心**: 在**等计算预算**条件下比较推理蒸馏 vs SFT 扩展
- **关键发现**:
  - IFT 在大多数配置下位于/接近 **Pareto 前沿**
  - 推理蒸馏只在 7B+ 模型和开放式任务上达 Pareto 前沿
  - 混合 25-50% 推理数据 + SFT 可以最低成本获取大部分收益
- **警示**: 选择推理蒸馏意味着隐式放弃了在同计算预算下训练更大 IFT 模型的机会

### 📊 混合线性注意力系统分析 (72 模型全因子实验)

- **链接**: [arXiv:2507.06457](https://arxiv.org/abs/2507.06457) (cs.CL)
- **核心**: 训练 72 个模型（36×340M + 36×1.3B），覆盖 6 种线性注意力变体 × 5 种混合比例
- **关键发现**:
  - 独立表现好的线性模型**不一定是好混合候选**
  - recall 在 full attention 比例 > 25% (即线性:满注意力 ≤ 3:1) 时显著提升
  - 推荐组合: **HGRN-2 或 GatedDeltaNet** + 线性:满比例 3:1 ~ 6:1

### 📊 Agent Harness Scaling Laws: 有效反馈计算

- **链接**: [arXiv:2605.29682](https://arxiv.org/abs/2605.29682) (cs.CL)
- **核心**: 提出 **Effective Feedback Compute (EFC)**，区分有用/冗余/不稳定反馈的 trace 级 scaling 坐标
- **结果**: NRS-EFC/D_task 在真实 traces 上 R²=0.93（原始计算 R² 近零或负）
- **应用**: EFC 作为协作控制层，平均 pass rate 从 61.2% → **68.2%**，成本从 213.8 → **85.1**
