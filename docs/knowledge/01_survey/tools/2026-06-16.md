## 1️⃣ GitHub — Copilot 用量指标接入服务端遥测，DAU 覆盖 +~5% 📊

- **来源**: [GitHub Changelog (Jun 15, 2026)](https://github.blog/changelog/2026-06-15-copilot-usage-metrics-now-include-more-of-your-active-users/)
- **工具**: GitHub Copilot Enterprise / Usage Metrics API
- **标签**: `copilot` `usage-metrics` `telemetry` `enterprise` `dau`

**核心内容**:

Copilot Enterprise 用量指标报告现在在客户端遥测之外，**额外接入服务端遥测**，以覆盖客户端数据丢失造成的活跃用户漏报。

**问题背景**:
- 此前 Copilot 用量完全依赖 IDE 等客户端的**客户端遥测**（Client-side Telemetry）
- 网络条件、代理配置、客户端设置等因素会阻止客户端上报活动数据
- 结果是**一个活跃的、已计费的用户可能从报告中消失**

**解决方案**:
- 新增**服务端遥测**（Server-side Telemetry）信号源，识别客户端未能上报的活跃用户
- 两种信号源**合并输出**，客户端已捕获的用户不变，**新增的纯服务端用户被补充到 DAU 计数中**

**量化影响**:

> 示例：原企业单日报告显示 **1,000 DAU**，更新后约显示 **1,050 DAU**。新增的 50 人是服务端确认活跃但客户端从未上报的用户。

**当前限制**：
- 这些新增用户被完全识别并计入活跃总数
- 但服务端遥测**尚未提供** IDE、功能特性、模型、代码行数等细粒度维度数据
- 因此这些用户的 `totals_by_ide` 和 `totals_by_feature` 维度暂时为空
- GitHub 表示这是**第一步**，后续版本会逐步为这些用户填充细粒度数据

**价值判断**:
- 这是 Copilot Enterprise ROI 核算和合规审计的**基础设施完善**——缺失的 DAU 会导致管理层低估 Copilot 的实际采用率
- 双源遥测设计使报告对**网络问题、代理配置**等不可控因素具有弹性
- 纯技术角度看，这标志着 GitHub 在大规模 AI 工具采用的**可观测性基建**上迈出了重要一步

**可用性**: 已在 Enterprise Cloud 客户可用

## 2️⃣ JetBrains CLion — C++26 全面支持 + Constexpr Debugger + C++20 模块自动导入 🦾

- **来源**: [JetBrains CLion Blog (Jun 2026)](https://blog.jetbrains.com/clion/2026/06/modern-cpp-support/)
- **工具**: JetBrains CLion (CLion Nova 引擎)
- **标签**: `jetbrains` `clion` `cpp26` `constexpr-debugger` `modules` `cpp`

**核心内容**:

CLion 发布了一篇综合性博客，汇总了最近版本（CLion 2026.1+，CLion Nova 引擎）中语言引擎的重要改进。以下是具体的技术更新：

### C++26 特性全面支持（除 reflection）
| 特性 | 说明 | 意义 |
|:-----|:-----|:-----|
| **`#embed`** 预处理指令 | 编译时将二进制文件（图片/图标等）直接嵌入为字节数组 | 消除外部脚本/自定义工具的二进制转 C 数组步骤 |
| **Pack indexing** | `values...[0]` 语法直接索引参数包元素 | 消除递归模板特化或辅助工具类 |
| **Variadic friends** | `friend Ts...;` 将友元授予包中所有类型 | 消除逐个声明友元的繁琐 |
| reflection | 计划在 **2026.2** 中实现 | 唯一未支持的主要 C++26 特性 |

### Constexpr Debugger 改进
- **首次 IDE 内置 Constexpr 调试器**（CLion 2025.3 引入）：
  - 逐步执行 `constexpr`/`consteval` 编译期评估
  - 检查局部变量、导航调用栈、**向后逐步执行**
- **新增**：IDE 检测 constexpr 评估失败，弹出评估轨迹提示，一键点击 **"Run evaluation until failure"** 直接用 Constexpr Debugger 停在失败点
- **扩展评估范围**：新增支持 `switch` 语句、带初始化器的 `if`、结构化绑定、`defaulted operator==` 等

### C++20 模块自动导入
- 使用已导出符号但缺少对应 `import` 声明时，CLion 建议**自动插入 import 语句**
- 目前支持从主模块接口单元和模块分区直接导出的符号

### 新增内置检查（Inspections）
| 检查项 | 行为 |
|:-------|:-----|
| **重复前向声明** | 灰化标注 + quick-fix 一键删除 |
| **C++20+ 中不必要的 `typename`** | 检测 + 删除修复 |
| **Designated initializer 顺序错误** | 直接在编辑器中标注顺序问题 |
| **覆盖函数访问级别不匹配** | 基类 public virtual 被派生类 private override 时告警 |

### 其他增强
- **不活跃代码块中的代码分析/补全/重构**：在 `#ifdef` 等不活跃预处理器块中，代码分析、补全和本地重构已完全可用
- **自动函数定义排序**：按头文件声明顺序自动排序源文件中的函数定义，支持单函数/文件/项目级执行
- **可编辑命名规则**：新的 Edit Naming Rule 对话框支持配置命名模式、前缀、后缀
- **GCC/Clang 扩展支持**：嵌套函数（GCC）、`_Nullable`/`_Nonnull` 限定符（Clang）、`x ?: y` 省略中间操作数简写、Designated initializer range 语法

**价值判断**: 这一批更新代表了 CLion Nova 引擎（2025.3 成为默认引擎）持续成熟的过程。C++26 `#embed` 和 pack indexing 虽非日常高频使用特性，但 C++20 模块自动导入和 Constexpr Debugger 对现代 C++ 项目的**日常开发效率有直接提升**。自动函数定义排序和命名规则编辑对大型 C++ 代码库的**代码规范一致性**意义明确。
