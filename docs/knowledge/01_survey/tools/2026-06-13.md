## 1️⃣ Cursor — Bugbot 3× 更快、22% 更便宜、多找 10% 的 Bug 🚀

- **来源**: [Cursor Changelog — Bugbot Performance (Jun 10, 2026)](https://www.cursor.com/changelog)
- **工具**: Cursor
- **标签**: `cursor` `bugbot` `code-review` `composer-2.5` `performance`
- **核心更新**:

Cursor Bugbot（代码审查 Agent）迎来 Composer 2.5 驱动的重大性能升级：

**性能对比 (before → after)**

| 指标 | 之前 | 之后 | 提升 |
|:-----|:----:|:----:|:----:|
| 平均审查时间 | ~5 分钟 | **~90 秒** | **3.3×** |
| 每次审查发现 Bug 数 | 0.56 | **0.62** | **+10.7%** |
| 单次运行成本 | 基准 | -22% | **↓22%** |

**新功能：`/review` 推前审查**
- 现在可在代码推送前通过 `/review` 运行 Bugbot + Security Review
- `/review-bugbot` 和 `/review-security` 可直接指定 Agent
- `/review` 与 **GitHub 和 GitLab Bugbot 同步**：如果本地已审查同一 diff，PR 阶段自动跳过并注释"已审查过"
- 可配置为**只审查上次审查以来的新代码**，聚焦最新变更

**可用范围**: Cursor 3.7+ 桌面端 + cursor.com/agents（CLI 即将支持）

**价值判断**: 这是 Composer 2.5 在 Code Review 场景的首次实战验证。90 秒的平均审查时间使"推前审查"成为可行工作流——不再打断开发者的 commit 节奏，同时保持审查质量。Bugbot 与 GitHub/GitLab 的 diff 同步意味着**审查不再是线性的"每次都跑"**，而是有状态、增量的持续过程。

## 2️⃣ Linear — Coding Sessions：Agent 从规划到编码的闭环 🎯

- **来源**: [Linear Changelog — Coding Sessions (Jun 11, 2026)](https://linear.app/changelog/2026-06-11-coding-sessions-in-linear)
- **工具**: Linear
- **标签**: `linear` `coding-agent` `agent-workflow` `plan-to-code`
- **核心更新**:

Linear Agent 现在可以使用 **Claude Code 和 OpenAI Codex** 直接编写代码，实现了从立项→规划→编码→审查→交付的完整闭环。

**能力矩阵**:

| 能力 | 说明 |
|:-----|:------|
| **Cloud 编码会话** | 无需搭建本地开发环境，Agent 在云端运行编码任务 |
| **Issue→PR 自动化** | 将 Issue 分配给 Agent，或在 Chat/Comment/Slack 中要求修改 |
| **Diff 预览** | Agent 完成后返回差异，可在 Linear 内审阅迭代 |
| **自动第一轮修复** | 新 Issue 提交后，Agent 自动调查 + 尝试修复 |

**性能数据**: Linear 内部使用此工作流，**约 30% 的新增 Bug 报告在第一轮自动化中解决**。

**自动化流程**:
1. 新 Bug 被提交 → Triage
2. Agent 通过 MCP 连接 Sentry/Datadog 等工具收集证据
3. 检查代码库追溯根本原因
4. 生成修复 → 返回 Diff → 通知团队审阅

**定价**: 适用于 Basic / Business / Enterprise 计划（需 GitHub 代码访问 + AI Credits）

**价值判断**: 这是业界首个**从 Issue 到代码的全链路闭环**。与 Cursor/GitHub Copilot 等 IDE 内 Agent 不同，Linear Agent 的起点是**问题定义**而非代码文件——它主动调查、归因、修复，完全不需要开发者切换上下文。30% 的一轮修复率意味着大量低复杂度 Bug 可以被 Agent 异步解决。

## 3️⃣ GitHub — Agentic Workflows 公开预览：自然语言定义 Agent 流水线 🆕

- **来源**: [GitHub Changelog — GitHub Agentic Workflows Public Preview (Jun 11, 2026)](https://github.blog/changelog/2026-06-11-github-agentic-workflows-is-now-in-public-preview/)
- **工具**: GitHub Actions / GitHub Copilot
- **标签**: `actions` `copilot` `agentic-workflow` `security-first` `public-preview`
- **核心更新**:

GitHub Agentic Workflows 进入公开预览——用自然语言 Markdown 文件定义 Agent 驱动的自动化，由 GitHub Actions 编译执行。

**核心能力**:
- **自然语言定义**: 用 Markdown 文件描述 Agent 任务（Issue 分类、CI 失败分析、文档更新等），自动编译为标准 Actions YAML
- **复用现有基础设施**: 直接使用现有的 Runner 组、策略约束和自托管 Runner
- **跨仓库操作**: 支持跨越多个仓库的变更

**安全架构（Security-first by design）**:

```
Agentic Workflow
     │
     ├─ 完整性过滤 (Integrity Filter) — Agent 访问的代码内容
     ├─ 默认只读权限 — Agent 默认无写入权限
     ├─ 沙箱容器 (Sandboxed Container)
     ├─ Agent Workflow Firewall — 专用防火墙
     ├─ Safe Outputs — 输出安全验证
     └─ Threat Detection — 威胁检测扫描所有变更
```

**引用**: "让 Agent 开 PR 从来不是难事。**信任它能被合并才是**。" — May Walter, CTO at Hud.io

**可用性**: 通过 CLI 扩展安装后即可使用。[GitHub Next Agentics 仓库](https://github.com/github-next/agentics) 提供了预建工作流（Triage、报告、合规等）。

**价值判断**: 这标志着 GitHub 在 Actions 层实现了**声明式 Agent 编排**——不再需要写 YAML 配置 Agent 的每一步，而是用 Markdown 描述"想做的事情"，系统自动决定执行路径。安全架构的六层设计也回应了企业对 Agent 写代码的核心担忧。结合同批发版的 **Agentic Workflows 不再需要 PAT**（PAT-free），降低了采用门槛。同类产品对比：

| 维度 | GitHub Agentic Workflows | Linear Coding Sessions |
|:-----|:-------------------------|:----------------------|
| 定位 | CI/CD 层 Agent 流水线 | Issue→Code 编码闭环 |
| 定义方式 | Markdown → Actions YAML | Linear Agent 对话 |
| 运行环境 | 现有 Actions Runner | Cloud sandbox |
| 安全模型 | 六层安全架构 | MCP 权限 + Triage AI |

## 4️⃣ GitHub — Copilot Chat 现在能看到你的 Agent 会话 🪟

- **来源**: [GitHub Changelog — Copilot Chat agent session visibility (Jun 10, 2026)](https://github.blog/changelog/2026-06-10-copilot-chat-now-sees-your-agent-sessions/)
- **工具**: GitHub Copilot Chat / Copilot Cloud Agent
- **标签**: `copilot` `chat` `agent-session` `observability`
- **核心更新**:

Copilot Chat 获得两个新工具，让 Agent 工作变得可追踪、可查询：

**1. 获取 Agent 日志 (Get agent logs)**
- 从 Copilot Cloud Agent 的 PR 工作中提取会话日志
- 直接在 Chat 对话中问：变更了什么、验证了什么、为什么这么做
- 无需离开 Chat 就了解 Agent 的完整工作过程

**2. 会话搜索 (Session search)**
- 按主题、标题或时间搜索过去的 Agent 会话
- 快速总结历史会话，接续之前的工作

**价值判断**: Agent 的"黑盒"问题是企业采纳的主要障碍。这组工具让 Agent 的工作从"提交一个变更不知道它怎么来的"变成"完全可追溯、可查询"。结合上周的第三方 Agent 安全验证 GA（每次变更自动过 CodeQL + Advisory DB + Secret Scanning），**GitHub 正在构建一个全面的 Agent 可观测性+安全层**。
