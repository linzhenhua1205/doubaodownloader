## 2️⃣ Linear — Issue Description Comments: 描述锚点评论 🆕

- **来源**: [Linear Changelog - Issue description comments (Jun 4, 2026)](https://linear.app/changelog)
- **工具**: Linear
- **核心更新**:

Linear 新增**描述锚点评论**功能——可以在 Issue 描述的文本或图片上选择任意位置，留下**精确定位的评论**。

**设计思路**:
> "Select any part of a description to leave a comment anchored to that exact spot."

**核心能力**:
- **精准锚定** — 选中描述中的某段文字或某张图片，评论直接附着在该位置
- **上下文保留** — 评论"属于"描述的具体部分，而非笼统的 Issue 评论
- **与普通评论区分** — 解决「对描述的反馈」vs「对 Issue 整体讨论」的分离

**典型场景**:
- PRD 评审：某段需求描述存在歧义 → 选中该段直接追问
- 设计稿反馈：某张截图中的 UI 表现 → 选中图片留下修改建议
- 技术方案审核：某段架构决策描述需要讨论 → 锚定该段文字发起讨论

- **价值说明**: 这是一个**细节设计**但**高频场景**的更新。之前 Linear 的评论系统聚焦于 Issue 整体讨论（Activity Feed），但「对描述本身的讨论」一直缺乏载体——评论要么写在全 Issue 层面（丢失定位上下文），要么靠评论者手动引用描述段落（信息熵高）。通过锚点评论，Linear 将 Issue 描述的**协作属性**提升到与代码 Review 中 inline comment 同等的精度——描述不再是"一次性写完就定稿的文档"，而是**可迭代、可协作、可追溯讨论上下文**的动态载体。与 Team Documents 形成互补：Team Docs 解决「归属存档」，锚点评论解决「过程中的协作」。
