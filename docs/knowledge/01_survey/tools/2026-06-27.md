## 1️⃣ Cursor 3.9 — Customize 统一管理页面 🎛️

- **来源**: [Cursor Changelog (Jun 22, 2026)](https://changelog.cursor.com/)
- **工具**: Cursor 3.9
- **标签**: `cursor` `3.9` `customize` `plugins` `mcp` `canvases` `marketplace`

**核心内容**: Cursor 3.9 版本的核心主题是「统一配置与生态扩展」，新增 Customize 页面作为所有扩展能力的中央控制台。

### 🔹 Customize 页面 — Plugin/Skill/MCP 统一管理

- 新增 **Centralized Customize Page**，将所有可扩展能力汇聚到一处：
  - Plugins、Skills、MCPs、Subagents
  - Rules、Commands、Hooks
- 支持 **三级配置范围**：User（个人）、Team（团队）、Workspace（工作空间）
- 支持 **自定义 MCP** 的导入和配置

> **意义**：此前 Cursor 的扩展点散落在设置、菜单、文件中，现在所有扩展能力统一到一个页面管理，降低配置复杂度。

### 🔹 Marketplace Leaderboard

- 新增 **插件/技能/MCP 排行榜**，按团队维度显示最受欢迎的工具
- **一键添加**：从排行榜点击即可安装到当前配置
- 帮助团队快速发现和采纳同事已验证的高价值扩展

### 🔹 Plugin Canvases — 共享配置模板

- Plugins 现在支持 **Prebuilt Canvases**（预置画布模板），团队可一键复用：
  - **Hex Canvas**: 构建数据可视化
  - **Atlassian Canvas**: 实时查看所有 Issues、Projects、Documents
- Canvases 本质是**共享配置模板**，团队可以创建和分发自己的 Canvas

### 🔹 Team Marketplace 扩展

- Team Marketplace 现在支持从 **GitLab、BitBucket、Azure DevOps** 导入插件仓库
- 便于企业统一分发内部插件到团队

> **趋势意义**: Cursor 从「编辑器工具」向「Agent 配置平台」演进。Customize 页面本质上是一个**轻量级内部开发者门户（IDP）**，统一管理 Agent 生态中的所有扩展资产。

## 2️⃣ v0 — Platform API v2 + MCP Chat Tools + Annotations Mode 🏗️

- **来源**: [v0 Changelog (Jun 19 & Jun 26, 2026)](https://v0.dev/changelog)
- **工具**: v0
- **标签**: `v0` `platform-api` `mcp` `annotations` `office-files` `wallet-checkout`

**核心内容**: v0 在 Jun 19 和 Jun 26 连续发布两个版本更新，涉及平台开放、交互模式和支付三个方向。

### 🔹 Platform API v2 (Beta) — Jun 26

- 支持**从 GitHub 仓库启动聊天**：API 可以直接基于一个 GitHub repo 创建新的 chat 会话
- 新增 v1 → v2 **迁移指南**和 **版本切换器**
- 标志着 v0 从「面向个人使用」的 Web UI 工具向「面向集成」的**平台化**演进

### 🔹 MCP Chat Tools — Jun 26

- v0 MCP Server 新增一套聊天管理工具集，可编程操作：
  - **创建**新聊天
  - **查找/读取**已有聊天
  - **获取预览**
  - **发送消息**
- 这意味着**开发者可通过 MCP 协议编程控制 v0 的聊天工作流**，将 v0 集成到自己的工具链中

### 🔹 Office 文件附件支持 — Jun 26

- 现在可以在 v0 聊天中直接附加 **Excel (.xlsx) 和 Word (.docx)** 文件
- 粘贴的图片和文件以 **内联 Chip** 形式显示在输入框中，带缩略图，页面重载后保持

### 🔹 Annotations Mode — Jun 19 ★ 交互模式创新

- 新增 **Annotations Mode**：在预览中点击元素，自动生成带数字编号的标注评论
- **批量发送**：多个标注可一次性作为反馈发送给 Agent
- 本质上是将「描述 → 理解 → 修改」的漫长交互压缩为「**点 + 发送 + 修改**」的两步操作
- 与 Figma 的协作评论模式类似，但 v0 的 Agent 可以**直接读取标注并执行修改**

### 🔹 In-Form Questions — Jun 19

- v0 的澄清问题现在直接显示在**输入表单**中，支持：
  - 单选项、多选项、跳过、内联自定义答案
- 取代了之前的「Enhance Prompt」按钮——模型需要澄清时会主动提问，不再依赖预设按钮

### 🔹 Wallet Checkout — Jun 19

- 结账时支持 **Apple Pay** 和 **Google Pay**

> **意义**: v0 在两周内连续推出两个版本，核心方向有三个：(1) **平台开放化**——API v2 + MCP Tools，让 v0 从独立应用变成嵌入开发者工具链的平台组件；(2) **交互密度提升**——Annotations Mode 大幅降低人机沟通成本；(3) **企业级**——Office 文件支持 + Wallet 支付，提升专业度。

## 3️⃣ GitHub Actions — Steps 可并行执行 ⚡

- **来源**: [GitHub Changelog (Jun 25, 2026)](https://github.blog/changelog/)
- **工具**: GitHub Actions
- **标签**: `github-actions` `parallel-steps` `ci-cd` `performance`

**核心内容**: GitHub Actions 现在支持在一个 Job 内**并行执行多个 Steps**。

### 🔹 用法

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm test
        with:
          parallel: true   # ← 与此 job 的其他 parallel steps 并行运行
      - run: npm run lint
        with:
          parallel: true
      - run: npm run build   # ← 串行执行，等待上述并行 steps 完成
```

### 🔹 能力

- 通过 `with: parallel: true` 标记，相关 Steps 将与其他标记为 parallel 的 Steps **并行执行**
- 并行 Steps 全部完成后，后续的串行 Steps 再继续
- **不需要拆分到多个 Job**，避免 Job 间协调的复杂度

> **意义**: 此前 Actions 要实现并行需要在 Job 级别用 `matrix` 或拆分多个 Job，增加了 workflow 复杂度。Step 级并行是用户长期诉求，大幅简化了 CI/CD pipeline 的编写。对 CI 时间敏感的大型 monorepo 尤为重要。

## 4️⃣ GitHub Desktop 3.6 — Worktrees + 深度 Copilot 集成 🖥️

- **来源**: [GitHub Changelog (Jun 26, 2026)](https://github.blog/changelog/)
- **工具**: GitHub Desktop
- **标签**: `github-desktop` `3.6` `worktrees` `copilot` `git`

**核心内容**: GitHub Desktop 3.6 发布，主要新增两大能力。

### 🔹 Git Worktrees 支持

- GitHub Desktop 现在支持 **Git Worktrees** —— 允许在同一个仓库的不同分支上**同时工作**
- Worktrees 是 Git 原生功能，但此前主要靠命令行操作
- GitHub Desktop 将其可视化，在 UI 中创建、切换和管理 worktrees

> **意义**: 对于并行开发多个功能或修复多个 Bug 的开发者，无需反复 stash 或 clone 多个仓库副本。尤其适合配合 AI Coding Agent 使用（Agent 在一个 worktree 上工作，人在另一个 worktree 上开发）。

### 🔹 更深的 Copilot 集成

- GitHub Desktop 中的 Copilot 集成进一步升级（具体功能细节待官方文档补充）
