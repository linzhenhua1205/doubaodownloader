## 1️⃣ VS Code 1.126 — Agents Window 多会话并行 + 会话级成本透明 🚀

- **来源**: [VS Code 1.126 Release Notes (Jun 24, 2026)](https://code.visualstudio.com/updates/v1_126)
- **工具**: Visual Studio Code
- **标签**: `vscode` `1.126` `agents-window` `cost-tracking` `copilot` `workspace-trust`

**核心内容**: VS Code 1.126 正式发布，聚焦"**看得见的成本 + 协同的 Agent 会话 + 更安全的浏览**"三大主题。

### 🔹 会话级成本可视化（Cost Management 首发功能）

- 现在可以看到**整个 Chat Session 的总成本**，而不仅是单个 turn 的消耗
- 帮助用户**快速识别"费 token 大户"会话**，管理 Copilot 用量和预算
- 这是 VS Code 首次在 Agent 体验中引入**成本透明度**，预计未来会有更多精细化管理

### 🔹 Agents Window（Preview）— 多会话并行

新增 **Agents Window**（专属 companion 窗口），专为探索、迭代和审查 Agent 会话而设计：

- **一个 Agent Host Copilot 会话可以运行多个聊天**：主对话在做 feature 实现，新建一个子聊天并行做 review 或写文档
- **会话持久化**：窗口 reload 后所有聊天保留，关机回来继续
- **可重命名聊天 Tab**：双击即可重命名，告别一堆叫"Chat"的标签页
- **Agentic Code Feedback**：Agent 可读取用户的 inline 评论（通过 addComment / listComments / resolveComments 等工具），形成**双向反馈闭环**
  - 用户留评论 → Agent 读评论并修改代码
  - Agent 自己也能生成评论（如 `/code-review` 自动加评论）
  - PR Review 评论同样支持：Agent 可以 resolve 所有 PR comments

### 🔹 Workspace Trust 改进

- 打开新文件夹**不再弹出信任对话框**，默认以 **Restricted Mode** 打开
- 用户可以**先浏览代码**，安全了再决定是否信任
- 移除了易误触的 **Trust Parent** 按钮（常被误点导致信任了过多上级目录）
- `security.workspace.trust.startupPrompt` 默认值从 `once` 改为 `never`

### 🔹 模型配置简化

- 上下文长度（context size）和推理深度（reasoning effort）合并为**统一模型定制选择器**
- 模型 hover 信息简化：一句话描述能力 + 深度链接按钮

### 🔹 文档结构重组

- VS Code 站点博客添加了**博客列表页**（不再直接跳到最后一篇）
- 文档 TOC 重构：所有 Agent 相关文档统一归入"Agents"章节，代码编辑和配置归入"Editor"章节，语言支持归入"Languages and Runtimes"

## 2️⃣ GitHub 企业凭证管理 — 一键吊销所有凭据 🛡️

- **来源**: [GitHub Changelog (Jun 24, 2026)](https://github.blog/changelog/2026-06-24-self-service-credential-revocation-for-incident-response/)
- **工具**: GitHub Enterprise
- **标签**: `github` `enterprise` `security` `credential-revocation` `incident-response`

**核心内容**: GitHub Enterprise 新增 **Break-Glass 级别的凭证吊销能力**，大幅缩短安全事件响应时间：

### 🔹 企业管理员可批量操作

针对**公司全部用户**或**特定用户**，支持以下操作：
- 批量**吊销 SSO 授权**的 credentials（Personal Access Tokens、SSH Keys、OAuth Tokens）
- 删除用户 tokens 和 SSH keys（即使没有 SSO 授权），**仅限 EMU 账户**
- 通过 org-level REST API 按组织列出和吊销 SSO 授权

### 🔹 个人自服务吊销

- 企业成员可在 **Settings → Credentials** 查看自己生成或 SSO 授权的凭证数量
- **一键自服务吊销/删除**所有凭证，无需逐 token/key 操作
- 吊销和删除操作通过 **Audit Logs** 和 **Email Notifications** 记录和通知

> **安全意义**：此前应对凭证泄露需要逐个撤销 token，时效性差。现在从发现到吊销可以缩短到**秒级**，大幅降低凭证泄露窗口。

## 3️⃣ Copilot Free/Student 计划 — 默认 Auto 模型选择 ⚡

- **来源**: [GitHub Changelog (Jun 24, 2026)](https://github.blog/changelog/2026-06-24-changes-to-model-selection-for-free-and-student-plans/)
- **工具**: GitHub Copilot
- **标签**: `copilot` `auto` `free-plan` `model-selection` `simplification`

**核心内容**:
- Copilot Free 和 Student 计划**默认且唯一使用 Auto 模型选择**
- Auto 为每个任务动态选择最佳模型，**不再需要手动选模型**
- 同时取消 Microsoft 发布模型的 **(Preview) 标签**——因为 Auto 自动管理路由，用户无需关心模型阶段标签
- 这是 Copilot 走向 **Zero-Friction**（零摩擦）体验的一步：用户聚焦任务本身，而非底层模型选择

## 4️⃣ Microsoft Agent Framework — "Build Your Own Claw" 系列启动 🦀

- **来源**: [Microsoft Dev Blog - Agent Framework (Jun 22-24, 2026)](https://devblogs.microsoft.com/agent/)
- **工具**: Microsoft Agent Framework
- **标签**: `microsoft` `agent-framework` `claw` `agent-harness` `tool-use` `planning`

**核心内容**: Microsoft 发布了系列博客 **"Build Your Own Claw and Agent Harness with Microsoft Agent Framework"**，从零讲解构建 Agent 基础设施：

### 🔹 Part 1: Meet Your Agent Harness and Claw（Wes Steyn）

- A **Claw** = capable, CLI-style agent that can **plan, use tools, remember things, and safely act on behalf**
- 底层是 **Agent Harness**：围绕 LLM 的循环，配以工具、规划、记忆等基础设施
- 从零搭建这个循环，给个人 agent（Claw）配备首次工具能力

### 🔹 配套第二篇："Stop Overloading Your Skills"

- 提醒开发者：不要在一个 skill 里塞太多内容
- 模型已经知道 API 参考和 SDK 模式——skill 应该只提供**上下文关键信息**，而非重复已知知识
- 过度打包 skill 只会浪费 token，**降低 agent 实际效率**

### 🔹 Azure Functions MCP Extension 更新 (Build 2026)

- Resource / Prompt triggers
- MCP Apps 支持
- 内置 MCP 认证
- Fluent .NET 配置 API
