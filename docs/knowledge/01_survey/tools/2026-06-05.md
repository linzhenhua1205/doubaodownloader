## 1️⃣ Linear — Team Documents: 每个团队的专属知识空间 🆕

- **来源**: [Linear Changelog - Team Documents (Jun 4, 2026)](https://linear.app/changelog)
- **工具**: Linear
- **核心更新**:

Linear 为每个团队推出了专属的 **Team Documents** 页面，解决"团队上下文散落在各处"的问题。

**设计思路**:
> "重要团队上下文并不总属于某个具体的 Issue、Project 或 Initiative。团队需要一个专门的空间来存放随时间的笔记、文档和共享参考。"

**核心能力**:
- **团队专属文档页面** — 每个团队都有一个独立页面，通过侧边栏的团队名称点击进入，或 `o, t` 快捷键搜索
- **多种文档来源** — 直接在团队主页创建文档，或从 Projects/Issues 中添加已有文档
- **分节组织** — 将资源按 section 分组（如"架构决策"、"Onboarding 指南"、"Sprint 回顾"）
- **与现有体系打通** — 不替代 Issues/Projects 中的文档，而是为"不属于特定 Issue/Project 的团队上下文"提供归宿

- **价值说明**: Team Documents 补齐了 Linear 在"团队级知识管理"上的缺口。此前 Linear 的文档附着于 Issue 或 Project → 适合与具体工作相关的上下文。但团队级别的 Know-how（架构原则、编码规范、复盘总结）缺乏独立载体。Team Documents 相当于在每个团队中内置了一个迷你知识库，配合 Linear Agent 可搜索引用，减少了"知识散落在 Slack/Notion/Confluence"的碎片化问题。与 Cursor Canvas 的趋势一致——**工具内部的知识附着面正在从"代码级"向"团队级"扩展**。

## 2️⃣ Linear — Shared Skills for Linear Agent: 团队级 Agent 行为标准化 🆕

- **来源**: [Linear Changelog - Shared Skills for Linear Agent (Jun 4, 2026)](https://linear.app/changelog)
- **工具**: Linear Agent
- **核心更新**:

Linear Agent 的 **Skills**（可复用的指令模板）现在支持团队共享，让个人高效的工作流变成团队标准流程。

**机制**:
- **Skills** 定义：一段可复用的指令文本，指导 Linear Agent 如何执行特定任务（如"写 Spec"、"做周计划"、"起草更新"）
- **共享方式**：在 Settings 中分享已有 Skill，或直接让 Linear Agent 将某个 Skill 分享给团队
- **调用方式**：在任何 Agent 对话中，点击 Skills 或使用 `/` 斜杠命令选择

**典型场景**:
- **Spec 写作标准化** — 团队统一使用同一份 Skill 模板写 PRD，确保结构一致
- **周计划自动化** — 每周运行"整理本周未完成任务 + 分类排优先级"的 Skill
- **更新通知起草** — 用 Skill 自动生成项目更新文本，包含固定的格式和内容检查点

- **价值说明**: Shared Skills 解决的是"AI Agent 行为一致性"的问题。单个用户调教好的 Agent 工作流，通过 Skills 共享变成团队资产——这是 AI 编码/项目管理工具从**个人效率工具**向**团队协作基础设施**演进的关键信号。与 Cursor 的 `/loop` Skill 和 GitHub Copilot 的 Agent Tasks 形成三路并行的趋势：**Agent 行为从"写死的回复"走向"可编写、可共享、可编排的技能库"**。
