## 1️⃣ Cursor 3.7 — Design Mode 改进：多选元素 + 语音输入 🆕

- **来源**: [Cursor Changelog — Design Mode Improvements (Jun 5, 2026)](https://www.cursor.com/changelog)
- **工具**: Cursor
- **标签**: `cursor` `design-mode` `voice-input` `ui-development`
- **核心更新**:

Cursor 3.7 在 Design Mode（浏览器预览模式）中新增两个关键交互能力：

**🔲 多选元素 (Multi-select elements)**
- 在浏览器中同时点击选择 **两个或以上元素**
- Cursor 能理解选中的元素、它们的代码、周围布局以及视觉关系
- 可要求 Agent："让这个匹配那个"、"删除重复内容"、"调整整组组件的样式"
- 🎯 **意义**: 从「逐元素编辑」升级到「多元素关系编辑」，理解布局中的视觉一致性

**🎤 语音输入 (Voice input)**
- 通过 Design Mode 浮层直接语音描述修改需求
- 麦克风在 Agent 执行过程中 **保持可用** — Agent 还在跑时即可语音下达下一个修改指令，无需等待前一个完成
- 🎯 **意义**: 消除「等待 Agent 完成 → 描述 → 再等待」的串行瓶颈，变成并行语音编排

**价值判断**: Design Mode 从「点击标注 + 文本描述」升级为「点击 + 语音 + 拖/画」三重交互范式。多选元素使 UI 编辑从逐元素修复变为**布局级关系操作**；语音输入则把等待时间转化为**编排时间**，形成更自然的 Agent 协作节奏。

## 2️⃣ GitHub Copilot — Claude Fable 5 GA：Anthropic Mythos-class 首个模型 🆕

- **来源**: [GitHub Changelog — Claude Fable 5 GA for GitHub Copilot (Jun 9, 2026)](https://github.blog/changelog/2026-06-09-claude-fable-5-is-generally-available-for-github-copilot/)
- **工具**: GitHub Copilot
- **标签**: `copilot` `claude` `anthropic` `model` `mythos-class`
- **核心更新**:

Anthropic 新一代 **Mythos-class 模型** Claude Fable 5 在 GitHub Copilot 中 GA 上线。

**模型定位**:
- Mythos 是 Anthropic 全新模型系列，高于 Opus-tier
- 专为 **long-horizon autonomous coding**（长周期自主编码）和知识工作设计

**性能表现**:
- GitHub 内部基准测试：在自主编码工作流中，Fable 5 用 **更少的工具调用 + 更低的 Token 消耗** 完成同等任务，超越 Opus-tier（Claude Opus 4.8 级别）
- 按 Provider List Pricing 计费（Usage Based Billing）

**可用范围**:
| 平台 | 覆盖 |
|:-----|:------|
| VS Code | Chat / Ask / Edit / Agent 全模式 |
| VS | ⚡ |
| Copilot CLI | ⚡ |
| Copilot Cloud Agent | ⚡ |
| Copilot App | ⚡ |
| github.com | ⚡ |
| Mobile iOS/Android | ⚡ |
| JetBrains | ⚡ |
| Xcode / Eclipse | ⚡ |

**⚠️ 数据保留要求**:
- Fable 5 是 Copilot 中**首个需要数据保留**的 Claude 模型
- Anthropic 保留 prompt/output 最多 **30 天** 用于安全分类器（检测有害/滥用使用）
- 30 天后删除，**不用于训练**
- Enterprise/Business 管理员需要在 Copilot 设置中**手动开启策略**（默认关闭）
- 其他 Claude 模型（Opus 4.8 / Sonnet 4.5 / Haiku 4.5）仍保持零数据保留

**价值判断**: Fable 5 是 Copilot 模型生态的重大信号——Anthropic 用 safety-first 架构换取更强的自主编码能力，而 GitHub 愿意接受数据保留约束来提供该能力。这意味着**企业级 AI 编码正在从"谁能写出好代码"转向"谁能安全地自主写代码"**。

## 3️⃣ GitHub — 第三方 Agent 自动安全验证 GA 🆕

- **来源**: [GitHub Changelog — Security validation for third-party coding agents GA (Jun 9, 2026)](https://github.blog/changelog/2026-06-09-security-validation-for-third-party-coding-agents/)
- **工具**: GitHub Copilot
- **标签**: `copilot` `security` `codeql` `secret-scanning` `third-party-agent`
- **核心更新**:

GitHub 将 **Copilot Cloud Agent 的安全验证能力** 扩展到第三方编程 Agent（Claude Code、OpenAI Codex 等）。

**工作原理**:
当第三方 Agent 在仓库中创建代码（通过 PR）时，GitHub 自动执行三层安全检查：

1. **CodeQL 扫描** — 分析代码中的潜在安全漏洞
2. **GitHub Advisory Database 匹配** — 检查新增依赖是否存在已知漏洞
3. **Secret Scanning** — 检测 API Key、Token 等敏感信息泄露

如果发现安全问题，Agent **自动尝试修复后再提交 PR**。

**覆盖范围**:
- 自 2025 年 10 月 Copilot Cloud Agent 安全验证上线以来，已**主动阻止数百起安全泄露和漏洞**
- 现在扩展到 Claude Code、OpenAI Codex 等所有第三方 Agent
- **默认启用**，跟随仓库的 Copilot 安全验证设置
- 无需 GitHub Advanced Security 许可证

**价值判断**: 这是跨 Agent 安全层的**标准化**关键步骤。无论开发者用哪个 Agent，GitHub 都在平台层提供统一的安全兜底。结合上周同批发布的 Fix with Copilot for Actions（CI/CD 修复）和 Agent Tasks REST API（Agent 编排），GitHub 正在将 Copilot 从"IDE 内的 AI 助手"升级为"**仓库级别的 AI 安全网关**"。

## 4️⃣ JetBrains — Koog 1.0：JVM/Kotlin Agent 框架稳定版 🆕

- **来源**: [JetBrains AI Blog — Koog 1.0 (KotlinConf 2026)](https://blog.jetbrains.com/ai/2026/05/koog-1-0-is-out-stable-core-better-interop-and-multiplatform-observability/)
- **工具**: JetBrains Koog
- **标签**: `jetbrains` `agent-framework` `kotlin` `java` `open-source`
- **核心更新**:

JetBrains 在 KotlinConf 2026 上宣布 Koog 1.0 —— 面向 Kotlin 和 Java 的 **开源 AI Agent 框架** 稳定版。

**稳定性承诺**:
- 稳定模块 **1 年零破坏性变更** 保障
- 为生产环境提供坚实基础

**1.0 新增能力**:

| 能力 | 说明 |
|:-----|:------|
| **本地 Android AI** | 支持在 Android 设备上本地运行 LiteRT 模型 |
| **重构 Java 互操作层** | API 更干净一致，降低 Java 项目集成门槛 |
| **解耦 HTTP 传输** | 灵活接入不同 HTTP 客户端 |
| **OpenTelemetry 支持** | 跨 Kotlin Multiplatform 的完整可观测性 |
| **持久化 & 记忆增强** | 支持长期运行的 Agent 状态保持 |
| **Anthropic Prompt Caching** | 减少重复 prompt 的延迟和 Token 成本 |

**核心模块**:
- **Tools** — 工具定义和调用
- **Workflows** — 工作流编排
- **Persistence** — Agent 状态持久化
- **Memory** — Agent 记忆管理
- **Observability** — OpenTelemetry 集成

**价值判断**: Koog 1.0 与 Mellum2（上周开源）构成了 JetBrains 的双层 Agent 基础设施：**Mellum2** 是高效推理模型，**Koog** 是 Agent 编排框架。JetBrains 在 JVM 生态的 Agent 化布局已经成型——从模型到框架到 IDE 集成的完整链路。
