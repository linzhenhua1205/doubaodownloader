## 1️⃣ GitHub Copilot for JetBrains IDEs — Claude Agent + 组织Agent + CLI 增强 🚀

- **来源**: [GitHub Changelog (Jun 22, 2026)](https://github.blog/changelog/2026-06-22-new-features-and-claude-as-agent-provider-preview-in-jetbrains-ides/)
- **工具**: GitHub Copilot / JetBrains IDE Plugin
- **标签**: `copilot` `jetbrains` `claude` `agent` `cli` `enterprise`

**核心内容**:

这是 Copilot for JetBrains 的一次**大版本级更新**，涉及 Agent 生态、CLI 交互、可观测性三个方向：

### Claude as Agent Provider — Public Preview

Claude 作为 Agent Provider 在 JetBrains IDE 中进入公测。用户可在 Copilot Chat 的 Agent 选择器中直接切换 Claude 作为 Agent Provider，无需离开 IDE：
- 需要本地安装 **Claude Code CLI**，并在 Settings → Tools → GitHub Copilot → Chat 中配置路径
- ⚠️ **当前限制**: Claude agent 运行在 bypass permissions 模式（文件编辑和工具调用自动批准），可配置权限将在未来版本中提供
- Copilot Business/Enterprise 用户需管理员启用 Editor preview features 策略

### 组织/企业级 Agent 支持

管理员可在 GitHub 组织或企业级定义 Agent 并发布，**自动分发**给所有用户：
- Agent 在组织/企业级别定义，**自动推送到成员 IDE 中**
- 成员在 Copilot Chat Agent 选择器中直接选用
- 确保 Agent 行为和配置一致，简化治理

### CLI 会话消息队列与转向

针对长任务场景（等待响应时不能中断），新增三种消息发送模式：
- **Add to Queue**: 消息排队，当前响应完成后自动处理
- **Steer with Message**: 当前工具执行完成后立即插入新消息（用于纠正方向）
- **Stop and Send**: 停止当前响应，立即发送新消息

### Agent Debug Logs Summary View

Agent Debug 面板新增日志汇总视图，提供活动的**聚合统计**，便于审查和调试会话行为。

### 其他增强

- **Cloud Agent 正式 GA**（不再需要 Editor Preview 功能标志）
- **Model Picker 增强**: `/models` 斜杠命令直接打开、大上下文窗口选择、最近使用模型显示
- **Per-turn AI Credits Indicator**: 本地/CLI/Claude Agent 会话可看到每次对话消耗的 credits
- **UX 改进**: 聊天输入布局、内联聊天状态重置、代码块渲染优化、Next Edit Suggestions 增强
- **Bug 修复**: diff 视图误开、completion 模型不显示、UI 卡顿问题

## 2️⃣ GitHub Actions Security 三连发 — checkout v7 + Workflow Protections + 自定义镜像构建 🛡️

- **来源**: [GitHub Changelog (Jun 18, 2026)](https://github.blog/changelog/2026-06-18-safer-pull_request_target-defaults-for-github-actions-checkout/) · [Workflow Execution Protections](https://github.blog/changelog/2026-06-18-control-who-and-what-triggers-github-actions-workflows/) · [Custom Images](https://github.blog/changelog/2026-06-18-actions-build-custom-images-from-custom-images/)
- **工具**: GitHub Actions
- **标签**: `actions` `security` `supply-chain` `checkout` `ci-cd` `pwn-request`

**核心内容**:

### ① actions/checkout v7 GA — 默认阻止 Pwn Request

`actions/checkout@v7` 正式 GA，**默认拒绝 pull_request_target 事件中来自 Fork PR 的不安全 checkout 模式**，直接阻断 "Pwn Request" 攻击：

- 阻止 `ref: refs/pull/${{ number }}/merge`、`ref: ${{ head.sha }}`、`repository: ${{ head.repo.full_name }}` 等常见攻击模式
- **7月16日后**将回滚到所有当前支持的 major 版本（v4/v5/v6的 floating tag）
- 提供 `allow-unsafe-pr-checkout` 显式 opt-out（命名设计为便于代码审查中识别）
- 仅影响 Fork PR 的 pull_request_target/workflow_run 事件，同仓库 PR 不受影响

### ② Workflow Execution Protections — 公开预览

新的**工作流执行保护**基于 GitHub Rulesets 框架，提供**企业级允许列表**控制谁能触发 Actions 工作流以及哪些事件可以触发：

- **Actor Rules**: 控制谁能触发工作流，支持用户、仓库角色（Read/Maintain/Admin）、GitHub App、Copilot、Dependabot
- **Event Rules**: 控制哪些事件被允许（push/pull_request/pull_request_target/workflow_dispatch 等）
- **核心价值**: 将"谁贡献代码"与"谁运行CI"分离，可给 contributor 写权限但禁止其执行工作流
- **安全管理**: 覆盖 poisoned pipeline execution、manual-trigger abuse、untrusted-actor execution 等攻击场景
- **Evaluate Mode**: 影子运行模式，可在强制前预览规则影响

### ③ Actions: Build Custom Images from Custom Images

Actions 现在支持**从自定义镜像构建新自定义镜像**，不再局限于官方仓库中构建镜像。

## 3️⃣ Copilot Auto Mode GA + Agent Finder + Copilot App GA 🎯

- **来源**: [GitHub Changelog (Jun 17, 2026)](https://github.blog/changelog/2026-06-17-auto-mode-in-copilot-chat-available-for-all-users/) · [Agent Finder](https://github.blog/changelog/2026-06-17-agent-finder-for-github-copilot-now-available/) · [Copilot App](https://github.blog/changelog/2026-06-17-github-copilot-app-generally-available/)
- **工具**: GitHub Copilot
- **标签**: `copilot` `auto-mode` `agent-finder` `ard` `ga`

**核心内容**:

### Auto Mode — GA（所有 Copilot 计划可用）

Copilot Chat 在 github.com 和 GitHub Mobile App 上的**自动模型选择**正式 GA：
- 根据请求复杂度和模型实时可用性动态路由（Claude Sonnet 4.6 / GPT-5.4 mini / GPT-5.4 / Haiku 4.5）
- 用户可悬停查看实际使用的模型
- 按次切换 Auto 或指定模型
- ❗ **Auto 模式享受 10% 折扣**（对 token 用量计费）

### Agent Finder — ARD 开放规范发布

全新的 **Agent Finder** 允许 Copilot 自动发现合适的 AI 资源（MCP servers / Skills / Canvases / Agents / Tools）：
- 用自然语言描述任务，Agent Finder 对可用资源进行**排序匹配**
- 实现开放的 **Agentic Resource Discovery (ARD)** 规范（与 Google、GoDaddy、Hugging Face、Microsoft 联合开发）
- 支持公共目录或私有注册表，企业可通过已管理的设置控制 Agent Finder 的发现范围
- **不自动安装**：Agent Finder 只发现，不自动连接，用户决定是否安装
- 覆盖所有 Copilot 计划

### GitHub Copilot App GA + 个人计划重开

Copilot App（桌面客户端）正式 GA，同时 Copilot **重新开放个人计划注册**（Student / Pro / Pro+ / Max）。

## 4️⃣ JetBrains Junie AI Coding Agent 退出 Beta → 正式 GA 🏆

- **来源**: [JetBrains Blog (Jun 22, 2026)](https://blog.jetbrains.com/junie/)
- **工具**: JetBrains Junie
- **标签**: `jetbrains` `junie` `ai-agent` `ga` `coding-agent`

**核心内容**:

JetBrains 宣布其 AI Coding Agent **Junie** **退出 Beta**，正式进入 GA 阶段。Junie 是 JetBrains 在 AI 编程 Agent 领域的核心产品，与 Cursor、Claude Code、GitHub Copilot 直接竞争。

**Junie 的定位特点**：
- **与 IDE 深度集成**：不仅生成代码，还能使用开发者所用的同款工具（build tools、test runners、linters）
- **LLM-agnostic**：支持多种模型后端（GPT、Claude 等）
- **Agent 模式**: 可自主完成任务（测试、修复、重构），也可在 AI Chat 中作为助手使用
- **CLI 支持**: Junie CLI 可连接 JetBrains IDE，实现终端和 IDE 的统一上下文

📌 **补充**: 6月16日 JetBrains Marketplace 发现15个**恶意第三方插件**窃取 AI 提供商 API key，已全部下架并将相关账户封禁，IDE 侧通过安全补丁禁用。
