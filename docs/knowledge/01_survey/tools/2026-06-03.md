## 1️⃣ GitHub Copilot App — Agent-native 桌面体验 🔥

- **来源**: [GitHub Blog - Mario Rodriguez (Jun 2, 2026)](https://github.blog/news-insights/product-news/github-copilot-app-the-agent-native-desktop-experience/)
- **工具**: GitHub Copilot App（新桌面客户端）
- **核心更新**:
  - **Copilot App 桌面客户端**（Technical Preview）— 统一的 Agent 控制中心，从 `My Work` 视图管理多 Agent 并行工作（bug 调查、功能开发、PR 审查），每个会话运行在隔离的 git worktree 中
  - **Canvas 工作面板** — 双向可编辑的可视化工作台，Agent 更新 canvas、开发者实时编辑/重定向/审批，包含 plan/PR/browser/terminal/deployment/dashboard 等多种类型
  - **Cloud & Local Sandboxes**（Public Preview）— 为 Agent 提供隔离执行环境。本地 sandbox 限制文件系统/网络/系统能力；云端 sandbox 是 GitHub 托管的完整隔离 Linux 环境，支持跨设备 remote control
  - **Code Review 增强** — 新增 `medium tier`（高推理模型审查），支持自定义 Agent skills、MCP server、`/security-review` 和 `/rubberduck` 技能，Azure Repos 原生支持
  - **Copilot SDK GA**— 支持 Node.js/TypeScript、Python、Go、.NET、Rust、Java，暴露与 Copilot App 相同的 agentic runtime
  - **Copilot CLI 重构** — 全新 TUI（/experimental 模式）、语音输入（on-device STT）、`/every` 定时任务调度
  - **Memory++ & /chronicle** — Agent 会话跨设备/跨时间连续性，可查询历史 session 上下文
  - **Partner Agent Apps** — LaunchDarkly、Amplitude、Sonar、PagerDuty、Miro 等生态集成
  - **Agent Merge** — Agent 可自主推进 PR 通过 CI/审查/合并全流程
- **价值说明**: 这是 GitHub 从一个代码托管平台向「Agent 原生开发系统」的范式级转型。Copilot App 解决了多 Agent 并行工作时的上下文碎片化问题，Canvas 是 Agent 与人类协作的新交互范式（AX），sandbox 解决了 Agent 安全执行的核心矛盾。SDK 开放意味着第三方可以构建与 Copilot 同等的 Agent 体验。

## 2️⃣ Cursor 3.6 & 3.5 — Auto-review / Shared Canvas / /loop Skill 🔥

- **来源**: [Cursor Changelog](https://cursor.com/changelog)
- **工具**: Cursor IDE
- **核心更新**:

### Cursor 3.6 — Auto-review Run Mode (May 29)
- **Auto-review 运行模式** — Agent 可更长时间自主工作，减少审批打断
  - Shell/MCP/Fetch 调用分类处理：allowlist 白名单立即执行，可 sandbox 的在 sandbox 执行
  - 其他操作由 **classifier subagent** 决策：允许执行、换方案、或请求审批
  - 支持自定义 classifier 指令
  - 在 Settings > Cursor Settings > Agents > Run Mode 中配置
- **价值**: 解决了 Agent 高频打断的核心痛点。Allowlist + Sandbox + Classifier 三级安全机制在自主性与安全性之间取得平衡，是 Agent 从"辅助"走向"自主执行"的关键能力

### Cursor 3.5 — Shared Canvases & /loop Skill (May 20)
- **Shared Canvases** — 将 Agent 创建的互动工作产物（dashboard/report/UI）以链接形式分享给团队，对方在浏览器中以只读方式查看
- **/loop Skill** — Agent 可按本地计划重复执行 prompt、直到达成某个条件（如"直到测试通过"）或手动停止。不指定间隔时 Agent 自主决定唤醒时机
- **Cursor Automations 增强**:
  - **多仓库自动化** — 跨代码库推理、交付、测试
  - **无仓库自动化** — 监控 Slack/Databricks/Stripe 等工具，发布 5 个模板：Slack digest agent、Product analytics agent、Product FAQ agent、Product finance agent、Customer health agent
- **价值**: Shared Canvases 让 Agent 产出可团队消费，/loop 实现了本地持续 Agent（区别于云端 scheduled agent），无仓库自动化让 Agent 扩展到代码之外的生产力场景（数据/运营/客服）

### Cursor in Jira (May 19)
- 在 Jira ticket 中 `@Cursor` 即可启动云 Agent，读取 ticket 上下文后自动完成 bug 修复/功能开发/测试更新，完成后更新 Jira 并附带 PR 链接
- **价值**: 将 Agent 嵌入项目管理工作流的里程碑，AI 可直接从 Jira ticket 驱动开发闭环

### Cursor Composer 2.5 (May 18)
- Composer 的大版本升级，长期任务执行更稳定、复杂指令更可靠
- 定价：Standard $0.50/M input, $2.50/M output；Fast $3.00/M input, $15.00/M output
- **价值**: Agent 持续工作能力的关键基础升级
