## 1️⃣ GitHub Copilot — Fix with Copilot for failing Actions 一键修复 🆕

- **来源**: [GitHub Changelog — Fix with Copilot for failing Actions (Jun 4, 2026)](https://github.blog/changelog/2026-06-04-fix-with-copilot-for-failing-actions-now-in-pro-pro-and-max/)
- **工具**: GitHub Copilot
- **标签**: `copilot` `ci-cd` `developer-experience`
- **核心更新**:

GitHub Actions 任务失败时，Copilot Pro/Pro+/Max 用户现在可以**一键修复**。

**工作流程**:
- 工作流运行日志页面上出现 **Fix with Copilot 按钮**
- 点击后 Copilot cloud agent（云端开发环境）自动：
  1. **调查失败原因** — 分析日志输出、错误上下文
  2. **推送修复到分支** — 直接在分支上提交代码修改
  3. **标记用户审核** — 修复完成后 tag 用户 Review

**覆盖场景**:
- 修复失败的测试用例（Flaky test / 断言失败）
- 修正 linter 报错（格式、风格检查）
- 其他可自动诊断的 CI 失败场景

**设计意图**: 将「简单但耗时」的 CI 修复工作委托给 Copilot，开发者可专注于真正的构建工作。

- **性能/功能数据**: 一键触发 → Copilot 自动完成调查、修复、推送、通知全流程，无需手动切上下文
- **价值判断**: 这是 Copilot 从「编码辅助」向「**CI/CD 全流程 Agent**」演进的关键一步。与 Agent Tasks REST API（同一天发布）配合，形成了 **Agent 化 CI/CD 修复管线**的完整闭环。

## 2️⃣ JetBrains — Mellum2 开源：12B MoE 模型专攻 AI 工作流 🆕

- **来源**: [JetBrains AI Blog — Mellum2 Goes Open Source (Jun 4, 2026)](https://blog.jetbrains.com/ai/2026/06/04/mellum2-goes-open-source/)
- **工具**: JetBrains Mellum2
- **标签**: `ai-model` `open-source` `moe` `routing`
- **核心更新**:

JetBrains 开源 Mellum2，一个从头训练的 **12B 参数 MoE 模型**，Apache 2.0 协议发布。

**架构与性能**:
- **MoE 设计** — 12B 总参数，每 token 仅激活 **2.5B 参数**，显著降低推理成本
- **专注领域** — 非多模态，专精**自然语言 + 代码**，适合软件工程场景
- **推理速度** — 同类模型推理时间「不到一半」（文中原话），适合高吞吐低延迟场景

**关键用例**:
| 场景 | 说明 |
|:-----|:------|
| **AI 工作流路由** | 分析输入 prompt，选择正确的模型/工具处理任务 |
| **低延迟 RAG 管线** | 检索上下文 → Mellum2 摘要 → 即时生成响应 |
| **子 Agent 执行** | Agent 管线中负责上下文收集、规划、验证等高频任务 |
| **私有本地部署** | 自托管运行，代码和数据完全受控 |

**JetBrains"焦点模型"哲学**:
> "前沿模型会继续推高天花板，但实用 AI 产品还需要**焦点模型（focal models）**：快速、专业化的组件，高效处理高频任务。"

Mellum2 定位为**协调系统**中的一环，而非替代大模型——聚焦 latency-sensitive、高频率、可路由的任务。

- **价值判断**: Mellum2 是 JetBrains 在 AI IDE 竞争中从「追赶者」转向「**基础设施提供者**」的战略布局。开源 + MoE 高效架构 + 代码领域专注，三个维度差异化明显。与 Cursor SDK / GitHub Copilot API 形成呼应——**整个行业都在将模型能力下沉为可组合的 API/基础设施**。

## 3️⃣ Cursor 3.7 — Canvas Context Usage Report: Agent Token 可视化 🆕

- **来源**: [Cursor Changelog — Canvas Design Mode and Context Usage Report (Jun 4, 2026)](https://www.cursor.com/changelog)
- **工具**: Cursor
- **标签**: `cursor` `context-management` `token-optimization` `agent`
- **核心更新**:

Cursor 3.7 新增 **Context Usage Report**，将 Agent 上下文消耗可视化呈现为交互式 Canvas 报告。

**报告内容**:
- 分解 Token 在以下部分的分布：
  - **System prompt**（系统提示词）
  - **Tool definitions**（工具定义）
  - **Rules**（用户规则）
  - **Skills**（技能模块）
  - 以及其他上下文组件

**交互能力**:
- Canvas 是**交互式**的，可以向 Agent 提问关于上下文使用的问题
- 内置 **Debug with Agent** 按钮，点击后 Agent 会分析如何减少上下文使用，并在新对话中应用优化建议

**同期 Canvas 改进**:
- 共享 Canvas 可**全屏打开**（适合展示给团队）
- Agent 可在 Canvas 中**嵌入可点击按钮**，点击执行指定 Prompt
- Canvas 类型错误自动修复增强
- 组件样式和图表定制增强

- **价值判断**: Context 管理是 Agent 开发的**核心痛点**——不了解 Token 分布就无法优化。Cursor 将「调试 Agent 的上下文消耗」变成了**可交互、可视化的协作体验**，而非枯燥的日志分析。Debug with Agent 按钮更是把 Agent 自身作为优化工具——形成 **Agent 调试 Agent** 的元循环。

## 4️⃣ GitHub Copilot — Copilot Chat PR 上下文增强 GA 🆕

- **来源**: [GitHub Changelog — Copilot Chat brings richer context to pull requests (Jun 4, 2026)](https://github.blog/changelog/2026-06-04-copilot-chat-brings-richer-context-to-pull-requests/)
- **工具**: GitHub Copilot
- **标签**: `copilot` `code-review` `pull-requests`
- **核心更新**:

Copilot Chat 在 **github.com 上的 PR/Diff 页面**中提供更丰富的上下文，从 Public Preview 转为 **GA**。

**核心能力**:
- **代码与聊天并排** — 在 PR 页面直接与 Copilot 对话，无需切换窗口
- **PR 理解 + Review + 摘要** — 新能力自动注入相关的 PR 和仓库上下文
- **性能提升** — 回答质量更快（"We've improved the performance"）

**入口方式**:
1. 每个 diff 顶部的 **Ask about this diff** 按钮
2. 顶部导航栏的 Copilot 按钮
3. 选中某行代码 → 右键 → Copilot

- **价值判断**: PR Review 是 Copilot 从「写代码」到「管代码」的自然延伸。GA 意味着 GitHub 认为该功能的**准确率和可用性已达到生产标准**。
