## 1️⃣ Cursor — Cloud Environment Setup + Cloud Subagents 🚀

- **来源**: [Cursor Changelog (Jun 17, 2026)](https://www.cursor.com/changelog)
- **工具**: Cursor 3.7
- **标签**: `cursor` `cloud` `subagents` `environment` `babysit` `shared-session`

**核心内容**: Cursor 发布了一次重大的 Cloud Agents 架构升级，引入了三个核心新能力：

### 🔧 Cloud Environment Setup（云环境设置，<10分钟）

- Cursor 现在可以在 **10 分钟内** 帮助你在云端建立完整的开发环境
- 通过共享终端会话实时观察 Agent 的进度（安装依赖等设置任务）
- 环境被捕获为**可复用的快照（snapshot）**，后续 Cloud Agent 启动更快
- Agent 可以运行软件来测试变更，在长时间的时间范围内迭代直到输出被验证
- 环境配置可提交到 `.cursor/environment.json`，**整个团队共享**

### ☁️ Cloud Subagents — `/in-cloud` 命令

- 使用 `/in-cloud` 在**独立 VM（独立分支）** 上启动 Cloud Subagent
- **本地工作区保持干净和响应**——编译/测试/依赖安装等放在云上
- 特别适用于隔离**长时间运行或并行工作**：修复 CI、调查问题、探索代码库

### 👶 `/babysit` PR 保姆模式

- 点击快速操作按钮或使用 `/babysit` 让 Cloud Subagent 在云端**远程迭代准备 PR 合并**
- 不占用本地会话资源
- Cloud Subagent 可在后台运行，**不中断父 Agent**（父 Agent 可继续在本地或云端运行）

### 🔄 本地 ⇄ 云端 Handoff

- Agent 会话可以在**本地电脑和云端之间更可靠地迁移**
- 将长时间运行的工作从本机卸载到云端
- 按需拉回本地测试变更
- **无并行 Agent 数量限制**

### 📊 价值判断

这是 Cursor 在 **Agent 执行环境** 维度上的重大扩展。之前 Cloud Agent 存在于 Cursor 云端但缺乏持久化环境管理。现在：

| 之前 | 现在 |
|:-----|:-----|
| Cloud Agent 只能用预置环境 | 可自定义设置 + 快照复用 |
| 单 Agent 单任务 | 多 Cloud Subagent 并行，不阻塞本地 |
| Agent 提交 PR 后需人工跟进 | `/babysit` 远程迭代到能合入 |
| 本地/云端切换不稳定 | 可靠 Handoff 机制 |

这与 **GitHub Copilot App GA**（见下文）形成直接竞争——两者都在抢"Agent 驱动开发的桌面入口"这个生态位。

## 2️⃣ GitHub Copilot — Auto Mode GA（自动模型选择） 🧠

- **来源**: [GitHub Changelog (Jun 17, 2026)](https://github.blog/changelog/2026-06-17-auto-mode-in-copilot-chat-available-for-all-users/)
- **工具**: GitHub Copilot Chat
- **标签**: `github` `copilot` `auto-mode` `model-selection` `ga`

**核心内容**: Copilot Chat 的 **Auto 自动模型选择** 现已在 github.com 和 GitHub Mobile App 上对所有 Copilot 套餐 GA。

### 工作原理

- **动态路由**：根据实时模型可用性和请求复杂度，自动选择最优模型
- **可路由的模型**: Claude Sonnet 4.6、GPT-5.4 mini、GPT-5.4、Haiku 4.5（取决于套餐和政策）
- 模型池会随时间变化

### 关键特性

| 特性 | 说明 |
|:-----|:------|
| **透明度** | 悬停模型响应可查看使用了哪个模型 |
| **任务路由** | Auto 评估任务复杂度并选择最优模型和实时系统健康 |
| **用户可控** | 可在**每次响应**时切换 Auto 或指定模型 |
| **遵守策略** | 尊重所有用户和管理员的模型设置 |

### 计费

- **Auto 模式享 10% 折扣**（对所有付费订阅用户）

### 📊 价值判断

Auto Mode GA 的意义在于**无需用户思考模型选择**。对于非技术用户或不想手动切模型的开发者，这是体验简化的重要一步。10% 折扣也是个不错的激励。配合 Agent Finder（见下文），Copilot 正在从"手动选工具"转向"自动发现最优能力"。

## 3️⃣ GitHub — Agent Finder for Copilot 🕵️

- **来源**: [GitHub Changelog (Jun 17, 2026)](https://github.blog/changelog/2026-06-17-agent-finder-for-github-copilot-now-available/)
- **工具**: GitHub Copilot
- **标签**: `github` `copilot` `agent-finder` `ard` `discovery` `mcp`

**核心内容**: **GitHub Agent Finder** 正式上线。实现开放的 **Agentic Resource Discovery (ARD)** 规范。

### 解决的问题

之前：需要手动配置每个 Agent 该用哪些 MCP 服务器、Skills、Canvases、Agents、Tools——不仅繁琐，还**塞满上下文窗口**。

现在：**用自然语言描述任务**，Agent Finder 自动搜索可用 AI 资源的索引，返回排序后的匹配结果供 Copilot 按需拉取。

### ARD 规范

- Agent Finder 实现了 **Agentic Resource Discovery (ARD)** 开放规范
- 与 **Google、GoDaddy、Hugging Face、Microsoft** 合作开发
- **任何 Registry 或 AI 客户端** 都可以采用相同的发现模型

### 安全管控

| 安全维度 | 说明 |
|:---------|:------|
| **自选 Registry** | 指向 GitHub 公共目录或自己的私有 Registry |
| **企业管控** | 管理员定义 Agent 允许发现和使用的资源范围 |
| **无自动安装** | Agent Finder **只找不装**，用户控制实际连接什么 |

### 📊 价值判断

Agent Finder 是**从"静态配置"到"动态发现"的范式转变**。它 + Auto Mode + ARD 规范构成了 Copilot Agent 生态的"三层发现体系"：

1. **Auto Mode**：自动选择模型
2. **Agent Finder**：自动发现能力（MCP/工具/技能）
3. **ARD 规范**：开放标准，任何注册中心都可接入

这不仅降低了用户心智负担，还为**企业级 Agent 治理**提供了"发现即管控"的路径。

## 4️⃣ GitHub Copilot App — GA（全平台桌面客户端） 💻

- **来源**: [GitHub Changelog (Jun 17, 2026)](https://github.blog/changelog/2026-06-17-github-copilot-app-generally-available/)
- **工具**: GitHub Copilot App
- **标签**: `github` `copilot` `app` `ga` `desktop` `agent`

**核心内容**: **GitHub Copilot App** 正式 GA，支持 macOS / Windows / Linux，是 **Agent 驱动开发的桌面入口**，原生构建在 GitHub 之上。

### 核心能力

| 能力 | 说明 |
|:-----|:------|
| **多源启动会话** | 从 Issue、PR 或 Prompt 启动 |
| **并行会话** | 跨仓库并行运行，每个在独立分支 + worktree |
| **Diff 审查** | 内置 Diff 审查 |
| **集成终端 + 浏览器** | 可直接验证 |
| **直接开 PR** | 使用团队的现有检查和合并要求 |

### 技术预览后的新增功能

- **Canvases**: 双向表面——你和 Agent 在同一 Plan/PR/终端/浏览器会话上操作，进度可视化可操控，不埋在聊天记录里
- **Cloud Automations**: 在云端**定时执行 Agent 工作**，不依赖本机开机
- **BYO Model & Tools**: 每个会话自选模型 + 通过 MCP 连接外部工具

### 企业要求

- Business/Enterprise 套餐需要管理员启用 **Copilot CLI** 策略

### 📊 价值判断

Copilot App GA 标志着 GitHub 在 **Agent 驱动开发入口** 上与 Cursor 正面对决。关键差异：

| 维度 | Cursor | GitHub Copilot App |
|:-----|:-------|:-------------------|
| IDE 依赖 | 独立 IDE（基于 VS Code） | 独立桌面 App |
| 触发入口 | IDE 内对话 | Issue/PR/Prompt |
| 云 Agent | ✅ Cloud Agents（6/17 新发布） | ✅ Cloud Automations |
| Canvas | ✅ Canvas | ✅ Canvases |
| 并行 | ✅ Cloud Subagents | ✅ 平行 Session |
| BYO 模型 | ✅ 支持 | ✅ 支持 |
| MCP | ✅ | ✅ |
| 代码库集成 | IDE 原生 | GitHub 原生 |

Cursor 在 **实时云环境设置** 上有优势（<10 分钟环境 + 快照复用），GitHub Copilot App 在 **Issue/PR 原生集成** 上更强。两者都在向"Agent OS"方向演进。

## 6️⃣ JetBrains Junie — AI 编程 Agent 正式退出 Beta 🎉

- **来源**: [JetBrains Blog — Junie 频道](https://blog.jetbrains.com/junie/)（搜"Junie: The JetBrains AI Coding Agent Leaves Beta"）
- **工具**: JetBrains Junie
- **标签**: `jetbrains` `junie` `ai-agent` `ga` `beta`

**核心内容**: JetBrains 的 AI 编程 Agent **Junie 正式退出 Beta 阶段**，成为 GA 产品。

### 背景

- Junie 最初是一个实验性项目——"如果一个 AI 编程 Agent 不只是猜测你的项目细节，而是实际使用与你相同的工具，会怎样？"
- **经过一年的实验**，从 JetBrains AI Chat 集成到独立的 CLI、MCP 支持、再到 ReSharper/.NET 集成，Junie 已经构建了完整的工具链

### 已知特性回顾

| 能力 | 说明 |
|:-----|:------|
| **IDE 原生集成** | 在 JetBrains IDE 中运行，使用项目真实上下文 |
| **Junie CLI** | 终端内使用，LLM-agnostic（不绑定特定模型） |
| **MCP 支持** | 通过 MCP 连接外部工具和数据源 |
| **Spec-Driven** | 支持规范先行的开发方式 |
| **多 IDE 支持** | IntelliJ IDEA、PyCharm、PhpStorm、GoLand、Rider 等 |

### 📊 价值判断

Junie GA 意味着 JetBrains 正式进入 AI 编程 Agent 竞争——与 Cursor、GitHub Copilot、Claude Code 同台竞技。JetBrains 的核心差异化在于 **IDE 深度集成**（而非独立 App 或 CLI），以及丰富的静态分析能力积累。

Junie 的市场定位不太可能是独立竞品，而更像是 **JetBrains IDE 生态的 AI 层**，让现有 JetBrains 用户无需切换工具就能获得 Agent 能力。

## 📈 趋势观察（2026-06-18）

### 1️⃣ Agent 驱动的开发入口大战白热化

6月17日一天之内：
- **Cursor** 发布了 Cloud Environment + Cloud Subagents（本地/云端 Handoff）
- **GitHub Copilot App** GA（独立桌面客户端）
- **JetBrains Junie** 退出 Beta（IDE 原生 Agent）

三大玩家在**同一周**发布了 Agent 开发环境的重要更新，说明市场正在从"AI 辅助编程"向"Agent 驱动的开发入口"快速演进。进入壁垒在快速建立——谁先占领习惯，谁就可能锁定用户。

### 2️⃣ 动态发现取代静态配置

Auto Mode（自动模型选择）+ Agent Finder（自动能力发现）+ ARD 规范（开放发现协议）= **Agent 行业正在从"人工配置"转向"自动发现"**。用户不再需要思考"该用哪个模型/工具/技能"——Agent 自己会去发现最优组合。

### 3️⃣ Cursor 的差异化方向

Cursor 的 Cloud Environment + Snapshots 在对标 **GitHub Codespaces** 和 **VS Code Dev Containers**——让 Agent 拥有"可复现的完整开发环境"，不仅仅是代码生成。这是 Cursor 相比单纯 IDE 插件的关键差异化。
