## 1️⃣ Cursor Mobile App for iOS — 从手机启动和管理 Agent 📱

- **来源**: [Cursor Changelog (Jun 29, 2026)](https://www.cursor.com/changelog)
- **工具**: Cursor Mobile App (iOS)
- **标签**: `cursor` `mobile` `ios` `cloud-agents` `remote-control`

**核心内容**: Cursor 发布 iOS 移动端 App（Public Beta），付费计划可用。这是 AI IDE 领域第一个真正意义上的移动端配套 App。

### 🔹 Cloud Agents on Mobile

- 在手机上打开 Cursor App，选择仓库，**像桌面端一样启动 Cloud Agent**
- 支持所有前沿模型选择
- **语音输入**描述想法，用 Slash Commands 引导 Agent
- Cloud Agent 运行在**隔离 VM** 中，有完整开发环境用于测试、验证、Demo
- 支持 **Local ↔ Cloud 切换**：关闭笔记本盖子后 Agent 继续在云端运行

### 🔹 Remote Control — 手机远程操控桌面 Agent

- 相当于 **Cursor 的「远程桌面」**：在手机上继续指挥正在电脑上运行的 Agent
- 开启 Keep Awake 设置后电脑保持唤醒，离开工位也能工作
- Team/Enterprise 计划需要管理员在 Dashboard 中启用

### 🔹 Live Activities + Push Notifications

- **锁屏 Live Activities** 实时追踪 Agent 状态
- **推送通知**：Agent 完成、需要输入、或待 Review 时推送

### 🔹 Artifacts & SCM 全功能

- 在手机上查看 Demos、截图、Logs、Diffs
- 直接留下后续指令，或直接**从 App 合并 PR**

> **趋势意义**: 这标志着 AI 编程工具的**「随时在线」转折点**。Cursor 没有选择做一个简单的「查看器」，而是把 Agent 启动、远程控制、PR 合并、通知推送的全套能力搬上手机。真正从"坐在电脑前编程"变成了"随时随地指挥 Agent 编程"。对需要频繁离席的工程师和管理者价值巨大。Remote Control + Cloud Agent 的组合意味着**任何人可以在任何地方启动、监控、指挥远端开发工作**。

## 3️⃣ GitHub Actions — Read-Only Cache 防不安全触发 🛡️

- **来源**: [GitHub Changelog (Jun 26, 2026)](https://github.blog/changelog/)
- **工具**: GitHub Actions Cache
- **标签**: `github-actions` `cache` `security` `supply-chain` `untrusted-triggers`

**核心内容**: Actions Cache 现在支持**只读模式**，当 Workflow 由不安全的触发事件（如 PR from fork、`issue_comment`、`workflow_run` 从不受信任来源）启动时，Actions 自动将 Cache 设为只读。

### 🔹 设计意图

- 此前攻击者可以通过 PR 的 Cache 注入恶意内容，后续安全的 Workflow 运行读取被污染的缓存→供应链攻击
- 只读 Cache 确保不受信任的 Workflow 不能写入新缓存，但可以读取已有缓存
- 工作流已通过 `actions/cache` 或 `actions/cache/restore` 的 `read-only` 输出变量获知只读状态

### 🔹 影响

- 不需要用户配置，GitHub 自动检测触发来源的安全性
- Fork PR、`pull_request_target` 使用不当、`issue_comment` 等触发的 Workflow 自动启用

> **意义**: 这是 GitHub 在 CI/CD 供应链安全方面的重要补强。Cache 污染是长期存在的攻击面——恶意 PR 注入恶意依赖到 Cache，后续构建拉取→自动投毒。只读模式从根源上切断了这条攻击链。对使用 Actions 的企业团队来说，这是零成本的供应链安全加固。
