### ① Cursor: Bugbot 3× 更快、22% 更便宜、10% 更多 Bug

| 维度 | 数值 |
|:-----|:-----|
| **来源** | [Cursor Changelog](https://changelog.cursor.com/) (Jun 10) |
| **平均审查时间** | ~90 秒 ↓ (**从 ~5 分钟缩短 3.3×**) |
| **Bug 发现率** | 0.62 bugs/review ↑ (从 0.56 提升 **10%**) |
| **运行成本** | **~22% 降低** per run |
| **底层引擎** | Composer 2.5（端到端训练，非微调） |

**新功能**:
1. **`/review` 可推前运行** — 在 push 之前即可执行 Bugbot 和 Security Review。`/review` 提示选择运行哪些 agent，或直接用 `/review-bugbot` / `/review-security`
2. **GitHub/GitLab 同步去重** — 若在本地用 `/review` 跑过审查，打开 PR 时 Bugbot 自动识别已审查的 diff，跳过重复分析，只在 PR 评论区标注"已审查过此 diff"
3. **增量审查** — 可配置 Bugbot **只审查上次以来新增的内容**，聚焦最新变更

> **🎯 价值**: Bugbot 真正成为"推前必检" 环节。审查时间从分钟级降到秒级+成本大幅下降，门槛显著降低。自动跳过重复分析设计也减少了 CI 冗余开销。

### ② GitHub: 第三方 Agent 代码自动安全验证 GA 🔒

| 维度 | 内容 |
|:-----|:-----|
| **来源** | [GitHub Changelog](https://github.blog/changelog/2026-06-09-security-validation-for-third-party-coding-agents/) (Jun 9) |
| **覆盖 Agent** | Claude Code、OpenAI Codex 等第三方编码 Agent |
| **验证方式** | CodeQL 安全分析 + Advisory DB 依赖审查 + Secret Scanning |
| **修复模式** | 发现问题 → Agent **自动修复后再提交 PR** |
| **生效条件** | **默认开启**，遵循仓库 Copilot 设置 |
| **前置验证** | 2025.10 已为 Copilot Cloud Agent 推出，验证期间"阻止了数百个潜在安全泄露" |

**关键点**:
- 第三方 Agent 生成的代码享受**与 Copilot Cloud Agent 同级别的自动安全审查**
- 无需 GitHub Advanced Security 许可证
- 三级防线: CodeQL（自定义查询）→ Advisory DB（第三方依赖 CVE）→ Secret Scanning（密钥/令牌泄漏）

> **🎯 价值**: 这是 Agent 安全治理的**分水岭时刻**。Claude Code/Codex 产出的代码从"需人工复核"进入"自动安全验证管线"，Agent 输出质量保障走向标准化。

### ③ GitHub: Copilot Chat 现可查看 Agent 会话全貌 🔍

| 维度 | 内容 |
|:-----|:-----|
| **来源** | [GitHub Changelog](https://github.blog/changelog/2026-06-10-copilot-chat-now-sees-your-agent-sessions/) (Jun 10) |
| **新增能力** | Chat ↔ Cloud Agent 双向感知、会话日志检索、会话搜索 |

**三大变化**:
1. **会话状态可见** — 从 Chat 发起 Agent 任务（创建 session/PR/深度研究仓库）后，Chat 实时反映任务进度，完成后可提出后续问题
2. **Get agent logs 工具** — 拉取 Cloud Agent 在 PR 上的完整会话日志，可追问"改变了什么"、"做了什么验证"、"为什么这样做"
3. **Session search 工具** — 按主题/标题/时间检索和总结过去的 Agent 会话，轻松接续之前的工作

> **🎯 价值**: 解决 Agent 工作的"黑盒"问题——之前 Agent 跑完留个 PR 就走，现在可追踪、可查询、可追问。对于多人协作和审计场景至关重要。

### ④ JetBrains: 发布 JetBrains Central — Agent 软件生产控制平台 🏗️

| 维度 | 内容 |
|:-----|:-----|
| **来源** | [JetBrains Blog](https://blog.jetbrains.com/blog/2026/06/10/introducing-jetbrains-central-an-open-system-for-agentic-software-development/) (Jun 10) |
| **定位** | "Agent 驱动软件生产的**控制与执行平面**" |
| **发布时间** | Q2 2026 Early Access Program（设计合作伙伴内测） |

**数据支撑**:
- JetBrains AI Pulse 2026.01 调研: 全球 11,000 开发者中 **90% 已在工作中使用 AI**，22% 已使用 AI 编码 Agent，**66% 公司计划 12 个月内采用**
- 但仅 **≤13%** 的开发者将 AI 用于全生命周期（代码审阅/发布管线）——存在巨大的"最后一公里"鸿沟

**三大核心能力**:
1. **治理与控制** — 策略执行、IAM、可观测性、可审计、成本归属（部分已通过 Central Console 可用）
2. **Agent 执行基础设施** — 云端 Agent 运行时 + 计算资源调配，跨开发环境可靠运行
3. **Agent 优化与上下文** — 跨仓库/项目的共享语义上下文层，实现任务路由到最合适的模型/工具

**关键架构设计**:
- **不锁定** — 可接入 Claude Agent、Codex、Gemini CLI、自定义 Agent，保留现有投资
- **语义层** — 持续聚合来自代码、架构、运行时行为、组织知识的结构化信息，使 Agent 获得"系统级理解"
- **Air / Air Team** — 开发者工作空间（Air App 已发布）+ 团队协调空间（Air Team 开发中），用于人和 Agent 协同工作
- **全工具集成** — 通过 Slack、Atlassian、Linear 等现有协作工具与 Agent 工作流对接

> **🎯 价值**: JetBrains 从 IDE 厂商正式转型为 **"AI-native 软件生产系统"基础设施提供商**。Central 对标的是 Cursor Enterprise / GitHub Copilot Enterprise 的企业治理层，但以"开放、不锁定"为差异化哲学。

### ⑤ Linear: Team Documents + Shared Skills + Issue Description Comments (Jun 4)

| 维度 | 内容 |
|:-----|:-----|
| **来源** | [Linear Changelog](https://linear.app/changelog) (Jun 4) |

**三大更新**:
1. **Team Documents** — 每个团队获得独立文档页面，用于存放不属于特定 Issue/Project 的团队级上下文（笔记、文档、参考链接），可按章节分组组织。从侧边栏点击团队名或 `o` + `t` 搜索进入
2. **Shared Skills** — Linear Agent 的 Skills 现支持**团队共享**。个人的高效工作流可以成为团队标准化流程，覆盖编写 specs、周计划、草拟更新等场景。可通过 Settings 或直接问 Linear Agent 分享
3. **Issue Description Comments** — 在 Issue 描述的任意文本或图片上**定点留言评论**，代替对整个 Issue 的泛化评论。反馈更精确
