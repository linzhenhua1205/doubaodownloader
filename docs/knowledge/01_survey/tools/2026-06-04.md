## 1️⃣ Cursor Enterprise Organizations — 多团队组织级治理 GA 🆕

- **来源**: [Cursor Blog - Introducing organizations for Cursor Enterprise (Jun 3, 2026)](https://cursor.com/blog/organizations)
- **工具**: Cursor Enterprise
- **核心更新**:

Cursor 为 Enterprise 客户引入了 **三层组织架构**，已于 6月3日 GA：

| 层级 | 说明 |
|:-----|:------|
| **Organization** | 公司身份/管理/成员关系的顶级容器，一个仪表盘管理所有团队 |
| **Teams** | 部门/子公司的运营单元，每个 team 有独立安全/预算/功能控制 |
| **Groups** | 跨 team 的轻量用户集合，无需新建 team 即可为特定人群设不同模型访问/消费限额/Agent 权限 |

**关键能力**:
- **IDP 一次性配置** — 组织级别配置身份提供商和 SCIM 目录源，自动同步成员身份
- **跨团队用量分析** — 组织仪表盘汇总所有团队的 spend/token 使用，可下钻到每个 team
- **用户可隶属多个 team** — 成员可在不同 team 拥有不同角色
- **管理 API + CSV 批量操作** — 通过 dashboard/API/CSV 在 team 间移动用户
- **新用户自动继承权限** — 加入 team 后自动应用设置
- **沙箱测试环境** — 可为新功能创建 staging team，先在有限范围验证再全量推广

**客户案例**: NVIDIA 使用 Cursor Organizations 隔离工程/产品 team（宽松网络+自动执行）与市场/财务 team（严格安全控制）。

- **价值说明**: 这是 AI 编程工具领域对标 GitHub Copilot Enterprise 多租户治理能力的标志性更新。当企业从几十人扩展到数千开发者时，统一的身份管理/预算控制/安全策略成为刚需。Cursor 的 `Organization → Teams → Groups` 三层模型比走 SaaS 管理的方向（如 Copilot 的 Business/Enterprise 两级）更灵活——Groups 层解决了"不需要新建团队但需要差异化权限"的长尾场景。结合此前 Auto-review Run Mode 的沙箱执行能力，Cursor 正在构建完整的 enterprise AI governance stack。

## 2️⃣ Bolt.new × Google Stitch — 一键设计交付 🔥

- **来源**: [Bolt Blog - Design in Stitch, build in Bolt (May 20, 2026)](https://bolt.new/blog/design-in-stitch-build-in-bolt)
- **工具**: Bolt.new + Google Stitch
- **核心更新**:

Google 的 AI 原生设计工具 **Stitch** 新增了 "Export to Bolt" 一键导出按钮。完整工作流：

1. **设计阶段** — 在 Stitch 中用 prompt/手绘/语音生成 UI 屏幕，支持实时渲染和原位编辑
2. **导出** — 点击 "Export to Bolt"，Stitch 打包完整上下文：布局、组件、设计符号、DESIGN.md
3. **构建** — Bolt 继承设计语言，直接开始构建带认证/数据库/API/业务逻辑的完整应用

**DESIGN.md 开放格式**：Google 与 Stitch 一同推出了 DESIGN.md 开放规范——一种 AI 原生可读的设计系统 Markdown 文件（颜色/字距/间距规则/组件模式）。Stitch 可以从 Figma 文件、线上网站或现有代码库生成 DESIGN.md。

**Stitch MCP 集成**：编码 Agent 可以从 Stitch 导入屏幕，并将视觉更改同步回代码库。设计师与开发者之间的反馈循环被极大缩短。

- **价值说明**: 这是"AI 工具链垂直分化"趋势的标志性事件。工具链正在拆分为设计层（Stitch/生成屏幕和视觉语言）、构建层（Bolt/接入数据库和业务逻辑），每一层做自己最擅长的事。DESIGN.md 作为开放格式可能成为 AI 时代的"sketch file"标准——让设计意图从 Figma → Stitch → Bolt 全程无损传递。对于开发者来说，这意味着不再需要用 prompt 描述视觉设计，设计决策随文件一起旅行。
