## 研究总纲

### 第一原则：分析原则

> **第一性原理、客观独立判断，不默认输入内容正确，严谨求证**

1. **不默认正确性**：任何信息来源（包括本知识库已有内容）都需要交叉验证，不因「之前写过」而默认成立
2. **循证优先**：结论须附带可追溯的证据链，避免诉诸权威或「众所周知」式陈述
3. **量化优先**：能用数据说话的问题，不用定性描述替代；数据缺位时明确标注「待量化」
4. **识别偏差**：对制造商/供应商/分析师提供的技术信息，需识别其立场偏向（如推新品的夸大、市场份额的自我报告偏差）

### 第二原则：研究流程

> **深度研究固定走「信息聚合 → 逻辑拆解 → 推理论证 → 方案沉淀 → 迭代复盘」闭环**

```
┌─────────────────────────────────────────────────────────┐
│                   研究闭环                              │
│                                                         │
│  信息聚合 ──→ 逻辑拆解 ──→ 推理论证 ──→ 方案沉淀        │
│     ↑                                      │           │
│     └────────── 迭代复盘 ←──────────────────┘           │
└─────────────────────────────────────────────────────────┘
```

| 阶段 | 目标 | 产出 |
|:----|:-----|:-----|
| **1. 信息聚合** | 收集一手来源信息，标注来源和日期 | 原始素材清单，含URL和时间戳 |
| **2. 逻辑拆解** | 拆解问题的构成要素，识别因果链和假设前提 | 问题分解树 + 关键假设清单 |
| **3. 推理论证** | 基于事实进行推理，对假设进行证伪验证 | 论证链：事实→推理→结论 |
| **4. 方案沉淀** | 将结论转化为可操作的方案或决策建议 | 行动项清单或架构决策说明 |
| **5. 迭代复盘** | 新信息到来时回溯验证已有结论，更新或修正 | 更新日志 + 修正理由说明 |

### 第三原则：信息来源

> **提供对应的信息来源，并进行充分验证，尽量采用第一手资料**

| 信息类型 | 一手来源优先 | 二手来源仅作发现入口 |
|:---------|:------------|:--------------------|
| 芯片/产品规格 | 厂商官方数据表/博客/白皮书 | 媒体报道、分析师评论 |
| 标准规范 | 标准组织官网（JEDEC/PCI-SIG/OCP/ODCC） | 第三方解读文章 |
| 学术研究 | 原始论文（arXiv/MLSys/ISCA/SIGMOD） | 新闻摘要或博客翻译 |
| 市场规模 | IDC/Gartner 官方报告（付费）或财报电话会 | 券商转引数据（需标注「转引自」） |
| 技术参数 | 实测数据或官方文档 | 论坛讨论、未经验证的声称 |
| 政策法规 | 政府/监管机构官方发布 | 媒体解读（区分事实和观点） |

**找不到一手来源时的处理规则**：
- 必须在输出中明确标注「未找到官方原文，以下内容来自[媒体名]报道，存在转述误差可能」
- 单一来源信息同样需标注
- 多个媒体引用同一来源不代表交叉验证

## ⚠️ 搜索质量记录（2026-05-30 更新）

本次搜索更新暴露了以下严重问题，需要记录在案：

### Bing 搜索中文质量极度下降
- **IDC 搜索**：Bing 对中文 "IDC" 完全混淆——搜索"AI 服务器 IDC 2026 份额"返回的是IDC数据中心/机房服务商，而非IDC国际数据公司（International Data Corporation）。所有相关关键词均受此影响
- **Vibe Coding 搜索**：Bing 的中文/英文搜索结果都只返回Vibe/Coding的词典释义和知乎基础介绍，无任何2026年新内容。英文关键词 "Vibe Coding trend 2026" 同样无效
- **企业治理搜索**：返回阿里千问、百度文心一言、豆包等广告页面占据整个首页，无有效结果
- **结论**：Bing 中文搜索当前已不适合作专业技术调研。如需有效搜索，需考虑：
  1. 切换至百度搜索（但本次百度也触发验证码阻止）
  2. 使用英文关键词 + site 限制
  3. 直接抓取已知权威源（如 arXiv / cursor.com / docs.anthropic.com / theverge.com）

### 成功获取信息的渠道（推荐）
- **学术论文**：arXiv.org（本次8篇高质量论文，全部来自arXiv，搜索直接用 site:arxiv.org + 精确关键词）
- **官方 changelog**：直接抓取产品官网 changelog（cursor.com/changelog、docs.anthropic.com）
- **聚合新闻站**：The Verge AI 频道、TechCrunch
