### 🔑 关键洞察

1. **InfiniBand直通GPU内核** — GICC证明InfiniBand上GPU可直接驱动NIC消除Host瓶颈，**229×延迟改善**暗示未来NIC需原生支持GPU-initiated通信
2. **互联带宽投资回报不对称** — CCL-Bench揭示：同样资金投在TPU互联上比GPU互联带来更大训练加速（中小规模），硬件投资决策需差异化
3. **NVLink-C2C的双重价值** — 既用作GPU互联（传统认知），也用作CPU↔GPU内存通道（C2CServe），GH200的异构内存架构潜力远未被充分利用
4. **RoCE的真实瓶颈不在线路** — SKT报告证实200Gbps RoCE利用率仅1.4-10.4%，瓶颈在存储栈(NFS RPC饱和)而非RoCE本身，这是**系统级而非网络级问题**
5. **光互联进入系统级设计** — ChipLight（DATE 2026）和3D SiN Interposer从"器件替代电互联"进入"跨层系统协同设计"阶段，标志着光互联成熟度提升

### 1️⃣ ASUS XA NB3I-E12 评测深度解构 — 9U 8×B300 空冷巨无霸（⭐⭐⭐⭐⭐）

- **来源**: [ServeTheHome](https://www.servethehome.com/asus-xa-nb3i-e12-review-a-massive-8x-nvidia-b300-gpu-server/), Patrick Kennedy, May 30, 2026（6月5日首页置顶）
- **架构全景**:

| 维度 | 详情 |
|:----|:-----|
| **尺寸** | **9U** 机架 — 比前代B200的8U更大，GPU散热器占了机箱一半以上高度 |
| **GPU子系统** | NVIDIA HGX B300 8-GPU（Blackwell Ultra），空冷设计 |
| **CPU** | Intel Xeon 6 (Granite Rapids P-core) |
| **网络** | **8×NVIDIA ConnectX-8 800Gbps XDR InfiniBand** — 每GPU独立800G链路，板载不是PCIe卡 |
| **总带宽** | >**6.4 Tbps** East-West GPU互联 |
| **存储** | 8×U.2 NVMe SSD（每GPU配一个）+ 额外PCIe扩展 |
| **管理** | IPMI + VGA + Q-CODE POST诊断屏 |
| **散热** | 纯空冷（ASUS策略：空冷可直接放入现有数据中心，不需要液冷改造） |
| **重量** | 4人才能搬动 — Patrick说以前80-100lb 8GPU他能单挑，现在要4个人了 |

- **与B200代相比的关键架构演进**:
  1. **ConnectX-8从PCIe插件→板载集成** — HGX B300 8 GPU母板上自带了8个ConnectX-8，不再需要额外插PCIe卡。这在B200代是定制模块，现在成了基础特性
  2. **OSFP笼编号不连续** — 官方走线要求按 `2,3,1,4,7,6,8,5` 顺序接线，而非简单1-8。说明高速信号层布局极其复杂
  3. **前端IO托盘可整体抽出** — 模块化设计，前端组件通过高密度连接器与CPU主板托架对接，便于PCIe卡/SSD热维修
  4. **机箱结构从简单加固→结构化钢架** — 散热器/GPU模块太重，必须加强整体钢性

- **架构意义**: ⭐⭐⭐⭐⭐ — 这是目前公开评测中最详细的 **B300 HGX 8-GPU空冷整机设计解剖**。核心信号：**NVIDIA正在将ConnectX从可选的PCIe生态组件变成HGX基板的基础组件**，进一步固化NVIDIA端到端生态锁定。同时，**空冷9U设计**证明了B300（1000W+/GPU）仍可空冷运行，但代价是机箱尺寸和重量的显著增长。

### 2️⃣ ODCC正式发布三项超节点互联规范 — Switchless+OISA+液冷超流体（⭐⭐⭐⭐⭐）

- **来源**: [ODCC官网](https://www.odcc.org.cn/) 成果发布区, 2026-05~06发布
- **URL**: odcc.org.cn → 成果发布 → 规范标准/白皮书

#### 新规范一：ODCC2504002 — Switchless Scale Up GPU超节点互联系统架构技术规范（🔥重点）

- **发布工作组**: 边缘计算工作组
- **核心定位**: 针对大规模AI算力集群在互联扩展性、效率与成本方面的挑战，围绕 **"Switchless Scale Up + Switch Scale Out"双向融合系统架构** 展开
- **关键技术方案**: **分布式光交换电路（dOCS）** 光互连技术
- **战略目标**: 打破当前GPU超节点互联的 **"闭源垄断困局"与"开源滞后瓶颈"** 两大痛点
- **架构创新**: 采用dOCS光互连，Switchless Scale Up域内GPU直接互联、无需交换机；Scale Out仍使用Switch架构
- **行业影响**: ⭐⭐⭐⭐⭐ — 这是中国首个**面向超节点互联的开放标准**，直接对标NVIDIA NVLink/NVSwitch封闭生态。dOCS光互连方案为国产GPU互联提供了开放替代路径

#### 新规范二：ODCC2504003 — OISA全向智感互联IO芯粒技术白皮书

- **发布工作组**: 服务器工作组
- **核心定位**: 通过 **IO芯粒（IO Chiplet）技术** 深度融合GPU与交换芯片
- **技术路线**: 
  - 打破传统板级互联的带宽与延迟瓶颈
  - IO芯粒作为GPU和Switch之间的桥梁
  - 面向下一代计算与网络一体化的技术路线

- **CVSS 3.1**: **5.3 MEDIUM** — `AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:L`
  - 攻击向量: **网络远程**
  - 权限要求: **无需认证**
  - 用户交互: **无需**
  - 影响: 仅可用性（拒绝服务），不影响机密性和完整性
- **漏洞类型**: 未认证远程攻击者可对 IBM OpenBMC 设备发起拒绝服务攻击
- **影响分析**: 🟡 **中等风险** — 虽然 CVSS 仅 5.3，但攻击条件极低（网络可达、无需认证），对部署 IBM OpenBMC 设备的运维环境构成实际威胁。IBM 已发布安全公告（需登录查看具体修复版本）
- **与上游 OpenBMC 的关系**: 此 CVE 影响的是 IBM 分支（FW1110），上游社区版尚未确认是否受影响，建议追踪上游是否合入对应修复
