## 1. 总览：三大 ML 技术 + EDA 的交叉点

### 1.1 ML 在 EDA 中解决的三大类问题

| ML 问题类型 | 传统方法的局限 | ML 方案 | 代表技术 |
|:-----------|:-------------|:--------|:---------|
| **预测/估计** | SPICE 仿真太慢、STA 计算量大 | 用 ML 模型替代昂贵仿真 | **GNN**（图结构预测） |
| **组合优化** | 布局/布线是 NP-Complete，解空间巨大 | 用 RL 学习搜索策略 | **RL**（强化学习） |
| **黑盒优化** | 工艺参数调优、SPICE 参数拟合，目标函数无梯度 | 用概率代理模型高效采样 | **贝叶斯优化**（BO） |

### 1.2 EDA 流程中的 ML 切入点

```
EDA 全流程 ML 切入点：
┌────────────────────────────────────────────────────────────┐
│ 逻辑设计                │ ┌─ GNN for 网表分析            │
│                         │ └─ ML 综合质量预测             │
├────────────────────────────────────────────────────────────┤
│ 物理设计                │ ┌─ RL for Macro Placement      │
│                         │ ├─ GNN for 布局拥塞预测        │
│                         │ ├─ RL for Global Routing       │
│                         │ └─ BO for STA 时序优化         │
├────────────────────────────────────────────────────────────┤
│ 模拟设计                │ ┌─ BO for 器件参数优化         │
│                         │ ├─ GNN for 电路性能预测        │
│                         │ └─ RL for 版图自动化          │
├────────────────────────────────────────────────────────────┤
│ 验证与测试              │ ┌─ GNN for 缺陷预测            │
│                         │ └─ ML for 测试覆盖优化         │
├────────────────────────────────────────────────────────────┤
│ 良率与可靠性            │ ┌─ BO for 工艺参数窗口         │
│                         │ └─ GNN for 良率预测            │
└────────────────────────────────────────────────────────────┘
```

### 1.3 三大技术的核心定位

| 技术 | 核心优势 | 核心缺陷 | 在 EDA 中最适合的阶段 |
|:-----|:---------|:---------|:-------------------|
| **GNN** (图神经网络) | 理解图结构关系（网表、电路图） | 需要大量标注数据、泛化性有限 | 时序/拥塞/功耗预测 |
| **RL** (强化学习) | 解决组合优化问题（布局、布线） | 训练时间长、Reward 设计困难 | 物理设计中的布局/布线 |
| **BO** (贝叶斯优化) | 高效黑盒优化（采样次数少） | 高维空间效果差 | 参数调优、工艺优化 |

## 2. GNN for EDA：图神经网络的芯片设计应用

### 2.1 为什么芯片设计天然适合图神经网络

### 2.1.1 芯片设计的图结构本质

芯片设计的每个抽象层次都可以表示为图：

```
芯片设计的层次化图结构：

网表级 (Netlist):
    ┌─────┐   net1   ┌─────┐
    │ AND │───────────│ OR  │
    │ 门   │          │ 门   │
    └─────┘          └─────┘
       │                │
    net2              net3
       │                │
       ▼                ▼
    ┌─────┐          ┌─────┐
    │ FF  │          │ AOI │
    └─────┘          └─────┘
    → 图表示：节点 = 标准单元/宏单元
              边 = 网 (net)

物理级 (Layout):
    ┌──────────────────┐
    │  ┌──┐ ┌──┐      │
    │  │FF│ │FF│──────│  → 图表示：节点 = 物理单元实例
    │  └──┘ └──┘      │    边 = 物理/电连接
    │  ┌──┐ ┌──┐      │    节点特征 = 位置(x,y)/尺寸/功耗
    │  │OR│ │AN│      │
    │  └──┘ └──┘      │
    └──────────────────┘

电路级 (Schematic):
    ┌─────────── VDD ─────────┐
    │                         │
    ├──┬──┬──┬──┬──┬──→       │  → 图表示：节点 = 器件/节点
    │M1│M2│M3│M4│R1│→        │    边 = 连接关系
    └──┴──┴──┴──┴──┴──→       │    节点特征 = W/L/类型
    │                         │
    └───── GND ──────────────┘
```

**这是 GNN 的天然战场**——芯片设计是极少数"完整、精确、层次化"的图结构数据源。

### 2.1.2 传统机器学习 vs GNN 在 EDA 中的差异

| 维度 | 传统 ML（如 MLP/RF/XGBoost） | GNN 方案 |
|:-----|:---------------------------|:---------|
| **输入形式** | 手工特征（如扇出、跳数、面积） | 原始图结构（网表/电路） |
| **特征工程** | 需要大量手工设计领域特征 | 自动学习节点和边特征 |
| **关系建模** | 无法直接建模拓扑关系 | 通过消息传递自然建模 |
| **泛化能力** | 对未见结构泛化差 | 学习结构模式，泛化更好 |
| **可扩展性** | 固定维度向量 | 处理变图大小 |
| **典型精度** | 预测误差 ~15-25% | 预测误差 ~5-15% |

### 2.2 GNN 核心技术原理（GCN/GAT/MPNN/GIN）

### 2.2.1 消息传递框架

几乎所有用于 EDA 的 GNN 都基于**消息传递（Message Passing）**框架：

```
消息传递神经网络 (MPNN) 通用框架：

初始化：
    h_v^(0) = x_v  (节点 v 的初始特征)

第 k 层消息传递：
    m_v^(k) = AGGREGATE({ M^(k)(h_v^(k-1), h_u^(k-1), e_uv) for u ∈ N(v) })
    h_v^(k) = UPDATE^(k)(h_v^(k-1), m_v^(k))

其中：
    M^(k) = 消息函数（通常为 MLP）
    AGGREGATE = 聚合函数（sum/mean/max）
    UPDATE = 更新函数（通常为 MLP + 残差连接）
    N(v) = 节点 v 的邻居集合
    e_uv = 边 (u,v) 的特征

读出：
    对于图级任务：h_G = READOUT({h_v^(K) for v ∈ G})
    对于节点级任务：直接使用 h_v^(K)
```

### 2.2.2 主流 GNN 变体对比

| GNN 类型 | 消息/聚合方式 | EDA 中的优势 | EDA 中的劣势 |
|:---------|:------------|:------------|:------------|
| **GCN** (Graph Convolutional Network) | 度归一化的邻域平均 | 计算简单、快速 | 无法区分不同邻居重要性 |
| **GAT** (Graph Attention Network) | 注意力权重的加权平均 | 自动学习邻居重要性 | 额外注意力参数 |
| **GIN** (Graph Isomorphism Network) | Sum 聚合 + MLP | 理论上最强表达力 | 对噪声敏感 |
| **MPNN** (Message Passing NN) | 可定制消息/聚合 | 最灵活、可针对网表定制 | 设计空间太大 |
| **GraphSAGE** | 采样邻居聚合 | 可扩展到超大规模图 | 信息丢失 |
| **Edge-Conditioned Conv** | 边特征加权 | 自然建模 net 属性 | 实现复杂 |

### 2.2.3 EDA 特有的 GNN 设计考虑

| 考虑 | 问题 | 常用方案 |
|:-----|:-----|:---------|
| **有向 vs 无向** | 网表是单向的（从输出到输入） | 使用有向 GNN 或双向 GNN |
| **边特征** | Net 含重要信息（RC 延时、扇出） | 边特征自然编码 |
| **层次化** | 芯片设计是层次化图 | Hierarchical GNN / 多级池化 |
| **超大规模** | 现代芯片 ~10M+ 节点 | 分块 + 采样 + GNN on subgraphs |
| **异质图** | 不同类型的节点（门/宏/端口） | Heterogeneous GNN / 节点类型嵌入 |

### 2.3 网表级应用：时序/拥塞/功耗预测

### 2.3.1 时序预测（Timing Prediction）

**为什么需要 GNN 预测时序？**

传统 STA 流程：
```
逻辑综合 → 网表 → 寄生提取 → SPICE → STA(PrimeTime) → 时序报告
                               ↑
                       布局前只能估算，误差可达 30-50%
```

GNN 方案：**在布局前或布局早期预测时序路径延迟**

| 方案 | 输入 | 输出 | 精度 | 速度提升 |
|:-----|:-----|:-----|:-----|:---------|
| **TimingGNN** (2022, UTexas) | 网表图 + 扇出/扇入特征 | 路径延迟分布 | MAE ~8-12ps | 1000× vs SPICE |
| **GraphPlanner** (2021, Google) | 网表图 + 物理约束 | 各节点 slack | R² >0.85 | — |
| **DeepTiming** (2020, UCLA) | 网表图 + 线负载估算 | 门级延迟 | RMSE ~15ps | 100× vs STA |
| **GAT4Timing** (2023, Cadence) | 异质网表图 | 路径延迟 | MAE ~6ps | — |

**GNN 时序预测的典型架构**：

```
输入：网表图
节点特征：门类型(独热编码)、扇出数、扇入数、驱动强度、输入转换时间
边特征：net 权重（估计的 RC 负载）

GNN encoder → 节点嵌入

预测头：
    - 门延迟 MLP head: h_v → 门延迟
    - 路径延迟 AGG: 通过求和/RNN 聚合路径上的节点嵌入
    - 建立到点延迟: 从启动 FF 沿路径到捕获 FF

输出：每条路径的建立时间/保持时间 slack
```

### 2.3.2 拥塞预测（Congestion Prediction）

**传统方法**：完成布局后运行拥塞分析工具（如 IC Compiler 或 Innovus 的拥塞地图），耗时数小时。

**GNN 方案**：在布局早期（甚至 pre-placement）预测拥塞。

| 方案 | 输入 | 输出 | 精度 | 应用 |
|:-----|:-----|:-----|:-----|:-----|
| **RouteNet** (2020, Cornell) | 网表 + RUDY 密度 | 每 bin 拥塞分数 | IoU ~0.7 | 拥塞热点预测 |
| **CongestionNet** (2021, UTexas) | GAT 网表图 | 全局/局部拥塞 | 真阳性率 >85% | 布局约束生成 |
| **GRCNN** (2022, Google) | 网表 + 布局密度图 | 拥塞热力图 | SSIM >0.8 | 布局质量评估 |
| **NVCell** (2022, NVIDIA) | GNN + CNN 混合 | 拥塞分布 | — | 标准单元布局 |

### 2.3.3 功耗预测（Power Prediction）

| 场景 | GNN 输入 | 输出 | 传统方法 | GNN 提升 |
|:-----|:---------|:-----|:---------|:---------|
| 动态功耗 | 网表图 + 开关活动因子 | 每门动态功耗 | SPICE 或向量仿真 | 100-1000× 更快 |
| 静态功耗 | 网表图 + 电压/温度 | 漏电分布 | 查找表 | 更精细粒度 |
| IR Drop | 网表 + 供电网络图 | 电压降分布 | Redhawk/Voltus | ~50× 更快 |

### 2.4 物理设计级应用：布局质量评估/IR Drop 预测

### 2.4.1 布局质量评估

GNN 可以在布局完成后快速评估布局质量，指导布局迭代：

```
布局质量评估的 GNN 方法：

输入：
    - 网表图 G = (V, E)
    - 物理位置特征 (x, y, width, height)
    - 宏单元/标准单元标记

GNN 处理流程：
    1. 构建物理-逻辑联合图
       - 逻辑边（网表连接）
       - 物理边（空间邻近连接）
    
    2. 消息传递：在联合图上传播信息
       - 聚合逻辑邻居的物理信息
       - 聚合物理邻居的逻辑信息
    
    3. 预测头：
       - 拥塞预测（每 bin）
       - 路径延迟预测（每条路径）
       - 总线长估计（HPWL + Steiner 修正）
       - 功耗密度估计

输出：布局质量评分（0-100）
```

### 2.4.2 IR Drop 预测

IR Drop 分析的 GNN 方案将供电网络建模为图：

```
供电网络图（Power Grid Network）：
    ┌───┬───┬───┬───┐
    │ P │ P │ P │ P │  节点 = 供电网络交叉点
    ├───┼───┼───┼───┤  边 = 金属线段（有电阻）
    │ P │ C │ C │ P │  节点特征：电压、电流负载
    ├───┼───┼───┼───┤  边特征：电阻、EM 约束
    │ P │ C │ C │ P │
    ├───┼───┼───┼───┤  预测目标：每节点的 IR Drop
    │ P │ P │ P │ P │
    └───┴───┴───┴───┘
```

| 方案 | 网络规模 | 精度 (MAPE) | 速度提升 |
|:-----|:--------|:------------|:---------|
| GNN-IR (2022, UTexas) | 10K 节点 | ~3-5% | 100× vs Redhawk |
| PowerNet (2023, UCLA) | 100K 节点 | ~5-8% | ~50× vs 商用工具 |
| GraphIR (2024, Synopsys) | 1M+ 节点 | ~2-4% | ~30× vs Voltus |

### 2.5 电路级应用：器件建模/良率预测

### 2.5.1 晶体管级 GNN 建模

在模拟/定制电路设计中，GNN 可以替代部分 SPICE 仿真：

| 应用 | 输入 | 输出 | 精度 | 加速比 |
|:-----|:-----|:-----|:-----|:-------|
| **器件 I-V 建模** | 电路拓扑图 + 偏置条件 | 支路电流/节点电压 | <5% 误差 | 10000× |
| **感测放大器偏移** | 晶体管配对图 + 工艺偏差 | Vth 失配 | <10% 误差 | — |
| **ADC 线性度** | 比较器阵列图 | DNL/INL | <3% 误差 | — |
| **PLL 相位噪声** | 环路框图 + 器件参数 | 相位噪声曲线 | ~5dB 精度 | — |

### 2.5.2 良率预测（Yield Prediction）

```
良率预测的 GNN 方法：

传统流程：制造 → 晶圆测试 → 良率数据 (耗时数周)
GNN 目标：在流片前预测良率

GNN 输入：
    - 芯片版图图 (layout graph)
    - 工艺缺陷参数矩阵
    - 关键面积 (Critical Area) 分布

预测：每芯片良率、主要缺陷类型

精度：与实测良率偏差 <3%（在已知工艺节点上）
```

### 2.6 GNN 的局限性与挑战

| 挑战 | 具体问题 | 当前方向 |
|:-----|:---------|:---------|
| **泛化性** | 在训练集外的电路上精度显著下降 | 预训练 + 微调、领域自适应 |
| **数据稀缺** | 高质量标注数据（STA/IR 结果）获取成本高 | 主动学习、弱监督、仿真→数据 |
| **图规模** | 现代芯片 ~10M 节点，全图 GNN 不可行 | 子图采样、层次化 GNN |
| **等变性** | 物理布局的平移/旋转等变性未被 GNN 保证 | 引入 SE(2)/SE(3) 等变层 |
| **时序动态** | 网表是静态的，但信号是动态的 | 时序-结构联合建模 |
| **可解释性** | 为什么预测该路径延迟高？ | GNNExplainer / 注意力可视化 |

## 3. RL 布局优化：强化学习在物理设计中的应用

### 3.1 布局问题的数学形式化

### 3.1.1 宏单元布局（Macro Placement）的标准形式

```
宏单元布局问题的形式化定义：

输入：
    - 一组宏单元 M = {m_1, m_2, ..., m_n}
    - 每组宏单元有尺寸 (w_i, h_i)
    - 网表连接的度量为：Hypergraph H 连接宏单元与标准单元
    
约束：
    - 宏单元间不重叠
    - 宏单元在芯片边界内
    - 固定宏（如 PHY/IO 块）位置固定
    
优化目标（多目标）：
    1. 总线长最小化: min ∑_{net ∈ Nets} WL(HPWL_net)
    2. 拥塞最小化: min ∑_{bin} Congestion_bin
    3. 时序最优: max slack(各路径)
    4. 功耗密度最小化: min max(PowerDensity_bin)

这是一个 NP-Complete 问题 + 多目标优化问题。
```

### 3.1.2 为何传统方法不足

| 传统方法 | 策略 | 问题 |
|:---------|:-----|:-----|
| **解析法** (ePlace/RePlAce/DREAMPlace) | 连续优化 + 合法化 + 详细布局 | 陷入局部最优 |
| **模拟退火** (VPR/TR) | 随机扰动 + 接受准则 | 极慢、无学习 |
| **划分法** (Capo/UMPack) | 递归二分 | 忽略全局时序 |

## 4. 贝叶斯优化在芯片设计中的应用

### 4.1 贝叶斯优化的核心原理

### 4.1.1 为什么需要贝叶斯优化？

芯片设计中有大量优化问题的目标函数是**黑盒、昂贵、无梯度**的：

| 场景 | 目标函数 f(x) | 每次评估成本 | 评估频率 |
|:-----|:-------------|:-----------|:---------|
| SPICE 仿真参数调优 | 电路仿真结果（如增益、带宽） | 分钟~小时 | 数百次 |
| TCAD 工艺参数优化 | 器件电学特性 | 小时~天 | 数十次 |
| STA 时序优化 | P&R 后的时序结果 | 小时 | 数十次 |
| 功耗优化 | P&R 后的功耗报告 | 小时 | 数十次 |

**贝叶斯优化（Bayesian Optimization, BO）** 是为这种场景设计的：用最少的评估次数找到最优解。

### 4.1.2 BO 的核心公式

```
贝叶斯优化的迭代循环：

1. 构建概率代理模型（Surrogate Model）：
   P(f | X, y) = GP(μ(x), k(x, x'))
   其中 GP 是高斯过程（Gaussian Process）
   
2. 选择采集函数（Acquisition Function）：
   α(x) = Expected Improvement (EI)
        = E[max(0, f(x) - f(x_best))]
   或
   α(x) = Upper Confidence Bound (UCB)
        = μ(x) + κ × σ(x)

3. 采集函数最大化：
   x_next = argmax α(x)

4. 评估实际目标函数：
   y_next = f(x_next)

5. 更新代理模型：
   添加 (x_next, y_next) 到历史数据，重新训练 GP
```

### 4.1.3 BO 在 EDA 中的优势与局限

| 维度 | BO 的优势 | BO 的局限 |
|:-----|:---------|:---------|
| **样本效率** | 只需数十~数百次评估 | 高维空间 (D>20) 下效率骤降 |
| **不确定性量化** | 天然提供预测置信度 | GP 不确定性估计在高维不准确 |
| **噪声鲁棒** | GP 可以处理噪声观测 | 需要正确设置噪声参数 |
| **并行化** | 支持批处理 BO | 复杂度增加 |
| **多目标** | 支持多目标 BO | 采集函数设计更复杂 |

### 4.2 模拟电路设计空间探索

### 4.2.1 问题形式化

```
模拟电路优化问题（以运放为例）：

设计变量 x：
    - 晶体管尺寸 (W1, W2, ..., Wn)
    - 偏置电流 Ibias
    - 补偿电容 Cc
    - 负载电容 CL

目标函数 f(x)：
    - 直流增益 (Av)              → 最大化
    - 单位增益带宽 (GBW)          → 最大化 
    - 相位裕度 (PM)               → 在 >60° 下最大化
    - 功耗                      → 最小化
    - 面积                      → 最小化

约束 g(x)：
    - 所有晶体管处于饱和区
    - 输出电压摆幅 > VDD - 0.3V
    - 共模输入范围 > 目标值

BO 流程：
    1. 定义设计空间（每个 W/L 的范围）
    2. 初始采样（Latin Hypercube, ~20 点）
    3. 运行 SPICE 仿真评估这些设计点
    4. 训练高斯过程代理模型
    5. 优化采集函数得到下一个要评估的点
    6. 重复 3-5 步 50-200 次
    7. 输出 Pareto 前沿
```

### 4.4 时序/功耗/面积的 Pareto 优化

### 4.4.1 多目标 BO 在 PPA 中的应用

```

变量 x：
    - 逻辑综合参数（retiming, resynthesis, 面积/时序权衡）
    - 物理设计参数（placement density, max_util, spare cells）
```

| **高维失效** | D>20 时 GP 插值效果差 | 深度核、贝叶斯神经网络 |
| **离散变量** | 工艺参数包含离散选择 | 混合 BO (GP + 随机森林) |
| **约束处理** | 可行性约束（如 Fmax 约束） | 约束 BO (EIC) |
| **多个局部最优点** | GP 易忽略多模态 | 集成的采集函数 |
| **并行评估** | 仿真可并行但标准 BO 串行 | QEI, LP-EI (批处理) |
| **时序依赖** | 设计参数随工艺漂移 | 动态 BO |

### 5.2 技术协同方案

### 5.2.1 GNN + RL 协同

```
GNN + RL 协同框架：
    1. GNN 为 RL 提供状态编码
       网表图 → GNN → 图嵌入 → RL 策略网络的输入
    
    2. GNN 为 RL 提供快速预测奖励
       布局中间状态 → GNN → 快速 PPA 预测 → 即时奖励
    
    3. 实际案例：
       - Google Floorplanning: GNN 编码网表拓扑 → RL 选择宏单元位置
       - MaskPlace: 注意力编码（类似 GNN）→ RL 放置
       - ChipPlace: Transformer（也可视为图模型）→ RL
```

### 5.2.2 GNN + BO 协同

```
GNN + BO 协同框架：
    1. GNN 作为 BO 的低维嵌入
       高维设计变量 → GNN encoder → 低维嵌入 → GP 代理模型
       解决 BO 在高维空间中的维度灾难
    
    2. GNN 提供快速预筛选
       BO 提出候选点 → GNN 快速预测性能 → 筛选关键点做 SPICE
    
    3. 实际案例：
       - 模拟优化: GNN 编码拓扑 + 设计参数 → BO 选出最值得 SPICE 的点
       - PPA 优化: GNN 预测路径延迟 → BO 优化综合参数
```

### 5.2.3 RL + BO 协同

```
RL + BO 协同框架：
    1. BO 为 RL 做超参数优化
       RL 训练中有大量超参数（学习率/权重/奖励系数）
       BO 自动调参，提升 RL 训练效率
    
    2. RL + BO 交替优化
       - RL 负责宏单元布局（组合优化部分）
       - BO 负责工艺/综合参数（连续优化部分）
       - 交替迭代，直至 PPA 收敛
    
    3. 实际案例：
       - DSO.ai: BO 优化综合参数 → 内部调用 RL 引擎进行布局分析
       - Cerebrus: BO 优化布局参数
```

### 7.2 数据匮乏问题的破解

ML for EDA 面临的核心问题是**标注数据匮乏**——每次评估（跑一次 P&R 或仿真）都需要数小时甚至数天：

| 方向 | 方法 | 进展 |
|:-----|:-----|:-----|
| **迁移学习** | 从旧设计微调到新设计 | 有效，减少 50% 训练数据 |
| **半监督学习** | 少量标注 + 大量无标注数据 | 研究阶段 |
| **弱监督学习** | 不精确标注（如线长估算代替完整 P&R） | Google 使用 |
| **仿真 to 真实** | 合成数据预训练后微调到真实数据 | 有进展 |
| **数据增强** | 网表扰动、布图变换 | 受限于物理合理性 |
| **主动学习** | 选择最有信息量的点标注 | 已用于 BO 流程 |

### 7.3 可解释性与信任

| 挑战 | 具体问题 | 重要性 |
|:-----|:---------|:-------|
| **为什么选择这个布局？** | RL/GNN 不提供决策解释 | 高——工程师需要信任工具 |
| **何时会失败？** | ML 模型在分布外样本的失效行为不可预测 | 高——流片成本高 |
| **约束满足保证** | ML 方法不保证满足所有 DRC/时序约束 | 高——需要验证层兜底 |
| **鲁棒性** | 输入噪声对 ML 预测的影响 | 中 |

**当前方案**：
- GNNExplainer 用于图模型解释
- RL 的奖励分析和动作热力图
- 约束始终由传统工具最终验证
- 人类-in-the-loop 的审批机制

### 7.4 端到端 ML-EDA 流程

```
理想的 ML for EDA 端到端流程：

RTL
  │
  ▼
┌─────────────────────────────────────────┐
│  Foundation Model for Chip Design       │
│  ┌───────────────────────────────────┐ │
│  │ • 从 RTL 直接预测高质量布局       │ │
│  │ • 一步到位满足 PPA 目标          │ │
│  │ • 零手工参数调整 (zero-shot)     │ │
│  └───────────────────────────────────┘ │
└─────────────────────────────────────────┘
  │
  ▼
传统 EDA 工具（验证 + 合规检查）
  │
  ▼
Tape-out
```

这是一种愿景，但当前离现实还有相当距离——**最大的障碍**是：
1. **ML 模型无法保证收敛性**——传统布局器至少能输出一个合法解
2. **验证负担转移**——ML 输出需要更多验证，而非更少
3. **碎片化设计**——每种芯片需要的约束都不同，ML 难以泛化

## 8. 综合方法论与交叉引用

### 8.1 核心洞察

1. **GNN 是 EDA 的"原生 ML 语言"**：芯片设计的所有抽象层次（网表、电路、版图）都是图结构。GNN 能自动学习拓扑特征，其效果远超手工特征工程。这是 ML for EDA 中最具潜力的方向。

2. **RL 解决的是 EDA 中最难的问题（布局/布线）**：布局是 NP-Complete，RL 通过学习搜索策略直接解组合优化。Google 的 Nature 2021 已证明可用于实际芯片，但**泛化性**和**训练成本**仍是主要障碍。

3. **BO 是 EDA 中落地最成功的 ML 技术**：DSO.ai/Cerebrus 等商用产品已在 160+ 个芯片 tape-out 中使用。BO 的"低样本需求"特性天然适配 EDA 场景——每次评估成本高，不允许大量采样。

4. **三大技术各有所长，最佳方案是协同**：GNN 负责快速预测（替代昂贵仿真）、RL 负责组合优化（布局/布线）、BO 负责参数调优。三者协同可以覆盖 EDA 全流程的大部分环节。

5. **ML for EDA 的落地速度远超预期**：2020 年 DSO.ai 发布时被认为是"噱头"，到 2024 年已有 160+ 个 tape-out 使用。DAC 2024 上 35% 的论文涉及 ML。趋势不可逆。
