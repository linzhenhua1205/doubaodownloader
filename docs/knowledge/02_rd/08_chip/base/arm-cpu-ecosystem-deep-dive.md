# ⚡ 国产 ARM 服务器 CPU 生态深度调研：博瑞晶芯 vs 飞腾 vs 华为鲲鹏

> **文档状态**: v1.0 首次创建 | **调研日期**: 2026-06-30 | **覆盖范围**: 三家国产 ARM 服务器 CPU 供应商的公司背景、技术栈、产品线与生态体系全景对比
>
> **核心发现**: 三家 ARM CPU 厂商代表了国产服务器芯片的三条截然不同的发展路径——鲲鹏是全栈垂直整合+开源生态，飞腾是「国家队」横向生态联盟，博瑞晶芯是初创企业 ARM v9 授权突围

## 2. 公司背景与资本结构

### 2.1 飞腾信息技术有限公司（Phytium）

**成立与股东背景**:
- 2014年8月21日在天津滨海新区成立
- 由中国电子信息产业集团（CEC）、天津市滨海新区政府、天津先进技术研究院联合支持成立
- 定位：**CPU 研发设计的「国家队」**，国内领先的自主核心芯片提供商
- 总部天津，在北京、长沙、成都、广州、深圳、上海、西安、南京设有子公司，武汉、银川、沈阳、海口等地设有办事处

**国家背书**:
- 2019年，习近平总书记天津考察时对飞腾工作给予充分肯定
- 获得国家科技进步一等奖、科技部重点领域创新团队等荣誉
- 属于国资委央企科技创新体系

**品牌来源**:
- 「飞腾」出自屈原《楚辞·离骚》：「吾令凤鸟飞腾兮，继之以日夜」
- 技术演进已走过20余年历史（从国防科大早期处理器研发算起）

**经营数据**:
- 累计销售芯片超过 **1300 万片**
- 生态伙伴超过 **8500 家**
- 硬件方案超 **7100 款**
- 软件适配超 **15 万种**
- 兼容 **200 万级** 移动 APP 应用
- 覆盖政务/金融/电信/电力/能源/交通/医疗/教育八大领域
- 政府信息化实现「三个全覆盖」：200+集成商、四级政府（部委-省-市-县乡）全覆盖、办公系统到核心应用全覆盖

### 2.2 华为鲲鹏（Kunpeng）/ 海思半导体

**母公司架构**:
- 鲲鹏处理器由 **海思半导体（HiSilicon）** 设计研发，海思是华为全资子公司（成立于2004年）
- 鲲鹏品牌正式发布于 **2019年**，定位通用计算处理器
- 鲲鹏计算产业生态由华为企业业务 BG 运营

**资本与抗风险能力**:
- 华为集团支撑，研发投入不受外部融资影响
- 受美国制裁影响：2020年起台积电断供，依靠库存和内部产能调配
- 工艺受限：鲲鹏920为7nm（2020年量产），后续迭代受阻于先进制程获取
- 通过 Chiplet 架构（鲲鹏930传闻）绕限制程限制

**品牌与市场影响**:
- 全球首款双线程 ARM 服务器处理器
- 「岭深（Lingshen）」超算——中国首个 CPU-only 百亿亿次超算，47,000 颗鲲鹏处理器实现 2 Exaflops
- 在中国 ARM 服务器 CPU 市场占据最大存量份额

### 2.3 博瑞晶芯科技（Borui Jingxin）

**公司简介与时间线**:
| 时间 | 事件 |
|:-----|:-----|
| 2021年4月 | 在深圳注册成立，初始注册资本 **80 亿人民币**（行业轰动级） |
| 2021-2023 | 前 Arm 中国高管团队组建，获 ARM v9 架构授权 + 全套最新 IP 授权 |
| 2024年底 | 爆发欠薪风波，公司陷入停摆，多位高管离职 |
| 2025年5月 | 总部从深圳迁至珠海 |
| 2026年1月 | **V 字回归**：获珠海新质生产力基金等国资机构超 **10 亿元** 融资，结清欠薪和供应商欠款 |
| 2026年至今 | 完成重组整合，轻装上阵，聚焦 ARM 服务器芯片设计 |

**股权与资本背景**:
- 初始 80 亿注册资本的来源引发外界诸多猜测（前 Arm 中国高管团队背景）
- 2024年欠薪风波后，**珠海新质生产力基金**（国资）等机构投资超 10 亿元
- 重组后运营趋于稳定

**创始团队**:
- 创始团队含多位 **前 Arm 中国（Arm China）高管**，引致外媒对其背景的广泛关注
- 创始团队与核心研发人员来自国际与国内知名处理器设计企业，平均 20 年以上经验
- 在上海、北京、成都、深圳设有研发中心

**战略定位**:
- 声称要做「**国产 ARM 服务器芯片领域的核心突破者**」
- 已获得的 ARM V9 架构授权和 ARM 全套最新 IP 授权是其最大差异化优势
- 瞄准信创市场，提供高性能、多元化的芯片解决方案
- 定位为「开放型计算芯片设计平台」

> **⚠️ 重要说明**: 截至2026年6月，博瑞晶芯**尚未有公开确认的流片或量产芯片**。以下信息基于公开报道和行业推测。

| 维度 | 已知信息 |
|:-----|:----------|
| 架构 | 已获 **ARM v9 架构授权** + ARM 全套最新 IP 授权 |
| 定位 | 服务器 CPU（ARM 架构），对标鲲鹏/飞腾服务器级产品 |
| 内核 | 大概率使用 **ARM Cortex 公版 IP**（非自研核） |
| 工艺 | 未公布（可能使用先进工艺） |

**差异化定位**:

- ARM v9 授权是其最大亮点——相比飞腾/鲲鹏的 ARM v8，v9 引入 SVE2 矢量扩展、更安全的架构

### 4.1 飞腾：信创「国家队」横向生态联盟

**生态规模**（截至2026年3月）:

| 指标 | 数据 |
|:-----|:----:|
| 生态伙伴 | **8500+** |
| 硬件方案 | **7100+** 款（飞腾平台设备上市数） |
| 软件适配 | **150000+** 种 |
| 移动 APP 兼容 | **200万+** 级 |
| 芯片累计销量 | **1300万+** 片 |
| 政府覆盖 | 部/省/市/县/乡镇全部覆盖 |

**生态架构**:
```
飞腾生态 → 三大计划
├── 飞腾生态伙伴计划（通用硬件/软件伙伴）
│   ├── 硬件适配列表（合作伙伴开发基于飞腾的设备）
│   ├── 软件适配认证（OS/数据库/中间件/应用）
│   └── 联合认证（深度合作）
├── 鹏腾生态伙伴计划（与华为鲲鹏共建）
│   └── 鹏腾生态 = 飞腾CPU + 鲲鹏CPU 统一生态
└── 产业创新
    ├── 生态服务中心
    └── 开放实验室（适配资源/5G通信/智能计算）
```

**软件栈支持**:
- **操作系统**: 麒麟OS、统信UOS、麒麟信安、湖南麒麟等全系列国产OS
- **数据库**: 达梦、人大金仓、南大通用、神舟通用、OceanBase、TiDB 等
- **中间件**: 东方通、中创、普元等
- **办公/应用**: 金山WPS、永中Office、数科OFD 等全系列办公软件
- **云平台**: 华为云Stack、阿里云、腾讯云、UCloud 等适配

**行业联合解决方案**（1000+套）:
- 党政办公系统全覆盖
- 电信核心业务系统
- 金融核心交易系统
- 电力调度系统
- 交通收费系统
- 医疗信息系统

> **飞腾生态特点**: 走「横向联盟」路线，与 8500+ 伙伴形成矩阵，强调「开放联合」。但多数适配停留在「跑通」层面，深度性能优化有限。

### 4.2 华为鲲鹏：全栈垂直整合 + 开源生态

**生态架构**:
```
鲲鹏计算产业
├── 🖥️ 硬件层
│   ├── TaiShan 服务器（华为自研整机）
│   ├── 鲲鹏主板（供合作伙伴开发自有品牌）
│   └── 鲲鹏模组（边缘/嵌入）
├── 💿 基础软件层
│   ├── openEuler（开源操作系统，已贡献给开放原子基金会）
│   ├── openGauss（开源数据库）
│   ├── openLooKeng（开源数据虚拟化引擎）
│   └── 鲲鹏 BoostKit（加速库套件，含基础/大数据/数据库/HPC/ARM加速）
├── 🛠️ 开发工具层
│   ├── 鲲鹏 DevKit（开发套件）
│   │   ├── 代码迁移工具
│   │   ├── 性能调优工具
│   │   ├── 编译器（基于 LLVM/GCC 优化）
│   │   └── 系统诊断工具
│   └── 鲲鹏社区（一站式开发者平台）
├── ☁️ 云服务层
│   ├── 华为云鲲鹏云服务
│   └── 鲲鹏云服务器实例（通用型/内存优化型）
└── 🏭 合作伙伴层
    └── ISV/OEM/Hardware Partner 生态
```

**开源生态关键指标**:

| 项目 | 说明 |
|:-----|:------|
| **openEuler** | 贡献给开放原子开源基金会，已有 20+ 发行版，支撑政务/金融/运营商核心系统 |
| **openGauss** | 企业级开源数据库，华为内部数据库替换核心，已在 3000+ 关键业务场景部署 |
| **鲲鹏 DevKit** | 1套代码 + 1套软件工程流水线，帮助开发者完成 x86 → ARM 迁移 |
| **鲲鹏 BoostKit** | 基础加速库（数学库/压缩/加密）+ 场景加速包（大数据/HPC/数据库/存储） |
| **鲲鹏社区** | 学习/实验/认证/技术论坛一站式平台 |

**垂直整合优势**:
- 鲲鹏 CPU + 昇腾 GPU + HCCS 互联 + openEuler OS + MindSpore AI 框架 → **全栈统一**
- 这意味着鲲鹏在数据库/大数据/HPC 场景的**深度优化能力**远超纯 CPU 供应商
- 例如：鲲鹏 BoostKit 中的加速库直接针对 TaiShan 核微架构调优，性能增益可达 20-50%

**生态风险**:
- **美国制裁持续**: 鲲鹏920 7nm 工艺受限，鲲鹏930 依赖 Chiplet 方案绕过
- 尽管 openEuler 已开源，但核心代码贡献仍以华为为主，社区独立性存疑
- 硬件合作伙伴（基于鲲鹏主板做自有品牌）对工艺节点更新慢有顾虑

### 4.3 博瑞晶芯：生态几乎空白

**当前生态状态**:

| 维度 | 现状 |
|:-----|:------|
| 芯片产量 | **0**（尚未有公开确认的流片/量产芯片） |
| 硬件伙伴 | 无公开披露 |
| 软件适配 | 无（依赖 ARM 原生生态） |
| OS 支持 | 理论上可运行所有 ARM 架构 OS（Linux 发行版/统信/麒麟） |
| 开发者社区 | 无 |
| 认证体系 | 无 |

**ARM v9 生态优势（理论）**:
- ARM v9 指令集向下兼容 ARM v8，可复用飞腾/鲲鹏的软件适配成果
- SVE2（可伸缩矢量扩展）对 AI/ML 推理负载有原生加速优势
- CCA（机密计算架构）提供硬件级安全隔离
- 如果有厂商已适配 ARM v8 平台，切换到 ARM v9 平台基本只在编译器层面

**生态建设路径推测**:
- 短期（2026-2027）：完成芯片流片/验证，加入 openEuler/统信/麒麟 等国产 OS 适配列表
- 中期（2028+）：以「ARM v9 + 更先进工艺 + 更低功耗/成本」差异化切入信创市场
- 长期挑战：需要与已有 15 万+ 软件适配的飞腾生态竞争，时间窗口有限

## 8. 战略评估与展望

### 8.1 关键结论

1. **鲲鹏和飞腾是国产 ARM 生态的双核**，短期内无可替代。两者在信创市场占有率合计占国产 ARM 服务器 CPU 的 95%+，且已形成极高的生态壁垒。

2. **博瑞晶芯的 ARM v9 授权是唯一的代际优势**，但挑战在于：
   - 将 IP 转化为可量产芯片需要 18-36 个月，研发风险极高
   - 错过信创第一波替换窗口（2027年前），市场机会大幅减少
   - 缺乏生态基础，需要从零建设

3. **鹏腾生态（飞腾+鲲鹏）**是最大的变量——如果 CEC 系（飞腾）和华为系（鲲鹏）真正统一底层软件生态，将极大压缩后来者的生存空间。

4. **ARM 服务器 CPU 面临来自 RISC-V 的长期竞争**——龙芯 LoongArch 是完全自主指令集，海光 x86 兼容方案也在加速。

### 8.2 对服务器整机厂商的决策建议

| 需求场景 | 推荐方案 | 理由 |
|:---------|:---------|:------|
| 党政信创服务器 | **飞腾 S5000C** | 适配最全、认证最齐、「国家队」背书 |
| 运营商集采通用服务器 | **飞腾 / 鲲鹏** | 两者均覆盖，看集采要求 |
| 高性能计算/数据库 | **鲲鹏 920（TaiShan）** | HCCS 互联+256核+openGauss 深度优化 |
| AI 训练推理主机 | **鲲鹏+昇腾组合** | 全栈最优，HCCS+CCAE 集群管理 |
| 边缘/嵌入式服务器 | **飞腾 E2000** | 柔性架构，功耗低，生态齐全 |
| 双供应商策略 | **飞腾+鲲鹏混部** | 鹏腾生态兼容性，互为备份 |
| 新创 CPU 评估 | 关注 **博瑞晶芯** 流片进展 | 2027年流片成功后再评估 |

### 8.3 需要持续跟踪的未知问题

1. 鲲鹏 930 的具体规格和量产出货时间
2. 飞腾 S5000C 实际性能对标（缺少公开的 SPEC 跑分）
3. 博瑞晶芯是否能在 2027 年前完成首款芯片流片
4. 鹏腾生态的实质融合进展
5. 美国制裁对华为海思（鲲鹏）的新限制
6. ARM 中国在 ARM v9 授权方面对国产厂商的政策变化
