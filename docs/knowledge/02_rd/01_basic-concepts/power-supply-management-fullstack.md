## 1. 引言：供电管理四层全景

### 1.1 供电管理的本质矛盾

从芯片到数据中心，供电管理者面对的**核心矛盾**是：

```
效率                    密度                 可靠性
 ↑                       ↑                     ↑
 更高转换效率 ←── 权衡 ──→ 更高功率密度 ←── 权衡 ──→ 更长MTBF
 ↓                       ↓                     ↓
 更大散热器/更慢开关     更小散热面积/更高温度    更多冗余/更大空间
```

这三个目标的**不可能三角**贯穿了供电管理的所有层级——从芯片上的电压调节器到数据中心的中压配电。

## 2. 芯片级供电管理

### 2.1 处理器供电的跨代演进

CPU/GPU供电从简单的Vcore供电演变为复杂的多域多模式电源架构：

#### 2.1.1 VRD/VRM标准演进

**VRD (Voltage Regulator Down)** 标准定义了主板VR控制器与CPU的交互协议：

| 世代 | 发布年 | 核心电压范围 | 最大电流 | SVID版本 | 关键特性 |
|:-----|:-------|:------------|:---------|:---------|:---------|
| VRD 10.x | 2006 | 0.8-1.6V | 100A | 1.0 | Core 2 Duo基础供电 |
| VRD 11.x | 2008 | 0.6-1.6V | 150A | 1.2 | LLC(负载线调节)引入 |
| VRD 12 | 2011 | 0.4-1.5V | 200A | 1.3 | C6/C7深度节能支持 |
| **VR13** | 2016 | 0.25-1.5V | 280A | 2.0 | Skylake Xeon, 数字SVID, PS4状态 |
| **VR14** | 2022 | 0.2-1.5V | 400A+ | 2.5 | Eagle Stream, DDR5 PMIC协同, 智能跳频 |
| **VR14.5 (VR15)** | 2025 | 0.15-1.5V | 600A+ | 2.7 | 支持背面供电(PowerVia), 每核LDO |

**IMVP (Intel Mobile Voltage Positioning)** 是移动版的VRD标准，在笔记本平台引入了更激进的节能状态：

```
IMVP演进:
├── IMVP6: Penryn时代, C4状态深度
├── IMVP7: Nehalem, 集成VR, C6
├── IMVP8: Haswell, FIVR(完全集成VR)
├── IMVP9: Skylake, C10深度休眠
├── IMVP10: Tiger Lake, DLVR(数字LDO)
└── IMVP IP: Lunar Lake, 全集成PMIC on-package
```

#### 2.1.2 FIVR — Fully Integrated Voltage Regulator

**FIVR**是Intel Haswell/Broadwell架构中的重大创新——将VR从主板移到CPU封装内：

```
传统方案(外部VR):
    主板VR(12V→1.2V) ─── 大电流(100A+) ─── 封装引脚 ─── CPU核
    
    ↑ 每路供电2-4个相位，占用大量PCB面积
    ↑ 大电流路径损耗显著(每相~5-10W I²R)
    ↑ VR切换速度受限(受主板走线电感制约)

FIVR方案(集成VR):
    主板VR(12V→1.8V中间总线) ─── 较低电流 ─── CPU封装
                                                  │
                                        ┌───────┴───────┐
                                        │  FIVR on-die   │
                                        │  每核独立VR    │
                                        │  (1.8V→核心电压)│
                                        └───────┬───────┘
                                                │
                                         CPU核心/LLC
```

**FIVR的关键优势**：
- **电压切换速度提升10x**：从10μs级→1μs级，支持更激进的电源门控
- **每核独立调压**：每个核心在P-State下有不同的最佳电压点
- **PCB大幅简化**：取消了主板上的多路VR相位
- **效率提升**：消除了封装引脚和PCB走线的IR损耗

**FIVR的致命缺陷**：
- **热量集中**：VR产生的热量在封装内，对散热要求极高
- **增大了封装尺寸**：需要在封装衬底上集成大电感/电容
- **良率影响**：多一个精密的analog模块在die上

**历史**：FIVR在Haswell上首次使用，但在Skylake上被简化(仅保留部分)，之后逐渐被DLVR替代。

#### 2.1.3 DLVR— Digital Linear Voltage Regulator

**DLVR**是Intel在Tiger Lake (2020)引入的数字LDO方案，是FIVR的精简版：

```
DLVR架构:
    VccIn (1.8V) ──→ DLVR(LDO mode) ──→ CPU核心
                       │
                        └─── 旁路模式(DLVR Bypass)──→ 直接VccIn

DLVR模式:
├── Full LDO: 完全调节，适用于轻度负载
├── Bypass: 旁路调节，高效率模式
├── Deep Bypass: 最小压差模式
└── Tri-state: 完全关闭(电源门控)
```

**DLVR vs FIVR**：
```
对比维度          FIVR               DLVR
原理             buck转换器          数字LDO
效率(重载)       90-92%            ~88%(bypass: 99%+)
效率(轻载)       85-87%            92-95%(LDO轻载优势)
硅面积           ~2mm²/相          ~0.5mm²
外接电感          需要on-pkg电感      不需要
集成度           高                  中
```

#### 2.1.4 背面供电(PowerVia)对VR的影响

Intel 18A (2025)引入的 **PowerVia (背面供电)** 从根本上改变了VR到核心的供电路径：

```
传统正面供电:
    ┌─────────────────────┐
    │       VR控制器        │
    └────────┬────────────┘
             │  (长距离PCB→封装→die走线)
    ┌────────┴────────────┐
    │   Die正面(信号层+电源) │ ← 电源和信号争抢正面金属层资源
    └────────┬────────────┘
             │  通过TSV到背面
    ┌────────┴────────────┐
    │   Die背面(PDN)       │
    └─────────────────────┘

背面供电(PowerVia):
    VR控制器 ──── 封装PDN
                     │
                     ↓
    ┌─────────────────────┐
    │   Die背面(电源层)     │ ← 完整的低阻抗电源网络
    └────────┬────────────┘
             │  Nano-TSV直接联通
    ┌────────┴────────────┐
    │   Die正面(仅为信号层) │ ← 信号层无电源干扰
    └─────────────────────┘
```

**PowerVia对供电架构的影响**：
1. **电压降减少50%+**：背面PDN的电阻仅为正面的1/3-1/4
2. **VR的负载线(Loadline)可以更平**：允许更小的电压余量(Vguard band)
3. **VR响应速度可以降低**：PDN阻抗低，瞬态响应不再需要极高频VR切换
4. **每相电流可以增大**：背面PDN支持更高电流密度

### 2.2 多相VR控制器架构

#### 2.2.1 基本原理

现代CPU/GPU VR的核心是**多相交错buck转换器**：

```
                    ┌───────┐
    12V Vin ──────┬─┤ Phase1 ├───┐
                  │ └───────┘    │
                  │ ┌───────┐    │  ┌──────────┐
                  ├─┤ Phase2 ├───┼──┤ 输出电容  ├──▶ Vcore
                  │ └───────┘    │  └──────────┘
                  │ ┌───────┐    │
                  └─┤ PhaseN ├───┘
                    └───────┘
                       ↑
                  ┌────┴────┐
                  │  VR     │ ← SVID ↔ CPU
                  │  Controller │
                  └─────────┘
```

**多相交错的关键好处**：
- **纹波抵消**：N相交错360°/N相位，输出纹波频率为N×fsw
- **热分散**：每相电流为总电流/N，热量均匀分布
- **瞬态响应**：多个相位并联，等效电感为L/N
- **可伸缩性**：按功耗需求增减相位数

#### 2.2.2 相位数与电流需求的演进

```
       主流CPU VR相位     主流GPU VR相位     每相最大电流
2010:   4-6相             6-8相             25-30A
2015:   6-8相             8-12相            40-50A
2020:   8-12相            12-16相           60-80A
2023:   12-16相(Xeon)     20-24相(H100)     80-100A
2026:   16-20+相          24-36相(B200)     100-120A
```

#### 2.2.3 VR控制器关键能力

| 功能 | 说明 | 接口/协议 |
|:-----|:------|:----------|
| **SVID** | 串行电压识别，CPU动态请求电压 + P-State | 2线(CLK+DATA), 3.4MHz |
| **PMBus** | 电源管理总线，监控遥测 + 配置 | I²C-based, 400kHz-1MHz |
| **AVSBus** | 自适应电压调节总线，GPU用 | 专有(英飞凌/瑞萨) |
| **IMVP9** | Intel移动电压定位 | SVID v2.0扩展 |
| **自适应电压调节** | 根据温度/老化自动补偿 | 模拟环路 |
| **Droop Compensation** | 负载线(Loadline)精确控制 | 数字控制器 |
| **电流平衡** | 多相间电流均分 | 每相电流采样 |
| **Phase Shedding** | 轻载时自动减少工作相数 | 数字控制器逻辑 |

**SVID (Serial Voltage Identification) 协议**：
```
CPU → VR (请求):
├── SETVID (VID码 + 转换速率)
├── SETVR_FS (频率切换)
├── SETPS (PS状态: PS0-PS4)
└── CONFIGURE (配置遥测参数)

VR → CPU (报告):
├── STATUS (电压就绪/故障)
├── MONITOR (电流/温度/功率)
└── ALERT (过流/过温/欠压)
```

### 2.3 内存供电

#### 2.3.1 DDR5 PMIC 革命

DDR5相比DDR4在供电管理上最大的变化是：**PMIC从主板移到DIMM上**。

```
DDR4供电:
    主板VR(12V→1.2V VDD) ─── 总线供电 ─── 所有RDIMM共享
    主板VR(12V→2.5V VPP) ─── 总线供电 ─── 所有RDIMM共享
    
    × 每根DIMM无法独立调节电压
    × 总线压降在远端DIMM处严重
    × PMIC在主板，DIMM上只有去耦

DDR5供电:
                 ┌──── DIMM 0 ────┐
    VIN_BULK ────┤  PMIC on-DIMM  ├── VDD(1.1V)
    (12V/5V)     │  ├── VDD(1.1V)  │── VDDQ(1.1V)
                 │  ├── VPP(1.8V)  │── VPP(1.8V)
                 │  └── VDDSPD(3.3)│── SPD Hub
                 └────────────────┘
    
                 ┌──── DIMM 1 ────┐
    VIN_BULK ────┤  PMIC on-DIMM  ├── ...
                 └────────────────┘
```

**DDR5 PMIC规范**：

| 参数 | DDR5 PMIC | 说明 |
|:-----|:----------|:------|
| 输入电压 | 5V ±10% | 主要输入轨 |
| 输出电压 | VDD 1.1V / VDDQ 1.1V / VPP 1.8V | 三路输出 |
| 最大输出 | 3A (VDD) + 2A (VDDQ) + 200mA (VPP) | 总~5.5W/DIMM |
| 开关频率 | 2-3MHz | 内部时钟，可同步 |
| 效率 | 85-90% (典型) | 取决于负载条件 |
| 接口 | I²C (1MHz) | 配置和遥测读取 |
| 保护 | OCP/OVP/OTP/UVP | 全保护功能 |

**PMIC关键遥测(通过I²C读取)**：
```
├── VDD输出电压
├── VDD输出电流
├── VDD输出功率
├── VPP输出电压
├── 温度
├── 输入电压(VRMON)
├── 效率
└── 故障状态(OC/OV/OT/UV)
```

#### 2.3.2 SPD Hub

DDR5将SPD从EEPROM升级为 **SPD Hub**——一个I²C从设备多路复用器：

```
                    SPD Hub
    I²C Bus ──→ ┌────────────────┐
                │  Config Regs   │
                ├────────────────┤
                │  MUX           │── SPD EEPROM
                │  (4通道)       │── Thermal Sensor(TSOD)
                │                │── PMIC Ctrl
                │                │── RCD(Register Clock Driver)
                └────────────────┘
```

SPD Hub的供电管理作用：
- 通过PMIC配置寄存器控制VDD/VPP的输出
- 读取TSOD温度触发PMIC throttling
- 在冷启动时先使能PMIC再使能RCD

#### 2.3.3 CKE (Clock Enable) 电源管理

**CKE Power Down**是DRAM的电源管理核心机制：

```
CKE=1 (正常运行)
    │  ┌─── Active Power-Down (快速退出)
    │  │    └── 保留所有bank数据，CKE拉低
    │  │        退出延迟: ~2-3个时钟周期
    │  │
    │  ┌─── Precharge Power-Down (更省电)
    │  │    └── 关闭所有bank的行
    │  │        退出延迟: ~5-6个时钟周期
    │  │
    │  ┌─── Self-Refresh (最省电)
    │  │    └── 内部自刷新，CKE拉低
    │  │        退出延迟: ~几百个时钟周期
    │  │
    │  └─── Maximum Power Saving Mode
    │       └── Self-Refresh + 温度补偿刷新率
    │           退出延迟: ~几μs
    │
CKE=0 (低功耗)
```

**温度补偿刷新(TCR)** 是内存CKE管理的核心智能：

```
温度范围      刷新率        省电收益
T < 45°C     x1 (标准)     基准
45 < T < 85  x2 (2倍)      0%
T > 85°C     x4 (4倍)      -5~10%功耗增加(刷新电流)
```

### 2.4 GPU/NPU 供电

#### 2.4.1 GPU大电流供电挑战

现代AI GPU的供电需求极其苛刻：

| GPU | TDP | Vcore电压 | 满载电流 | VR相位数 | VR方案 |
|:----|:----|:----------|:---------|:---------|:-------|
| NVIDIA A100 | 400W | 0.8-0.95V | ~450A | 12-14相 | 板载多相VR |
| NVIDIA H100 | 700W | 0.75-0.85V | ~870A | 16-20相 | 板载多相VR |
| NVIDIA B200 | 1000W+ | 0.7-0.8V | ~1400A | 24-36相 | 模块化VR |
| AMD MI300X | 750W | 0.75-0.9V | ~900A | 16-24相 | 板载多相VR |

**GPU VR的核心设计约束**：
```
┌─────────────────────────────────────────────────────┐
│  GPU供电核心约束                                     │
├─────────────────────────────────────────────────────┤
│ 1. 极低电压(0.7-0.95V) × 极大电流(500-1500A+)       │
│     → 导通损耗(I²R)成为主要损耗来源                   │
│     → 每相并联MOSFET的Rds(on)必须<1mΩ                │
│                                                      │
│ 2. 瞬态响应(Ton/Toff)需满足GPU workload快速切变       │
│     → 从空闲到满负载时间<1μs                          │
│     → 需要Smart Power Stage + 极低ESL输出电容          │
│                                                      │
│ 3. 热管理: VR效率损失(~5-8%)在700W级GPU上=35-56W热  │
│     → 每相VR的结温必须<125°C                          │
│     → 需要紧贴散热器/风道布局                          │
│                                                      │
│ 4. PCB压降: 1500A在PCB上的压降可达~10-30mV           │
│     → 需要多层铜皮(4oz+)、大电流过孔阵列                │
│     → 或采用overkill: 在VR输出端用铜汇流排             │
└─────────────────────────────────────────────────────┘
```

#### 2.4.2 Smart Power Stage (SPS)

现代GPU VR的核心组件是**智能功率级**——集成了MOSFET驱动器和电流/温度传感的模块：

```
传统分立方案:
    Gate Driver ──┬── High-side MOSFET ──→ Switch Node
                  │
                  └── Low-side MOSFET ──→ GND
    
    × 驱动回路寄生电感大
    × 电流采样精度差(需额外sense电阻)
    × 温度监测需要用外部NTC

Smart Power Stage (SPS):
    ┌───────────────────────────┐
    │  SPS Module                │
    │                            │
    │  ┌─────────┐  ┌───────┐   │
    │  │ HS+LSS  │  │ Driver│   │
    │  │ MOSFET  │  │  +    │   │
    │  │ 集成    │  │ Sense │   │
    │  └─────────┘  └───────┘   │
    │                            │

#### 2.5.1 NVMe SSD供电架构

企业级NVMe SSD通常集成完整的供电子系统：

```
PCIe Slot (3.3V/12V)
    │
    ├── 3.3V ──→ LDO ──→ 1.8V (控制器I/O)
    │          └──→ LDO ──→ 1.2V (NAND接口)
    │
    └── 12V ───→ DC-DC ──→ 3.3V (NAND供电)
                └──→ DC-DR ──→ Vcore (控制器核心)

#### 2.5.2 掉电保护(Power Loss Protection, PLP)

企业级SSD的关键供电特性——掉电时的数据保护：

```
正常供电                    掉电事件
    │                         │
    ▼                         ▼
┌──────────┐            ┌──────────────┐
│ 主控制器  │            │ 检测Vcc <阈值  │
│ + NAND   │            └──────┬───────┘
└──────────┘                   │
                              ▼
                    ┌──────────────────┐
                    │ PLP电容阵列放电     │
                    │ (钽聚合物/超级电容) │
                    ├──────────────────┤
                    │ 紧急操作序列:       │
                    │ 1. 停止所有IO      │
                    │ 2. 刷写DRAM缓存    │
                    │ 3. 更新FTL映射表   │
                    │ 4. 记录Safety Status│
                    │ 5. 关闭电源        │
                    └──────────────────┘

PLP电容容量计算:
    E_required = P_total × t_hold
    其中:
    - P_total: SSD在紧急写模式下的功耗(5-10W)
    - t_hold: 需要维持的时间(10-50ms)
    - 电容: C = 2 × E / (V_initial² - V_final²)

    典型值: 2-4颗 100-330μF 钽聚合物电容
    维持时间: 10-30ms (足够完成刷写操作)
```

#### 3.2.2 中间总线架构(IBA / IBC)

从12V转向48V是数据中心供电架构的重大变革, 引入**中间总线转换器(Intermediate Bus Converter, IBC)**：

```
48V IBC架构:
                     ┌──────────┐
  48V ───────────────┤   IBC    ├── 12V/1.8V ──→ POL VR
                     │ (1/4砖)   │
                     │ 1kW       │
                     │ 48:12/1.8 │
                     │ ~97%效    │
                     └──────────┘
                        │
                        ├── 效率优势:
                        │   48V到POL的转换, 省去48V→12V→Vcore多级
                        │   IBC固定比率转换(SAC拓扑), 极高效率
                        │
                        ├── 电流优势:
                        │   相同功率下48V电流仅为12V的1/4
                        │   减少I²R损耗 ~16x
                        │
                        └── PCB优势:
                            大电流路径减短54'
                            铜皮要求降低(2oz→1oz)
                            去耦电容需求降低
```

**IBC拓扑对比**：

| 拓扑 | 效率(满载) | 功率密度 | 隔离 | 开关频率 | 应用场景 |
|:-----|:----------|:---------|:-----|:---------|:---------|
| **SAC (Sine Amplitude)** | 96-98% | 最高 | 是 | 1-2MHz | 数据中心48V→POL |
| **LLC谐振** | 95-97% | 高 | 是 | 300-500kHz | 隔离式IBC |
| **相移全桥(PSFB)** | 93-95% | 中 | 是 | 100-200kHz | 传统48V方案 |
| **Buck** | 96-98% | 高 | 否 | 500kHz-2MHz | 非隔离48V→12V |

### 3.3 Standby供电

#### 3.3.1 Standby电源轨

服务器主电源关闭时，Standby电源为管理电路供电：

```
Standby电源轨:
├── 12VSB: BMC主控制器, 管理网口PHY, IPMI/KCS
├── 5VSB: BMC副控制器, USB root hub, legacy设备
├── 3.3VSB: SPI Flash, RTC, BIOS SRAM, 数字逻辑
├── VCC_RTC: RTC时钟(钮扣电池/超级电容) <1mW
└── HotSwap供电: PCIe槽热插拔控制器

Standby功耗:
├── 单路服务器: 15-25W (仅BMC+管理)
├── 双路服务器: 20-35W (BMC+管理+FAN standby)
├── AI服务器: 30-60W (BMC+管理+MCTP+NVSwitch standby)
└── 全机架管理器: 100-200W (RMC+交换机+...)
```

**Standby供电设计约束**：
```
挑战 #1: 效率极低负载
    Standby PSU在2-5%负载下运行
    效率可能降至50-60%
    因此: 很多方案采用PSU内部的独立standby绕组

挑战 #2: 长时间运行(7x24)
    服务器寿命5-7年
    Standby -> 即使主系统关机也不断电
    对PSU standby组件的寿命要求极高

挑战 #3: 低噪声
    管理网络需要安静运行
    BMC上的PMBus轮询不能产生电磁干扰影响
    需要独立的低噪LDO为管理PHY供电

典型Standby PSU方案:
    PSU内部集成独立standby DC-DC
    主PSU关闭时，standby绕组仍供电
    效率: 60-70%@5%负载
    功耗: ~3-5W (仅PSU内部)
```

#### 3.3.2 DPU/NIC的带外保持供电

现代DPU和SmartNIC要求在主系统掉电时仍保持**管理网口活跃**：

```
服务器主电源关 ── 但BMC/管理网口仍在工作
          │
          │  需要: DPU/NIC的物理层仍供电
          │        管理网口保持link up
          │        Redfish/IPMI可访问
          │        
          ▼
┌──────────────────────────────┐
│  DPU/SmartNIC                │
│ ┌────────────┐  ┌──────────┐ │
│ │ 应用处理器  │  │ 管理处理器 │ │

│ └────────────┘  └──────────┘ │
│ ┌──────────────────────────┐ │
│ │ 网口PHY                    │ │
│ │ 主供电(12V) + Standby (3.3VSB)│

供电需求:

### 3.4 Power Good (PG) 信号链

#### 3.4.1 PG信号的作用

PG(Power Good)是电源管理中最关键的**同步信令**机制：

```
┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
│ PSU_PG │───►│ P12V_PG│───►│ P5V_PG │───►│ P3.3_PG│
│ (PSU)  │    │ (12V)  │    │ (5V)   │    │ (3.3V) │
└────────┘    └────────┘    └────────┘    └────────┘
                  │
                  ▼
            ┌──────────┐
            │  System   │
            │  SIO/CPLD │──→ PG总信号 → BMC → 系统就绪
            │   (上电   │
            │   时序逻辑)│──→ VR使能信号 → CPU → Vcore_PG
            └──────────┘       │
                               └──→ 内存VR → VDD_PG → DDR5 PMIC

PG延迟需求(典型):
├── PSU PG → 12V: <5ms (PSU输出稳定后)
├── 12V → 5V PG: 10-50ms (5V启动时间)
├── 5V → 3.3V PG: 5-20ms (3.3V启动时间)
├── 3.3V → VR_EN: 1-10ms (VR准备就绪)
├── VR_EN → Vcore_PG: 1-5ms (VR软启动)
└── Vcore_PG → SYS_PWROK: 1-100ms (系统复位释放)
```

#### 3.4.2 PG链的设计约束

**最大PG链长度**：
```
信号链路径: PSU → 12V VR → 5V VR → 3.3V VR → VR13 → CPU Vcore PG → SYS_PWROK

总共 ~5-7级: 每级延迟~1-10ms
总延迟: ~50-200ms
系统复位释放: 需在上电后500ms内完成

超时风险:
    如果某级VR故障 → PG不升高 → 下一级不启动
    → CPLD/BMC的Watchdog检测到"超时"→ 报故障
    → 某些方案用"跳线"旁路PG链(工厂调试用)
```

**PG链的故障模式**：

| 故障模式 | 现象 | 定位手段 | 影响范围 |
|:---------|:-----|:---------|:---------|
| PSU无输出 | PSU_PG始终为低 | PSU LED/PMBus读取 | 整机不上电 |
| 12V VR故障 | 12V_PG不升高 | 万用表测12V输出 | 所有12V负载不工作 |
| VR使能信号开路 | VR不启动 | 示波器测EN引脚 | 特定电压域缺失 |
| PG信号延迟偏大 | 系统复位时序异常 | 时序分析仪 | 随机启动失败 |
| PG glitch | 系统不稳定重启 | 示波器捕获 | 偶发重启 |

### 3.5 功率预算与PMBus监控

#### 3.5.1 系统功率预算

服务器功率预算决定了所有供电组件的选型：

```
┌──── 系统功率预算 ────────────────────┐
│                                       │
│  CPU TDP = 350W (×2 = 700W)          │
│  GPU TDP = 700W (×8 = 5600W)         │
│  Memory = 30W (×16 DIMMs = 480W)     │
│  Storage = 25W (×6 SSD = 150W)       │
│  Network = 25W (×4 NIC = 100W)       │
│  VR/PMIC损耗(~8%) = ~560W            │
│  FAN = 25W (×6 = 150W)               │
│  BMC/管理 = 30W                       │
│                                       │
├──── 总计 = ~7,770W ──────────────────┤
│                                       │
│  PSU配置: 4×3000W (N+1冗余)           │
│  可用功率: 3×3000 = 9000W             │
│  冗余裕量: ~15%                        │
│                                       │
│  实际运行: ~6000-7000W                 │
│  PSU负载率: 50-58% → 效率最优区域       │
└───────────────────────────────────────┘
```

#### 3.5.2 动态功率管理

**PMBus监控频率**决定了功率控制的时效性：

```
BMC PMBus轮询频率:
├── 电压监控: 每100ms (快速)
├── 电流监控: 每200ms (快速)
├── 功率监控: 每500ms (中速)
├── 温度监控: 每1s (慢)
└── 故障响应: 可配置(OCP/OVP在VR内部微秒级响应)

功率封顶(Power Capping)流程:
    1. BMC读取总功率(P_total = PSU P_out / 效率)
    2. 如果 P_total > P_limit:
       a. 降低CPU P-State (通过PMM沟通)
       b. 降低GPU频率(通过NVML/ROCm)
       c. 降低内存频率(通过DIMM PMIC)
       d. 限制SSD IO(最后选择)
    3. 每100ms重新评估 → 逐步恢复
```

**功率封顶的分级策略**：

| 等级 | 功率限制 | 执行时间 | 手段 | 性能影响 |
|:-----|:---------|:---------|:-----|:---------|
| L1 | P_limit + 0% | 0-500ms | 硬件：VR降低电流限值 | 无/极微 |
| L2 | P_limit - 5% | 500ms-2s | 探测DPTF，降低CPU P-State | ~3-5% |
| L3 | P_limit - 10% | 2-5s | 降低GPU频率/MEM CLK | ~10-15% |
| L4 | P_limit - 20% | 5-30s | 限制内存访问、I/O | ~20-30% |
| L5 | P_limit - 50% | 30s+ | 关闭部分GPU/DIMM | ~50%+ |

## 4. 系统级供电策略

### 4.1 上电时序

#### 4.1.1 逐级上电序列

服务器上电有严格的**电压域启动顺序**，违反时序可能导致芯片Latch-up或永久损坏：

```
时序(相对时间):
  0ms ── PSU启动
    │  ┌─ PSU_PG升高 → CPUD检测PG
    │  │
  10ms ── 12V输出稳定
    │  └─ 12V_PG升高
    │
  20ms ── 5V输出稳定
    │  └─ 5V_PG升高 → VR_EN升高
    │
  25ms ── 3.3V输出稳定
    │  └─ 3.3V_PG升高
    │
  30ms ── VR开始软启动 → Vcore开始升高
    │  └─ 内存VR启动 → VDD/VPP开始升高
    │
  50ms ── Vcore稳定
    │  └─ VCCP_PG升高 → CPU复位释放(RSMRST#)
    │
  60ms ── CPU取第一条指令(BIST开始)
    │  └─ 内存初始化(SPD Hub → DDR5 PMIC)
    │
  100ms ── 内存通道training
    │
  200ms ── POST开始(BIOS开始执行)
    │
  800ms ── PCIe枚举
    │
  2-5s ── OS开始启动
```

**方案A: CPLD/SIO时序逻辑** (主流方案)

### 4.2 随机间隔上电(Staggered Power-On)

#### 4.2.1 问题背景

多PSU/多节点场景下，同时上电会产生**巨大浪涌电流**：

```
同时上电问题:
    8台GPU服务器同时上电
    ├── 每台PSU input电容: ~1000μF
    ├── 每台PSU浪涌电流: ~40A peak (230V AC)
    ├── 8台同时浪涌: ~320A peak
    │
    └── 断路器可能跳闸
        PDU断路器: 32A/64A
        320A peak远超断路器额定
        即使每台只有10ms的浪涌...
```

#### 4.2.2 随机延迟策略

标准解决方案是**随机化上电启动延迟**：

```
PSU内部实现:
    每个PSU在上电时生成随机延迟
    ├── 延迟范围: 100ms - 2s (随机)
    ├── 通过PSU内部MCU/CPLD生成
    ├── 精度: ±5ms
    └── 每次AC上电重新生成

整机柜实现:
    RMC(Rack Management Controller)分配上电时隙
    ├── 节点0: 延迟0ms (立即)
    ├── 节点1: 延迟500ms
    ├── 节点2: 延迟1000ms
    ├── ...
    └── 节点N: 延迟N×500ms
    
    控制精度: ±50ms (网络延迟影响)
    最多支持: 32-48节点/(PDU 30kW)

Open Rack V3的实现:
    ┌─────────────────────────────────┐
    │  RMC供电时隙表                    │
    ├─────────────────────────────────┤
    │  Slot 0: T+0.0s (PDU breaker 63A)│
    │  Slot 1: T+0.5s                   │
    │  Slot 2: T+1.0s                   │
    │  ...                              │
    │  Slot 20: T+10.0s (全部上线)       │
    └─────────────────────────────────┘
    
    PDU breaker额定值: 63A
    单节点浪涌: 30A peak × 50ms
    错开时间: 500ms(远大于50ms浪涌持续期)
    最大同时浪涌: 1节点 × 30A = 30A ← 安全
```

#### 4.2.3 浪涌电流限制电路

除了随机延迟，各节点自身也配备**浪涌电流限制器(ICL)**：

```
被动方案(NTC热敏电阻):
    AC入 ── NTC(5Ω@25°C→0.1Ω@热态) ── 整流 ── PFC ── 主电容
    
    ↑ 简单可靠、成本低
    ↑ 但热态NTC冷却需要时间(~10s)
    ↑ 短时间内重复上电→NTC仍为低阻→浪涌抑制失效

主动方案(MOSFET+电阻):
    AC入 ── R(10Ω/50W)── 预充 ── 主电容
              │
              MOSFET(旁路)
    
    上电过程:
    1. MOSFET关断 → 电流经10Ω电阻限制
    2. 主电容充电至~90% → MOSFET导通
    3. 短路电阻 → 正常运行
    
    循环可重复(不同于NTC需要冷却)
    但需要控制电路(增加成本)
```

### 4.3 异常设备发现与电源控制

#### 4.3.1 异常检测分类

```
供电异常
├── 过流(OC)
│   ├── 瞬时OC(μs级) → VR内部关断
│   └── 持续OC(ms级) → PMBus ALERT → BMC
│
├── 过压(OV)
│   ├── 输出超调(瞬态) → VR OVP → 钳位/关断
│   └── 持续过压 → PSU关闭+锁死(需AC cycle恢复)
│
├── 欠压(UV)
│   ├── 负载阶跃 → VR dropout → VR关闭
│   └── 输入跌落 → PSU holdup测试 → PSU关闭
│
├── 过温(OT)
│   └── VR/PSU结温超限 → 降频/关闭
│
├── 功率限制
│   └── 超过PSU额定功率 → 功率封顶/重启
│
├── 时序异常
│   ├── PG信号未按时到达 → CPLD超时 → 关闭所有
│   └── 上电顺序错误 → 系统无法启动 → BMC事件记录
│
└── 电源质量
    ├── 输入谐波 > THD限制 → PSU效率降低
    └── 输入电压跌落(Sag) → PSU bypass → 电池/BBU接管
```

#### 4.3.2 异常设备隔离

当系统上电后发现某个设备(如GPU/内存)供电异常：

```
检测流程:
1. 系统启动后，BMC轮询所有PMBus设备
2. 发现GPU3的VR上报FAULT
   ├── STATUS_WORD = OCP (过流)
   └── READ_IOUT = 短路级电流

响应动作:
├── 策略A: 关闭GPU3 VR → 隔离故障 → 剩余设备继续启动
│   └── 系统降级运行(7/8 GPU可用)
│
├── 策略B: 重启GPU3 (VR重置 → 重新软启动)
│   ├── 如果成功 → 恢复正常
│   └── 如果失败 → 转策略A
│
├── 策略C: 记录故障信息 → 通知运维
│   ├── PMBus故障日志提取
│   ├── VR_FAULT原因(OC/OV/OT)
│   ├── 电压/电流/温度快照
│   └── BMC SEL记录 → Redfish事件推送

硬件实现:
    GPU3的VR使能信号由CPLD/BMC独立控制
    ┌──────────┐        ┌────────────┐
    │  CPLD    │── EN0 ──→ GPU0 VR    │→ 0.8V
    │  (独立使能)│── EN1 ──→ GPU1 VR    │→ 0.8V
    │           │── ...                │
    │           │── EN7 ──→ GPU7 VR    │→ 0.8V
    │           │── EN_NVSW ──→ NVSwitch VR
    └──────────┘        └────────────┘
    
    每个GPU的VR有独立使能和独立PG
    CPLD/BMC可逐GPU控制 → 精确定位故障
```

### 4.4 电源异常控制策略

#### 4.4.1 PSU故障场景处理

| 故障类型 | BMC响应 | 系统影响 | 恢复方式 |
|:---------|:---------|:---------|:---------|
| 单PSU掉电(N+1) | 记录SEL+触发告警 | 无(冗余供电) | 热插拔替换 |
| 多PSU掉电(超过冗余) | 立即触发Power Capping | 性能降级 | 更换PSU |
| PSU输出不稳 | OVP/UVP触发VR关闭 | 系统重启 | PMBus重置 |
| PSU过热 | 提高风扇转速 + 负载降低 | 性能降级 | 温度恢复后自动恢复 |
| PSU风扇故障 | PMBus告警 | 可能触发PSU过温保护 | 更换PSU |
| 输入掉电 | PSU holdup(20ms)维持 | 无(短时) | 切换到BBU/UPS |

#### 4.4.2 系统级电源管理策略

```
策略: 自动关机条件 (BMC固件控制)

条件1: 输入电压 < 176V AC (宽范围PSU)
    ├── 等待300ms(去抖动)
    ├── 如果仍然低于阈值 → 触发Graceful Shutdown
    ├── 30s内完成OS关闭 → 断PSU
    └── 电压恢复后 → BMC通过RTC/定时器恢复

条件2: PSU过温 > 65°C (PSU内部OTP)
    ├── 立即记录SEL
    ├── 降低系统负载(CPU P-State限制)
    ├── 提高风扇转速
    └── 如果温度继续升高 → 触发Force Shutdown

条件3: 系统功率超过PSU额定容量
    ├── 触发硬件功率封顶(VR限流)
    ├── 触发PMBus Power Limit
    ├── 触发OS级Power Capping
    └── 全部失败 → 立即关机(防硬件损坏)

条件4: 关键电压轨异常
    ├── Vcore欠压 → CPU故障 → 系统立即挂起
    ├── Memory VDD异常 → DDR5 PMIC报告 → 系统MCE
    └── P3.3VSB丢失 → BMC掉电 → 无法恢复
```

### 4.5 48V vs 12V 配电

#### 4.5.1 12V到48V的驱动力

```
配电电压与电流关系:
    功率 = V × I
    
    3000W PSU:
    12V输出: 3000/12 = 250A ← 超大规模铜皮
    48V输出: 3000/48 = 62.5A ← 显著减少

PCB铜皮厚度需求对比(同等压降3%):

| 功率 | 12V(铜厚) | 48V(铜厚) | 节省比例 |
|:-----|:----------|:----------|:---------|
| 500W | 2oz | 1oz | 50% |
| 1000W | 3oz | 1oz | 67% |
| 3000W | 4oz+ | 2oz | 50%+ |
| 5000W | 6oz(特殊工艺) | 3oz | 50%+ |

I²R损耗对比:
    3000W传输5m线缆:
    12V @ 250A: P_loss = I²R = 250² × 0.001Ω = 62.5W
    48V @ 62.5A: P_loss = 62.5² × 0.001Ω = 3.9W
    
    省电: 58.6W × 8760h = 513kWh/年/rack
```

#### 4.5.2 48V去中心化供电

48V机架让**供电更加分散化**：

```
传统12V机架:
    PDU(400V AC)
      │
      └── 每节点内部PSU(AC→12V)
            └── 主板VR(12V→Vcore)
    
    ↑ 每节点都需要AC/DC转换
    ↑ 热量在节点内部
    ↑ PSU体积约束在1U高度内

48V机架(ORv3):
    PDU(400V AC)
      │
      └── 机架级PSU(AC→48V) 
            │   集中式, 更大效率
            │   热管理在机架级
            │
            ├── Node 0: 48V→IBC→POL→Vcore
            ├── Node 1: 48V→IBC→POL→Vcore
            └── Node N: ...

    ↑ 单级AC/DC在大PSU中效率更高
    ↑ 节点内无AC高压 → 安全
    ↑ 48V传输损耗小
    ↑ BBU可直接接48V汇流排
```

| 参数            | Si IGBT         | SiC MOSFET      | GaN HEMT       |
|:----------------|:----------------|:----------------|:---------------|
| 击穿电压         | 600-1200V       | 1200-1700V      | 600-900V       |
| 开关频率         | 10-50kHz        | 50-200kHz       | 500kHz-2MHz    |
| 导通电阻Rds(on)  | 高(IGBT饱和压降)  | 极低(~15mΩ)      | 极低(~20mΩ)    |
| 开关损耗         | 高(拖尾电流)      | 极低              | 极低(无体二极管) |
| 工作结温         | 150°C           | 200°C+          | 150°C          |
| 效率提升(vs Si)  | 基准             | +2-5%            | +3-6%          |
| 系统成本         | 基准             | ~1.5-2x          | ~1.5-3x        |

800V PSU采用SiC后的效果:
    传统Si PSU(3000W, 400VAC→48V): 效率93-94%
    SiC PSU(3000W, 800VDC→48V): 效率96-97%
    
    3%效率提升在300kW机架上:
    每天省电: 300,000 × 3% × 24 = 216 kWh
    每年省电: ~78,840 kWh
    按0.6元/kWh计: ~4.7万元/年/机架
    
    数据中心100个机架: 470万元/年
```

#### 5.3.4 800V HVDC落地挑战

| 挑战 | 具体问题 | 当前进展 |
|:-----|:---------|:---------|
| 安全标准 | 800V DC属于高压, 操作安全要求高 | IEC 62477-1 (安全标准), SEV/接触防护方案已成熟 |
| 弧闪风险 | DC电弧不自动过零, 难熄灭 | 快速熔断器+电弧检测, 已有DC专用断路器 |
| 绝缘监测 | DC对地绝缘要求高(漏电流小) | IMD(绝缘监测设备)在800V系统中已使用 |
| 连接器 | 800V/300A+连接器可靠性 | TE ELCON MICRO+, Amphenol 定制方案 |
| 运维培训 | 运维人员需高压资质 | 行业培训体系正在建立 |
| 成本 | SiC器件比Si贵~50-100% | 随工艺成熟, 2026年成本差已缩小至30-50% |
| 兼容性 | 现有12V设备不能直连800V | 节点内仍使用48V IBC(POL最后一级不变) |

### 5.4 备用电源技术路线

#### 5.4.1 传统UPS vs 分布式BBU

```
传统集中式UPS:
    400V AC ----> UPS ----> rPDU ----> PSU ----> 12V/48V
                    │
                    └── 大型电池组(铅酸/锂电)
    
    ↑ 单点故障: 整个UPS下电→所有设备掉电
    ↑ 效率: UPS本身~92% (8%损耗)
    ↑ 维护: 电池组需专业维护
    ↑ 扩展: 机架增加需UPS扩容

分布式BBU (Battery Backup Unit):
    800V/48V汇流排 ──┬── PSU ──→ 负载
                      │
                      └── BBU (机架级/节点级)
    
    ↑ 无单点: 每个机架/节点独立
    ↑ 效率: 无UPS损耗, BBU直连汇流排
    ↑ 扩展: 按需增加BBU
    ↑ 维护: 热插拔BBU, 即插即用
```

#### 5.4.2 BBU技术对比

| 类型 | 能量密度 | 响应时间 | 保持时间 | 循环寿命 | 成本 | 适用场景 |
|:-----|:---------|:---------|:---------|:---------|:-----|:---------|
| **锂离子(LFP)** | 150-200 Wh/kg | <2ms | 3-10min | 5000+次 | 中 | 主流BBU方案 |
| **锂离子(NMC)** | 200-260 Wh/kg | <2ms | 5-15min | 2000-3000次 | 高 | 高密度短时备用 |
| **超级电容** | 5-10 Wh/kg | <100μs | 10-30s | 500,000+次 | 中 | 瞬态补偿+写缓存 |
| **飞轮** | 10-50 Wh/kg | <2ms | 10-30s | 极长 | 高 | UPS级短时备用 |
| **铅酸** | 30-50 Wh/kg | <5ms | 5-30min | 500次 | 极低 | 传统UPS(正在淘汰) |

**NVIDIA HGX BBU方案** (以Grace Hopper为例)：
```
机架级BBU模块:
┌─────────────┐
│  48V BBU    │  1U/2U模块
├─────────────┤
│  LFP电池组   │  1-3kWh容量
│  BMS(电池管理)│
│  DC-DC转换   │
│  热插拔设计   │
│  前面板LED   │
└─────────────┘

工作模式:
1. 正常: BBU充电(恒流→恒压)
2. 掉电: BBU <2ms切换为供电
    ├── 保持系统3-5分钟
    ├── 足够完成:
    │   - GPU checkpoint (dirty cache→HBM→SSD)
    │   - Graceful Shutdown
    │   - 或发电机启动
    └── BBU放电至~10%后关断
3. 恢复: BBU重新充电
```

### 5.5 液冷对供电架构的影响

当整机功率密度超过30kW/rack，风冷已不可行，液冷改变了供电架构：

```
风冷 vs 液冷供电差异:

风冷节点:
    PSU(AC→12V) ── 主板VR ── 芯片
                  风扇供电(5-15%总功耗)

液冷节点:
    PSU(AC→48V) ── IBC(48→1.8V) ── 芯片
                  无风扇! (CDU泵在机架/行级)
                  节省: ~200-500W/节点
                  但: CDU泵功耗增加(机架共享)

整机液冷供电:
    ┌──────────────┐
    │  CDU泵组      │← 48V/800V供电(大型泵功率~2-10kW)
    │  (冷量分配单元)│
    └──────────────┘
         │
         ├── 冷板(冷头) → 热量通过液体带走
         │     ↑ VR产生的废热也被冷却液带走
         │     所以VR可以更紧凑
         │
         └── CDU内部: 板换/干冷器/冷水机
    
    液冷对PSU的额外约束:
    ├── 液冷节点: PSU不能在节点内
    │    → 需要机架级PSU + 汇流排分配
    ├── 工作温度: 冷板冷却液入水温度~35-45°C
    │    → PSU吸入空气可能更热
    │    → PSU降额(derating)风险
    └── 维修: 液冷节点不能随便拔插PSU
         → 需要快接头 → 增加成本
```

## 6. 全栈统一分析矩阵

### 6.1 各层级关键指标对比

| 层级 | 关键参数 | 典型值 | 管理接口 | 控制粒度 |
|:-----|:---------|:-------|:---------|:---------|
| **芯片级(CPU VR)** | Vcore、Icore、TDP | 0.7-1.5V, 600A | SVID v2.x, PMBus | 每核/μs级 |
| **芯片级(内存PMIC)** | VDD/VPP效率 | 1.1V/1.8V, 3A | I²C (1MHz) | 每DIMM |
| **芯片级(GPU VR)** | VGPU、IGPU、TDP | 0.7-0.9V, 1500A | AVSBus/PMBus | 每GPU ~10μs |
| **芯片级(SSD PMIC)** | Vcore/VNAND | 0.8/3.3V, 2-5W | 内部状态机 | PS状态切换 |
| **单板级(PSU)** | 输入/输出电压 | 400VAC/48V | PMBus, IPMI | 10-100ms |
| **单板级(电源树)** | 中间总线电压 | 12V/48V/1.8V | CPLD时序逻辑 | ~10ms |
| **系统级(电源时序)** | 电压域顺序 | 6-12个电压域 | CPLD/BMC Sequencer | ~ms级 |
| **系统级(功率封顶)** | 总功率限制 | 机架额定值 | PMBus + OS/Driver | 100ms-30s |
| **DC级(UPS/BBU)** | 保持时间/效率 | 3-10min/92-97% | SNMP/IPMI | ~秒级 |
| **DC级(配电)** | 电压等级 | 48V/400V/800V | 配电管理系统 | ~分钟级 |

### 6.2 效率链分解

```
典型AI服务器效率链(800V HVDC→GPU核心):

800V DC (输入)
  │  PSU: 96% (SiC, 800VDC→48V, 3kW)
  ▼
48V 汇流排
  │  IBC: 97% (48V→1.8V, Vicor DCD)
  ▼
1.8V 中间总线
  │  GPU VR: 90% (1.8V→0.8V, 24相, 1500A)
  ▼
GPU核心电压(0.8V)
  │  GPU PDN: 98% (on-die IR损耗)
  ▼
GPU晶体管

逐级效率叠加:
    96% × 97% × 90% × 98% = 82.2%

所以: 1000W AC输入 → 实际到GPU晶体管 ~822W
      ~178W 在供电链中损失为热量
```

### 6.3 各层级PMBus/MCTP集成

```
PMBus / MCTP在供电管理中的角色:

芯片层:
├── VR控制器 ← SVID → CPU P-State
├── DDR5 PMIC ← I²C → SPD Hub → BMC
├── GPU VR ← AVSBus → GPU ASIC
└── SSD PMIC ← 内部 → SSD主控

单板层:
├── PSU PMBus ← I²C → BMC (或通过MCTP)
├── VR PMBus ← I²C → BMC (轮询)
├── IBC PMBus ← I²C → BMC
└── CPLD Sequencer ← GPIO/SPI → BMC

系统层:
├── BMC ← IPMI/Redfish → 管理网络
├── BMC ← MCTP over PCIe → 主机OS
└── BMC ← NC-SI → 管理网口

DC层:
├── rPDU ← SNMP → DCIM
├── BBU ← CAN/Modbus → BMS → DCIM
└── UPS ← SNMP → DCIM

MCTP (Management Component Transport Protocol):
├── 统一传输层(PCIe/I2C/SMBus)
├── 承载PLDM(平台级遥测)
├── 承载SPDM(安全认证)
└── 2026+平台标配: 替代旧式PMBus直连
```

### 6.4 供电可靠性FTA (部分)

```
系统运行(AC输入正常)
  │
  ├── [OR] PSU故障
  │     ├── 单PSU: 有冗余 → 继续
  │     └── 多PSU: 超冗余 → 触发Power Capping或关机
  │
  ├── [OR] 输入电源异常
  │     ├── 电压跌落 < 200ms: PSU holdup → 继续
  │     ├── 断电 < 30s: BBU接管 → 平稳切换
  │     └── 断电 > 30s: 发电机启动失败 → Graceful Shutdown
  │
  ├── [AND] 主板VR故障
  │     ├── VR过流: CPLD拉低EN → 关闭对应设备
  │     └── VR过压: VR OVP保护 → 系统完全关断
  │
  ├── [AND] 时序异常
  │     └── PG超时 → CPLD Watchdog关断所有
  │
  └── [OR] 过温
        ├── PSU OT: PSU自身关断 → 冗余切换
        └── VR OT: VR降额 → 性能损失
```

## 7. 未来趋势

### 7.1 800V HVDC 成为基准

到2028-2030年，800V HVDC预计成为AI数据中心的事实标准：
- **推动力**: 100kW+机架密度 + 液冷 + SiC器件成本下降
- **中间状态**: 2026-2028 400V AC与800V DC共存期
- **兼容性**: 节点内48V IBC方案保证800V与旧12V设备并存

### 7.2 全SiC/GaN供电链

从PSU到POL，正在逐步用SiC/GaN取代Si器件：

| 器件 | 2024 | 2026 | 2028(预测) |
|:-----|:-----|:------|:-----------|
| PSU主开关 | Si IGBT | SiC MOSFET | SiC/GaN混合 |
| IBC开关 | Si MOSFET | SiC/GaN | GaN fully |
| VR功率级 | Si MOSFET | Si + GaN hybrid | GaN Co-Packaged |
| Cost premium | 2-3x | 1.2-1.5x | ~1.0x(平价) |

### 7.3 AI驱动的电源管理

- **预测性功率分配**: 基于AI工作负载预测，提前调整PSU负载分布
- **动态电压频率缩放进化**: 从简单P-State到每核毫秒级AI预测调压
- **故障预测**: 从PMBus遥测数据训练模型，提前预警PSU/VR退化

### 7.4 MCTP/SPDM统一安全供电管理

- **SPDM (Security Protocol and Data Model)**: 对PSU/VR进行身份认证
- **加密封装**: 供电遥测数据加密传输
- **防篡改**: PMBus固件签名验证（防恶意修改供电参数）

### 7.5 能量回收与电池互补

- **GPU负载快速变化**: 从空闲到满载仅需<1ms
- **传统PSU响应**: 负载变化率限制(约 50W/ms)
- **Li-ion/超级电容补充**: 快速响应瞬态负载
