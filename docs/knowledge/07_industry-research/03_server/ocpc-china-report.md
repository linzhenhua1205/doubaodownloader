#### 🔥🔥 ④ 09:45-10:05 超大规模MoE对AI基础设施的挑战和机遇 — 阿里云·卢晓伟

**演讲者**：**卢晓伟**，阿里云基础设施异构硬件和系统解决方案资深总监

**背景**：卢晓伟（原阿里云异构硬件总监/资深总监级别）负责阿里云AI基础设施的硬件方案和系统级优化，长期深度参与ODCC/OCP开放标准制定。阿里云2025年发布了OAM/UBB开放加速模块规范，2026年推进ScaleUP互连标准。

**主旨要点**：
- MoE（Mixture of Experts）架构成为大模型主流（DeepSeek-V4/Kimi-K2/Claude Opus 4/GPT-5系列）后，对AI基础设施带来的根本性挑战
- **Expert路由负载不均衡**对GPU利用率和显存分配的冲击
- **All-to-All通信模式**对Scale-Up互联带宽的指数级需求增长（与Dense模型的全Reduce模式本质不同）
- 训练与推理阶段MoE通信模式差异（训练：All-to-All + 全 Reduce；推理：路由稀疏 + 部分 Expert 激活）
- 阿里云在MoE推理集群中的实践经验——含负载均衡策略和GPU利用率数据

**与分论坛联动**：
> 下午14:00 开放系统设计论坛，阿里云孔阳/胡文普将分享 **「数据中心服务器高性能ScaleUP互连系统与实践」**，与本议题形成完整的「MoE负载→互连需求→ScaleUP方案」技术叙事链。

**战略意义**：⭐⭐⭐⭐ — MoE是2026年AI架构的最大变量，阿里云在这方面的公开分享将直接揭示**中国最大公有云厂商的技术判断和投资方向**。

#### 🔥 ⑦ 10:35-10:50 从类脑计算集群走向智算超节点集群 — 清华大学·刘学

**演讲者**：**刘学**，清华大学类脑计算研究中心工程研究员

**背景**：清华大学类脑计算中心是国内类脑计算核心研究团队，此前推出**天机芯片**（Tianjic）——同时支持人工神经网络（ANN）和脉冲神经网络（SNN）的混合芯片架构。团队与华为、浪潮有长期合作。本次议题将类脑计算与超节点架构结合，是学术界探索新型AI系统架构的重要尝试。

**主旨要点（预期）**：
- 类脑计算集群的架构设计理念——**事件驱动（Event-driven）vs 时钟驱动（Clock-driven）**
- 与传统GPU集群的架构差异对比——能效/延迟/扩展性
- 脑启发范式在超节点互联中的应用——**脉冲通信 vs 数据包通信**
- 类脑计算在大规模AI推理中的能效优势数据
- **「从类脑集群走向超节点」的路线图**——如何将神经形态计算从实验室推向超大规模部署

**战略意义**：⭐⭐⭐ — 虽然类脑计算目前仍处于研究阶段，但清华类脑中心与产业界的紧密合作（华为/浪潮），使得这一路线有可能在未来2-3年找到特定场景落地（如边缘推理、实时决策）。**该议题与主论坛超节点主题的呼应，暗示学术界也在主动向超节点架构靠拢**。

**主旨要点（预期）**：

- 高速连接器的创新设计（从传统IO连接器到正交背板/BTB/线缆背板）

**战略意义**：⭐⭐⭐⭐ — 立讯是产业链上少有的能同时评估「铜/光/CPO」三条路线的实体，李承伟对物理层趋势的判断具有产业链权威性。本议题是理解超节点互连**物理层天花板**的关键。

| 🧊 **开放系统设计论坛** | 抖音基于业务优化的液冷服务器架构 | **业务液冷设计** | 高晓军（二次登台） |

**字节四线布局深度解读**：

> 字节在OCPC的参与规模（4条技术线、4位演讲者、跨4个论坛）超过了除浪潮外的任何一家公司。这暗示字节在开放计算社区的战略是**蓄力已久、系统性布局、全栈出击**。

> 从大禹整机架构（硬件）→ OpenBMC（固件管理层）→ 云固件（固件运维层）→ 液冷（热管理层），字节展示的是一条从硬件到管理到运维到物理实现的**全栈开放技术栈**。这对于评估字节对开放标准（dOCS/OISA/CXL/UALink）的真实投入程度有重要意义——字节不是在「顺路参与」，而是在「系统推进」。

### 4.2 阿里云 vs 字节跳动：超节点路线正面交锋

| 维度 | 阿里云 | 字节跳动 |
|:-----|:-------|:---------|
| **主论坛议题** | MoE对AI基础设施的挑战（卢晓伟） | 「大禹」开放架构首秀（高晓军） |
| **分论坛议题** | ScaleUP互连系统与实践（孔阳/胡文普） | 抖音液冷服务器架构（高晓军） |
| **技术路线特征** | **开放标准制定者**（OAM/UBB/dOCS/OISA驱动） | **内部自研→逐步开放**（大禹全栈自研） |
| **社区参与方式** | OCP/OCTC标准制定者，推动生态 | OCP/OCTC新参与者，展示成果后逐步共建 |
| **隐含方向** | **MoE超节点 + 开放标准ScaleUP** | **大禹整机柜 + 开放架构** |
| **定位差异** | 从标准到实践（上而下） | 从实践到标准（下而上） |
| **关键词** | 生态共建、MoE负载、ScaleUP互连 | 自研开放、业务驱动、全栈布局 |

**路线交锋的核心意义**：

> 阿里云和字节跳动（国内最大的两个AI推理集群拥有者）在OCPC的同台交锋实际上是中国AI基础设施路线选择的一次缩影：
> - **阿里路线**：通过标准先行的方式，构建可跨厂商复用的开放生态，降低长期演进成本
> - **字节路线**：先内部打磨到极致，再逐步开放成熟模块，注重实效和性能
>
> 两种路线没有绝对对错，但它们的实践成果将决定中国开放计算生态的未来走向。值得注意的是，字节虽然本次以「新参与者」面貌出现，但其在开放基础设施上的投入（4条线并发）已远超大多数「老牌参与者」。

### 4.3 CXL产业链全景展示

**开放系统设计论坛**构建了完整的CXL对话：

```
Intel 赵森林 (Xeon平台/CXL 3.0) → 协议定义 + CPU平台
                      ↓
        三星 何兴 (CXL产品/解决方案) → 芯片/模组
                      ↓
   浪潮 王海梦/赵伟康 (CXL系统 + PCIe光互连) → 系统集成
                      ↓
       阿里云 孔阳/胡文普 (ScaleUP实践) → 应用落地
```

**这是CXL从技术标准走向生态落地的关键标志**：2025年以前CXL更多停留在技术规范和原型阶段，2026年OCPC的完整产业链展示表明CXL在中国已进入产品化和商用评估阶段。

**关注要点**：
- 三星CXL产品的具体规格（容量/带宽/延迟/功耗/TCO）
- Intel Xeon 6+对CXL 3.0的启用状态和软件生态成熟度
- 浪潮CXL+PCIe光互连方案的技术新颖性——光互连CXL在全球范围内仍处于探索阶段
- 阿里云ScaleUP与CXL的关系——是互补还是竞争？

## 附录 B：术语表

| 术语 | 全称 | 说明 |
|:-----|:-----|:------|
| OCP | Open Compute Project | 开源硬件设计社区，由Facebook于2011年发起 |
| OCTC | 开放计算标准工作委员会 | 中国电子工业标准化技术协会下属的开放计算标准组织 |
| ODCC | 开放数据中心委员会 | 中国开放数据中心联盟，与OCTC并行运作 |
| dOCS | 去交换机互联架构 | 超节点内部GPU直接互联的去交换机化架构 |
| OISA | 开放智能加速架构 | ODCC提出的开放AI加速器互联规范 |
| UALink | Ultra Accelerator Link | OCP发起的加速器高速互联开放标准 |
| UBB | Universal Baseboard | 开放加速器基板通用规范 |
| OAM | Open Accelerator Module | 开放加速器模块外形规范 |
| CXL | Compute Express Link | 基于PCIe的高速缓存一致性互联协议 |
| UQD | Universal Quick Disconnect | 液冷系统快速接头，正在标准化 |
| MoE | Mixture of Experts | 稀疏激活的混合专家模型架构 |
| ScaleUP | — | GPU/加速器之间的高速互联域 |
| ScaleOUT | — | 服务器/机柜之间的集群互联域 |
| CPO | Co-Packaged Optics | 共封装光学，将光引擎与交换芯片封装在一起 |
| BMC | Baseboard Management Controller | 基板管理控制器，服务器远程管理核心固件 |
